# <a name="responsibilities-of-winlogon"></a><span data-ttu-id="a237b-103">Responsabilità di Winlogon</span><span class="sxs-lookup"><span data-stu-id="a237b-103">Responsibilities of Winlogon</span></span>

<span data-ttu-id="a237b-104">[*Winlogon*](../secgloss/w-gly.md) ha le responsabilità seguenti:</span><span class="sxs-lookup"><span data-stu-id="a237b-104">[*Winlogon*](../secgloss/w-gly.md) has the following responsibilities:</span></span>

-   <span data-ttu-id="a237b-105">Windows Station e Desktop Protection</span><span class="sxs-lookup"><span data-stu-id="a237b-105">Window station and desktop protection</span></span>

    <span data-ttu-id="a237b-106">Winlogon imposta la protezione della stazione della finestra e dei desktop corrispondenti per assicurarsi che ogni sia accessibile correttamente.</span><span class="sxs-lookup"><span data-stu-id="a237b-106">Winlogon sets the protection of the window station and corresponding desktops to ensure that each is properly accessible.</span></span> <span data-ttu-id="a237b-107">In generale, questo significa che il sistema locale avrà accesso completo a questi oggetti e che un utente connesso in modo interattivo disporrà dell'accesso in lettura all'oggetto della stazione della finestra e dell'accesso completo all'oggetto desktop dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="a237b-107">In general, this means that the local system will have full access to these objects and that an interactively logged-on user will have read access to the window station object and full access to the application desktop object.</span></span>

-   <span data-ttu-id="a237b-108">Riconoscimento SAS standard</span><span class="sxs-lookup"><span data-stu-id="a237b-108">Standard SAS recognition</span></span>

    <span data-ttu-id="a237b-109">In Winlogon sono presenti Hook speciali nel server user32 che consentono di monitorare gli eventi di firma di accesso condiviso (SAS, [*Secure Attention Sequence*](../secgloss/s-gly.md) ).</span><span class="sxs-lookup"><span data-stu-id="a237b-109">Winlogon has special hooks into the User32 server that allow it to monitor CTRL+ALT+DEL [*secure attention sequence*](../secgloss/s-gly.md) (SAS) events.</span></span> <span data-ttu-id="a237b-110">Winlogon rende disponibili le informazioni sull'evento SAS a GINAs da usare come firma di accesso condiviso o come parte della firma di accesso condiviso.</span><span class="sxs-lookup"><span data-stu-id="a237b-110">Winlogon makes this SAS event information available to GINAs to use as their SAS, or as part of their SAS.</span></span> <span data-ttu-id="a237b-111">In generale, GINAs deve monitorare SASs autonomamente; Tuttavia, qualsiasi [*Gina*](../secgloss/g-gly.md) con la firma di accesso condiviso standard con la combinazione di tasti CTRL + ALT + CANC come uno dei SASs it riconosce deve usare il supporto Winlogon fornito a questo scopo.</span><span class="sxs-lookup"><span data-stu-id="a237b-111">In general, GINAs should monitor SASs on their own; however, any [*GINA*](../secgloss/g-gly.md) that has the standard CTRL+ALT+DEL SAS as one of the SASs it recognizes should use the Winlogon support provided for this purpose.</span></span>

-   <span data-ttu-id="a237b-112">Invio di routine SAS</span><span class="sxs-lookup"><span data-stu-id="a237b-112">SAS routine dispatching</span></span>

    <span data-ttu-id="a237b-113">Quando Winlogon rileva un evento SAS o quando una firma di accesso condiviso viene recapitata a Winlogon da GINA, Winlogon imposta lo stato in base alle modifiche apportate al desktop Winlogon e chiama una delle funzioni di elaborazione della firma di accesso condiviso di GINA.</span><span class="sxs-lookup"><span data-stu-id="a237b-113">When Winlogon encounters a SAS event or when a SAS is delivered to Winlogon by the GINA, Winlogon sets the state accordingly, changes to the Winlogon desktop, and calls one of the SAS processing functions of the GINA.</span></span>

-   <span data-ttu-id="a237b-114">Caricamento del profilo utente</span><span class="sxs-lookup"><span data-stu-id="a237b-114">User profile loading</span></span>

    <span data-ttu-id="a237b-115">Quando gli utenti accedono, i relativi profili utente vengono caricati nel registro di sistema.</span><span class="sxs-lookup"><span data-stu-id="a237b-115">When users log on, their user profiles are loaded into the registry.</span></span> <span data-ttu-id="a237b-116">In questo modo, i processi dell'utente possono usare la chiave del registro di sistema speciale HKEY \_ Current \_ User.</span><span class="sxs-lookup"><span data-stu-id="a237b-116">In this way, the processes of the user can use the special registry key HKEY\_CURRENT\_USER.</span></span> <span data-ttu-id="a237b-117">Winlogon esegue questa operazione automaticamente dopo un accesso riuscito, ma prima dell'attivazione della Shell per l'utente appena connesso.</span><span class="sxs-lookup"><span data-stu-id="a237b-117">Winlogon does this automatically after a successful logon but before activation of the shell for the newly logged-on user.</span></span>

-   <span data-ttu-id="a237b-118">Assegnazione di sicurezza alla shell utente</span><span class="sxs-lookup"><span data-stu-id="a237b-118">Assignment of security to user shell</span></span>

    <span data-ttu-id="a237b-119">Quando un utente esegue l'accesso, GINA è responsabile della creazione di uno o più processi iniziali per tale utente.</span><span class="sxs-lookup"><span data-stu-id="a237b-119">When a user logs on, the GINA is responsible for creating one or more initial processes for that user.</span></span> <span data-ttu-id="a237b-120">Winlogon fornisce una funzione di supporto affinché GINA applichi la sicurezza dell'utente appena connesso a questi processi.</span><span class="sxs-lookup"><span data-stu-id="a237b-120">Winlogon provides a support function for the GINA to apply the security of the newly logged-on user to these processes.</span></span> <span data-ttu-id="a237b-121">Tuttavia, il modo migliore per eseguire questa operazione è che GINA chiama la funzione di Windows [**CreateProcessAsUser ha**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-createprocessasusera)e consente al sistema di fornire il servizio.</span><span class="sxs-lookup"><span data-stu-id="a237b-121">However, the preferred way to do this is for the GINA to call the Windows function [**CreateProcessAsUser**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-createprocessasusera), and let the system provide the service.</span></span>

-   <span data-ttu-id="a237b-122">Controllo screen saver</span><span class="sxs-lookup"><span data-stu-id="a237b-122">Screen saver control</span></span>

    <span data-ttu-id="a237b-123">Winlogon monitora l'attività della tastiera e del mouse per determinare quando attivare i salvaschermi.</span><span class="sxs-lookup"><span data-stu-id="a237b-123">Winlogon monitors keyboard and mouse activity to determine when to activate screen savers.</span></span> <span data-ttu-id="a237b-124">Dopo l'attivazione del screen saver, Winlogon continua a monitorare l'attività della tastiera e del mouse per determinare quando terminare l'screen saver.</span><span class="sxs-lookup"><span data-stu-id="a237b-124">After the screen saver is activated, Winlogon continues to monitor keyboard and mouse activity to determine when to terminate the screen saver.</span></span> <span data-ttu-id="a237b-125">Se la screen saver è contrassegnata come sicura, Winlogon considera la workstation bloccata.</span><span class="sxs-lookup"><span data-stu-id="a237b-125">If the screen saver is marked as secure, Winlogon treats the workstation as locked.</span></span> <span data-ttu-id="a237b-126">Quando si verifica un'attività di mouse o tastiera, Winlogon richiama la funzione [**WlxDisplayLockedNotice**](/windows/desktop/api/Winwlx/nf-winwlx-wlxdisplaylockednotice) di Gina e il comportamento della workstation bloccato riprende.</span><span class="sxs-lookup"><span data-stu-id="a237b-126">When there is mouse or keyboard activity, Winlogon invokes the [**WlxDisplayLockedNotice**](/windows/desktop/api/Winwlx/nf-winwlx-wlxdisplaylockednotice) function of the GINA and locked workstation behavior resumes.</span></span> <span data-ttu-id="a237b-127">Se il screen saver non è sicuro, qualsiasi attività della tastiera o del mouse termina l'screen saver senza notifica a GINA.</span><span class="sxs-lookup"><span data-stu-id="a237b-127">If the screen saver is not secure, any keyboard or mouse activity terminates the screen saver without notification to the GINA.</span></span>

-   <span data-ttu-id="a237b-128">Supporto per più provider di rete</span><span class="sxs-lookup"><span data-stu-id="a237b-128">Multiple network provider support</span></span>
