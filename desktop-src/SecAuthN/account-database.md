<span data-ttu-id="2e3e7-104">Active Directory fornisce il database degli account utilizzato dall' [*centro distribuzione chiavi*](/windows/desktop/SecGloss/k-gly) (KDC) per ottenere informazioni sulle [*entità di sicurezza*](/windows/desktop/SecGloss/s-gly) nel dominio.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-104">Active Directory provides the account database that the [*Key Distribution Center*](/windows/desktop/SecGloss/k-gly) (KDC) uses to obtain information about [*security principals*](/windows/desktop/SecGloss/s-gly) in the domain.</span></span> <span data-ttu-id="2e3e7-105">Ogni entità è rappresentata da un oggetto account nella directory.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-105">Each principal is represented by an account object in the directory.</span></span> <span data-ttu-id="2e3e7-106">La chiave di [*crittografia*](/windows/desktop/SecGloss/e-gly) utilizzata per la comunicazione con un utente, un computer o un servizio viene archiviata come attributo dell'oggetto account di quell'entità di sicurezza.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-106">The [*encryption*](/windows/desktop/SecGloss/e-gly) key used in communicating with a user, computer, or service is stored as an attribute of the account object of that security principal.</span></span>

<span data-ttu-id="2e3e7-107">Solo i controller di dominio sono server Active Directory.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-107">Only domain controllers are Active Directory all other replicas.</span></span> <span data-ttu-id="2e3e7-110">Windows replica l'archivio informazioni per la Active Directory utilizzando un protocollo di replica multimaster proprietario che data-ttu-id="2e3e7-111">La connessione utilizza il [*protocollo di autenticazione Kerberos*](/windows/desktop/SecGloss/k-gly) per fornire [*l'autenticazione*](/windows/desktop/SecGloss/a-gly) e la <span data-ttu-id="2e3e7-112">L'archiviazione fisica dei dati dell'account viene gestita dall' [agente di sistema di directory](/windows/desktop/AD/directory-system-agent), un processo protetto integrato con l'autorità di [*protezione locale*](/windows/desktop/SecGloss/l-gly) (LSA) nel controller di dominio.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-112">Physical storage of account data is managed by the [Directory System Agent](/windows/desktop/AD/directory-system-agent), a protected process integrated with the [*Local Security Authority*](/windows/desktop/SecGloss/l-gly) (LSA) on the domain controller.</span></span> <span data-ttu-id="2e3e7-113">Ai client del servizio directory non viene mai concesso l'accesso diretto all'archivio dati.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-113">Clients of the directory service are never given direct access to the data store.</span></span> <span data-ttu-id="2e3e7-114">Tutti i client che desiderano accedere alle informazioni della directory devono connettersi all'agente del sistema di directory e quindi cercare, leggere e scrivere oggetti directory e i relativi attributi.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-114">Any client that wants access to directory information must connect to the Directory System Agent and then search for, read, and write directory objects and their attributes.</span></span>

<span data-ttu-id="2e3e7-115">Le richieste di accesso a un oggetto o a un attributo nella directory sono soggette alla convalida da parte dei meccanismi di controllo di accesso di Windows.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-115">Requests to access an object or attribute in the directory are subject to validation by Windows access control mechanisms.</span></span> <span data-ttu-id="2e3e7-116">Analogamente agli oggetti di file e cartelle nel file system NTFS, gli oggetti nel Active Directory sono protetti da [*elenchi di controllo di accesso*](/windows/desktop/SecGloss/a-gly) (ACL) che specificano chi può accedere all'oggetto e in che modo.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-116">Like file and folder objects in the NTFS file system, objects in the Active Directory are protected by [*access control lists*](/windows/desktop/SecGloss/a-gly) (ACLs) that specify who can access the object and in what way.</span></span> <span data-ttu-id="2e3e7-117">A differenza di file e cartelle, tuttavia, gli oggetti Active Directory hanno un ACL per ognuno dei relativi attributi.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-117">Unlike files and folders, however, Active Directory objects have an ACL for each of their attributes.</span></span> <span data-ttu-id="2e3e7-118">Gli attributi per le informazioni riservate sull'account possono pertanto essere protetti da autorizzazioni più restrittive rispetto a quelle concesse per altri attributi dell'account.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-118">Thus attributes for sensitive account information can be protected by more restrictive permissions than those granted for other attributes of the account.</span></span>

<span data-ttu-id="2e3e7-119">Le informazioni più sensibili su un account sono ovviamente la relativa password.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-119">The most sensitive information about an account is of course its password.</span></span> <span data-ttu-id="2e3e7-120">Sebbene l'attributo password di un oggetto account archivi una chiave di crittografia derivata da una password, non la password stessa, questa chiave è altrettanto utile per un intruso.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-120">Although the password attribute of an account object stores an encryption key derived from a password, not the password itself, this key is just as useful to an intruder.</span></span> <span data-ttu-id="2e3e7-121">Di conseguenza, l'accesso all'attributo password di un oggetto account viene concesso solo al titolare dell'account, mai a chiunque, non ad altri amministratori.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-121">Therefore, access to the password attribute of an account object is granted only to the account holder, never to anyone else, not even administrators.</span></span> <span data-ttu-id="2e3e7-122">Solo i processi con privilegi di base Trusted Computing, ovvero i processi in esecuzione nel [*contesto*](/windows/desktop/SecGloss/c-gly) di sicurezza di LSA, possono leggere o modificare le informazioni sulla password.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-122">Only processes with Trusted Computing Base privilege—processes running in the security [*context*](/windows/desktop/SecGloss/c-gly) of the LSA—are allowed to read or change password information.</span></span>

<span data-ttu-id="2e3e7-123">Per ostacolare un attacco offline da parte di un utente che ha accesso al nastro di backup di un controller di dominio, l'attributo password di un oggetto account è ulteriormente protetto da una seconda crittografia mediante una chiave di sistema.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-123">To hinder an offline attack by someone with access to the backup tape of a domain controller, the password attribute of an account object is further protected by a second encryption using a system key.</span></span> <span data-ttu-id="2e3e7-124">Questa chiave di crittografia può essere archiviata su supporti rimovibili in modo che sia possibile proteggerla separatamente oppure archiviarla sul controller di dominio, ma protetta da un meccanismo di dispersione.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-124">This encryption key can be stored on removable media so that it can be safeguarded separately, or it can be stored on the domain controller but protected by a dispersal mechanism.</span></span> <span data-ttu-id="2e3e7-125">Agli amministratori viene offerta la possibilità di scegliere la posizione in cui viene archiviata la chiave di sistema e quale dei diversi algoritmi viene utilizzata per crittografare gli attributi delle password.</span><span class="sxs-lookup"><span data-stu-id="2e3e7-125">Administrators are given the option to choose where the system key is stored and which of several algorithms is used to encrypt password attributes.</span></span>
