## <a name="remarks"></a><span data-ttu-id="9513f-209">Commenti</span><span class="sxs-lookup"><span data-stu-id="9513f-209">Remarks</span></span>

<span data-ttu-id="9513f-210">La funzione **AcceptSecurityContext (Negotiate)** è la controparte del server della funzione [**InitializeSecurityContext (Negotiate)**](initializesecuritycontext--negotiate.md) .</span><span class="sxs-lookup"><span data-stu-id="9513f-210">The **AcceptSecurityContext (Negotiate)** function is the server counterpart to the [**InitializeSecurityContext (Negotiate)**](initializesecuritycontext--negotiate.md) function.</span></span>

<span data-ttu-id="9513f-211">Quando il server riceve una richiesta da un client, il server usa il parametro *fContextReq* per specificare cosa richiede la sessione.</span><span class="sxs-lookup"><span data-stu-id="9513f-211">When the server receives a request from a client, the server uses the *fContextReq* parameter to specify what it requires of the session.</span></span> <span data-ttu-id="9513f-212">In questo modo, un server può specificare che i client devono essere in grado di usare una sessione riservata o controllata dall' [*integrità*](../secgloss/i-gly.md)e rifiutare i client che non sono in grado di soddisfare tale richiesta.</span><span class="sxs-lookup"><span data-stu-id="9513f-212">In this fashion, a server can specify that clients must be capable of using a confidential or [*integrity*](../secgloss/i-gly.md)-checked session, and it can reject clients that cannot meet that demand.</span></span> <span data-ttu-id="9513f-213">In alternativa, un server può non richiedere alcun elemento e qualsiasi altro elemento che il client può fornire o richiede viene restituito nel parametro *pfContextAttr* .</span><span class="sxs-lookup"><span data-stu-id="9513f-213">Alternatively, a server can require nothing, and whatever the client can provide or requires is returned in the *pfContextAttr* parameter.</span></span>
