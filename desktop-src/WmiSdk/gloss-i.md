<dl> <dt>

<span id="wmi.gloss_impersonation"></span><span id="WMI.GLOSS_IMPERSONATION"></span>**Rappresentazione**
</dt> <dd>

Possibilità di un thread in un processo di assumere l'identità di sicurezza e i privilegi di un altro processo.

</dd> <dt>

<span id="wmi.gloss_instance_provider"></span><span id="WMI.GLOSS_INSTANCE_PROVIDER"></span>**provider di istanze**
</dt> <dd>

Un server COM che fornisce istanze di classi e implementa metodi [**dell'interfaccia IWbemServices**](/windows/desktop/api/WbemCli/nn-wbemcli-iwbemservices) per supportare il recupero, la modifica, l'eliminazione, l'enumerazione o l'elaborazione di query.

</dd> <dt>

<span id="wmi.gloss_intrinsic_event"></span><span id="WMI.GLOSS_INTRINSIC_EVENT"></span>**evento intrinseco**
</dt> <dd>

Evento fornito da WMI, ovvero una notifica di una modifica nel modello di dati WMI standard. Ad esempio, la creazione di una nuova istanza di [**Win32 \_ Process**](/windows/desktop/CIMWin32Prov/win32-process) è un [**\_ \_ tipo InstanceCreationEvent**](--instancecreationevent.md) di evento intrinseco. Vedere anche [*l'evento etrinsic*](gloss-e.md).

</dd> <dt>

<span id="wmi.gloss_instrumentation"></span><span id="WMI.GLOSS_INSTRUMENTATION"></span>**Strumentazione**
</dt> <dd>

Meccanismo usato per segnalare lo stato dell'hardware e del software di sistema. WMI implementa la strumentazione modellando un sistema usando classi basate su [*Common Information Model (CIM).*](gloss-c.md) WMI nell'.NET Framework crea classi instrumentate aggiungendo attributi.

</dd> </dl>
