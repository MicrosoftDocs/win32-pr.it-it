# <a name="i-wmi"></a>I (WMI)

<dl> <dt>

<span id="wmi.gloss_impersonation"></span><span id="WMI.GLOSS_IMPERSONATION"></span>**rappresentazione**
</dt> <dd>

La capacità di un thread in un processo di assumere l'identità di sicurezza e i privilegi di un altro processo.

</dd> <dt>

<span id="wmi.gloss_instance_provider"></span><span id="WMI.GLOSS_INSTANCE_PROVIDER"></span>**provider di istanze**
</dt> <dd>

Server COM che fornisce istanze delle classi e implementa i metodi dell'interfaccia [**IWbemServices**](/windows/desktop/api/WbemCli/nn-wbemcli-iwbemservices) per supportare il recupero, la modifica, l'eliminazione, l'enumerazione o l'elaborazione di query dei dati.

</dd> <dt>

<span id="wmi.gloss_intrinsic_event"></span><span id="WMI.GLOSS_INTRINSIC_EVENT"></span>**evento intrinseco**
</dt> <dd>

Un evento fornito da WMI, ovvero una notifica di una modifica nel modello di dati WMI standard. La creazione di una nuova istanza del [**\_ processo Win32**](/windows/desktop/CIMWin32Prov/win32-process) , ad esempio, è un tipo [**\_ \_ InstanceCreationEvent**](--instancecreationevent.md) di evento intrinseco. Vedere anche [*evento estrinseco*](gloss-e.md).

</dd> <dt>

<span id="wmi.gloss_instrumentation"></span><span id="WMI.GLOSS_INSTRUMENTATION"></span>**Strumentazione**
</dt> <dd>

Meccanismo utilizzato per segnalare lo stato dell'hardware e del software di sistema. WMI implementa la strumentazione mediante la modellazione di un sistema utilizzando le classi basate sul [*Common Information Model (CIM)*](gloss-c.md). WMI nella .NET Framework crea classi instrumentate aggiungendo attributi.

</dd> </dl>
