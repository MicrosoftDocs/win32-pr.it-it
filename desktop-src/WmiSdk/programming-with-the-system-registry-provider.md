# <a name="programming-with-the-system-registry-provider"></a>Programmazione con il provider del registro di sistema

Negli argomenti seguenti viene descritto il modo in cui le applicazioni possono utilizzare il provider del registro di sistema per fornire i dati per le classi create.



| Argomento                                                                                                                      | Descrizione                                                                                                                                                                                                                    |
|----------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [Definizione delle classi per il provider del registro di sistema](defining-classes-for-the-system-registry-provider.md)                 | Per accedere al registro di sistema, è necessario creare classi Windows che archiviano le informazioni del registro di sistema nel repository Strumentazione gestione Windows (WMI).                                                                     |
| [Uso del provider del registro di sistema come provider di proprietà](using-the-system-registry-provider-as-a-property-provider.md) | Il provider del registro di sistema richiede l'utilizzo di diversi qualificatori per visualizzare le proprietà del registro di sistema.                                                                                                                             |
| [Descrizione di una risorsa per il registro di sistema](describing-a-resource-for-the-registry.md)                                       | È possibile accedere ai dati nel registro di sistema che descrive una risorsa di sistema. Per modificare le informazioni, il provider del registro di sistema richiede una sintassi specifica.                                                                      |
| [Accesso a un valore del registro di sistema senza nome](accessing-an-unnamed-registry-value.md)                                             | Per accedere alla maggior parte dei valori del registro di sistema, creare una classe per ricevere i valori in WMI oppure è possibile esaminare specifiche proprietà del registro di sistema. Tuttavia, il provider del registro di sistema richiede una sintassi speciale per visualizzare i valori del registro di sistema senza nome. |
