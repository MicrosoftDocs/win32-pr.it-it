# <a name="w-wmi"></a>W (WMI)

[A](gloss-a.md) B [C](gloss-c.md) [d](gloss-d.md) [E](gloss-e.md) [F](gloss-f.md) G [H](gloss-h.md) [i](gloss-i.md) J [K](gloss-k.md) [L](gloss-l.md) [M](gloss-m.md) [N](gloss-n.md) [O](gloss-o.md) [P](gloss-p.md) [Q](gloss-q.md) [R](gloss-r.md) [S](gloss-s.md) [T](gloss-t.md) U V W X Y Z

<dl> <dt>

<span id="wmi.gloss_win32_schema"></span><span id="WMI.GLOSS_WIN32_SCHEMA"></span>**Schema Win32**
</dt> <dd>

Estensione Microsoft dello [*schema CIM*](gloss-c.md) che contiene le definizioni di classi e istanze per rappresentare [*gli oggetti gestiti*](gloss-m.md) presenti in un ambiente Windows tipico.

</dd> <dt>

<span id="wmi.gloss_wmi_class"></span><span id="WMI.GLOSS_WMI_CLASS"></span>**Classe WMI**
</dt> <dd>

Modello per un tipo di [*oggetto gestito*](gloss-m.md). Le classi WMI definiscono le unità di base della gestione.

</dd> <dt>

<span id="wmi.gloss_wmi_class_provider"></span><span id="WMI.GLOSS_WMI_CLASS_PROVIDER"></span>**Provider di classi WMI**
</dt> <dd>

Server COM che fornisce le definizioni delle classi. I provider di classi implementano i metodi nell'interfaccia [**IWbemServices**](/windows/desktop/api/WbemCli/nn-wbemcli-iwbemservices) per supportare il recupero, la modifica, l'eliminazione, l'enumerazione e l'elaborazione di query dei dati.

</dd> <dt>

<span id="wmi.gloss_wmi_control"></span><span id="WMI.GLOSS_WMI_CONTROL"></span>**Controllo WMI**
</dt> <dd>

Uno snap-in di Microsoft Management Console (MMC) che consente a un utente di configurare le impostazioni WMI in un computer locale o remoto.

</dd> <dt>

<span id="wmi.gloss_wmi_for_windows_driver_model"></span><span id="WMI.GLOSS_WMI_FOR_WINDOWS_DRIVER_MODEL"></span>**WMI per Windows Driver Model (WDM)**
</dt> <dd>

Componente di basso livello che fornisce l'accesso tra i dispositivi WMI e hardware.

</dd> <dt>

<span id="wmi.gloss_wmi_odbc_adapter"></span><span id="WMI.GLOSS_WMI_ODBC_ADAPTER"></span>**Adapter ODBC WMI**
</dt> <dd>

Interfaccia che consente a un'applicazione di ottenere informazioni dalla gestione di Windows utilizzando *WMI Query Language*. Questo fornisce un'API standard che consente alle applicazioni basate su ODBC di usare i dati nel [*repository CIM*](gloss-c.md) come se si trattasse di un database.

</dd> <dt>

<span id="wmi.gloss_wmi_query_language"></span><span id="WMI.GLOSS_WMI_QUERY_LANGUAGE"></span>**WMI Query Language (WQL)**
</dt> <dd>

Un subset di SQL (Structured Query Language) con estensioni per supportare la notifica degli eventi WMI e altre funzionalità specifiche di WMI.

</dd> <dt>

<span id="wmi.gloss_wmi_method"></span><span id="WMI.GLOSS_WMI_METHOD"></span>**Metodo WMI**
</dt> <dd>

Metodo implementato dall'API [com](com-api-for-wmi.md) WMI o dall'API di [Scripting](scripting-api-for-wmi.md). Due esempi sono il metodo COM [**IWbemServices:: ExecMethod**](/windows/desktop/api/WbemCli/nf-wbemcli-iwbemservices-execmethod) e il metodo API di scripting simile [**SWbemServices.ExecMethod**](swbemservices-execmethod.md).

</dd> <dt>

<span id="wmi.gloss_wmi_repository"></span><span id="WMI.GLOSS_WMI_REPOSITORY"></span>**Repository WMI**
</dt> <dd>

Area di archiviazione centrale gestita dal [*CIM Object Manager*](gloss-c.md).

</dd> <dt>

<span id="wmi.gloss_wql"></span><span id="WMI.GLOSS_WQL"></span>**WQL**
</dt> <dd>

Vedere *WMI Query Language (WQL)*.

</dd> </dl>
