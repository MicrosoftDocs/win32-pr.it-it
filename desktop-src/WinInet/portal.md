# <a name="windows-internet"></a>Windows Internet

## <a name="purpose"></a>Scopo

L'API Microsoft Windows Internet (WinINet) consente alle applicazioni di accedere ai protocolli Internet standard, ad esempio FTP e HTTP. Per semplicità d'uso, WinINet astrae questi protocolli in un'interfaccia di alto livello.

## <a name="where-applicable"></a>Se applicabile

WinINet non supporta le implementazioni del server. Inoltre, non deve essere usato da un servizio. Per le implementazioni o i servizi server, [usare Microsoft Windows HTTP Services (WinHTTP)](/windows/desktop/WinHttp/winhttp-start-page).

## <a name="developer-audience"></a>Sviluppatori

WinINet è progettato per l'uso da parte dei programmatori C/C++. Richiede una conoscenza di base dei protocolli FTP e HTTP.

## <a name="run-time-requirements"></a>Requisiti di runtime

Le applicazioni che usano l'API WinINet richiedono Windows NT 4.0 o versione successiva o Windows Me/98/95. Per altre informazioni sui sistemi operativi o sui componenti necessari per usare un particolare elemento di programmazione, vedere la sezione Requisiti della documentazione.
