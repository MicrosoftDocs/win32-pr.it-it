# <a name="logon-count-attribute"></a>Logon-Count attributo

Numero di volte in cui l'account ha eseguito l'accesso. Il valore 0 indica che il valore è sconosciuto.



| Voce | Valore |
|-------------------|--------------------------------------|
| CN                | Logon-Count                          |
| Ldap-Display-Name | logonCount                           |
| Dimensione              | 4 byte                              |
| Privilegio di aggiornamento  | Amministratore di dominio                 |
| Frequenza di aggiornamento  | Ogni volta che l'utente esegue l'accesso.          |
| Attribute-Id      | 1.2.840.113556.1.4.169               |
| System-Id-Guid    | bf9679aa-0de6-11d0-a285-00aa003049e2 |
| Sintassi            | [**Enumerazione**](s-enumeration.md) |

## <a name="remarks"></a>Commenti

Questo attributo non viene replicato e viene mantenuto in ogni controller di dominio nel dominio. Per ottenere un valore accurato per il numero totale di tentativi di accesso riusciti dell'utente nel dominio, è necessario eseguire query su ogni controller di dominio nel dominio e usare la somma dei valori. Tenere presente che l'attributo non viene replicato, pertanto i controller di dominio ritirati potrebbero avere contati anche gli accessi per l'utente e questi non saranno presenti nel conteggio.

> [!IMPORTANT]
> A causa della compatibilità con le versioni a 16 bit di LAN Manager, l'attributo ha un limite superiore di 65535. Dopo aver raggiunto questo limite, non è possibile usarlo come indicatore dell'attività dell'utente in questo controller di dominio.
