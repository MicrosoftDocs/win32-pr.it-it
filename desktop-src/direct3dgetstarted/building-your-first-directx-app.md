# <a name="create-your-first-windows-app-using-directx"></a>Creare la prima app Windows con DirectX

Se si conosce DirectX, è possibile sviluppare un'app DirectX usando C++ e HLSL nativi per sfruttare al meglio l'hardware grafico.

Usare questa esercitazione di base per iniziare a usare lo sviluppo di app DirectX, quindi usare la roadmap per continuare a esplorare DirectX.

## <a name="windows-desktop-app-with-c-and-directx"></a>Windows'app desktop con C++ e DirectX

Un Windows app desktop con DirectX è un'app sviluppata usando API C++ e DirectX native. Questo modello è più complesso di un'app scritta in un framework gestito, ma offre maggiore flessibilità e maggiore accesso alle risorse di sistema, in particolare ai dispositivi grafici. Si tratta quindi di un buon modello per lo sviluppatore esperto.

## <a name="why-develop-a-windows-app-with-directx"></a>Perché sviluppare un'app Windows con DirectX?

La risposta è semplice: si vuole creare un gioco a elevato utilizzo di grafica o multimediale e usare le funzionalità supportate da molti dispositivi grafici. Questo non sarà facile se non si ha la prima idea di sviluppo di giochi o di sviluppo di Windows e C/C++, ma ci sono alcune buone notizie: DirectX 11 è la versione più semplice e coesa di Microsoft DirectX. È anche il più potente e ricco di funzionalità. Se l'obiettivo è quello di imparare a usare le tecniche di rendering più avanzate, DirectX può offrire la possibilità di farlo.

Detto questo, la pianificazione del gioco (o app interattiva in tempo reale) è essenziale. Se non si ha la possibilità di sviluppare giochi e il gioco non ha requisiti di grafica impegnativi, è consigliabile svilupparlo con .NET Framework. Inoltre, molti pacchetti di grafica e sviluppo di giochi "middleware" sono disponibili Windows piattaforme e alcuni non richiedono competenze di programmazione significative.

Se si è certi o si ha semplicemente il sogno di creare un gioco con grafica ad alta fedeltà (o un'app con contenuto grafico complesso), leggere.
