# <a name="i-isolated-applications-and-side-by-side-assemblies"></a>I (applicazioni isolate e assembly side-by-side)

<dl> <dt>

<span id="_win32_isolated_application_gly"></span><span id="_WIN32_ISOLATED_APPLICATION_GLY"></span>**applicazione isolata**
</dt> <dd>

Applicazione che usa assembly affiancati, ma che non condividono gli assembly. Un'applicazione isolata è sempre accompagnata da un file manifesto dell'applicazione. Il manifesto specifica le versioni degli assembly side-by-side condivise a cui l'applicazione esegue il binding in fase di esecuzione e può contenere alcuni metadati per gli assembly side-by-side privati. Il manifesto contiene informazioni tradizionalmente archiviate nel registro di sistema; un'applicazione completamente isolata può essere indipendente dal registro di sistema.

</dd> </dl>
