# <a name="m-isolated-applications-and-side-by-side-assemblies"></a>M (applicazioni isolate e assembly side-by-side)

[A](a-sbscs-gly.md) B C [d](d-sbscs-gly.md) E F [G](g-sbscs-gly.md) H [i](i-sbscs-gly.md) J K L M N O [P](p-sbscs-gly.md) Q [R](r-sbscs-gly.md) [S](s-sbscs-gly.md) T U V W X Y Z

<dl> <dt>

<span id="_win32_manifest_file_gly"></span><span id="_WIN32_MANIFEST_FILE_GLY"></span>**file manifesto**
</dt> <dd>

File XML che descrive un assembly affiancato o un'applicazione isolata. Un file manifesto può contenere i metadati di associazione e attivazione tradizionalmente archiviati nel registro di sistema, ad esempio le classi COM, le interfacce e le librerie dei tipi. Ogni applicazione isolata o assembly affiancato è accompagnato da un file manifesto. Un manifesto dell'applicazione descrive un'applicazione completamente o parzialmente isolata e un manifesto dell'assembly descrive un assembly side-by-side.

</dd> <dt>

<span id="_win32_metadata_gly"></span><span id="_WIN32_METADATA_GLY"></span>**metadati**
</dt> <dd>

Informazioni archiviate in un file manifesto sull'associazione o sull'attivazione di un'applicazione isolata o di un assembly affiancato. Ad esempio, le informazioni sulle classi COM, le interfacce e le librerie dei tipi, tradizionalmente archiviate nel registro di sistema, possono essere archiviate come metadati in un manifesto. I metadati descrivono gli elementi che costituiscono un assembly e le relative dipendenze da altri assembly affiancati. I metadati specificati nel manifesto dell'applicazione e gli assembly dipendenti possono essere utilizzati per popolare il contesto di attivazione.

</dd> </dl>
