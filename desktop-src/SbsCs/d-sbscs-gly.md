# <a name="d-isolated-applications-and-side-by-side-assemblies"></a>D (applicazioni isolate e assembly side-by-side)

<dl> <dt>

<span id="_win32_def_context_assemblyidentity_gly"></span><span id="_WIN32_DEF_CONTEXT_ASSEMBLYIDENTITY_GLY"></span>**AssemblyIdentity del contesto DEF**
</dt> <dd>

**Sottoelemento assemblyIdentity** che definisce l'assembly side-by-side descritto dal manifesto. Un sottoelemento **DEF-context assemblyIdentity** è sempre il primo sottoelemento di un **elemento assembly.**

</dd> <dt>

<span id="_win32_dll_versioning_conflict_gly"></span><span id="_WIN32_DLL_VERSIONING_CONFLICT_GLY"></span>**Conflitto di controllo delle versioni delle DLL**
</dt> <dd>

Problema di compatibilità. I problemi di condivisione degli assembly si verificano quando un'applicazione installa una versione di un assembly condiviso che non è compatibile con le versioni precedenti con la versione installata in precedenza. Una soluzione per i conflitti di controllo delle versioni delle DLL consiste nell'usare la condivisione di assembly side-by-side e le applicazioni isolate. Con la condivisione di assembly side-by-side, più versioni dello stesso Windows assembly possono essere eseguite contemporaneamente. Gli sviluppatori possono scegliere l'assembly side-by-side da usare.

</dd> </dl>
