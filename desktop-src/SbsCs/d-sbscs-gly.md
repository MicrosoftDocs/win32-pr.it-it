# <a name="d-isolated-applications-and-side-by-side-assemblies"></a>D (applicazioni isolate e assembly side-by-side)

<dl> <dt>

<span id="_win32_def_context_assemblyidentity_gly"></span><span id="_WIN32_DEF_CONTEXT_ASSEMBLYIDENTITY_GLY"></span>**AssemblyIdentity del contesto DEF**
</dt> <dd>

Sottoelemento **assemblyIdentity** che definisce l'assembly affiancato descritto dal manifesto. Un sottoelemento **assemblyIdentity** del contesto def è sempre il primo sottoelemento di un elemento **assembly** .

</dd> <dt>

<span id="_win32_dll_versioning_conflict_gly"></span><span id="_WIN32_DLL_VERSIONING_CONFLICT_GLY"></span>**Conflitto di controllo delle versioni DLL**
</dt> <dd>

Problema di compatibilità. Si verificano problemi di condivisione degli assembly quando un'applicazione installa una versione di un assembly condiviso che non è compatibile con la versione precedente installata. Una soluzione per i conflitti di controllo delle versioni delle DLL consiste nell'utilizzare la condivisione di assembly affiancata e le applicazioni isolate. Con la condivisione di assembly affiancati, è possibile eseguire contemporaneamente più versioni dello stesso assembly di Windows. Gli sviluppatori possono scegliere l'assembly affiancato da usare.

</dd> </dl>
