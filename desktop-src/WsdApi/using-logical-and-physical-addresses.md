# <a name="using-logical-and-physical-addresses"></a><span data-ttu-id="44206-103">Uso di indirizzi logici e fisici</span><span class="sxs-lookup"><span data-stu-id="44206-103">Using Logical and Physical Addresses</span></span>

<span data-ttu-id="44206-104">[WS-Discovery](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf) definisce l'indirizzamento logico mediante gli URI in base al `urn:uuid:` formato.</span><span class="sxs-lookup"><span data-stu-id="44206-104">[WS-Discovery](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf) defines logical addressing using URIs based on the `urn:uuid:` format.</span></span> <span data-ttu-id="44206-105">Lo scopo di questo schema di indirizzamento è differenziare l'identità del dispositivo dal relativo indirizzo IP corrente.</span><span class="sxs-lookup"><span data-stu-id="44206-105">The purpose of this addressing scheme is to differentiate the identity of the device from its current IP address.</span></span> <span data-ttu-id="44206-106">Questo schema fornisce essenzialmente la funzionalità dei nomi DNS senza richiedere un server dei nomi.</span><span class="sxs-lookup"><span data-stu-id="44206-106">This scheme essentially provides the functionality of DNS names without requiring a name server.</span></span> <span data-ttu-id="44206-107">[Devices Profile for Web Services](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS) suggerisce che i dispositivi usino questo schema di indirizzamento.</span><span class="sxs-lookup"><span data-stu-id="44206-107">[Devices Profile for Web Services](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS) recommends that devices use this addressing scheme.</span></span>

<span data-ttu-id="44206-108">DPWS consiglia inoltre ai servizi di utilizzare indirizzi fisici (detti anche trasporto).</span><span class="sxs-lookup"><span data-stu-id="44206-108">DPWS also recommends that services use physical (also called transport) addresses.</span></span> <span data-ttu-id="44206-109">Questo consente ai client che non supportano in modo nativo WS-Discovery meccanismi di indirizzamento per comunicare con i servizi DPWS.</span><span class="sxs-lookup"><span data-stu-id="44206-109">This enables clients that do not natively support WS-Discovery addressing mechanisms to communicate with DPWS services.</span></span> <span data-ttu-id="44206-110">Ogni servizio può inoltre definire i relativi indirizzi, che consente l'indirizzamento a livello di trasporto per le implementazioni dei dispositivi che gestiscono l'invio di servizi a un livello inferiore.</span><span class="sxs-lookup"><span data-stu-id="44206-110">Also, each service can define its addresses, which enables transport level addressing for device implementations that manage service dispatch at a lower layer.</span></span> <span data-ttu-id="44206-111">Infine, l'utilizzo di indirizzi fisici ottimizza l'interoperabilità.</span><span class="sxs-lookup"><span data-stu-id="44206-111">Finally, using physical addresses maximizes interoperability.</span></span>

<span data-ttu-id="44206-112">Lo svantaggio dell'indirizzamento fisico è che aggiunge complessità alle implementazioni dei dispositivi, in quanto l'indirizzo IP o di trasporto corrente deve essere rilevato e i metadati del dispositivo devono essere modificati di conseguenza.</span><span class="sxs-lookup"><span data-stu-id="44206-112">The disadvantage of physical addressing is that it adds complexity to device implementations, as the current IP or transport address must be tracked and device metadata must be modified accordingly.</span></span> <span data-ttu-id="44206-113">Per questo motivo, DPWS non richiede l'utilizzo di indirizzi di trasporto da servizi.</span><span class="sxs-lookup"><span data-stu-id="44206-113">For this reason, DPWS does not require services to use transport addresses.</span></span>

<span data-ttu-id="44206-114">Se si usano indirizzi logici, esistono alcuni scenari in cui il comportamento di implementazione non è definito.</span><span class="sxs-lookup"><span data-stu-id="44206-114">If logical addresses are used, then there are some scenarios where the implementation behavior is undefined.</span></span> <span data-ttu-id="44206-115">La specifica WS-Discovery non descrive il significato di un servizio che si trova in un indirizzo logico.</span><span class="sxs-lookup"><span data-stu-id="44206-115">The WS-Discovery specification does not describe what it means for a service to reside at a logical address.</span></span> <span data-ttu-id="44206-116">R1001 della specifica WS-Discovery consiglia di non usare WS-Discovery sui servizi ospitati a causa del chiacchiericcio di rete associato.</span><span class="sxs-lookup"><span data-stu-id="44206-116">R1001 of the WS-Discovery specification recommends against using WS-Discovery on hosted services because of the associated network chatter.</span></span>

<span data-ttu-id="44206-117">Non è consigliabile che i servizi si trovino in indirizzi logici in quanto questo riduce l'interoperabilità.</span><span class="sxs-lookup"><span data-stu-id="44206-117">It is not recommended that services reside at logical addresses as this reduces interoperability.</span></span> <span data-ttu-id="44206-118">Se un'implementazione deve trovarsi in un indirizzo logico, il servizio deve usare lo stesso indirizzo logico del dispositivo.</span><span class="sxs-lookup"><span data-stu-id="44206-118">If an implementation absolutely must reside at a logical address, then the service should use the same logical address as the device.</span></span> <span data-ttu-id="44206-119">Se questa operazione aggiunge una complessità eccessiva al modello di invio sul dispositivo, la soluzione consigliata consiste nell'usare i parametri di riferimento per distinguere i servizi.</span><span class="sxs-lookup"><span data-stu-id="44206-119">If this adds too much complexity to the dispatch model on the device, then the recommended solution is to use reference parameters to differentiate the services.</span></span> <span data-ttu-id="44206-120">WSDAPI invierà correttamente messaggi ai servizi se utilizzano lo stesso indirizzo endpoint del dispositivo.</span><span class="sxs-lookup"><span data-stu-id="44206-120">WSDAPI will correctly send messages to services if they use the same endpoint address as the device.</span></span>
