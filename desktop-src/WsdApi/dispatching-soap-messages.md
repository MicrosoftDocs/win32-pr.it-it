# <a name="dispatching-soap-messages"></a><span data-ttu-id="e9f4d-104">Invio di messaggi SOAP</span><span class="sxs-lookup"><span data-stu-id="e9f4d-104">Dispatching SOAP Messages</span></span>

## <a name="transport-level-dispatch"></a><span data-ttu-id="e9f4d-107">Dispatch a livello di trasporto</span><span class="sxs-lookup"><span data-stu-id="e9f4d-107">Transport level dispatch</span></span>

<span data-ttu-id="e9f4d-108">Con la distribuzione a livello di trasporto, il server HTTP sottostante (ad esempio l' [API HTTP](/windows/desktop/Http/http-api-start-page)) viene usato per gestire il routing delle richieste al dispositivo e ai relativi servizi.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-108">With transport level dispatch, the underlying HTTP server (such as the [HTTP API](/windows/desktop/Http/http-api-start-page)) is used to manage routing of requests to the device and its services.</span></span> <span data-ttu-id="e9f4d-109">Il server fornisce un URL diverso per ogni servizio e per il dispositivo e i sink diversi sono registrati per ogni URL.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-109">The server provides a different URL for each service and for the device and different sinks are registered for each URL.</span></span> <span data-ttu-id="e9f4d-110">In questo modo è possibile progettare il codice in modo che ogni servizio sia isolato dall'altro, in esecuzione come componenti distinti all'interno dello stesso processo o in esecuzione come processi distinti.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-110">This allows code to be designed such that each service is isolated from the other, either running as separate components within the same process or running as separate processes.</span></span>

<span data-ttu-id="e9f4d-111">La distribuzione a livello di trasporto presenta alcuni vantaggi.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-111">Transport level dispatch has a few advantages.</span></span> <span data-ttu-id="e9f4d-112">I messaggi possono essere inviati al componente appropriato senza prima analizzare la busta SOAP o il corpo del messaggio.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-112">Messages can be dispatched to the appropriate component without first parsing the SOAP envelope or message body.</span></span> <span data-ttu-id="e9f4d-113">Inoltre, è possibile riutilizzare il meccanismo esistente per il routing dei messaggi forniti dalla maggior parte delle implementazioni del server HTTP, il che significa che il codice di invio personalizzato non è necessario.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-113">Also, the existing mechanism for routing messages provided by most HTTP server implementations can be reused, which means custom dispatching code is unnecessary.</span></span> <span data-ttu-id="e9f4d-114">Isola inoltre il codice di elaborazione SOAP tra i servizi, che fornisce un livello di sicurezza, poiché i servizi sicuri evitano che i messaggi viaggino attraverso codice comune.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-114">It also isolates the SOAP processing code between services, which provides a level of security, as secure services avoid having messages travel through common code.</span></span>

## <a name="address-and-action-dispatch"></a><span data-ttu-id="e9f4d-115">Indirizzo e invio di azioni</span><span class="sxs-lookup"><span data-stu-id="e9f4d-115">Address and action dispatch</span></span>

<span data-ttu-id="e9f4d-118">Questo modello incoraggia il riutilizzo del codice in uno stack di messaggistica a più livelli, poiché tutto il codice fino al processore SOAP è condiviso da tutti i servizi.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-118">This model encourages code reuse throughout a layered messaging stack, as all code up to the SOAP processor is shared by all services.</span></span> <span data-ttu-id="e9f4d-119">Inoltre, non sono necessari indirizzi di trasporto distinti per i servizi, il che significa che gli indirizzi UUID possono essere utilizzati per gli endpoint di servizio.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-119">Also, distinct transport addresses for services are not required, which means that UUID addresses can be used for service endpoints.</span></span> <span data-ttu-id="e9f4d-120">La distribuzione degli indirizzi e delle azioni converte inoltre più direttamente in un modello di programmazione.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-120">Address and action dispatch also translates more directly to a programming model.</span></span> <span data-ttu-id="e9f4d-121">Gli sviluppatori possono collegare servizi e dispositivi a un singolo componente che gestisce il routing, anziché dover associare un livello HTTP o creare componenti distinti per ogni servizio.</span><span class="sxs-lookup"><span data-stu-id="e9f4d-121">Developers can plug services and devices into a single component which manages routing, rather than having to tie into a HTTP layer or creating separate components for each service.</span></span>
