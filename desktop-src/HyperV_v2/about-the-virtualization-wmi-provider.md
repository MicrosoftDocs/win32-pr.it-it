# <a name="about-the-hyper-v-wmi-provider"></a>Informazioni sul provider WMI Hyper-V

Il provider WMI per Hyper-V consente agli sviluppatori e agli scripter di creare rapidamente strumenti, utilità e miglioramenti personalizzati per la piattaforma di virtualizzazione. Le interfacce WMI possono gestire tutti gli aspetti dei servizi Hyper-V.

## <a name="about-microsoft-windows-server-2008-r2-hyper-v"></a>Informazioni su Microsoft Windows Server 2008 R2 Hyper-V

Microsoft Windows Server 2008 R2 Hyper-V consente agli amministratori di sistema di consolidare server hardware separati in un singolo server che esegue Microsoft Windows Server 2008 R2 come sistema operativo host.

Ognuna delle macchine virtuali ospitate viene eseguita nel proprio ambiente virtuale separato e isolato. Ciò consente all'amministratore di gestire, fornire e gestire facilmente un numero elevato di macchine virtuali, riducendo al tempo stesso la necessità di eseguire più soluzioni hardware per l'uso di vari prodotti e sistemi operativi.

Un altro vantaggio dell'ambiente di macchine virtuali è il test e il supporto. Le nuove configurazioni di server e di sistema possono essere distribuite e testate in parallelo con il sistema esistente senza richiedere tempi di inattività diss costosi durante la fase di implementazione. Ogni macchina virtuale può essere impostata in modo da usare configurazioni variabili durante l'esecuzione in una piattaforma standard per produrre risultati di test migliori. Con il supporto per sistemi operativi precedenti, è possibile supportare e testare hardware, sistemi e applicazioni legacy. Il supporto snapshot consente di creare uno snapshot dello stato di una macchina virtuale in modo da poter tornare a un evento precedente, se necessario.

Hyper-V espone un'interfaccia ricca che consente all'utente di monitorare e controllare l'ambiente della macchina virtuale. La maggior parte di Hyper-V può essere controllata tramite il provider WMI, che consente una personalizzazione semplice ma potente delle macchine virtuali.
