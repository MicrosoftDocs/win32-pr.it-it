# <a name="structured-storage-api-functions"></a>Funzioni API di archiviazione strutturata

Alcune funzioni API per l'archiviazione strutturata sono funzioni helper che eseguono una sequenza di chiamate ad altre funzioni API e metodi di interfaccia. È possibile utilizzare queste funzioni helper come scorciatoie.

Altre funzioni API consentono di accedere all'implementazione di COM di archiviazione strutturata, file composti.

Un altro set di funzioni API consente di convertire gli oggetti OLE 1 in un archivio strutturato. È possibile utilizzare queste funzioni per determinare se una classe di oggetti è da OLE 1 e per convertire gli oggetti tra OLE 1 e i formati di archiviazione COM correnti.

Infine, sono disponibili funzioni API per la conversione e l'emulazione di altri oggetti. Queste funzioni forniscono ai servizi che un'applicazione server funziona con i dati di un'altra applicazione. I dati possono essere convertiti nel formato nativo dell'applicazione server leggendo i dati dal formato originale, ma scrivendoli nel formato nativo dell'applicazione dell'oggetto. In alternativa, i dati possono rimanere nel formato originale mentre l'applicazione server legge e scrive i dati nel formato originale.

Per ulteriori informazioni, vedere la sezione seguente [modalità di accesso all'archiviazione strutturata](structured-storage-access-modes.md).
