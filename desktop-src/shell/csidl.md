# <a name="csidl"></a>CSIDL

> [!Note]  
> Al Windows Vista, questi valori sono stati sostituiti da [**valori KNOWNFOLDERID.**](knownfolderid.md) Per un elenco delle nuove costanti e dei valori CSIDL corrispondenti, vedere questo argomento. Per praticità, vengono anche specificati **i valori KNOWNFOLDERID** corrispondenti per ogni valore CSIDL.
>
> Il sistema CSIDL è supportato in Windows Vista per motivi di compatibilità. Tuttavia, i nuovi progetti di sviluppo [**devono usare i valori KNOWNFOLDERID**](knownfolderid.md) anziché i valori CSIDL.

 

I valori CSIDL (elenco di ID elemento speciale costante) forniscono un modo univoco indipendente dal sistema per identificare le cartelle speciali usate di frequente dalle applicazioni, ma che potrebbero non avere lo stesso nome o percorso in un sistema specifico. Ad esempio, la cartella di sistema può essere "C: Windows" in un sistema e \\ "C: \\ Winnt" in un altro. Queste costanti sono definite in Shlobj.h.

## <a name="remarks"></a>Commenti

Questi valori sorrendono l'uso di variabili di ambiente a questo scopo. Vengono a loro volta sostituiti in Windows Vista e versioni successive dai [**valori KNOWNFOLDERID.**](knownfolderid.md)

Un file CSIDL viene usato insieme a una delle quattro funzioni [**shell, SHGetFolderLocation**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetfolderlocation), [**SHGetFolderPath**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetfolderpatha), [**SHGetSpecialFolderLocation**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetspecialfolderlocation)e [**SHGetSpecialFolderPath**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetspecialfolderpatha), per recuperare il percorso o il puntatore di una cartella speciale a un elenco di identificatori di elemento (PIDL).

Combinare ****CSIDL FLAG CREATE**** con qualsiasi altro CSIDL, ad eccezione di \_ \_ ****CSIDL \_ FLAG \_ DONT VERIFY****, per forzare la creazione della cartella \_ associata. I CSID rimanenti corrispondono a file system cartelle virtuali. Dove CSIDL identifica una file system, come esempio viene fornito un percorso di uso comune. È possibile usare altri percorsi. È possibile eseguire il mapping di alcuni CSID A una variabile di ambiente *%VariableName%* equivalente. I CSIDL sono tuttavia più affidabili e devono essere usati, se possibile.
