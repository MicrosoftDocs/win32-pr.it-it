<span data-ttu-id="1cb6f-180">In rari casi, la coerenza non offre alcun vantaggio per l'utente e può anche peggiorare l'esperienza utente.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-180">In rare cases, consistency provides no benefit to the user and can even degrade the user experience.</span></span> <span data-ttu-id="1cb6f-181">Non rendere coerente l'interfaccia utente se la coerenza non è in grado di eseguire un'attività.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-181">Do not make the UI consistent if that consistency impairs the ability to accomplish a task.</span></span> <span data-ttu-id="1cb6f-182">La coerenza non garantisce l'usabilità.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-182">Consistency in itself does not guarantee usability.</span></span> <span data-ttu-id="1cb6f-183">Si tratta di un errore a pensare che la coerenza nell'interfaccia provocherà una progettazione corretta.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-183">It is a mistake to think that consistency in the interface will lead to good design.</span></span>

<span data-ttu-id="1cb6f-184">Ad esempio, l'interfaccia utente del gioco video è in genere molto specifica per il tipo di gioco.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-184">For example, video game UI is typically very specific to the kind of game.</span></span> <span data-ttu-id="1cb6f-185">Il tentativo di progettare un'interfaccia utente generica che funziona bene per due giochi specializzati, uno che richiede una rotellina di sterzata e un altro che funziona meglio con un joystick e pulsanti, probabilmente non riuscirà per entrambi i giochi.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-185">Trying to design a generic UI that works well for two specialized games, one that requires a steering wheel and another that works best with a joystick and buttons, is likely not going to be successful for either game.</span></span> <span data-ttu-id="1cb6f-186">È probabile che si ottenga una posizione intermedia che non è ottimale per nessuno dei due.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-186">At best a middle ground is likely to be achieved that is not good for either.</span></span>

<span data-ttu-id="1cb6f-187">La presenza di dati appropriati sulle modalità di utilizzo è la chiave per prendere questa decisione.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-187">Having good data about how things are used is the key to making this decision.</span></span> <span data-ttu-id="1cb6f-188">È possibile chiarire i vantaggi e gli svantaggi di ogni compromesso (velocità e affidabilità, facilità di apprendimento rispetto alla competenza degli esperti, coerenza globale rispetto all'ottimizzazione locale) e prendere le decisioni migliori per la funzionalità in relazione all'intero prodotto.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-188">Be clear about the pros and cons of each tradeoff (speed versus reliability, ease of learning versus expert proficiency, global consistency versus local optimization) and make the best decisions for the feature in relation to the whole product.</span></span>

<span data-ttu-id="1cb6f-189">La progettazione è la scelta della modalità di errore: l'ottimizzazione per un elemento comporta un errore in un altro.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-189">Design is choosing how to fail: optimizing for one thing means failing at another.</span></span> <span data-ttu-id="1cb6f-190">La chiave per una buona progettazione dell'interfaccia utente è la possibilità di decidere quali caratteristiche dell'applicazione sono le più importanti e quali possono essere tagliate.</span><span class="sxs-lookup"><span data-stu-id="1cb6f-190">The key to good UI design is being able to decide which characteristics of the application are the most important, and which ones can be cut.</span></span>
