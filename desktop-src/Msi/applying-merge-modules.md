# <a name="applying-merge-modules"></a>Applicazione di moduli unione

I moduli unione devono essere uniti in un pacchetto di installazione tramite uno strumento di unione. L'alternativa migliore consiste nell'ottenere uno strumento di merge distribuito liberamente o acquistare uno degli strumenti di unione disponibili da fornitori di software indipendenti. Ad esempio, è possibile usare la funzionalità fornita da Mergemod.dll.

I moduli unione in più lingue possono essere aperti in lingue diverse e quindi uniti in una destinazione per produrre una versione diversa del pacchetto. Per altre informazioni, vedere:

-   [Uso dell'automazione per unire un modulo di merge in un database](using-automation-to-merge-a-merge-module-into-a-database.md)
-   [Uso dell'API per unire un modulo di merge in un database](using-the-api-to-merge-a-merge-module-into-a-database.md)
-   [Apertura di un Multiple-Language merge in un linguaggio specifico](opening-a-multiple-language-merge-module-in-a-specific-language.md)
-   [Unione di un modulo per più linguaggi nello stesso pacchetto più volte](merging-a-multiple-language-module-into-the-same-package-multiple-times.md)
-   [Connessione di un modulo unione a più funzionalità](connecting-a-merge-module-to-multiple-features.md)
