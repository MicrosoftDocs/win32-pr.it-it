# <a name="using-isolated-components"></a>Utilizzo di componenti isolati

Utilizzando i [componenti isolati](isolated-components.md), gli autori dei pacchetti di installazione possono specificare che il programma di installazione copia i file condivisi di un'applicazione direttamente nella cartella dell'applicazione anziché in un percorso condiviso. Questo set di file privato viene quindi usato esclusivamente dall'applicazione.

Per ulteriori informazioni, vedere [installazione di un componente com in un percorso privato](installing-a-com-component-to-a-private-location.md), [installazione di un componente non com in un percorso privato](installing-a-non-com-component-to-a-private-location.md), [rendere privato un componente com in un pacchetto esistente](make-a-com-component-in-an-existing-package-private.md) o [creare un componente non com in un pacchetto esistente privato](make-a-non-com-component-in-an-existing-package-private.md).
