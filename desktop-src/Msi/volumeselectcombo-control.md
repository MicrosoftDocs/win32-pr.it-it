# <a name="volumeselectcombo-control"></a>Controllo VolumeSelectCombo

Il controllo VolumeSelectCombo consente all'utente di selezionare un volume da un elenco alfabetico di volumi. I tipi di volumi visualizzati nell'elenco vengono specificati usando i bit associati agli attributi di controllo [RemovableVolume](removablevolume-control-attribute.md), [FixedVolume](fixedvolume-control-attribute.md), [RemoteVolume](remotevolume-control-attribute.md), [CDROMVolume](cdromvolume-control-attribute.md), [RAMDiskVolume](ramdiskvolume-control-attribute.md) [e FloppyVolume](floppyvolume-control-attribute.md) .

È possibile associare questo controllo a una proprietà immettendo il nome della proprietà nella colonna Proprietà della [tabella Control](control-table.md).

## <a name="remarks"></a>Commenti

Questo controllo può essere creato dalla classe COMBOBOX usando la [**funzione CreateWindowEx.**](/windows/win32/api/winuser/nf-winuser-createwindowexa) Include gli stili **CBS \_ DROPDOWNLIST,** **CBS \_ OWNERDRAWFIXED,** **CBS \_ HASSTRINGS,** **WS \_ VSCROLL,** **WS \_ CHILD,** **WS \_ GROUP,** **WS \_ TABSTOP** e **CBS \_ SORT.** Per informazioni sullo sviluppo di un'interfaccia utente con Windows, [vedere Interfaccia utente Design and Development](/previous-versions/aa286531(v=msdn.10)).

Per garantire la compatibilità con le utilità per la lettura dello schermo, quando si crea una finestra di dialogo con un controllo VolumeSelectCombo come primo controllo attivo, è necessario impostare il campo di testo appartenente al campo di modifica come primo controllo attivo nella tabella Della finestra di [dialogo.](dialog-table.md) Poiché il testo statico non può assumere lo stato attivo, quando viene creata la finestra di dialogo, il campo di modifica avrà inizialmente lo stato attivo come previsto. Ciò garantisce che le utilità per la lettura dello schermo mostrino le informazioni corrette.
