# <a name="listview-control"></a>Controllo ListView

Controllo ListView che visualizza una singola colonna di valori con un'icona accanto a ogni elemento. Consente all'utente di selezionare una singola stringa o un valore intero per una proprietà da un elenco predeterminato. I valori possibili vengono letti dalla [tabella ListView](listview-table.md). È possibile associare il controllo a un valore intero o stringa immettendo il nome della proprietà nella colonna Proprietà della [tabella Control](control-table.md).

Il contenuto del campo Text della [tabella Control](control-table.md) non viene mai visualizzato dal controllo ListView. Questo campo specifica invece lo stile del testo che deve essere visualizzato dal controllo e contiene una descrizione del controllo utilizzato dalle utilità di revisione dello schermo. Per impostare il tipo di carattere e lo stile del carattere di una stringa di testo, antessare alla stringa di caratteri visualizzati { style} o \\ {&*style*}. Dove style è un identificatore elencato nella colonna TextStyle della [tabella TextStyle](textstyle-table.md). Se nessuna di queste proprietà è presente, ma la [**proprietà DefaultUIFont**](defaultuifont.md) è definita come stile di testo valido, verrà usato tale tipo di carattere. Le informazioni seguenti vengono lette dalle utilità di revisione dello schermo come descrizione del controllo. Vedere [Accessibilità](accessibility.md).

## <a name="remarks"></a>Commenti

Questo controllo può essere creato dalla classe LISTVIEW WC \_ usando la [**funzione CreateWindowEx.**](/windows/win32/api/winuser/nf-winuser-createwindowexa) Include gli stili **LVS \_ REPORT,** **LVS \_ NOCOLUMNHEADER,** **WS \_ VSCROLL,** **WS \_ HSCROLL,** **LVS \_ SHAREIMAGELISTS, LVS** **\_ SINGLESEL,** **LVS \_ SHOWSELALWAYS,** **WS \_ BORDER** e **WS \_ TABSTOP.** Se il bit di stile Sorted non è on, anche il controllo ha lo stile **LVS \_ SORTASCENDING.**

Per compatibilità con le utilità per la lettura dello schermo, quando si crea una finestra di dialogo con un controllo ListView come primo controllo attivo, è necessario impostare il campo di testo appartenente al campo di modifica come primo controllo attivo nella tabella [Dialog.](dialog-table.md) Poiché il testo statico non può assumere lo stato attivo, quando viene creata la finestra di dialogo, il campo di modifica avrà inizialmente lo stato attivo come previsto, ma questa operazione garantisce che le utilità per la lettura dello schermo mostrino le informazioni corrette.
