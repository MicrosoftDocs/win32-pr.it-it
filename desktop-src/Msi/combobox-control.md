# <a name="combobox-control"></a>Controllo ComboBox

Il controllo ComboBox visualizza un elenco a discesa di valori predefiniti e un campo di modifica in cui l'utente può immettere un valore. Per associare questo controllo a una proprietà stringa o integer, immettere il nome della proprietà nella colonna Proprietà della [tabella Control](control-table.md).

## <a name="control-attributes"></a>Attributi del controllo

Per modificare il valore di un attributo usando un evento , sottoscrivere il controllo a un oggetto ControlEvent nella tabella [EventMapping](eventmapping-table.md) ed elencare l'identificatore dell'attributo nella colonna Attribute . Immettere l'identificatore di ControlEvent nella colonna Event .

## <a name="remarks"></a>Commenti

Questo controllo può essere creato dalla classe COMBOBOX usando la [**funzione CreateWindowEx.**](/windows/win32/api/winuser/nf-winuser-createwindowexa) Include gli stili **CBS \_ AUTOHSCROLL,** **WS \_ TABSTOP,** **WS \_ GROUP** e **WS \_ CHILD.** Se il bit ComboList è on, ha anche lo stile **\_ CBS DROPDOWNLIST,** in caso contrario ha lo **stile CBS \_ DROPDOWN.**

La lunghezza del testo che può essere immesso può essere limitata inserendo un numero da 0 a 2147483646 tra parentesi graffe all'inizio del campo Text nella tabella [Control](control-table.md). Ad esempio, se il campo di testo inizia con {80} , la lunghezza della stringa è limitata a 80 caratteri. Se tale limite non viene specificato nella tabella o se viene specificato 0, la lunghezza viene impostata sul massimo possibile (2147483646 caratteri). Un valore negativo o non numerico genererà un errore.

Per compatibilità con le utilità per la lettura dello schermo, quando si crea una finestra di dialogo con un controllo ComboBox come primo controllo attivo, è necessario impostare il campo di testo appartenente al campo di modifica come primo controllo attivo nella tabella [Dialog](dialog-table.md). Poiché il testo statico non può assumere lo stato attivo, quando viene creata la finestra di dialogo, il campo di modifica avrà inizialmente lo stato attivo come previsto. In questo modo si garantisce che le utilità per la lettura dello schermo mostrino le informazioni corrette.
