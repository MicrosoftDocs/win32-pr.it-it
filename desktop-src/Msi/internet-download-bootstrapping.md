# <a name="internet-download-bootstrapping"></a>Bootstrap di download Internet

Con Windows Installer, un file eseguibile bootstrap configurabile (Setup.exe) e uno strumento di configurazione ([Msistuff.exe](msistuff-exe.md)) sono inclusi in [Windows SDK Components for Windows Installer Developers](platform-sdk-components-for-windows-installer-developers.md). Usando Msistuff.exe per configurare le risorse in Setup.exe, gli sviluppatori possono creare facilmente un'installazione Web di un pacchetto Windows Installer.

La versione minima del programma di installazione richiesta dal file eseguibile bootstrap Windows installer versione 2.0. Le applicazioni possono ottenere la versione del programma di installazione usando [**DllGetVersion**](/windows/win32/api/shlwapi/nc-shlwapi-dllgetversionproc). Per altre informazioni, vedere [Determinare la Windows del programma di installazione](determining-the-windows-installer-version.md)

Il file eseguibile bootstrap fornito con Windows Installer SDK esegue le operazioni seguenti:

-   Chiama [**WinVerifyTrust**](/windows/win32/api/wintrust/nf-wintrust-winverifytrust) per verificare la firma digitale del file .msi file. Windows Il programma di installazione versione 2.0 e versioni successive fornisce un pacchetto DIS (Subject Interface Package) per abilitare la firma Windows pacchetti del programma di installazione.
-   Se necessario, aggiorna la versione del Windows installer nel computer.

Le risorse di Setup.exe possono essere visualizzate o configurate usando [Msistuff.exe](msistuff-exe.md).

È possibile usare solo una di esse alla volta.

| **/a** | setup.exe **/a**                                                           | Avvia un'installazione amministrativa. | **/a** | setup.exe **/a<** percorso completo di un'installazione *amministrativa esistente*> | Se la proprietà PATCH ISETUPPROPNAME è configurata in Setup.exe, questa opzione può essere usata per applicare patch a un'installazione amministrativa \_ esistente del prodotto.
