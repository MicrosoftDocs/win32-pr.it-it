# <a name="installation-context"></a>Contesto di installazione

Windows Installer possibile installare un pacchetto in un computer in due contesti di installazione: per computer e per utente. Per consentire a tutti gli utenti del computer di accedere e usare l'applicazione, è necessaria un'installazione per computer del pacchetto. Poiché un'installazione per computer apporta modifiche al sistema che interessano tutti gli utenti, è possibile che agli utenti standard con privilegi limitati venga impedito di installare un pacchetto nel contesto per computer senza prima ottenere l'autorizzazione.

È possibile specificare il contesto di installazione creando il pacchetto per l'installazione per utente o per computer e utilizzando le proprietà [**ALLUSERS**](allusers.md) e [**MSIINSTALLPERUSER**](msiinstallperuser.md) . In base a queste proprietà, Windows Installer reindirizza automaticamente i valori delle proprietà e delle registrazioni delle cartelle ai percorsi per il contesto per utente o per computer.

> [!Note]  
> La proprietà [**MSIINSTALLPERUSER**](msiinstallperuser.md) , disponibile a partire da Windows Installer 5,0 e windows Server 2008 R2 e Windows 7, può facilitare lo sviluppo di un singolo pacchetto in grado di essere installato nel contesto per computer o per utente. Per informazioni sullo sviluppo di un pacchetto a doppio scopo che conferisce all'utente la possibilità di scegliere il contesto di installazione al momento dell'installazione, vedere [creazione di singoli pacchetti](single-package-authoring.md). Windows Installer ignora la proprietà **MSIINSTALLPERUSER** se il valore di [**ALLUSERS**](allusers.md) non è 2. Windows Installer Reimposta sempre il valore di **ALLUSERS** su 1 quando viene installato nel contesto per computer e reimposta il valore di **ALLUSERS** su una stringa vuota ("") quando viene installato nel contesto per utente.
