# <a name="installation-context"></a>Contesto di installazione

Windows Il programma di installazione può installare un pacchetto in un computer in due contesti di installazione: per computer e per utente. È necessaria un'installazione per computer del pacchetto per consentire a tutti gli utenti del computer di accedere e usare l'applicazione. Poiché un'installazione per computer apporta modifiche al sistema che interessano tutti gli utenti, agli utenti standard con privilegi limitati potrebbe essere impedito di installare un pacchetto nel contesto per computer senza prima ottenere l'autorizzazione.

È possibile specificare il contesto di installazione mediante la creazione del pacchetto per l'installazione per utente o per computer e usando le proprietà [**ALLUSERS**](allusers.md) e [**MSIINSTALLPERUSER.**](msiinstallperuser.md) In base a queste proprietà, Windows installer reindirizza automaticamente i valori delle proprietà della cartella e delle registrazioni ai percorsi per il contesto per utente o per computer.

> [!Note]  
> La proprietà [**MSIINSTALLPERUSER,**](msiinstallperuser.md) disponibile a partire da Windows Installer 5.0 e Windows Server 2008 R2 e Windows 7, può semplificare lo sviluppo di un singolo pacchetto in grado di essere installato nel contesto per computer o per utente. Per informazioni sullo sviluppo di un pacchetto per doppio scopo che offre all'utente la possibilità di scegliere il contesto di installazione in fase di installazione, vedere [Creazione di singoli pacchetti.](single-package-authoring.md) Windows Il programma di installazione **ignora la proprietà MSIINSTALLPERUSER** se il valore di [**ALLUSERS**](allusers.md) non è 2. Windows Il programma di installazione reimposta sempre il valore di **ALLUSERS** su 1 quando viene installato nel contesto per computer e reimposta il valore di **ALLUSERS** su una stringa vuota ("") quando viene installato nel contesto per utente.
