# <a name="list-view-control-msaa-ui-element-reference"></a>List-View (riferimento all'elemento DELL'interfaccia utente MSAA)

> [!Note]  
> Questo argomento descrive gli oggetti **controllo List-View** ai fini delle informazioni di riferimento sugli elementi dell'interfaccia utente MSAA. La procedura per creare **oggetti controllo List-View** in diversi framework dell'interfaccia utente non è descritta qui. Vedere la documentazione di riferimento dell'API per il framework dell'interfaccia utente in uso.

 

Un controllo visualizzazione elenco visualizza una raccolta di elementi, ognuno costituito da un'icona e un'etichetta, e offre diversi modi per visualizzare e disporre gli elementi. Ad esempio, le informazioni aggiuntive su ogni elemento vengono visualizzate nelle colonne a destra dell'icona e dell'etichetta. Windows Explorer usa un controllo visualizzazione elenco per visualizzare i file.

Il nome della classe della finestra per un controllo visualizzazione elenco è WC \_ LISTVIEW, definito come "SysListView" in Commctrl.h.

## <a name="notes"></a>Note

-   Microsoft Active Accessibility non espone il testo dell'elemento di visualizzazione elenco nelle applicazioni che usano una funzione di callback per aggiungere il testo all'elemento.
-   Per gli elementi della visualizzazione elenco ritagliati non è impostato [**STATE \_ SYSTEM \_ INVISIBLE.**](object-state-constants.md) Per determinare se un elemento dell'elenco viene ritagliato, i client possono confrontare le coordinate dell'elemento restituite da [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) con l'area di ritaglio della finestra.
-   Microsoft Active Accessibility non supporta l'uso del metodo [**accNavigate**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) per passare al controllo intestazione all'interno di un controllo visualizzazione elenco nella visualizzazione Dettagli. Tuttavia, i client usano [**la funzione AccessibleObjectFromPoint**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfrompoint) per ottenere il controllo intestazione.
-   Per gli oggetti non rettangolari, ad esempio gli elementi di visualizzazione elenco nella visualizzazione Dettagli, le coordinate del rettangolo di delimitazione dell'elemento recuperato da [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) hanno esito negativo se testate con [**accHitTest**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acchittest) perché **accHitTest** determina i limiti dell'oggetto pixel per pixel.
