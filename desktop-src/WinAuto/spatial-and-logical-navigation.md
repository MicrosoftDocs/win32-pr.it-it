## <a name="establishing-the-order-in-logical-navigation"></a><span data-ttu-id="46961-125">Definizione dell'ordine nell'esplorazione logica</span><span class="sxs-lookup"><span data-stu-id="46961-125">Establishing the Order in Logical Navigation</span></span>

<span data-ttu-id="46961-126">Nella navigazione logica gli sviluppatori che progettano gli oggetti stabiliscono le relazioni tra di essi.</span><span class="sxs-lookup"><span data-stu-id="46961-126">In logical navigation, the developers who design the objects establish the relationships between them.</span></span> <span data-ttu-id="46961-127">La navigazione logica è più soggettiva rispetto alla navigazione spaziale.</span><span class="sxs-lookup"><span data-stu-id="46961-127">Logical navigation is more subjective than spatial navigation.</span></span> <span data-ttu-id="46961-128">Inoltre, l'ordine nell'esplorazione logica non corrisponde all'ordine utilizzato con gli ID figlio.</span><span class="sxs-lookup"><span data-stu-id="46961-128">Also, the order in logical navigation is not the same as the order used with child IDs.</span></span>

<span data-ttu-id="46961-129">Per gli oggetti che dispongono di posizioni dello schermo, gli sviluppatori del server devono stabilire l'ordine di navigazione nel modo in cui la maggior parte degli utenti considera logica.</span><span class="sxs-lookup"><span data-stu-id="46961-129">For objects that have screen locations, server developers should establish the navigation order in the way that most users would consider logical.</span></span> <span data-ttu-id="46961-130">Nei paesi/aree di lingua inglese, ad esempio, ciò significa un ordine da sinistra a destra, dall'alto verso il basso.</span><span class="sxs-lookup"><span data-stu-id="46961-130">In English-speaking countries/regions, for instance, this means a left-to-right, top-to-bottom ordering.</span></span>

<span data-ttu-id="46961-131">L'ordine di navigazione logico deve essere l'ordine di navigazione della tastiera parallelo.</span><span class="sxs-lookup"><span data-stu-id="46961-131">Logical navigation order must parallel keyboard navigation order.</span></span> <span data-ttu-id="46961-132">Ad esempio, una finestra di dialogo contiene i pulsanti **OK** e **Annulla** push e alcuni controlli di modifica.</span><span class="sxs-lookup"><span data-stu-id="46961-132">For example, a dialog box contains **OK** and **Cancel** push buttons and a few edit controls.</span></span> <span data-ttu-id="46961-133">Un client che chiama [**IAccessible:: accNavigate**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-accnavigate) per passare all'oggetto successivo o precedente nella finestra di dialogo viene spostato nello stesso ordine in cui si preme TAB o MAIUSC + TAB per spostare lo stato attivo tra gli elementi.</span><span class="sxs-lookup"><span data-stu-id="46961-133">A client that calls [**IAccessible::accNavigate**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-accnavigate) to navigate to the next or previous object in that dialog box moves in the same order as a user pressing TAB or SHIFT+TAB to move the focus between items.</span></span>

<span data-ttu-id="46961-134">Per gli oggetti che non hanno definito posizioni dello schermo, l'ordine logico viene deciso dagli sviluppatori del server e gli sviluppatori client non devono fare supposizioni.</span><span class="sxs-lookup"><span data-stu-id="46961-134">For objects that do not have defined screen locations, the logical order is decided by server developers, and client developers should make no assumptions about it.</span></span> <span data-ttu-id="46961-135">Ad esempio, è accettabile per gli oggetti non visibili, ad esempio gli oggetti che sono solo temporaneamente nascosti, da sparpagliare con gli oggetti visibili.</span><span class="sxs-lookup"><span data-stu-id="46961-135">For instance, it is acceptable for non-visible objects, such as objects that are only temporarily hidden, to be interspersed with visible objects.</span></span>
