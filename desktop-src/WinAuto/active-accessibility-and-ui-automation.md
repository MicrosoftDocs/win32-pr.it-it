# <a name="active-accessibility-and-ui-automation"></a>Active Accessibility e Automazione interfaccia utente

Microsoft Active Accessibility è progettato per l'uso con Windows XP e sistemi operativi precedenti. Gli sviluppatori di controlli personalizzati e applicazioni client tecnologiche accessibili per Windows XP e Windows Vista devono prendere in considerazione l'uso di Microsoft [Automazione interfaccia utente.](entry-uiauto-win32.md)

Microsoft Automazione interfaccia utente è un sistema completo che fornisce informazioni più precise sull'interfaccia utente e offre all'utente maggiore possibilità di interagire con i controlli. In particolare, fornisce un supporto notevolmente migliorato per il testo.

Un set di oggetti proxy fornisce Automazione interfaccia utente per i controlli Microsoft Win32 e Windows Form standard. È disponibile un supporto più ricco per i controlli scritti in modo specifico Automazione interfaccia utente, inclusi gli elementi Windows Presentation Foundation (WPF) nativi, che non supportano direttamente Microsoft Active Accessibility. I controlli personalizzati che supportano Automazione interfaccia utente possono essere scritti in codice gestito o non gestito.

Microsoft Active Accessibility client hanno accesso limitato, tramite un livello di bridging, agli elementi dell'interfaccia utente che supportano solo Automazione interfaccia utente. Per altre informazioni, vedere [Appendice G: Active Accessibility Bridge to Automazione interfaccia utente](appendix-g--active-accessibility-bridge-to-ui-automation.md).

Per altre informazioni sulle analogie e le differenze tra Microsoft Active Accessibility e Automazione interfaccia utente, vedere Microsoft Active Accessibility [e Automazione interfaccia utente Compared](microsoft-active-accessibility-and-ui-automation-compared.md).
