# <a name="object-state-constants"></a>Costanti di stato dell'oggetto

Questo argomento descrive i valori costanti usati per descrivere lo stato degli oggetti in un'interfaccia utente dell'applicazione. Le costanti di stato sono definite in oleacc. h.

Un oggetto è associato a uno o più valori di stato in qualsiasi momento. Non vengono utilizzate le costanti di stato dell'oggetto seguenti: \_ avviso sistema di stato \_ \_ alto \_ , \_ medio avviso sistema stato \_ , \_ avviso sistema stato \_ \_ basso e sistema di stato a \_ \_ virgola mobile.

I client recuperano lo stato di un oggetto chiamando [**IAccessible:: Get \_ accState**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-get_accstate), che restituisce un numero intero che è una combinazione dei flag di bit seguenti. I client chiamano [**GetStateText**](/windows/desktop/api/Oleacc/nf-oleacc-getstatetexta) con il valore di stato per recuperare una stringa localizzata che descrive lo stato dell'oggetto.

Quando lo stato di un oggetto viene modificato, i server devono chiamare [**NotifyWinEvent**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) con la costante evento [**\_ \_ STATECHANGE dell'oggetto evento**](event-constants.md) . Tuttavia, gli oggetti con il sistema di stato \_ \_ Invisible, lo stato attivo del sistema di stato e le costanti di \_ stato dell' \_ oggetto selezionato del sistema di stato \_ \_ hanno le proprie costanti di evento. Per questi oggetti, non usare l' **\_ oggetto evento \_ STATECHANGE**. Usare invece la costante di evento individuale.
