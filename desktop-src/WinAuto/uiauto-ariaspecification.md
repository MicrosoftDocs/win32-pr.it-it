# <a name="ui-automation-for-w3c-accessible-rich-internet-applications-specification"></a>Automazione interfaccia utente per la specifica di applicazioni Internet avanzate con W3C accessibile

I siti Web aumentano l'utilità con contenuto dinamico e controlli dell'interfaccia utente avanzati supportati da tecnologie quali Asynchronous JavaScript and XML (AJAX), DHTML e JavaScript. Tuttavia, le tecnologie per l'accessibilità non possono interagire spesso con questi controlli complessi o esporre contenuti dinamici agli utenti. Internet Applications (ARIA) accessibile è una specifica tecnica W3C per lo sviluppo di contenuto Web e applicazioni in modo che siano accessibili a utenti con particolari esigenze.

Per supportare la specifica ARIA, la specifica di automazione interfaccia utente Microsoft consente agli sviluppatori di associare gli attributi AriaRole e AriaProperties di automazione interfaccia utente con i ruoli e gli Stati di W3C ARIA e le proprietà. Questo consente alle applicazioni utente come Microsoft Internet Explorer di supportare il modello a oggetti ARIA nel contesto di automazione interfaccia utente mantenendo un modello a oggetti di accessibilità di base.

Questa appendice fornisce il mapping suggerito delle informazioni W3C a Microsoft Active Accessibility e alle specifiche di automazione interfaccia utente, sfruttando le due nuove proprietà di automazione interfaccia utente.

In questo argomento sono incluse le sezioni seguenti:

-   [Ruolo di W3C ARIA mappato a Microsoft Active Accessibility e automazione interfaccia utente](#w3c-aria-role-mapped-to-microsoft-active-accessibility-and-ui-automation)
-   [Stati e proprietà del W3C ARIA mappati a Microsoft Active Accessibility e automazione interfaccia utente](#w3c-aria-states-and-properties-mapped-to-microsoft-active-accessibility-and-ui-automation)
-   [Considerazioni aggiuntive](#additional-considerations)

## <a name="w3c-aria-role-mapped-to-microsoft-active-accessibility-and-ui-automation"></a>Ruolo di W3C ARIA mappato a Microsoft Active Accessibility e automazione interfaccia utente

I ruoli di W3C ARIA possono essere mappati a Microsoft Active Accessibility ruoli o ai tipi di controllo di automazione interfaccia utente. Con l'automazione dell'interfaccia utente, il ruolo originale W3C ARIA può essere supportato anche dalla proprietà AriaRole. Gli agenti utente possono offrire una descrizione localizzata del ruolo W3C ARIA con la proprietà LocalizedControlType di automazione interfaccia utente. Tuttavia, questa opzione è facoltativa. Se non si specifica una stringa localizzata, il sistema fornirà la stringa LocalizedControlType predefinita. Gli agenti utente possono anche offrire ruoli secondari nella proprietà AriaRole usando lo spazio come separatore, come definito nello standard ARIA W3C.

## <a name="w3c-aria-states-and-properties-mapped-to-microsoft-active-accessibility-and-ui-automation"></a>Stati e proprietà del W3C ARIA mappati a Microsoft Active Accessibility e automazione interfaccia utente

Gli Stati e le proprietà del W3C ARIA sono mappati a diverse funzioni e proprietà di automazione interfaccia utente di Microsoft Active Accessibility. La maggior parte degli Stati e delle proprietà di ARIA è supportata dalla proprietà AriaProperties di automazione interfaccia utente. Le eccezioni sono proprietà ARIA che accettano riferimenti a oggetti (ad esempio, la proprietà DescribedBy ARIA) e le proprietà ARIA supportate dal modello a oggetti di accessibilità. Ad esempio, la proprietà ARIA activedescendent dovrebbe essere rappresentata come elemento con lo stato attivo nel modello a oggetti di accessibilità. Quando viene specificata, la proprietà ARIA possiede anche il riflesso nel modello a oggetti di accessibilità.

AriaProperties è una proprietà di stringa con una raccolta di coppie nome/valore i cui delimitatori sono il segno di uguale (=) e il punto e virgola (;), ad esempio, "Checked = true; disabled = false". Una barra rovesciata ( \) viene utilizzata come escape quando i caratteri delimitatori, o una barra rovesciata, vengono visualizzati nei valori. Il codice del provider può aggiungere la verifica per convalidare le proprietà ARIA originali nell'Document Object Model, ma non è obbligatorio.

## <a name="additional-considerations"></a>Ulteriori considerazioni

Automazione interfaccia utente offre anche un modello a oggetti di testo semplice con il pattern di controllo [Text](uiauto-implementingtextandtextrange.md) , che supporta l'incorporamento di oggetti in un oggetto documento. Ciò consente agli agenti utente e alle applicazioni client di trattare il contenuto Web come documento HTML o come interfaccia utente desktop tradizionale, a seconda degli scenari dell'utente finale.
