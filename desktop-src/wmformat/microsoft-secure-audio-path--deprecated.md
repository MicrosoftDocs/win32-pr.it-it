# <a name="microsoft-secure-audio-path-deprecated"></a>Microsoft Secure Audio Path (deprecato)

Questa pagina documenta una funzionalità che verrà supportata con una soluzione tecnica diversa nelle versioni future di Windows.

I creatori di contenuti e i distributori possono specificare in una licenza DRM che un file audio può essere riprodotto solo in un sistema con componenti Microsoft Secure Audio Path (SAP). Il percorso audio sicuro offre un livello di protezione più elevato al contenuto audio rendendo praticamente impossibile per le applicazioni non attendibili o i driver audio accedere ai bit audio non crittografati. Il percorso audio sicuro è supportato in Microsoft Windows® me e Windows XP. Protegge la musica digitale nel kernel del sistema operativo. Inoltre, il Digital Millennium Copyright Act consente di aggirare le misure antipirateria nel software.

Il percorso audio sicuro viene attivato e implementato completamente automaticamente dal componente Microsoft DRM quando una licenza DRM richiede SAP. Per le applicazioni in esecuzione su Windows® XP Service Pack 1, è possibile abilitare la crittografia SAP in qualsiasi contenuto audio, al di fuori del contesto della soluzione DRM Microsoft, usando Secure Audio Path SDK. Per ulteriori informazioni su Secure Audio Path SDK, visitare il [sito Web Microsoft](/documentation/).
