# <a name="microsoft-secure-audio-path-deprecated"></a>Percorso audio sicuro Microsoft (deprecato)

Questa pagina documenta una funzionalità che verrà supportata con una soluzione tecnica diversa nelle versioni future di Windows.

I creatori di contenuto e i distributori possono specificare in una licenza DRM che un file audio può essere riprodotto solo in un sistema con componenti SAP (Microsoft Secure Audio Path). Percorso audio sicuro offre un livello di protezione molto più elevato per il contenuto audio rendendo praticamente impossibile per le applicazioni non attendibili o i driver audio accedere ai bit audio non crittografati. Percorso audio sicuro è supportato in Microsoft Windows® Me e Windows XP. Protegge la musica digitale nel kernel del sistema operativo. Inoltre, il Digital Millennium Copyright Act rende un reato aggirare le misure antipiracy nel software.

Percorso audio sicuro viene attivato e implementato completamente automaticamente dal componente Microsoft DRM quando una licenza DRM richiede SAP. Per le applicazioni in esecuzione Windows® XP Service Pack 1, è possibile abilitare la crittografia SAP per qualsiasi contenuto audio, all'esterno del contesto della soluzione Microsoft DRM, usando Secure Audio Path SDK. Per altre informazioni su Secure Audio Path SDK, vedere il [sito Web Microsoft](/documentation/).
