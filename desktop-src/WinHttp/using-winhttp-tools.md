# <a name="using-winhttp-tools"></a>Uso degli strumenti WinHTTP

Per i comandi che è possibile usare in Windows Vista e versioni successive per configurare le impostazioni proxy e di traccia per Windows HTTP, vedere [Netsh Commands for Windows Hypertext Transfer Protocol (WINHTTP) (Comandi Netsh per Windows Hypertext Transfer Protocol (WINHTTP)](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc731131(v=ws.10))).

**Windows Server 2003 e versioni precedenti:** Sono disponibili diversi strumenti che consentono di scrivere ed eseguire il debug delle applicazioni. Sono illustrati negli argomenti seguenti:

-   [WinHttpCfg.exe,](winhttpcertcfg-exe--a-certificate-configuration-tool.md) uno strumento di configurazione dei certificati consente agli [](glossary.md) amministratori di installare e configurare i certificati client in qualsiasi archivio certificati accessibile dall'account IWAM (Web Application Manager) di Internet Server. Lo strumento elimina anche la necessità di eseguire qualsiasi operazione speciale per gli account, ad esempio l'account IWAM, per ottenere l'accesso ai certificati quando si usa asp (Active Server Pages).
-   [Netsh.exe o ProxyCfg.exe proxy Strumenti di configurazione](proxycfg-exe--a-proxy-configuration-tool.md) possibile usare per configurare i server proxy con Microsoft Windows HTTP Services (WinHTTP).
-   [WinHttpTraceCfg, uno strumento di configurazione della traccia,](winhttptracecfg-exe--a-trace-configuration-tool.md) consente di monitorare le funzioni WinHTTP e il traffico di rete corrispondente. Il debug di applicazioni che utilizzano protocolli di rete crittografati, ad esempio Secure Sockets Layer (SSL), preclude l'analisi dei pacchetti a livello di trasporto di rete e richiede la possibilità di intercettare il traffico tra il client e il server dopo che è stato decrittografato. La funzionalità di traccia WinHTTP include una gamma di funzionalità per intercettare il flusso di traffico tra il client e il server.
