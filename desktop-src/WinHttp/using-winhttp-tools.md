# <a name="using-winhttp-tools"></a>Uso degli strumenti WinHTTP

Per i comandi che è possibile utilizzare in Windows Vista e versioni successive per configurare le impostazioni del proxy e della traccia per il protocollo HTTP di Windows, vedere [comandi Netsh per windows Hypertext Transfer Protocol (WinHTTP)](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc731131(v=ws.10)).

**Windows Server 2003 e versioni precedenti:** Sono disponibili diversi strumenti che consentono di scrivere ed eseguire il debug delle applicazioni. Sono illustrati negli argomenti seguenti:

-   [WinHttpCfg.exe, uno strumento di configurazione dei certificati](winhttpcertcfg-exe--a-certificate-configuration-tool.md) consente agli amministratori di installare e configurare i certificati client in qualsiasi [*archivio certificati*](glossary.md) a cui è possibile accedere tramite l'account IWAM (Internet server Web Application Manager). Lo strumento Elimina inoltre la necessità di eseguire operazioni speciali per gli account, ad esempio l'account IWAM, per ottenere l'accesso ai certificati quando si utilizza Active Server Pages (ASP).
-   [Netsh.exe o ProxyCfg.exe proxy strumenti di configurazione](proxycfg-exe--a-proxy-configuration-tool.md) può essere utilizzato per configurare i server proxy con i servizi HTTP di Microsoft Windows (WinHTTP).
-   [Winhttptracecfg, uno strumento di configurazione della traccia](winhttptracecfg-exe--a-trace-configuration-tool.md) consente di monitorare le funzioni WinHTTP e il traffico di rete corrispondente. Il debug di applicazioni che utilizzano protocolli di trasmissione crittografati, ad esempio Secure Sockets Layer (SSL), impedisce l'analisi dei pacchetti a livello di trasporto di rete e richiede la possibilità di intercettare il traffico tra il client e il server dopo che è stato decrittografato. La funzionalità di traccia WinHTTP dispone di una gamma di funzionalità per intercettare il flusso di traffico tra il client e il server.
