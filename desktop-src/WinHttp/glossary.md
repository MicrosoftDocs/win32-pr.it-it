# <a name="glossary-winhttp"></a>Glossario (WinHTTP)

<dl> <dt>

<span id="term_authentication_data"></span><span id="TERM_AUTHENTICATION_DATA"></span>**dati di autenticazione**
</dt> <dd>

Blocco di dati specifico dello schema scambiato tra il server e il client durante l'autenticazione. Per dimostrare la propria identità, il client crittografa alcuni o tutti i dati con un nome utente e una password. Il client invia i dati crittografati al server, che li decrittografa e li confronta con quelli originali. Se i dati decrittografati corrispondono ai dati originali, il client viene autenticato.

</dd> <dt>

<span id="term_base64_encoding"></span><span id="TERM_BASE64_ENCODING"></span>**codifica Base64**
</dt> <dd>

Schema utilizzato per la trasmissione di dati binari. Base64 elabora i dati come gruppi a 24 bit, eseguendo il mapping di questi dati a quattro caratteri codificati. Viene talvolta definito codifica da 3 a 4. Ogni 6 bit del gruppo a 24 bit viene utilizzato come indice in una tabella di mapping (alfabeto Base64) per ottenere un carattere per i dati codificati. I dati codificati hanno lunghezze di riga limitate a 76 caratteri.

</dd> <dt>

<span id="term_certificate_store"></span><span id="TERM_CERTIFICATE_STORE"></span>**Archivio certificati**
</dt> <dd>

In genere, archiviazione permanente in cui vengono archiviati i certificati, gli elenchi di revoche di certificati (CRL) e gli elenchi di certificati attendibili (CTL). Un archivio certificati può essere anche temporaneo quando si utilizzano certificati basati sulla sessione.

</dd> <dt>

<span id="term_cobranding"></span><span id="TERM_COBRANDING"></span>**cobranding**
</dt> <dd>

Possibilità per un sito di Microsoft Passport partecipante di fornire i propri elementi di personalizzazione e spiegazioni nelle pagine del servizio di rete Passport. Il cobranding include la personalizzazione dell'aspetto, del testo specifico e del comportamento specifico nelle pagine della rete Passport.

</dd> <dt>

<span id="term_code_page"></span><span id="TERM_CODE_PAGE"></span>**tabella codici**
</dt> <dd>

Tabella che mette in correlazione i codici carattere binario utilizzati da un programma ai tasti della tastiera o all'aspetto dei caratteri sul monitor. L'utilizzo di tabelle codici è un modo per fornire supporto per i set di caratteri e i layout di tastiera utilizzati in paesi e aree diverse. È possibile configurare dispositivi quali il monitoraggio e la tastiera per l'utilizzo di una tabella codici specifica e per passare da una tabella codici (ad esempio Stati Uniti) a un'altra (ad esempio, Portogallo) in base alla richiesta dell'utente.

</dd> <dt>

<span id="term_http_verb"></span><span id="TERM_HTTP_VERB"></span>**Verbo HTTP**
</dt> <dd>

Il verbo HTTP (o il metodo HTTP) è un'istruzione inviata in un messaggio di richiesta che notifica a un server HTTP l'azione da eseguire sulla risorsa specificata. Ad esempio, "GET" specifica che una risorsa viene recuperata dal server. I verbi comuni includono "GET", "POST" e "HEAD". Per ulteriori informazioni e per un elenco completo dei verbi HTTP standard, vedere la specifica HTTP/1.1.

</dd> <dt>

<span id="term_ticket"></span><span id="TERM_TICKET"></span>**ticket**
</dt> <dd>

Cookie che contiene un profilo utente per l'autenticazione Passport. Ogni ticket corrisponde a un solo URL. Il server di accesso di un'autorità di dominio Passport fornisce i ticket inviati dal client a un membro Passport per l'autenticazione.

</dd> <dt>

<span id="term_user_agent"></span><span id="TERM_USER_AGENT"></span>**agente utente**
</dt> <dd>

L'agente utente è l'applicazione client che richiede un documento da un server HTTP. Quando il client invia una richiesta a un server HTTP, in genere invia il nome dell'agente utente con l'intestazione della richiesta, in modo che il server possa determinare le funzionalità del software client.

</dd> </dl>
