# <a name="peer-networking-scenarios"></a>Scenari di rete peer

L'infrastruttura peer-to-peer offre molte opzioni per lo sviluppo di applicazioni di rete peer-to-peer. In questo argomento vengono fornite informazioni sugli scenari di rete peer-to-peer seguenti:

-   Applicazioni di collaborazione
-   Messaggistica immediata o applicazioni di chat
-   Applicazioni di distribuzione del contenuto

## <a name="collaborative-applications"></a>Applicazioni di collaborazione

È possibile utilizzare l'infrastruttura peer-to-peer per creare applicazioni collaborative per consentire agli utenti di condividere in modo interattivo le idee tra loro, indipendentemente dal fatto che si trovino nella stessa sala riunioni o in Internet. È ad esempio possibile creare un'applicazione di tipo lavagna condivisa.

> [!Note]  
> L'infrastruttura peer-to-peer non supporta lo streaming audio o video.

 

## <a name="instant-messaging-or-chat-applications"></a>Messaggistica immediata o applicazioni di chat

Per l'utilizzo dell'infrastruttura peer-to-peer è possibile aggiornare le applicazioni di messaggistica immediata e di chat che utilizzano server centrali. Ad esempio, un'applicazione di chat può utilizzare l'infrastruttura peer-to-peer come alternativa a un server centralizzato. L'infrastruttura peer-to-peer migliora la scalabilità e consente agli utenti di partecipare alle attività di chat e messaggistica senza un server.

## <a name="content-distribution-applications"></a>Applicazioni di distribuzione del contenuto

È possibile utilizzare l'infrastruttura peer-to-peer come meccanismo di distribuzione per il contenuto. Ad esempio, è possibile distribuire una newsletter ai peer che sottoscrivono un servizio. Poiché i dati nell'infrastruttura peer-to-peer sono sincronizzati, i sottoscrittori ricevono la newsletter anche quando il server di pubblicazione non è online.
