# <a name="non-default-backup-and-restore-locations"></a>Percorsi di backup e ripristino non predefiniti

Le operazioni di backup e ripristino in genere copiano i file da un determinato percorso predefinito su disco al supporto di backup e quindi ripristinano da tale supporto nello stesso percorso predefinito su disco.

Esistono molti motivi, in particolare quando si gestiscono processi in esecuzione, per non seguire questo modello semplice. VSS supporta l'uso di origini non standard per il backup e destinazioni alternative per il ripristino e include meccanismi per lavorare con subset di file e per modificare il mapping dei file.

-   [Uso di percorsi alternativi durante il backup](working-with-alternate-paths-during-backup.md)
-   [Uso di percorsi alternativi durante il ripristino](working-with-alternate-locations-during-restore.md)
-   [Uso di nuove destinazioni durante il ripristino](working-with-new-targets-during-restore.md)
-   [Uso di file parziali](working-with-partial-files.md)
-   [Uso di destinazioni indirizzate](working-with-directed-targets.md)
