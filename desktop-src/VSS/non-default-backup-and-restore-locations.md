# <a name="non-default-backup-and-restore-locations"></a>Percorsi di backup e ripristino non predefiniti

Le operazioni di backup e ripristino in genere copiano i file da un determinato percorso predefinito sul disco a un supporto di backup e quindi il ripristino da tale supporto nello stesso percorso predefinito sul disco.

Esistono diversi motivi, in particolare quando si gestiscono i processi in esecuzione, senza seguire questo semplice modello. VSS supporta l'utilizzo di origini non standard per il backup e destinazioni alternative per il ripristino e include meccanismi che consentono di utilizzare subset di file e di modificare il mapping dei file.

-   [Utilizzo di percorsi alternativi durante il backup](working-with-alternate-paths-during-backup.md)
-   [Utilizzo di percorsi alternativi durante il ripristino](working-with-alternate-locations-during-restore.md)
-   [Utilizzo di nuove destinazioni durante il ripristino](working-with-new-targets-during-restore.md)
-   [Utilizzo di file parziali](working-with-partial-files.md)
-   [Utilizzo di destinazioni dirette](working-with-directed-targets.md)
