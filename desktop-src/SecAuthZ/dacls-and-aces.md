# <a name="dacls-and-aces"></a><span data-ttu-id="236d0-103">Elenchi DACL e ACE</span><span class="sxs-lookup"><span data-stu-id="236d0-103">DACLs and ACEs</span></span>

<span data-ttu-id="236d0-104">Se un oggetto Windows non dispone di un [*elenco di controllo di accesso discrezionale*](/windows/desktop/SecGloss/d-gly) (DACL), il sistema consente a tutti gli utenti di accedervi.</span><span class="sxs-lookup"><span data-stu-id="236d0-104">If a Windows object does not have a [*discretionary access control list*](/windows/desktop/SecGloss/d-gly) (DACL), the system allows everyone full access to it.</span></span> <span data-ttu-id="236d0-105">Se un oggetto dispone di un DACL, il sistema consente solo l'accesso consentito in modo esplicito dalle [*voci di controllo di accesso*](/windows/desktop/SecGloss/a-gly) (ACE) nell'elenco DACL.</span><span class="sxs-lookup"><span data-stu-id="236d0-105">If an object has a DACL, the system allows only the access that is explicitly allowed by the [*access control entries*](/windows/desktop/SecGloss/a-gly) (ACEs) in the DACL.</span></span> <span data-ttu-id="236d0-106">Se non sono presenti voci ACE nell'elenco DACL, il sistema non consente l'accesso a nessuno.</span><span class="sxs-lookup"><span data-stu-id="236d0-106">If there are no ACEs in the DACL, the system does not allow access to anyone.</span></span> <span data-ttu-id="236d0-107">Analogamente, se un elenco DACL include ACE che consentono l'accesso a un set limitato di utenti o gruppi, il sistema nega in modo implicito l'accesso a tutti i trustee non inclusi nelle voci ACE.</span><span class="sxs-lookup"><span data-stu-id="236d0-107">Similarly, if a DACL has ACEs that allow access to a limited set of users or groups, the system implicitly denies access to all trustees not included in the ACEs.</span></span>

<span data-ttu-id="236d0-108">Nella maggior parte dei casi, è possibile controllare l'accesso a un oggetto usando le voci di accesso consentito. non è necessario negare esplicitamente l'accesso a un oggetto.</span><span class="sxs-lookup"><span data-stu-id="236d0-108">In most cases, you can control access to an object by using access-allowed ACEs; you do not need to explicitly deny access to an object.</span></span> <span data-ttu-id="236d0-109">L'eccezione si verifica quando una voce ACE consente l'accesso a un gruppo e si vuole negare l'accesso a un membro del gruppo.</span><span class="sxs-lookup"><span data-stu-id="236d0-109">The exception is when an ACE allows access to a group and you want to deny access to a member of the group.</span></span> <span data-ttu-id="236d0-110">A tale scopo, inserire una voce ACE di accesso negato per l'utente nell'elenco DACL in anticipo rispetto all'ACE consentito per l'accesso per il gruppo.</span><span class="sxs-lookup"><span data-stu-id="236d0-110">To do this, place an access-denied ACE for the user in the DACL ahead of the access-allowed ACE for the group.</span></span> <span data-ttu-id="236d0-111">Si noti che l' [ordine delle voci ACE](order-of-aces-in-a-dacl.md) è importante perché il sistema legge le voci ACE in sequenza fino a quando l'accesso non viene concesso o negato.</span><span class="sxs-lookup"><span data-stu-id="236d0-111">Note that the [order of the ACEs](order-of-aces-in-a-dacl.md) is important because the system reads the ACEs in sequence until access is granted or denied.</span></span> <span data-ttu-id="236d0-112">La voce ACE di accesso negato dell'utente deve essere visualizzata per prima. in caso contrario, quando il sistema legge l'ACE consentito per l'accesso al gruppo, consentirà l'accesso all'utente con restrizioni.</span><span class="sxs-lookup"><span data-stu-id="236d0-112">The user's access-denied ACE must appear first; otherwise, when the system reads the group's access allowed ACE, it will grant access to the restricted user.</span></span>
