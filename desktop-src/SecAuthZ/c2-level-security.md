# <a name="c2-level-security"></a>Sicurezza a livello C2

L'elenco seguente include alcuni dei requisiti più importanti per la sicurezza a livello C2, definiti dal Ministero della difesa degli Stati Uniti:

-   È necessario che sia possibile controllare l'accesso a una risorsa concedendo o negando l'accesso a singoli utenti o a gruppi di utenti denominati.
-   La memoria deve essere protetta, in modo che il relativo contenuto non possa essere letto dopo che è stato liberato da un [*processo*](/windows/desktop/SecGloss/p-gly) . Analogamente, è necessario che un file system protetto, ad esempio NTFS, protegga i file eliminati dalla lettura.
-   Gli utenti devono identificarsi in modo univoco, ad esempio tramite password, al momento dell'accesso. Tutte le azioni controllabili devono identificare l'utente che esegue l'azione.
-   Gli amministratori di sistema devono essere in grado di controllare gli eventi correlati alla sicurezza. Tuttavia, l'accesso ai dati di controllo degli eventi correlati alla sicurezza deve essere limitato agli amministratori autorizzati.
-   Il sistema deve essere protetto da interferenze esterne o manomissioni, ad esempio la modifica del sistema in esecuzione o dei file di sistema archiviati su disco.
