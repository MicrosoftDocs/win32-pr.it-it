<span data-ttu-id="adcce-104">Come tutti i [*processi*](/windows/desktop/SecGloss/p-gly), un server protetto dispone di un [*token di accesso primario*](/windows/desktop/SecGloss/p-gly) che ne descrive il [*contesto di sicurezza*](/windows/desktop/SecGloss/s-gly).</span><span class="sxs-lookup"><span data-stu-id="adcce-104">Like all [*processes*](/windows/desktop/SecGloss/p-gly), a protected server has a [*primary access token*](/windows/desktop/SecGloss/p-gly) that describes its [*security context*](/windows/desktop/SecGloss/s-gly).</span></span> <span data-ttu-id="adcce-105">Quando un client si connette a un server protetto, il server potrebbe voler eseguire azioni utilizzando il contesto di sicurezza del client anziché il contesto di sicurezza del server.</span><span class="sxs-lookup"><span data-stu-id="adcce-105">When a client connects to a protected server, the server may want to perform actions using the client's security context instead of the server's security context.</span></span> <span data-ttu-id="adcce-106">Ad esempio, quando un client in una conversazione DDE (Dynamic Data Exchange) richiede informazioni da un server DDE, il server deve verificare che il client sia autorizzato ad accedere alle informazioni.</span><span class="sxs-lookup"><span data-stu-id="adcce-106">For example, when a client in a dynamic data exchange (DDE) conversation requests information from a DDE server, the server needs to verify that the client is allowed access to the information.</span></span>

<span data-ttu-id="adcce-107">Un server può agire nel contesto di sicurezza del client in due modi:</span><span class="sxs-lookup"><span data-stu-id="adcce-107">There are two ways that a server can act in the client's security context:</span></span>

-   <span data-ttu-id="adcce-108">Un thread del processo server può rappresentare il client.</span><span class="sxs-lookup"><span data-stu-id="adcce-108">A thread of the server process can impersonate the client.</span></span> <span data-ttu-id="adcce-109">In questo caso, il thread del server dispone di un token di [*accesso*](/windows/desktop/SecGloss/a-gly) di rappresentazione che identifica il client, i gruppi del client e i [*privilegi*](/windows/desktop/SecGloss/p-gly)del client.</span><span class="sxs-lookup"><span data-stu-id="adcce-109">In this case, the server's thread has an impersonation [*access token*](/windows/desktop/SecGloss/a-gly) that identifies the client, the client's groups, and the client's [*privileges*](/windows/desktop/SecGloss/p-gly).</span></span> <span data-ttu-id="adcce-110">Per ulteriori informazioni, vedere [rappresentazione client](client-impersonation.md).</span><span class="sxs-lookup"><span data-stu-id="adcce-110">For more information, see [Client Impersonation](client-impersonation.md).</span></span>
-   <span data-ttu-id="adcce-111">Il server può ottenere le [*credenziali*](/windows/desktop/SecGloss/c-gly) del client e registrare il client nel computer del server.</span><span class="sxs-lookup"><span data-stu-id="adcce-111">The server can get the client's [*credentials*](/windows/desktop/SecGloss/c-gly) and log the client on to the server's computer.</span></span> <span data-ttu-id="adcce-112">Viene creata una nuova [*sessione*](/windows/desktop/SecGloss/s-gly) di accesso e viene generato un token di accesso primario per il client.</span><span class="sxs-lookup"><span data-stu-id="adcce-112">This creates a new logon [*session*](/windows/desktop/SecGloss/s-gly) and produces a primary access token for the client.</span></span> <span data-ttu-id="adcce-113">Il server può quindi usare il token di accesso del client per rappresentare il client o per avviare un nuovo processo eseguito nel contesto di sicurezza del client.</span><span class="sxs-lookup"><span data-stu-id="adcce-113">The server can then use the client's access token to impersonate the client or to start a new process that runs in the security context of the client.</span></span> <span data-ttu-id="adcce-114">Per ulteriori informazioni, vedere [sessioni di accesso client](client-logon-sessions.md).</span><span class="sxs-lookup"><span data-stu-id="adcce-114">For more information, see [Client Logon Sessions](client-logon-sessions.md).</span></span>

<span data-ttu-id="adcce-115">Nella maggior parte dei casi, è sufficiente rappresentare il client.</span><span class="sxs-lookup"><span data-stu-id="adcce-115">In most cases, impersonating the client is sufficient.</span></span> <span data-ttu-id="adcce-116">La rappresentazione consente al server di controllare l'accesso del client agli oggetti a protezione diretta, controllare i privilegi del client e generare audit trail voci che identificano il client.</span><span class="sxs-lookup"><span data-stu-id="adcce-116">Impersonation enables the server to check the client's access to securable objects, check the client's privileges, and generate audit trail entries that identify the client.</span></span> <span data-ttu-id="adcce-117">In genere, un server deve avviare una sessione di accesso client solo se è necessario usare il contesto di sicurezza del client per accedere alle risorse di rete.</span><span class="sxs-lookup"><span data-stu-id="adcce-117">Typically, a server needs to start a client logon session only if it needs to use the client's security context to access network resources.</span></span>
