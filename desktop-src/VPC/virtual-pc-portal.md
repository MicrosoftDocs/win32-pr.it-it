# <a name="windows-virtual-pc"></a>Windows Virtual PC

\[Windows Virtual PC non è più disponibile per l'uso a partire da Windows 8. Usare invece il [provider WMI Hyper-V (v2)](../hyperv_v2/windows-virtualization-portal.md).\]

## <a name="purpose"></a>Scopo

In questa documentazione vengono fornite informazioni sull'interfaccia COM di Windows Virtual PC.

## <a name="where-applicable"></a>Se applicabile

Windows Virtual PC è la più recente tecnologia di virtualizzazione Microsoft; consente di eseguire molte applicazioni di produttività in un ambiente Windows virtuale, con un solo clic, direttamente da Windows 7. È possibile creare macchine virtuali separate nella parte superiore del desktop di Windows 7. Ogni macchina virtuale emula un sistema hardware completo, dal processore alla scheda di rete, in un ambiente software autosufficiente e isolato, consentendo il funzionamento simultaneo di sistemi altrimenti incompatibili.

## <a name="developer-audience"></a>Sviluppatori

Le interfacce COM per PC virtuali Windows sono destinate agli sviluppatori che creano applicazioni client in grado di automatizzare la distribuzione e il funzionamento delle macchine virtuali.

## <a name="run-time-requirements"></a>Requisiti di runtime

Windows Virtual PC richiede una delle edizioni di Windows 7 seguenti: Home Basic, Home Premium, Professional, Ultimate o Enterprise Edition.

Per ottenere i file di intestazione e di libreria correlati, installare il Windows SDK per Windows 7 dall' [area download Microsoft](https://www.microsoft.com/download/details.aspx?id=8279).
