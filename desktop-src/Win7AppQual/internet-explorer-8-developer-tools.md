# <a name="internet-explorer-8-developer-tools"></a>Strumenti di sviluppo di Internet Explorer 8

Windows Internet Explorer 8 include la funzionalità Strumenti di sviluppo funzionalità. Questa funzionalità consente di eseguire rapidamente il debug di Microsoft JScript, analizzare un comportamento specifico Windows Internet Explorer o scorrere rapidamente per prototipare o testare una nuova soluzione di progettazione.

Le sezioni seguenti descrivono le caratteristiche importanti del Strumenti di sviluppo in Internet Explorer 8.

## <a name="integrated-and-simple-to-use"></a>Integrato e semplice da usare

I Strumenti di sviluppo sono inclusi in ogni installazione di Internet Explorer 8, quindi è possibile eseguire il debug dei problemi in qualsiasi computer che esegue Internet Explorer 8. Inoltre, gli strumenti vengono caricati solo quando sono necessari per limitare l'impatto sulle prestazioni del browser. Usando il Strumenti di sviluppo, è possibile eseguire rapidamente il debug solo del processo Internet Explorer corrente, anziché eseguire il debug per tutte le Internet Explorer.

## <a name="provide-a-visual-interface-to-the-platform"></a>Fornire un'interfaccia visiva alla piattaforma

Anziché reverse engineering il funzionamento del sito Web o la modifica del sito per l'output delle informazioni di debug, il Strumenti di sviluppo consente di esaminare Internet Explorer per visualizzarne la rappresentazione del sito. Questa visualizzazione riduce il tempo necessario per il debug di siti dinamici, in cui l'ispezione dell'origine non è utile, o l'analisi di un comportamento specifico di Internet Explorer, in cui uno strumento di creazione generico non può essere utile.

## <a name="enable-fast-experimentation"></a>Abilitare la sperimentazione rapida

Quando si crea una nuova progettazione o si testano correzioni nelle versioni precedenti di Internet Explorer, è probabile che si modifica l'origine, la si salva, si aggiorna la pagina nel browser e si ripete. Il Strumenti di sviluppo semplificare questo scenario consentendo di modificare il sito all'interno del browser e visualizzare immediatamente le modifiche.

## <a name="optimize-application-performance"></a>Ottimizzare le prestazioni dell'applicazione

Per identificare e risolvere i problemi di prestazioni, è probabile che l'iterazione si concentri su uno scenario alla volta. Usando il profiler di script nel Strumenti di sviluppo, è possibile raccogliere statistiche, tra cui il tempo di esecuzione e il numero di volte in cui viene chiamata una funzione JavaScript. È possibile usare queste informazioni durante il test dell'applicazione e il report del profilo per identificare e correggere rapidamente i colli di bottiglia delle prestazioni.

Per accedere al Strumenti di sviluppo, premere F12 mentre si visualizza una pagina **Web o** fare clic Strumenti di sviluppo **dal** menu Strumenti.
