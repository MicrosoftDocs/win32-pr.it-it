# <a name="fiddler-web-debugger-tool"></a>Strumento di debugger Web Fiddler

Lo strumento Fiddler consente di eseguire il debug di applicazioni Web acquisendo il traffico di rete tra Internet e i computer di test. Lo strumento consente di esaminare i dati in ingresso e in uscita per monitorare e modificare richieste e risposte prima che il browser le riceva. Fiddler include anche un potente sottosistema di scripting basato su eventi, che è possibile estendere usando qualsiasi .NET Framework linguaggio.

Fiddler e le opzioni di riproduzione HTTP consentono di risolvere i problemi sul lato client con le applicazioni Web effettuando una copia offline del sito di test. Con questi strumenti è possibile creare immagini offline dell'esperienza di esplorazione e quindi creare un pacchetto e analizzare i risultati per ottenere informazioni di debug più dettagliate.

Per scaricare il componente aggiuntivo Fiddler, passare alla [Internet Explorer componenti aggiuntivi.](https://www.microsoft.com/iegallery) Per altre informazioni su come risolvere i problemi usando Fiddler e gli strumenti correlati, vedere il post di blog How [to use fiddler and HTTP replay to have an offline copy of your site](https://blogs.msdn.com/b/ricardo/archive/2010/06/22/how-to-user-fiddler-and-http-replay-to-have-an-offline-copy-of-your-site.aspx) blog (Come usare fiddler e riproduzione HTTP per avere una copia offline del sito) e il post di blog Troubleshooting Authentication with Fiddler (Risoluzione dei problemi di autenticazione con [Fiddler).](https://blogs.msdn.com/b/ieinternals/archive/2009/11/22/clipboard-paste-bug-in-windows-7-password-prompt.aspx)
