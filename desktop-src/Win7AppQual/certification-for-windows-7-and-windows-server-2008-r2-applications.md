# <a name="windows-7-client-software-logo-program"></a>Windows 7 Client Software Logo Program

## <a name="affected-platforms"></a>Piattaforme interessate

 **Client** - Windows 7  


## <a name="description-client-application-certification"></a>Descrizione: Certificazione dell'applicazione client

Nell'Windows di tempo 7, si vuole continuare a costruire sulla riuscita di Windows ed espandere il programma Logo in modo che sia più pertinente per i clienti. In base al feedback dei partner e dei clienti, il programma logo Windows 7 si concentrerà sul garantire che dispositivi, sistemi e applicazioni siano compatibili, affidabili e possano essere eserciti per soddisfare gli standard Windows standard. Il programma proposto per Windows 7 avrà un solo logo per sistemi, dispositivi e applicazioni. Questo argomento si concentra in particolare sulla compatibilità del software client.

È stato migliorato il processo per qualificare le applicazioni software per il logo "Compatibile con Windows 7". Microsoft non richiederà più agli sviluppatori di testare le applicazioni per la certificazione da parte di autorità di test autorizzate. Microsoft sta invece sviluppando un toolkit di logo che è possibile usare per garantire che l'applicazione sia compatibile con Windows 7 e soddisfi i requisiti del programma Windows 7 Client Software Logo Program. Il numero di requisiti è stato notevolmente ridotto. Analogamente al programma hardware, sono stati creati alcuni criteri. anche se non sono collegati a test case specifici, questi criteri verranno applicati. I partner devono soddisfare tutti i requisiti e osservare tutti i criteri in modo che le applicazioni software siano idonee per il logo "Compatibile con Windows 7".

## <a name="process"></a>Processo

Quando le applicazioni software sono pronte per il test, passare alla pagina Microsoft Connessione Windows 7 Software Logo Program e seguire le istruzioni:

1.  Iniziare in anticipo Esaminare i requisiti e identificare eventuali modifiche al codice necessarie per soddisfarli.
2.  Usare lo strumento di auto-test alfa per la verifica Scaricare e installare la versione più recente del toolkit. Si notino l'architettura a 64 bit e i requisiti del sistema operativo nel file readme.txt file. Eseguire l'applicazione tramite il toolkit come illustrato nella panoramica per verificare la conformità dell'applicazione in base ai requisiti.
3.  Apportare eventuali modifiche al codice (se necessario) ed eseguire di nuovo il test Se sono necessarie modifiche all'applicazione per soddisfare i requisiti, apportare le modifiche necessarie e ripetere il test con il toolkit per verificare la conformità.
4.  Acconsentire esplicitamente a ricevere notifiche tramite posta elettronica Microsoft inizierà ad accettare gli invii di logo con il rilascio del toolkit beta in scadenza alla fine di maggio. È consigliabile acconsentire esplicitamente alla ricezione di notifiche tramite posta elettronica quando il toolkit viene rilasciato e quando il programma viene aperto per gli invii.

    La procedura seguente può essere eseguita solo dopo il rilascio del toolkit beta:

5.  Scaricare il toolkit beta, testare e inviare il logo una volta che il toolkit beta è disponibile, scaricare, installare e testare l'applicazione. Inviare i risultati che passano a Microsoft.
6.  Firmare il Contratto di licenza del logo software Dopo aver inviato i risultati, firmare il [contratto di licenza](/windows-hardware/drivers/dashboard/winqual-submission-tool--winqualexe-). È quindi possibile scaricare e iniziare a usare la grafica logo e sarà idoneo a ricevere tutti i vantaggi del logo disponibili.
