# <a name="windows-troubleshooting"></a>Windows Risoluzione dei problemi relativi

## <a name="affected-platforms"></a>Piattaforme interessate

**Client** - Windows 7  
**Server** - Windows Server 2008 R2  









## <a name="description"></a>Descrizione

Una nuova funzionalità Windows 7 Action Center (precedentemente nota come Centro soluzioni), Windows Troubleshooting, offre un'esperienza di risoluzione dei problemi sistematica per l'utente. Il Centro notifiche è una delle cinque icone aggiunte nel systray. Windows Risoluzione dei problemi consente di cercare o cercare i pacchetti di risoluzione dei problemi incorporati e i pacchetti di risoluzione dei problemi archiviati in un server Microsoft in Internet, un'esperienza BWC (Better When Connected). È possibile selezionare ed eseguire un pacchetto di risoluzione dei problemi per tentare di risolvere il problema. Se non è possibile identificare una risoluzione del problema, è possibile cercare contenuto della Guida, contenuto della community e articoli di supporto o altri pacchetti di risoluzione dei problemi per il contenuto correlato pertinente. Se non si risponde al problema, è possibile ripristinare il PC a un'ora precedente a quando si è verificato il problema o ottenere assistenza tramite assistenza remota. Lo scopo è consentire di trovare una soluzione al problema in modo semplice e rapido.

## <a name="usage"></a>Utilizzo

Il Centro operativo è chiaramente visibile e disponibile da diverse posizioni. È possibile avviarlo dal menu di scelta rapida del Centro notifiche nel systray, dal pannello di controllo come collegamento rapido in Sistema e manutenzione, dalla pagina principale del Centro notifiche e dal contenuto della Guida.

I Pacchetti di risoluzione dei problemi sono basati su script di PowerShell. Il processo di creazione e pubblicazione di un pacchetto di risoluzione dei problemi verrà reso pubblico per consentire a OEM, IHV, ISV e professionisti IT di sviluppare e elaborare contenuti personalizzati per la risoluzione dei problemi.

Per un'esperienza utente coerente, assicurarsi di seguire le procedure consigliate e le linee guida descritte in Windows troubleshooting toolkit quando si progettano pacchetti di risoluzione dei problemi personalizzati.
