# <a name="packaging-and-deployment-tools-and-powershell-cmdlets"></a>Creazione di pacchetti e strumenti di distribuzione e cmdlet di PowerShell

Usare questi strumenti e cmdlet per creare, distribuire e inventariare i pacchetti dell'applicazione per le app di Windows.

## <a name="app-packaging-tools-and-guidance"></a>Strumenti e linee guida per la creazione di pacchetti di app

Informazioni su come creare un pacchetto di un'app, creare un certificato di prova e firmare l'app usando il prompt dei comandi e PowerShell. Per altre info, vedi:

-   [Creare un pacchetto dell'app con lo strumento MakeAppx.exe](/windows/msix/package/create-app-package-with-makeappx-tool)
-   [Creare un certificato per la firma dei pacchetti](/windows/msix/package/create-certificate-package-signing)
-   [Firmare il pacchetto di un'app tramite SignTool](/windows/msix/package/sign-app-package-using-signtool)

In alternativa, è possibile usare Visual Studio per creare e firmare un pacchetto dell'app. Per altre informazioni, vedere creare [un pacchetto di un'app UWP con Visual Studio](/windows/msix/package/packaging-uwp-apps).

## <a name="powershell-cmdlets"></a>Cmdlet PowerShell

Usare Windows PowerShell per installare, inventariare e disinstallare app di Windows LOB (line-of-business) che non sono firmate dall'archivio. Per altre informazioni, vedere [cmdlet del modulo appx](/powershell/module/appx/index?view=win10-ps).
