# <a name="creating-web-based-presentations"></a>Creazione di presentazioni Web-Based

Il controllo Windows Media Player consente di creare con facilità presentazioni di diapositive basate sul Web che combinano audio e video con HTML. Aggiungendo i comandi script di tipo URL ai file multimediali, è possibile fare in modo che le pagine Web specificate vengano visualizzate in un frame di Web browser specificato in momenti specifici durante la riproduzione di file multimediali digitali.

Windows Media Player include anche flussi multimediali avanzati per consentire una distribuzione efficiente dei dati HTML in una rete all'interno di un singolo flusso o file di Windows Media. Il lettore e il server gestiscono il recapito uniforme e tempestivo di audio, video e HTML al browser. Analogamente alle presentazioni Web che non utilizzano flussi multimediali avanzati, i comandi di script incorporati eseguono il rendering della presentazione in un frame del browser specificato a intervalli specificati.

Un layout tipico per le presentazioni basate sul Web usa due frame. Un frame contiene una pagina Web che incorpora il controllo Media Player di Windows e visualizza la parte video di una presentazione. L'altro frame Visualizza le pagine Web che cambiano in diversi momenti durante la riproduzione del video.

Se il supporto digitale che accompagna le pagine Web contiene solo audio, è possibile nascondere il frame che contiene il controllo Media Player di Windows impostando la larghezza su zero. In questo modo, l'audio può suonare senza interruzioni in background mentre le pagine Web vengono visualizzate nella finestra del browser completo.

Le sezioni seguenti descrivono le tecniche comuni per l'abilitazione di presentazioni basate sul Web:

-   [Capovolgimento URL](url-flipping.md)
-   [Streaming multimediale avanzato](rich-media-streaming.md)
