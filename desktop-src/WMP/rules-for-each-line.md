# <a name="rules-for-each-line"></a>Regole per ogni riga

Ogni definizione nel file di definizione dell'interfaccia deve trovarsi su una riga separata. È necessario immettere il testo nelle righe usando le regole seguenti:

-   Ogni riga deve terminare con un restituzione di paragrafo. Alcune righe possono essere più lunghe della lunghezza della riga visibile dell'editor di testo, ma non interrompere le righe. Potrebbe essere necessario disattivare il ritorno a capo riga per assicurarsi che le righe non si interrompano.
-   Usare solo i caratteri visibili come definito nel riferimento all'interfaccia.
-   Una riga è costituita da uno o più elementi, separati da spazi.
-   È possibile immettere il numero di spazi tra gli elementi desiderati per migliorare la leggibilità.
-   Non usare caratteri di tabulazione in qualsiasi punto del file.
-   Le righe che iniziano con due barre consecutive (//) vengono ignorate. Usare questi per i commenti. Anche le righe vuote vengono ignorate.
-   Includere spazi tra le parentesi quadre e il nome della sezione.
