# <a name="using-asynchronous-rpc-with-dce-pipes"></a>Uso di RPC asincrone con pipe DCE

Microsoft RPC supporta l'uso di pipe DCE. Sebbene condividono un nome simile, le pipe DCE non sono correlate alle [named pipe](asynchronous-rpc-over-the-named-pipe-protocol.md). Le named pipe sono un protocollo di trasporto. Le pipe DCE sono un metodo indipendente dal protocollo della comunicazione client/server.

Questa sezione presenta una discussione sulla RPC che usa le pipe DCE asincrone. È divisa negli argomenti seguenti:

-   [Pipe asincrone](asynchronous-pipes.md)
-   [Dichiarazione di pipe asincrone](declaring-asynchronous-pipes.md)
-   [Gestione della pipe asincrona sul lato client](client-side-asynchronous-pipe-handling.md)
-   [Gestione della pipe asincrona sul lato server](server-side-asynchronous-pipe-handling.md)

Per un'applicazione RPC pipe asincrona di esempio, vedere l'esempio FileRep nel Software Development Kit (SDK) della piattaforma.
