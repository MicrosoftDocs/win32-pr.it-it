# <a name="s-rpc"></a>S (RPC)

<dl> <dt>

<span id="rpc.s_glos_1_gly"></span><span id="RPC.S_GLOS_1_GLY"></span>**Scalabilità**
</dt> <dd>

Possibilità di utilizzare completamente la potenza di elaborazione aggiuntiva in un sistema multiprocessore (2, 4, 8, 32 o più processori). La scalabilità è anche la possibilità di gestire un numero elevato di client.

</dd> <dt>

<span id="_rpcl_spp_glos"></span><span id="_RPCL_SPP_GLOS"></span>**Sequenced Packet Protocol (SPP)**
</dt> <dd>

Protocollo di comunicazione orientato [*alla*](c-glos.md) connessione di Banyan Vines per il routing dei pacchetti di informazioni sulle reti locali.

</dd> <dt>

<span id="_rpc_spx_glos"></span><span id="_RPC_SPX_GLOS"></span>**Sequenced Packet Exchange (SPX)**
</dt> <dd>

Novell NetWare [*connection-oriented*](c-glos.md) communication protocol for routing information packets over local area networks and wide area networks (Protocollo di comunicazione orientato alla connessione NetWare per il routing di pacchetti di informazioni su reti locali e reti WIDE).

</dd> <dt>

<span id="_rpc_serialization_glos"></span><span id="_RPC_SERIALIZATION_GLOS"></span>**Serializzazione**
</dt> <dd>

Processo di marshalling dei dati verso (codifica) e annullamento del marshalling dei dati dai buffer (decodifica) che si controllano. Questo comportamento è in contrasto con l'utilizzo tradizionale di RPC, in cui gli stub e il runtime RPC controllano i buffer di marshalling. Chiamato anche pickling. Vedere anche [*serializzazione delle procedure,*](p-glos.md) [*serializzazione dei tipi.*](t-glos.md)

</dd> <dt>

<span id="_rpc_server_stub_glos"></span><span id="_RPC_SERVER_STUB_GLOS"></span>**server stub**
</dt> <dd>

Codice sorgente in linguaggio C generato da MIDL che contiene tutte le funzioni necessarie all'applicazione server per gestire le richieste remote tramite chiamate di routine locali. Vedere anche [*stub client.*](c-glos.md)

</dd> <dt>

<span id="_rpc_session_glos"></span><span id="_RPC_SESSION_GLOS"></span>**Sessione**
</dt> <dd>

È stata stabilita una relazione tra un'applicazione client e un'applicazione server. Vedere anche [*associare*](b-glos.md), [*handle di associazione*](b-glos.md).

</dd> <dt>

<span id="_rpc_static_callback_function_glos"></span><span id="_RPC_STATIC_CALLBACK_FUNCTION_GLOS"></span>**funzione di callback statica**
</dt> <dd>

Procedura remota che fa parte del lato client di un'applicazione distribuita, che un server può chiamare per ottenere informazioni dal client. \[ [L'attributo di callback](/windows/desktop/Midl/callback) definisce una funzione di callback \] statica.

</dd> <dt>

<span id="_rpc_static_identity_tracking_glos"></span><span id="_RPC_STATIC_IDENTITY_TRACKING_GLOS"></span>**rilevamento delle identità statiche**
</dt> <dd>

Il rilevamento delle identità statiche specifica che il run-time RPC usa le credenziali di sicurezza nell'handle di associazione del client per tutte le chiamate RPC. Vedere anche [*Rilevamento dinamico delle identità.*](d-glos.md)

</dd> <dt>

<span id="_rpc_string_binding_glos"></span><span id="_RPC_STRING_BINDING_GLOS"></span>**associazione di stringhe**
</dt> <dd>

Stringa di caratteri costituita [*dall'UUID*](u-glos.md)dell'oggetto [*,*](p-glos.md)dalla sequenza di [*protocollo,*](n-glos.md)dall'indirizzo di rete, [*dall'endpoint*](e-glos.md)e dalle opzioni dell'endpoint, che possono essere tutte usate per creare un [*handle*](b-glos.md) di associazione per il server specificato.

</dd> <dt>

<span id="_rpc_strong_typing_glos"></span><span id="_RPC_STRONG_TYPING_GLOS"></span>**tipizzazione forte**
</dt> <dd>

Imposizione da parte del compilatore di un controllo rigoroso sui tipi di dati. In MIDL e RPC la tipizzazione forte viene usata per garantire che i dati siano interpretati in modo coerente da computer diversi in un ambiente distribuito.

</dd> </dl>
