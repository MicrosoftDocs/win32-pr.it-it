# <a name="s-rpc"></a>S (RPC)

<dl> <dt>

<span id="rpc.s_glos_1_gly"></span><span id="RPC.S_GLOS_1_GLY"></span>**scalabilità**
</dt> <dd>

Possibilità di usare completamente la potenza di elaborazione aggiuntiva in un sistema multiprocessore (2, 4, 8, 32 o più processori). La scalabilità è anche la possibilità di servire un numero elevato di client.

</dd> <dt>

<span id="_rpcl_spp_glos"></span><span id="_RPCL_SPP_GLOS"></span>**Sequenced Packet Protocol (SPP)**
</dt> <dd>

Protocollo di comunicazione [*orientato alla connessione*](c-glos.md) Banyan VINES per il routing dei pacchetti di informazioni su reti locali.

</dd> <dt>

<span id="_rpc_spx_glos"></span><span id="_RPC_SPX_GLOS"></span>**Sequenced Packet Exchange (SPX)**
</dt> <dd>

Protocollo di comunicazione [*orientato alla connessione*](c-glos.md) di Novell NetWare per il routing dei pacchetti di informazioni su reti locali e reti wide area.

</dd> <dt>

<span id="_rpc_serialization_glos"></span><span id="_RPC_SERIALIZATION_GLOS"></span>**serializzazione**
</dt> <dd>

Processo di marshalling dei dati in (codifica) e di unmarshalling dei dati dai buffer (Decodifica) che si controllano. Si tratta di una differenza rispetto all'utilizzo RPC tradizionale, in cui gli stub e il runtime RPC controllano i buffer di marshalling. Detto anche selezione. Vedere anche [*serializzazione delle procedure*](p-glos.md), [*serializzazione del tipo*](t-glos.md).

</dd> <dt>

<span id="_rpc_server_stub_glos"></span><span id="_RPC_SERVER_STUB_GLOS"></span>**Stub server**
</dt> <dd>

Codice sorgente del linguaggio C generato da MIDL che contiene tutte le funzioni necessarie per l'applicazione server per gestire le richieste remote tramite chiamate a procedure locali. Vedere anche [*Stub client*](c-glos.md).

</dd> <dt>

<span id="_rpc_session_glos"></span><span id="_RPC_SESSION_GLOS"></span>**sessione**
</dt> <dd>

Relazione stabilita tra un'applicazione client e un'applicazione server. Vedere anche [*Bind*](b-glos.md), [*handle di associazione*](b-glos.md).

</dd> <dt>

<span id="_rpc_static_callback_function_glos"></span><span id="_RPC_STATIC_CALLBACK_FUNCTION_GLOS"></span>**funzione di callback statica**
</dt> <dd>

Procedura remota che fa parte del lato client di un'applicazione distribuita, che può essere chiamata da un server per ottenere informazioni dal client. L' \[ [](/windows/desktop/Midl/callback) \] attributo di callback designa una funzione di callback statica.

</dd> <dt>

<span id="_rpc_static_identity_tracking_glos"></span><span id="_RPC_STATIC_IDENTITY_TRACKING_GLOS"></span>**rilevamento identità statica**
</dt> <dd>

Il rilevamento statico delle identità specifica che la fase di esecuzione RPC usa le credenziali di sicurezza nell'handle di associazione del client per tutte le chiamate RPC. Vedere anche [*Dynamic Identity Tracking*](d-glos.md).

</dd> <dt>

<span id="_rpc_string_binding_glos"></span><span id="_RPC_STRING_BINDING_GLOS"></span>**Associazione stringa**
</dt> <dd>

Stringa di caratteri costituita dalle opzioni [*UUID*](u-glos.md)dell'oggetto, [*sequenza di protocollo*](p-glos.md), [*indirizzo di rete*](n-glos.md), [*endpoint*](e-glos.md)ed endpoint, che possono essere utilizzate per creare un [*handle di associazione*](b-glos.md) al server specificato.

</dd> <dt>

<span id="_rpc_strong_typing_glos"></span><span id="_RPC_STRONG_TYPING_GLOS"></span>**tipizzazione forte**
</dt> <dd>

Imposizione del compilatore di un controllo rigoroso sui tipi di dati. In MIDL e RPC, la tipizzazione forte viene utilizzata per garantire che i dati vengano interpretati in modo coerente da computer diversi in un ambiente distribuito.

</dd> </dl>
