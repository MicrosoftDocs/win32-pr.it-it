# <a name="configuring-the-name-service-provider"></a>Configurazione del provider di servizi dei nomi

Se l'applicazione distribuita registra la propria interfaccia con un servizio dei nomi, sia il client che il server devono usare lo stesso servizio nome. Microsoft RPC interagisce con Microsoft Locator e con qualsiasi provider di servizi dei nomi (NSP) conforme all'interfaccia NSI (Microsoft RPC Name Service Interface), ad esempio il servizio dce cell directory accessibile tramite il daemon del servizio dei nomi (NSID) di Digital Equipment Corporation. Microsoft Locator, progettato per l'uso in ambienti Windows, è il provider di servizi dei nomi predefinito.

In questa sezione viene illustrata l'installazione e la configurazione dei provider di servizi dei nomi:

-   [Configurazione del servizio dei nomi](configuring-the-name-service-for-windows-xp-windows-2000-or-windows-nt.md)
-   [Configurazione del servizio dei nomi per Windows 95](configuring-the-name-service-for-windows-95.md)
-   [Configurazione del servizio dei nomi per Windows 3.x o MS-DOS](configuring-the-name-service-for-windows-3-x-or-ms-dos.md)
-   [Avvio e arresto di Microsoft Locator](starting-and-stopping-microsoft-locator.md)
