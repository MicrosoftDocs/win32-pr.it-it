<dl> <dt>

<span id="_rpc_cds_glos"></span><span id="_RPC_CDS_GLOS"></span>**Servizio directory celle (CDS)**
</dt> <dd>

Nome-provider di servizi per l'ambiente di elaborazione distribuita di Open Software Foundation.

</dd> <dt>

<span id="_rpc_client_stub_glos"></span><span id="_RPC_CLIENT_STUB_GLOS"></span>**Stub client**
</dt> <dd>

Codice sorgente del linguaggio C generato da MIDL. Contiene tutte le funzioni necessarie affinché l'applicazione client effettui chiamate a procedure remote utilizzando il modello di una chiamata di funzione tradizionale in un'applicazione autonoma. Lo stub del client è responsabile del marshalling dei parametri di input e dell'unmarshalling dei parametri di output. Vedere anche [*stub server*](s-glos.md), [*stub proxy*](p-glos.md).

</dd> <dt>

<span id="_rpc_conformant_array_glos"></span><span id="_RPC_CONFORMANT_ARRAY_GLOS"></span>**matrice conforme**
</dt> <dd>

Matrice la cui dimensione viene determinata in fase di esecuzione da un altro parametro, da un campo della struttura o da un'espressione.

</dd> <dt>

<span id="_rpc_connection_oriented_glos"></span><span id="_RPC_CONNECTION_ORIENTED_GLOS"></span>**orientato alla connessione**
</dt> <dd>

Protocollo di comunicazione o trasporto che fornisce un circuito virtuale attraverso il quale i pacchetti di dati vengono ricevuti nello stesso ordine in cui sono stati trasmessi. Se la connessione tra computer ha esito negativo, l'applicazione riceve una notifica. [*TCP*](t-glos.md) e [*SPX*](s-glos.md) sono esempi di protocolli orientati alla connessione. Vedere anche [*datagramma*](d-glos.md).

</dd> <dt>

<span id="_rpc_connectionless_glos"></span><span id="_RPC_CONNECTIONLESS_GLOS"></span>**connessione**
</dt> <dd>

Vedere [*datagramma*](d-glos.md).

</dd> <dt>

<span id="_rpc_context_rundown_glos"></span><span id="_RPC_CONTEXT_RUNDOWN_GLOS"></span>**Rundown del contesto**
</dt> <dd>

Notifica del server risultante da una chiusura imprevista dell'associazione tra applicazioni client e server.

</dd> </dl>
