<dl> <dt>

<span id="_rpc_cds_glos"></span><span id="_RPC_CDS_GLOS"></span>**Cell Directory Service (CDS)**
</dt> <dd>

Provider di servizi dei nomi per Distributed Computing Environment di Open Software Foundation.

</dd> <dt>

<span id="_rpc_client_stub_glos"></span><span id="_RPC_CLIENT_STUB_GLOS"></span>**stub client**
</dt> <dd>

Codice sorgente in linguaggio C generato da MIDL. Contiene tutte le funzioni necessarie all'applicazione client per effettuare chiamate di procedura remota usando il modello di una chiamata di funzione tradizionale in un'applicazione autonoma. Lo stub client è responsabile del marshalling dei parametri di input e dell'annullamento del marshalling dei parametri di output. Vedere anche [*stub del server*](s-glos.md), [*stub proxy.*](p-glos.md)

</dd> <dt>

<span id="_rpc_conformant_array_glos"></span><span id="_RPC_CONFORMANT_ARRAY_GLOS"></span>**matrice conforme**
</dt> <dd>

Matrice la cui dimensione è determinata in fase di esecuzione da un altro parametro, campo della struttura o espressione.

</dd> <dt>

<span id="_rpc_connection_oriented_glos"></span><span id="_RPC_CONNECTION_ORIENTED_GLOS"></span>**orientato alla connessione**
</dt> <dd>

Protocollo di comunicazione o trasporto che fornisce un circuito virtuale tramite il quale i pacchetti di dati vengono ricevuti nello stesso ordine in cui sono stati trasmessi. Se la connessione tra computer non riesce, l'applicazione viene notificata. [*TCP*](t-glos.md) e [*SPX sono*](s-glos.md) esempi di protocolli orientati alla connessione. Vedere anche [*datagramma.*](d-glos.md)

</dd> <dt>

<span id="_rpc_connectionless_glos"></span><span id="_RPC_CONNECTIONLESS_GLOS"></span>**senza connessione**
</dt> <dd>

Vedere [*datagramma*](d-glos.md).

</dd> <dt>

<span id="_rpc_context_rundown_glos"></span><span id="_RPC_CONTEXT_RUNDOWN_GLOS"></span>**rundown del contesto**
</dt> <dd>

Notifica server che risulta da una chiusura imprevista dell'associazione tra applicazioni client e server.

</dd> </dl>
