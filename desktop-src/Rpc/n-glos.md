# <a name="n-rpc"></a>N (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) [d](d-glos.md) [E](e-glos.md) [F](f-glos.md) G H [i](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) N [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_name_service_glos"></span><span id="_RPC_NAME_SERVICE_GLOS"></span>**nome servizio**
</dt> <dd>

Servizio che esegue il mapping dei nomi agli oggetti e archivia le coppie nome/oggetto in un database. Il servizio nome RPC, ad esempio, esegue il mapping di un nome logico a un [*handle di binding*](b-glos.md) in modo che le applicazioni client possano fare riferimento a tale nome logico, anziché una sequenza di protocollo e un indirizzo di rete. Vedere anche Name Service-Interface daemon ([NSID)](/windows), client Directory Service ([*CDS)*](c-glos.md), [*Locator*](l-glos.md).

</dd> <dt>

<span id="_rpc_nca_glos"></span><span id="_RPC_NCA_GLOS"></span>**Architettura di Network Computing (garante)**
</dt> <dd>

Raccolta di linee guida per l'elaborazione distribuita. I protocolli di comunicazione RPC seguono queste linee guida.

</dd> <dt>

<span id="_rpc_nsi_glos"></span><span id="_RPC_NSI_GLOS"></span>**Nome servizio indipendente (NSI)**
</dt> <dd>

Standard per le funzioni API che consentono a un'applicazione distribuita di accedere agli elementi del database del servizio nome RPC tramite diversi provider di servizi nome, ad esempio OSF-DCE Cell Directory Service o Microsoft Locator. Vedere anche [-daemon di interfaccia](/windows) (NSID).

</dd> <dt>

<span id="_rpc_nsid_glos"></span><span id="_RPC_NSID_GLOS"></span>**nome Service-daemon Interface (NSID)**
</dt> <dd>

Servizio che fornisce un'interfaccia tra Microsoft [*Locator*](l-glos.md) e i database del servizio directory delle celle OSF-DCE per le funzioni del servizio nome RPC.

</dd> <dt>

<span id="_rpc_named_pipe_glos"></span><span id="_RPC_NAMED_PIPE_GLOS"></span>**named pipe**
</dt> <dd>

Protocollo orientato alla connessione basato su Server Message Blocks (PMI) e [NetBIOS](/windows), usato per la comunicazione tra un processo server e uno o più processi client.

</dd> <dt>

<span id="_rpc_netbeui_glos"></span><span id="_RPC_NETBEUI_GLOS"></span>**NetBEUI (Extended User Interface) NetBIOS**
</dt> <dd>

Il protocollo di trasporto nativo di LAN Manager e il driver di dispositivo di rete. Vedere anche [NetBIOS](/windows).

</dd> <dt>

<span id="_rpc_netbios_glos"></span><span id="_RPC_NETBIOS_GLOS"></span>**Rete Basic Input/Output System (NetBIOS)**
</dt> <dd>

Interfaccia software tra il sistema operativo Microsoft MS-DOS, il bus di I/O e una rete locale.

</dd> <dt>

<span id="_rpc_ndr_glos"></span><span id="_RPC_NDR_GLOS"></span>**Rappresentazione dei dati di rete (NDR)**
</dt> <dd>

Formato standard utilizzato durante la trasmissione di rete indipendente dal formato del tipo di dati in una particolare architettura del computer. I dati trasmessi includono informazioni che specificano il relativo formato NDR.

</dd> <dt>

<span id="_rpc_network_address_glos"></span><span id="_RPC_NETWORK_ADDRESS_GLOS"></span>**Indirizzo di rete**
</dt> <dd>

Indirizzo che identifica un server in una rete.

</dd> <dt>

<span id="_rpc_nonencapsulated_union_glos"></span><span id="_RPC_NONENCAPSULATED_UNION_GLOS"></span>**Unione non incapsulata**
</dt> <dd>

[*Unione discriminata*](d-glos.md) meno restrittiva rispetto a un'Unione incapsulata in quanto il discriminante e l'Unione non sono strettamente associati. Se l'Unione è un parametro, discriminante è un altro parametro. Se l'Unione è un campo della struttura, discriminante è un altro campo della struttura. L'opzione parole chiave IDL \[ [**\_ è**](/windows/desktop/Midl/switch-is) \] e \[ il [**\_ tipo di opzione**](/windows/desktop/Midl/switch-type) \] identifica discriminante e il relativo tipo. Vedere anche [*Unione incapsulata*](e-glos.md).

</dd> <dt>

<span id="_rpc_nonidempotent_glos"></span><span id="_RPC_NONIDEMPOTENT_GLOS"></span>**nonidempotent**
</dt> <dd>

Indica che una chiamata di procedura remota non può essere eseguita più di una volta perché restituirà un valore diverso o modificherà uno stato.

</dd> </dl>
