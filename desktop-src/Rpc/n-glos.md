# <a name="n-rpc"></a>N (RPC)

<dl> <dt>

<span id="_rpc_name_service_glos"></span><span id="_RPC_NAME_SERVICE_GLOS"></span>**name service**
</dt> <dd>

Servizio che esegue il mapping dei nomi agli oggetti e archivia le coppie nome/oggetto in un database. Ad esempio, il servizio nomi RPC esegue il mapping di un nome logico a un [*handle*](b-glos.md) di associazione in modo che le applicazioni client possano fare riferimento a tale nome logico, anziché a una sequenza di protocollo e a un indirizzo di rete. Vedere anche assegnare un nome al daemon service-interface ([nsid),](/windows)al servizio directory client [*(CDS)*](c-glos.md)e [*al localizzatore*](l-glos.md).

</dd> <dt>

<span id="_rpc_nca_glos"></span><span id="_RPC_NCA_GLOS"></span>**Architettura di elaborazione di rete (NCA)**
</dt> <dd>

Raccolta di linee guida per l'elaborazione distribuita. I protocolli di comunicazione RPC seguono queste linee guida.

</dd> <dt>

<span id="_rpc_nsi_glos"></span><span id="_RPC_NSI_GLOS"></span>**Indipendente dal servizio dei nomi (NSI)**
</dt> <dd>

Standard per le funzioni API che consente a un'applicazione distribuita di accedere agli elementi del database nome-servizio RPC tramite vari provider di servizi di nomi, ad esempio OSF-DCE Cell Directory Service o Microsoft Locator. Vedere anche [–interface daemon](/windows) (nsid).

</dd> <dt>

<span id="_rpc_nsid_glos"></span><span id="_RPC_NSID_GLOS"></span>**name service-interface daemon (nsid)**
</dt> <dd>

Servizio che fornisce un'interfaccia tra Microsoft [*Locator*](l-glos.md) e i database del servizio nome OSF-DCE Cell Directory Service per le funzioni name-service RPC.

</dd> <dt>

<span id="_rpc_named_pipe_glos"></span><span id="_RPC_NAMED_PIPE_GLOS"></span>**named pipe**
</dt> <dd>

Protocollo orientato alla connessione, basato su Server Message Blocks (SMB) e [NetBIOS,](/windows)usato per la comunicazione tra un processo server e uno o più processi client.

</dd> <dt>

<span id="_rpc_netbeui_glos"></span><span id="_RPC_NETBEUI_GLOS"></span>**NetBIOS Extended Interfaccia utente (NetBEUI)**
</dt> <dd>

Protocollo di trasporto nativo e driver di dispositivo di rete di LAN Manager. Vedere anche [NetBIOS.](/windows)

</dd> <dt>

<span id="_rpc_netbios_glos"></span><span id="_RPC_NETBIOS_GLOS"></span>**Network Basic Input/Output System (NetBIOS)**
</dt> <dd>

Interfaccia software tra il sistema operativo Microsoft MS-DOS, il bus di I/O e una rete locale.

</dd> <dt>

<span id="_rpc_ndr_glos"></span><span id="_RPC_NDR_GLOS"></span>**Rappresentazione dei dati di rete (NDR)**
</dt> <dd>

Formato standard utilizzato durante la trasmissione di rete indipendente dal formato del tipo di dati in una particolare architettura del computer. I dati trasmessi includono informazioni che specificano il formato NDR.

</dd> <dt>

<span id="_rpc_network_address_glos"></span><span id="_RPC_NETWORK_ADDRESS_GLOS"></span>**indirizzo di rete**
</dt> <dd>

Indirizzo che identifica un server in una rete.

</dd> <dt>

<span id="_rpc_nonencapsulated_union_glos"></span><span id="_RPC_NONENCAPSULATED_UNION_GLOS"></span>**unione non incapsulata**
</dt> <dd>

[*Unione discriminante*](d-glos.md) meno restrittiva rispetto a un'unione incapsulata in quanto la discriminante e l'unione non sono strettamente associate. Se l'unione è un parametro, il discriminante è un altro parametro; se l'unione è un campo di struttura, il discriminante è un altro campo della struttura. L'opzione delle parole chiave IDL è e il tipo di opzione \[ [**\_**](/windows/desktop/Midl/switch-is) identifica \] la \[ [**\_**](/windows/desktop/Midl/switch-type) \] discriminante e il relativo tipo. Vedere anche [*unione incapsulata.*](e-glos.md)

</dd> <dt>

<span id="_rpc_nonidempotent_glos"></span><span id="_RPC_NONIDEMPOTENT_GLOS"></span>**nonidempotente**
</dt> <dd>

Indicatore che una chiamata di procedura remota non può essere eseguita più di una volta perché restituirà un valore diverso o modificherà uno stato.

</dd> </dl>
