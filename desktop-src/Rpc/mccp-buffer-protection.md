# <a name="mccp-buffer-protection"></a><span data-ttu-id="52897-104">Protezione del buffer MCCP</span><span class="sxs-lookup"><span data-stu-id="52897-104">MCCP Buffer Protection</span></span>

<span data-ttu-id="52897-105">A partire da Windows Vista, il motore di marshalling RPC esegue ulteriori passaggi per evitare sovraccarichi del buffer sul lato client a causa dei dati restituiti.</span><span class="sxs-lookup"><span data-stu-id="52897-105">Starting with Windows Vista, the RPC Marshalling Engine takes further steps to try to prevent client-side buffer overruns due to returned data.</span></span> <span data-ttu-id="52897-106">Questa funzionalità è detta protezione della conformità di calcolo minima (MCCP).</span><span class="sxs-lookup"><span data-stu-id="52897-106">This facility is called Mini Compute Conformance Protection (MCCP).</span></span>

<span data-ttu-id="52897-107">Quando il client passa un puntatore a un buffer esistente a un \[ parametro [**out**](/windows/desktop/Midl/out-idl) \] o \[ [**in**](/windows/desktop/Midl/in)**out** \] , i dati restituiti per il parametro vengono copiati nel buffer esistente.</span><span class="sxs-lookup"><span data-stu-id="52897-107">When the client passes a pointer to an existing buffer to an \[[**out**](/windows/desktop/Midl/out-idl)\] or \[[**in**](/windows/desktop/Midl/in),**out**\] parameter, returned data for that parameter is copied into the existing buffer.</span></span> <span data-ttu-id="52897-108">Se i dati restituiti sono maggiori del buffer passato, può verificarsi un sovraccarico del buffer quando RPC copia i dati restituiti nel buffer troppo piccolo.</span><span class="sxs-lookup"><span data-stu-id="52897-108">If the returned data is larger than the passed buffer, a buffer overrun can occur when RPC copies the returned data into the too-small buffer.</span></span> <span data-ttu-id="52897-109">Vedere [puntatori di primo livello e incorporati](top-level-and-embedded-pointers.md).</span><span class="sxs-lookup"><span data-stu-id="52897-109">See [Top-Level and Embedded Pointers](top-level-and-embedded-pointers.md).</span></span>

<span data-ttu-id="52897-110">Con MCCP, RPC tenta di rilevare questa condizione e di rifiutare la chiamata se viene rilevata.</span><span class="sxs-lookup"><span data-stu-id="52897-110">With MCCP, RPC attempts to detect this condition and reject the call if it is detected.</span></span> <span data-ttu-id="52897-111">Per i buffer con un valore di correlazione, ad esempio \[ [**size \_**](/windows/desktop/Midl/size-is) \] , se i dati restituiti non rientrano nella dimensione del buffer specificata, la chiamata viene rifiutata e \_ \_ viene generata l'eccezione RPC X Bad \_ Stub \_ Data.</span><span class="sxs-lookup"><span data-stu-id="52897-111">For buffers with a correlation value, such as \[[**size\_is**](/windows/desktop/Midl/size-is)\], if the returned data does not fit in the specified buffer size, the call is rejected and RPC\_X\_BAD\_STUB\_DATA exception is raised.</span></span> <span data-ttu-id="52897-112">Per le stringhe non dimensionate, la chiamata viene rifiutata se la dimensione della stringa esistente (lunghezza fino al carattere di terminazione **null** ) non è sufficiente per conservare la stringa restituita, la chiamata viene rifiutata.</span><span class="sxs-lookup"><span data-stu-id="52897-112">For unsized strings, the call is rejected if the existing string size (length until the **null** terminator) is insufficient to hold the returned string, the call is rejected.</span></span> <span data-ttu-id="52897-113">RPC non è in grado di rilevare i sovraccarichi del buffer in tutte le condizioni, quindi lo sviluppatore deve continuare a adottare le normali precauzioni per i sovraccarichi del buffer.</span><span class="sxs-lookup"><span data-stu-id="52897-113">RPC cannot detect buffer overruns in all conditions, so the developer is advised to continue to take normal precautions against buffer overruns.</span></span>

<span data-ttu-id="52897-114">Se il client non passa un buffer esistente per un parametro \[ [**out**](/windows/desktop/Midl/out-idl) \] , ma passa invece un puntatore dereferenziato a **null**, RPC seguirà le normali regole per allocare un nuovo buffer per conto del client.</span><span class="sxs-lookup"><span data-stu-id="52897-114">If the client does not pass an existing buffer for an \[[**out**](/windows/desktop/Midl/out-idl)\] parameter, but instead passes a dereferenced pointer to **NULL**, RPC will follow normal rules to allocate a new buffer on the client’s behalf.</span></span> <span data-ttu-id="52897-115">Questo buffer verrà allocato con spazio sufficiente per contenere i dati restituiti.</span><span class="sxs-lookup"><span data-stu-id="52897-115">This buffer will be allocated with sufficient space to hold the returned data.</span></span>

<span data-ttu-id="52897-116">Una seconda protezione è che, per i parametri correlati, RPC imporrà che viene passato un buffer non **null** quando la variabile di conteggio delle correlazioni è non **null**.</span><span class="sxs-lookup"><span data-stu-id="52897-116">A second protection is that for correlated parameters, RPC will enforce that a non-**null** buffer is passed when the correlation count variable is non-**null**.</span></span>
