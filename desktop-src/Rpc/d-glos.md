# <a name="d-rpc"></a>D (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) d [E](e-glos.md) [F](f-glos.md) G H [i](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_datagram_glos"></span><span id="_RPC_DATAGRAM_GLOS"></span>**datagramma**
</dt> <dd>

Protocollo di comunicazione o trasporto in cui i pacchetti di dati vengono instradati in modo indipendente. Possono seguire diverse route e arrivare in un ordine diverso da quello in cui sono stati inviati. [*UDP*](u-glos.md) e [*IPX*](i-glos.md) sono esempi di protocolli di livello trasporto-datagramma. Vedere anche [*orientato alla connessione*](c-glos.md).

</dd> <dt>

<span id="_rpc_discriminant_glos"></span><span id="_RPC_DISCRIMINANT_GLOS"></span>**discriminante**
</dt> <dd>

Variabile che specifica i tipi di dati che possono essere archiviati in un'Unione.

</dd> <dt>

<span id="_rpc_discriminated_union_glos"></span><span id="_RPC_DISCRIMINATED_UNION_GLOS"></span>**unione discriminata**
</dt> <dd>

(o *record variante*) Unione che include un discriminatore come parte della struttura dei dati in modo che il tipo di dati attualmente valido venga trasmesso insieme all'Unione. Vedere anche [*Unione*](e-glos.md)incapsulata, Unione non [*incapsulata*](n-glos.md).

</dd> <dt>

<span id="_rpc_dce_glos"></span><span id="_RPC_DCE_GLOS"></span>**Ambiente di calcolo distribuito (DCE)**
</dt> <dd>

Specifica di Open Software Foundation per un set di servizi integrati, incluse le chiamate a procedure remote, i file system distribuiti e i servizi di sicurezza. Lo standard RPC OSF-DCE è la base per Microsoft RPC.

</dd> <dt>

<span id="_rpc_dynamic_endpoint_glos"></span><span id="_RPC_DYNAMIC_ENDPOINT_GLOS"></span>**endpoint dinamico**
</dt> <dd>

Specifico della rete: indirizzo del server richiesto e assegnato in fase di esecuzione. Vedere anche [*endpoint noto*](w-glos.md).

</dd> <dt>

<span id="_rpc_dynamic_identity_tracking_glos"></span><span id="_RPC_DYNAMIC_IDENTITY_TRACKING_GLOS"></span>**rilevamento dinamico delle identità**
</dt> <dd>

Specifica che la libreria di runtime RPC utilizzerà le credenziali del thread chiamante, anziché l'handle di binding, per l'autenticazione ogni volta che il client chiama una procedura remota. Vedere anche [*Rilevamento statico delle identità*](s-glos.md).

</dd> </dl>
