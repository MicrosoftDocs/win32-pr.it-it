</dt> <dd>

[*Handle di associazione*](b-glos.md) che non include informazioni sull' [*endpoint*](e-glos.md) . Vedere anche [*handle completamente associato*](f-glos.md).

</dd> <dt>

<span id="_rpc_pickling_glos"></span><span id="_RPC_PICKLING_GLOS"></span>**decapaggio**
</dt> <dd>

Vedere [*serializzazione*](s-glos.md).

</dd> <dt>

<span id="_rpc_pipe_glos"></span><span id="_RPC_PIPE_GLOS"></span>**tubo**
</dt> <dd>

Costruttore di tipo [*IDL*](i-glos.md)che supporta la trasmissione di un flusso di dati aperto tra applicazioni client e server.

</dd> <dt>

<span id="_rpc_primary_enterprise_controller_glos"></span><span id="_RPC_PRIMARY_ENTERPRISE_CONTROLLER_GLOS"></span>**Controller Enterprise primario**
</dt> <dd>

Database MQIS (Master Message Queue Information Service) per una rete. Vedere anche [*Message Queue Information Service (MQIS)*](m-glos.md).

</dd> <dt>

<span id="_rpc_primary_site_controller_glos"></span><span id="_RPC_PRIMARY_SITE_CONTROLLER_GLOS"></span>**controller del sito primario**
</dt> <dd>

Database Message Queue Information Service (MQIS) per un sito specifico all'interno di un'azienda. Vedere anche [*Message Queue Information Service (MQIS)*](m-glos.md).

</dd> <dt>

<span id="_rpc_procedure_serialization_glos"></span><span id="_RPC_PROCEDURE_SERIALIZATION_GLOS"></span>**serializzazione delle procedure**
</dt> <dd>

[*Serializzazione*](s-glos.md) dei dati che usa uno stub di serializzazione generato da MIDL per eseguire la codifica e la decodifica di uno o più tipi con una singola chiamata di procedura. La serializzazione delle procedure viene eseguita applicando gli \[ [](/windows/desktop/Midl/encode) \] attributi Encode e \[ [Decode](/windows/desktop/Midl/decode) \] a un prototipo di funzione nel file ACF. Vedere anche [*serializzazione del tipo*](t-glos.md).

</dd> <dt>

<span id="_rpc_protocol_sequence_glos"></span><span id="_RPC_PROTOCOL_SEQUENCE_GLOS"></span>**sequenza di protocollo**
</dt> <dd>

Stringa di caratteri che rappresenta una combinazione valida di un protocollo RPC, un protocollo di livello rete e un protocollo del livello di trasporto. La sequenza di protocollo NCACN IP TCP, ad esempio, \_ \_ descrive una connessione del [*garante*](n-glos.md) della conformità a un protocollo Internet (ip) con un Transmission Control Protocol (TCP) come trasporto.

</dd> <dt>

<span id="_rpc_proxy_stub_glos"></span><span id="_RPC_PROXY_STUB_GLOS"></span>**Stub proxy**
</dt> <dd>

Linguaggio C o C++ generato da MIDL: codice sorgente che contiene tutte le funzioni necessarie per un'interfaccia OLE personalizzata.

</dd> </dl>
