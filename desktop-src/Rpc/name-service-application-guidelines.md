# <a name="name-service-application-guidelines"></a><span data-ttu-id="4386e-103">Linee guida sull'applicazione di servizio nome</span><span class="sxs-lookup"><span data-stu-id="4386e-103">Name Service Application Guidelines</span></span>

<span data-ttu-id="4386e-104">Quando si sviluppa l'applicazione distribuita, è necessario fornire agli utenti dell'applicazione un metodo per specificare il nome con cui è possibile registrare l'applicazione nel database del servizio nome.</span><span class="sxs-lookup"><span data-stu-id="4386e-104">When you develop your distributed application, you need to provide the application users with a method for specifying the name under which they can register the application in the name service database.</span></span> <span data-ttu-id="4386e-105">Questo metodo può essere costituito da un file di dati, dall'input della riga di comando o dalla finestra di dialogo.</span><span class="sxs-lookup"><span data-stu-id="4386e-105">This method can consist of a data file, command-line input, or dialog box.</span></span>

<span data-ttu-id="4386e-106">Sebbene l'architettura del servizio nome RPC supporti diversi metodi per organizzare le voci del server di un'applicazione, è ottimizzata per le ricerche.</span><span class="sxs-lookup"><span data-stu-id="4386e-106">Although the RPC name service architecture supports various methods for organizing an application's server entries, it is optimized for lookups.</span></span> <span data-ttu-id="4386e-107">Di conseguenza, gli aggiornamenti frequenti possono compromettere le prestazioni del servizio nome e dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="4386e-107">As a result, frequent updates can hinder the performance of both the name service and the application.</span></span> <span data-ttu-id="4386e-108">Per evitare di esportare le informazioni inutilmente, scegliere una progettazione che consenta al server di determinare se le relative informazioni si trovino nel database del servizio nome.</span><span class="sxs-lookup"><span data-stu-id="4386e-108">To avoid exporting information unnecessarily, choose a design that lets the server determine whether its information is in the name service database.</span></span> <span data-ttu-id="4386e-109">Inoltre, ogni istanza del server deve essere esportata nel nome della propria voce.</span><span class="sxs-lookup"><span data-stu-id="4386e-109">In addition, each server instance should export to its own entry name.</span></span> <span data-ttu-id="4386e-110">In caso contrario, sarà difficile per un'istanza modificare gli UUID degli oggetti supportati o le sequenze di protocollo senza compromettere le informazioni di un'altra istanza.</span><span class="sxs-lookup"><span data-stu-id="4386e-110">Otherwise, it will be difficult for an instance to change its supported object UUIDs or protocol sequences without disturbing another instance's information.</span></span>

<span data-ttu-id="4386e-111">Il metodo seguente consente di evitare questi problemi e fornisce prestazioni ottimali, indipendentemente dal servizio dei nomi usato dalla rete.</span><span class="sxs-lookup"><span data-stu-id="4386e-111">The following method avoids these pitfalls and provides good performance, no matter what name service your network uses.</span></span>

<span data-ttu-id="4386e-112">Per iniziare, progettare l'applicazione in modo che la prima volta che viene avviata un'istanza del server specificata selezioni un nome univoco per la voce del server e salva il nome nel registro di sistema insieme alle altre informazioni di configurazione dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="4386e-112">To begin with, design your application so that the first time a given server instance starts, it picks a unique server-entry name and saves this name in the registry along with the application's other configuration information.</span></span> <span data-ttu-id="4386e-113">Quindi, fare in modo che esporti gli handle di binding e gli UUID degli oggetti, se presenti, alla relativa voce di servizio del nome.</span><span class="sxs-lookup"><span data-stu-id="4386e-113">Then, have it export its binding handles and object UUIDs, if any, to its name service entry.</span></span>

<span data-ttu-id="4386e-114">Le chiamate successive dell'istanza del server devono verificare che la voce del servizio del nome sia presente e che contenga il set corretto di UUID degli oggetti e degli handle di associazione.</span><span class="sxs-lookup"><span data-stu-id="4386e-114">Subsequent invocations of the server instance should check that the name service entry is present and contains the correct set of object UUIDs and binding handles.</span></span> <span data-ttu-id="4386e-115">Una voce mancante può indicare che un amministratore l'ha rimossa o che un'interruzione dell'alimentazione ha causato la perdita delle informazioni sul servizio del nome.</span><span class="sxs-lookup"><span data-stu-id="4386e-115">A missing entry may mean that an administrator removed it, or that a power outage caused the name service information to be lost.</span></span> <span data-ttu-id="4386e-116">È importante verificare che gli handle di associazione nella voce siano corretti; Se un amministratore aggiunge il supporto TCP/IP a un computer, ad esempio, i server RPC saranno in ascolto su tale sequenza di protocollo quando chiamano [**RpcServerUseAllProtSeqs**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcserveruseallprotseqs).</span><span class="sxs-lookup"><span data-stu-id="4386e-116">It is important to verify that the binding handles in the entry are correct; if an administrator adds TCP/IP support to a computer, for example, RPC servers will listen on that protocol sequence when they call [**RpcServerUseAllProtseqs**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcserveruseallprotseqs).</span></span> <span data-ttu-id="4386e-117">Tuttavia, se il server non aggiorna la voce del servizio del nome, i client non saranno informati che TCP è supportato.</span><span class="sxs-lookup"><span data-stu-id="4386e-117">However, if the server does not update the name service entry, clients will not be informed that TCP is supported.</span></span>

<span data-ttu-id="4386e-118">Quando il client importa, deve specificare **null** come nome di voce.</span><span class="sxs-lookup"><span data-stu-id="4386e-118">When the client imports, it should specify **NULL** as the entry name.</span></span> <span data-ttu-id="4386e-119">Se si specifica **null** , le funzioni della libreria RPC di Microsoft cercano l'interfaccia in tutte le voci del servizio nome del dominio o del gruppo di lavoro del computer client, individuando quindi le informazioni per ogni istanza.</span><span class="sxs-lookup"><span data-stu-id="4386e-119">Specifying **NULL** causes the Microsoft RPC library functions to search for the interface in all name service entries in the client computer's domain or workgroup, thus finding the information for every instance.</span></span>

<span data-ttu-id="4386e-120">Se si utilizzano UUID di oggetti per rappresentare oggetti noti, ad esempio stampanti, è possibile utilizzare una variante di questo metodo.</span><span class="sxs-lookup"><span data-stu-id="4386e-120">If you use object UUIDs to represent well-known objects such as printers, you can use a variation of this method.</span></span> <span data-ttu-id="4386e-121">Anziché esportare i binding in una voce, progettare l'applicazione in modo che ogni istanza crei una voce per ogni oggetto supportato, ad esempio "/.:/ stampanti/laser1 "e"/.:/ stampanti/Laser2 ".</span><span class="sxs-lookup"><span data-stu-id="4386e-121">Instead of exporting bindings to one entry, design your application so that each instance creates an entry for each supported object, such as "/.:/printers/Laser1" and "/.:/printers/Laser2."</span></span> <span data-ttu-id="4386e-122">Quindi, fare in modo che il server esporti gli handle di binding a ogni voce del server, insieme all'UUID dell'oggetto pertinente per tale voce.</span><span class="sxs-lookup"><span data-stu-id="4386e-122">Then, have the server export its binding handles to each server entry, along with the object UUID relevant to that entry.</span></span>

<span data-ttu-id="4386e-123">In questo caso, un client può cercare una risorsa in base al nome importando dalla voce del server pertinente. non richiede l'UUID dell'oggetto della risorsa.</span><span class="sxs-lookup"><span data-stu-id="4386e-123">In this case, a client can look up a resource by name by importing from the relevant server entry; it does not require the object UUID of the resource.</span></span> <span data-ttu-id="4386e-124">Se ha l'UUID della risorsa ma non il nome, può importare dalla voce **null** .</span><span class="sxs-lookup"><span data-stu-id="4386e-124">If it has the resource UUID but not the name, it can import from the **null** entry.</span></span>
