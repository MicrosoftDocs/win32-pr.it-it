# <a name="policy-object"></a><span data-ttu-id="80fa8-103">Oggetto Policy</span><span class="sxs-lookup"><span data-stu-id="80fa8-103">Policy Object</span></span>

<span data-ttu-id="80fa8-104">L'oggetto **criterio** viene utilizzato per controllare l'accesso al database dell' [*autorità di protezione locale*](/windows/desktop/SecGloss/l-gly) (LSA) e contiene informazioni applicabili all'intero sistema o stabilisce impostazioni predefinite per il sistema.</span><span class="sxs-lookup"><span data-stu-id="80fa8-104">The **Policy** object is used to control access to the [*Local Security Authority*](/windows/desktop/SecGloss/l-gly) (LSA) database and contains information that applies to the entire system or establishes defaults for the system.</span></span> <span data-ttu-id="80fa8-105">Ogni sistema dispone di un solo oggetto **criteri** .</span><span class="sxs-lookup"><span data-stu-id="80fa8-105">Each system has only one **Policy** object.</span></span> <span data-ttu-id="80fa8-106">Questo oggetto **criterio** viene creato dall'LSA all'avvio del sistema e le applicazioni non possono crearlo o eliminarlo definitivamente.</span><span class="sxs-lookup"><span data-stu-id="80fa8-106">This **Policy** object is created by the LSA when the system starts up, and applications cannot create or destroy it.</span></span>

<span data-ttu-id="80fa8-107">Le informazioni archiviate in un oggetto **criteri** includono:</span><span class="sxs-lookup"><span data-stu-id="80fa8-107">The information stored in a **Policy** object includes:</span></span>

<span data-ttu-id="80fa8-120">I campi **AccountDomain** e **PrimaryDomain** dell'oggetto **policy** vengono utilizzati per scopi diversi a seconda del tipo di relazioni di sistema e di trust:</span><span class="sxs-lookup"><span data-stu-id="80fa8-120">The **AccountDomain** and **PrimaryDomain** fields of the **Policy** object are used for different purposes depending upon the type of system and trust relationships:</span></span>

-   <span data-ttu-id="80fa8-121">In un sistema che non dispone di un dominio primario, il campo **AccountDomain** contiene il nome e il SID del dominio dell'account locale del sistema, che corrisponde al nome del computer.</span><span class="sxs-lookup"><span data-stu-id="80fa8-121">On a system that does not have a primary domain, the **AccountDomain** field contains the name and SID of the system's local account domain, which is the same as the computer name.</span></span> <span data-ttu-id="80fa8-122">Il campo **PrimaryDomain** contiene il nome del gruppo di lavoro di cui è membro il computer.</span><span class="sxs-lookup"><span data-stu-id="80fa8-122">The **PrimaryDomain** field contains the name of the workgroup this machine is a member of.</span></span> <span data-ttu-id="80fa8-123">Gli oggetti [**trustedDomain**](trusteddomain-object.md) vengono ignorati con un'eccezione. non può essere presente un oggetto **trustedDomain** con lo stesso nome del gruppo di lavoro perché verrà visualizzato come se fosse il dominio primario del computer.</span><span class="sxs-lookup"><span data-stu-id="80fa8-123">[**TrustedDomain**](trusteddomain-object.md) objects are ignored with one exception—there cannot be a **TrustedDomain** object with the same name as the workgroup because it will appear as though it is the machine's primary domain.</span></span>
-   <span data-ttu-id="80fa8-124">In un sistema che dispone di un dominio primario, il campo **AccountDomain** identifica il nome e il SID del dominio dell'account locale, come prima.</span><span class="sxs-lookup"><span data-stu-id="80fa8-124">On a system that has a primary domain, the **AccountDomain** field identifies the name and SID of the local account domain, as before.</span></span> <span data-ttu-id="80fa8-125">Tuttavia, il campo **PrimaryDomain** contiene il nome e il SID del dominio primario per il sistema.</span><span class="sxs-lookup"><span data-stu-id="80fa8-125">However, the **PrimaryDomain** field contains the name and SID of the primary domain for the system.</span></span> <span data-ttu-id="80fa8-126">Deve inoltre essere presente un oggetto [**trustedDomain**](trusteddomain-object.md) con il nome e il SID identificati nel campo **PrimaryDomain** .</span><span class="sxs-lookup"><span data-stu-id="80fa8-126">Additionally, there should be a [**TrustedDomain**](trusteddomain-object.md) object with the name and SID identified in the **PrimaryDomain** field.</span></span> <span data-ttu-id="80fa8-127">Questo oggetto **trustedDomain** contiene le informazioni relative all'account e al server necessarie per stabilire un canale sicuro per un controller di dominio nel dominio primario.</span><span class="sxs-lookup"><span data-stu-id="80fa8-127">This **TrustedDomain** object contains the account and server information necessary to establish a secure channel to a domain controller in the primary domain.</span></span> <span data-ttu-id="80fa8-128">Tutti gli altri oggetti **trustedDomain** vengono ignorati.</span><span class="sxs-lookup"><span data-stu-id="80fa8-128">Any other **TrustedDomain** objects are ignored.</span></span>
-   <span data-ttu-id="80fa8-129">Nei controller di dominio il campo **AccountDomain** identifica il dominio dell'account locale per il sistema; il nome dell'account viene tuttavia assegnato dall'utente anziché essere un nome noto.</span><span class="sxs-lookup"><span data-stu-id="80fa8-129">On domain controllers, the **AccountDomain** field identifies the local account domain for the system; however, the account name is user assigned rather than being a well-known name.</span></span> <span data-ttu-id="80fa8-130">Poiché il dominio primario è lo stesso del dominio dell'account, il campo **PrimaryDomain** deve contenere lo stesso valore del campo **AccountDomain** .</span><span class="sxs-lookup"><span data-stu-id="80fa8-130">Since the primary domain is the same as the account domain, the **PrimaryDomain** field must contain the same value as the **AccountDomain** field.</span></span> <span data-ttu-id="80fa8-131">Inoltre, è previsto che tutti gli oggetti [**trustedDomain**](trusteddomain-object.md) siano validi e rappresentino relazioni di trust con altri domini.</span><span class="sxs-lookup"><span data-stu-id="80fa8-131">Furthermore, all [**TrustedDomain**](trusteddomain-object.md) objects are expected to be valid and represent trust relationships with other domains.</span></span> <span data-ttu-id="80fa8-132">Se il sistema non considera attendibili altri domini, non devono essere presenti oggetti **trustedDomain** .</span><span class="sxs-lookup"><span data-stu-id="80fa8-132">If the system does not trust any other domains, there should not be any **TrustedDomain** objects.</span></span>
