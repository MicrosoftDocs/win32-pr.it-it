# <a name="security-management"></a>Gestione della sicurezza

## <a name="purpose"></a>Scopo

Le tecnologie di gestione della [](/windows/desktop/SecGloss/l-gly) sicurezza possono essere usate per gestire i criteri dell'autorità di sicurezza locale (LSA) e i criteri di filtro [*delle password,*](/windows/desktop/SecGloss/p-gly) eseguire query sulla capacità dei programmi provenienti da origini esterne e per gli allegati del servizio che estendono le funzionalità dello strumento Di configurazione della sicurezza.

Le tecnologie di gestione della sicurezza Microsoft includono l'API Criteri LSA, l'API Filtro password, l'API Più sicura e l'API Allegati di sicurezza del servizio. Queste tecnologie consentono agli sviluppatori di software di creare applicazioni che gestiscono sistemi e applicazioni.

## <a name="developer-audience"></a>Sviluppatori

L'API Criteri LSA, l'API Filtro password, l'API Safer e l'API Service Security Attachments sono destinate agli sviluppatori di applicazioni che consentono agli amministratori di gestire e proteggere i propri sistemi. Gli sviluppatori devono avere familiarità con i linguaggi di programmazione C e C++ e con la Windows basata su linguaggio.
