# <a name="how-object-pooling-works"></a><span data-ttu-id="7cc39-104">Funzionamento del pool di oggetti</span><span class="sxs-lookup"><span data-stu-id="7cc39-104">How Object Pooling Works</span></span>

<span data-ttu-id="7cc39-105">Quando si configura un componente da raggruppare, COM+ ne manterrà le istanze in un pool, pronto per essere attivato per qualsiasi client che richiede il componente.</span><span class="sxs-lookup"><span data-stu-id="7cc39-105">When you configure a component to be pooled, COM+ will maintain instances of it in a pool, ready to be activated for any client requesting the component.</span></span> <span data-ttu-id="7cc39-106">Tutte le richieste di creazione di oggetti verranno gestite tramite Gestione pool.</span><span class="sxs-lookup"><span data-stu-id="7cc39-106">Any object creation requests will be handled through the pool manager.</span></span>

<span data-ttu-id="7cc39-107">I pool sono configurati e gestiti in base ai singoli componenti.</span><span class="sxs-lookup"><span data-stu-id="7cc39-107">Pools are configured and maintained on a per-component basis.</span></span> <span data-ttu-id="7cc39-108">Un pool è costituito da oggetti omogenei che condividono lo stesso CLSID.</span><span class="sxs-lookup"><span data-stu-id="7cc39-108">A pool consists of homogeneous objects that share the same CLSID.</span></span> <span data-ttu-id="7cc39-109">L'unica eccezione riguarda gli oggetti transazionali, per i quali i pool vengono mantenuti contenenti oggetti con affinità di transazione mentre una transazione è in sospeso.</span><span class="sxs-lookup"><span data-stu-id="7cc39-109">The only exception is for transactional objects, for which subpools are maintained containing objects that have transaction affinity while a transaction is pending.</span></span>

<span data-ttu-id="7cc39-110">Quando l'applicazione viene avviata, il pool verrà popolato fino al livello minimo che è stato specificato in modo amministrativo, a condizione che la creazione dell'oggetto abbia esito positivo.</span><span class="sxs-lookup"><span data-stu-id="7cc39-110">When the application starts, the pool will be populated up to the minimum level that you have administratively specified, as long as object creation succeeds.</span></span> <span data-ttu-id="7cc39-111">Poiché le richieste dei client per il componente sono disponibili, vengono soddisfatte per la prima volta dal pool.</span><span class="sxs-lookup"><span data-stu-id="7cc39-111">As client requests for the component come in, they are satisfied on a first-come first-served basis from the pool.</span></span> <span data-ttu-id="7cc39-112">Se non sono disponibili oggetti in pool e il pool non è ancora al livello massimo specificato, viene creato e attivato un nuovo oggetto per il client.</span><span class="sxs-lookup"><span data-stu-id="7cc39-112">If no pooled objects are available and the pool is not yet at its specified maximum level, a new object is created and activated for the client.</span></span>

<span data-ttu-id="7cc39-113">Quando il pool raggiunge il livello massimo, le richieste client vengono accodate e riceveranno il primo oggetto disponibile dal pool.</span><span class="sxs-lookup"><span data-stu-id="7cc39-113">When the pool reaches its maximum level, client requests are queued and will receive the first available object from the pool.</span></span> <span data-ttu-id="7cc39-114">Il numero di oggetti, inclusi sia attivato che disattivato, non supererà mai il valore massimo del pool.</span><span class="sxs-lookup"><span data-stu-id="7cc39-114">The number of objects, including both activated and deactivated, will never exceed the maximum pool value.</span></span> <span data-ttu-id="7cc39-115">Le richieste di creazione di oggetti si timeout dopo un periodo di tempo specificato amministrativa, in modo da poter controllare il tempo di attesa dei client per la creazione dell'oggetto.</span><span class="sxs-lookup"><span data-stu-id="7cc39-115">Object creation requests will time out after an administratively specified period so that you can control how long clients will wait for object creation.</span></span> <span data-ttu-id="7cc39-116">Quando si verifica un errore di timeout, il client otterrà il \_ timeout di errore E da [**CoCreateInstance**](/windows/desktop/api/combaseapi/nf-combaseapi-cocreateinstance).</span><span class="sxs-lookup"><span data-stu-id="7cc39-116">Upon a time-out failure, the client will get back the error E\_TIMEOUT from [**CoCreateInstance**](/windows/desktop/api/combaseapi/nf-combaseapi-cocreateinstance).</span></span>

<span data-ttu-id="7cc39-117">Quando possibile, COM+ tenterà di riutilizzare un oggetto dopo che è stato rilasciato da un client, fino a quando il pool non raggiunge il livello massimo.</span><span class="sxs-lookup"><span data-stu-id="7cc39-117">Whenever possible, COM+ will attempt to reuse an object after a client releases it, until the pool reaches its maximum level.</span></span> <span data-ttu-id="7cc39-118">L'oggetto è responsabile del monitoraggio dello stato per determinare se è possibile riutilizzarlo e deve restituire un valore appropriato per [**IObjectControl:: CanBePooled**](/windows/desktop/api/ComSvcs/nf-comsvcs-iobjectcontrol-canbepooled).</span><span class="sxs-lookup"><span data-stu-id="7cc39-118">The object is responsible for monitoring its state to determine whether it can be reused and should return an appropriate value for [**IObjectControl::CanBePooled**](/windows/desktop/api/ComSvcs/nf-comsvcs-iobjectcontrol-canbepooled).</span></span>

<span data-ttu-id="7cc39-119">Quando viene creato un oggetto in pool, questo viene aggregato in un oggetto più grande che gestirà la durata dell'oggetto.</span><span class="sxs-lookup"><span data-stu-id="7cc39-119">When a pooled object is created, it is aggregated into a larger object that will manage the object's lifetime.</span></span> <span data-ttu-id="7cc39-120">L'oggetto esterno chiama i metodi su [**IObjectControl**](/windows/desktop/api/ComSvcs/nn-comsvcs-iobjectcontrol) in momenti appropriati nel ciclo di vita dell'oggetto, come indicato di seguito:</span><span class="sxs-lookup"><span data-stu-id="7cc39-120">The outer object calls methods on [**IObjectControl**](/windows/desktop/api/ComSvcs/nn-comsvcs-iobjectcontrol) at appropriate times in the object's life cycle, as follows:</span></span>

-   <span data-ttu-id="7cc39-121">Il metodo [**Activate**](/windows/desktop/api/ComSvcs/nf-comsvcs-iobjectcontrol-activate) viene chiamato ogni volta che l'oggetto viene restituito a un client, attivato in un contesto specifico.</span><span class="sxs-lookup"><span data-stu-id="7cc39-121">The [**Activate**](/windows/desktop/api/ComSvcs/nf-comsvcs-iobjectcontrol-activate) method is called whenever the object is returned to a client, activated in a specific context.</span></span>
-   <span data-ttu-id="7cc39-122">Il metodo [**Deactivate**](/windows/desktop/api/ComSvcs/nf-comsvcs-iobjectcontrol-deactivate) viene chiamato ogni volta che un oggetto viene rilasciato dal client o, nel caso di un oggetto attivato da JIT, quando viene disattivato.</span><span class="sxs-lookup"><span data-stu-id="7cc39-122">The [**Deactivate**](/windows/desktop/api/ComSvcs/nf-comsvcs-iobjectcontrol-deactivate) method is called whenever an object is released by the client or, in the case of a JIT-activated object, when it is deactivated.</span></span>
-   <span data-ttu-id="7cc39-123">Il metodo [**CanBePooled**](/windows/desktop/api/ComSvcs/nf-comsvcs-iobjectcontrol-canbepooled) viene chiamato ogni volta che un oggetto deve essere restituito al pool generale.</span><span class="sxs-lookup"><span data-stu-id="7cc39-123">The [**CanBePooled**](/windows/desktop/api/ComSvcs/nf-comsvcs-iobjectcontrol-canbepooled) method is called whenever an object is to be returned to the general pool.</span></span> <span data-ttu-id="7cc39-124">Se l'oggetto rileva che alcune risorse riutilizzabili sono in uno stato non valido, deve restituire **false** per questo metodo e la gestione pool eliminerà l'oggetto.</span><span class="sxs-lookup"><span data-stu-id="7cc39-124">If the object detects that some reusable resource is in a bad state, it should return **FALSE** for this method and the pool manager will discard the object.</span></span>

<span data-ttu-id="7cc39-125">Un oggetto non deve necessariamente implementare [**IObjectControl**](/windows/desktop/api/ComSvcs/nn-comsvcs-iobjectcontrol).</span><span class="sxs-lookup"><span data-stu-id="7cc39-125">An object does not necessarily need to implement [**IObjectControl**](/windows/desktop/api/ComSvcs/nn-comsvcs-iobjectcontrol).</span></span> <span data-ttu-id="7cc39-126">In caso contrario, le istanze verranno sempre riutilizzate fino a quando non viene raggiunto il livello massimo del pool.</span><span class="sxs-lookup"><span data-stu-id="7cc39-126">If it does not, instances will always be reused, until the pool maximum level is reached.</span></span>

<span data-ttu-id="7cc39-127">Per informazioni dettagliate su come configurare i componenti da raggruppare, vedere [configurazione di un componente da raggruppare](configuring-a-component-to-be-pooled.md).</span><span class="sxs-lookup"><span data-stu-id="7cc39-127">For details about how to configure components to be pooled, see [Configuring a Component to Be Pooled](configuring-a-component-to-be-pooled.md).</span></span>
