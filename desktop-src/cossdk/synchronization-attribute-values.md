# <a name="synchronization-attribute-values"></a><span data-ttu-id="21603-103">Valori degli attributi di sincronizzazione</span><span class="sxs-lookup"><span data-stu-id="21603-103">Synchronization Attribute Values</span></span>

<span data-ttu-id="21603-104">L'attributo di sincronizzazione è una proprietà dichiarativa che specifica il tipo di sincronizzazione che si desidera che i componenti abbiano quando vengono attivati.</span><span class="sxs-lookup"><span data-stu-id="21603-104">The synchronization attribute is a declarative property that specifies what type of synchronization you want your components to have when they are activated.</span></span> <span data-ttu-id="21603-105">Quando si include l'attributo di sincronizzazione, COM+ gestisce i dettagli della sincronizzazione per conto dell'utente; non è necessario effettuare altre chiamate.</span><span class="sxs-lookup"><span data-stu-id="21603-105">When you include the synchronization attribute, COM+ handles the details of synchronization on your behalf; you do not have to make any other calls.</span></span>

<span data-ttu-id="21603-106">A seconda dei requisiti, un oggetto può condividere la sincronizzazione del chiamante, richiedere una nuova sincronizzazione o funzionare senza sincronizzazione.</span><span class="sxs-lookup"><span data-stu-id="21603-106">Depending on its requirements, an object can share its caller's synchronization, require a new synchronization, or operate without synchronization.</span></span>

<span data-ttu-id="21603-107">COM+ fornisce i seguenti valori di attributo di sincronizzazione:</span><span class="sxs-lookup"><span data-stu-id="21603-107">COM+ provides the following synchronization attribute values:</span></span>

-   <span data-ttu-id="21603-108">**Disabilitati.**</span><span class="sxs-lookup"><span data-stu-id="21603-108">**Disabled.**</span></span> <span data-ttu-id="21603-109">Quando si disabilita l'attributo di sincronizzazione, COM+ ignora i requisiti di sincronizzazione del componente nel determinare il contesto per l'oggetto.</span><span class="sxs-lookup"><span data-stu-id="21603-109">When you disable the synchronization attribute, COM+ ignores the synchronization requirements of the component in determining context for the object.</span></span> <span data-ttu-id="21603-110">Di conseguenza, l'oggetto può condividere o meno il contesto del chiamante (e la sincronizzazione).</span><span class="sxs-lookup"><span data-stu-id="21603-110">As a result, the object may or may not share its caller's context (and synchronization).</span></span>

    <span data-ttu-id="21603-111">In generale, è consigliabile usare questo valore di attributo quando si sa che il componente non accede mai a una gestione risorse.</span><span class="sxs-lookup"><span data-stu-id="21603-111">In general, you should use this attribute value when you know the component never accesses a resource manager.</span></span> <span data-ttu-id="21603-112">Quando si esegue la migrazione di componenti COM a COM+, è necessario disabilitare l'attributo di sincronizzazione per mantenere lo stesso comportamento del componente COM non configurato.</span><span class="sxs-lookup"><span data-stu-id="21603-112">When migrating COM components to COM+, you must disable the synchronization attribute to maintain the same behavior as the unconfigured COM component.</span></span> <span data-ttu-id="21603-113">Un componente non configurato è un componente COM che non è stato installato in un'applicazione COM+.</span><span class="sxs-lookup"><span data-stu-id="21603-113">An unconfigured component is a COM component that has not been installed in a COM+ application.</span></span>

-   <span data-ttu-id="21603-114">**Non supportato.**</span><span class="sxs-lookup"><span data-stu-id="21603-114">**Not Supported.**</span></span> <span data-ttu-id="21603-115">Un oggetto con questo valore non fa mai parte di una sincronizzazione, a prescindere dallo stato del relativo chiamante.</span><span class="sxs-lookup"><span data-stu-id="21603-115">An object with this value never participates in synchronization, regardless of the status of its caller.</span></span> <span data-ttu-id="21603-116">Questa impostazione è disponibile solo per i componenti che non sono transazionali e non utilizzano il servizio [di attivazione JIT (just-in-Time) com+](com--just-in-time-activation.md) .</span><span class="sxs-lookup"><span data-stu-id="21603-116">This setting is available only for components that are non-transactional and do not use the [COM+ just-in-time (JIT) activation](com--just-in-time-activation.md) service.</span></span>

-   <span data-ttu-id="21603-117">**Supportato.**</span><span class="sxs-lookup"><span data-stu-id="21603-117">**Supported.**</span></span> <span data-ttu-id="21603-118">Un oggetto con questo valore partecipa alla sincronizzazione, se esistente.</span><span class="sxs-lookup"><span data-stu-id="21603-118">An object with this value participates in synchronization if it exists.</span></span> <span data-ttu-id="21603-119">Questo valore viene dichiarato quando si desidera che un oggetto condivida la sincronizzazione del chiamante, ma non richiede la sincronizzazione.</span><span class="sxs-lookup"><span data-stu-id="21603-119">You declare this value when you want an object to share in its caller's synchronization but not require synchronization of its own.</span></span>

    <span data-ttu-id="21603-120">Un buon motivo per impostare l'attributo di sincronizzazione su supportato è che questa impostazione può essere meno costosa in termini di risorse di sistema.</span><span class="sxs-lookup"><span data-stu-id="21603-120">A good reason to set your synchronization attribute to Supported is that this setting can be less expensive in terms of system resources.</span></span> <span data-ttu-id="21603-121">Tuttavia, è più difficile scrivere il componente a causa della necessità di proteggerlo dalle chiamate simultanee.</span><span class="sxs-lookup"><span data-stu-id="21603-121">However, it is more difficult to write your component because of the need to protect it from concurrent calls.</span></span> <span data-ttu-id="21603-122">L'implicazione dell'impostazione dell'attributo di sincronizzazione su supportato è che, in determinate circostanze, un'istanza dell'oggetto può essere creata in modo tale che non sia sincronizzata.</span><span class="sxs-lookup"><span data-stu-id="21603-122">The implication of setting the synchronization attribute to Supported is that, under certain circumstances, an instance of your object may be created in such a way that it's not synchronized.</span></span> <span data-ttu-id="21603-123">Se il modello di threading del componente è libero o entrambi, sarà necessario proteggere i dati dell'istanza con un tipo di meccanismo di blocco.</span><span class="sxs-lookup"><span data-stu-id="21603-123">If the component's threading model is Free or Both, you will have to protect your instance data with some type of locking mechanism.</span></span> <span data-ttu-id="21603-124">Se il modello di threading è Apartment (STA), non è necessario proteggere i dati dell'istanza.</span><span class="sxs-lookup"><span data-stu-id="21603-124">If the Threading Model is Apartment (STA), you won't need to protect your instance data.</span></span>

-   <span data-ttu-id="21603-125">**Obbligatoria.**</span><span class="sxs-lookup"><span data-stu-id="21603-125">**Required.**</span></span> <span data-ttu-id="21603-126">Quando si imposta questo attributo, COM+ garantisce che tutti gli oggetti creati dal componente verranno sincronizzati.</span><span class="sxs-lookup"><span data-stu-id="21603-126">When you set this attribute, COM+ ensures that all objects created from the component will be synchronized.</span></span> <span data-ttu-id="21603-127">In effetti, ogni volta che COM+ crea un'istanza del componente, verifica che sia presente un solo thread attraverso questa istanza alla volta.</span><span class="sxs-lookup"><span data-stu-id="21603-127">In effect, whenever COM+ creates an instance of your component, it makes sure there's only one thread going through this instance at a time.</span></span>

    <span data-ttu-id="21603-128">Quando COM+ attiva un oggetto, esamina lo stato di sincronizzazione del chiamante.</span><span class="sxs-lookup"><span data-stu-id="21603-128">As COM+ activates an object, it looks at the synchronization status of its caller.</span></span> <span data-ttu-id="21603-129">Se il chiamante è sincronizzato, COM+ scorre il limite di sincronizzazione del chiamante per includere il nuovo oggetto.</span><span class="sxs-lookup"><span data-stu-id="21603-129">If the caller is synchronized, COM+ flows the caller's synchronization boundary to include the new object.</span></span> <span data-ttu-id="21603-130">In caso contrario, COM+ avvia la sincronizzazione.</span><span class="sxs-lookup"><span data-stu-id="21603-130">Otherwise, COM+ begins the synchronization.</span></span>

-   <span data-ttu-id="21603-131">**Richiede un nuovo.**</span><span class="sxs-lookup"><span data-stu-id="21603-131">**Requires New.**</span></span> <span data-ttu-id="21603-132">Un oggetto con questo valore deve partecipare a una nuova sincronizzazione, in cui COM+ gestisce contesti e Apartment per conto di tutti i componenti coinvolti nella chiamata.</span><span class="sxs-lookup"><span data-stu-id="21603-132">An object with this value must participate in a new synchronization, where COM+ manages contexts and apartments on behalf of all components involved in the call.</span></span> <span data-ttu-id="21603-133">COM+ avvia automaticamente una nuova sincronizzazione, che è diversa dalla sincronizzazione del chiamante.</span><span class="sxs-lookup"><span data-stu-id="21603-133">COM+ automatically initiates a new synchronization, which is distinct from the caller's synchronization.</span></span>

    <span data-ttu-id="21603-134">Un buon motivo per impostare l'attributo di sincronizzazione su richiede nuovo è che questa impostazione consente di effettuare in modo più efficiente chiamate esterne a un'istanza del componente.</span><span class="sxs-lookup"><span data-stu-id="21603-134">A good reason to set your synchronization attribute to Requires New is that this setting allows you to make outside calls to an instance of your component more efficiently.</span></span> <span data-ttu-id="21603-135">Tuttavia, effettua anche chiamate tra l'oggetto e l'oggetto che lo ha creato più costoso in termini di risorse di sistema.</span><span class="sxs-lookup"><span data-stu-id="21603-135">However, it also makes calls between your object and the object that created it more expensive in terms of system resources.</span></span>

    <span data-ttu-id="21603-136">Si supponga, ad esempio, un caso in cui l'oggetto e il relativo oggetto Creator condividono lo stesso limite di sincronizzazione.</span><span class="sxs-lookup"><span data-stu-id="21603-136">For example, assume a case where your object and its creator object share the same synchronization boundary.</span></span> <span data-ttu-id="21603-137">Se il client A chiama l'oggetto Creator e il client B chiama l'oggetto, la seconda chiamata dovrà attendere fino al completamento della prima chiamata.</span><span class="sxs-lookup"><span data-stu-id="21603-137">If client A calls the creator object and client B calls your object, the second call will have to wait until the first call is completed.</span></span> <span data-ttu-id="21603-138">Se si imposta requires New, l'oggetto viene creato in un limite di sincronizzazione separato.</span><span class="sxs-lookup"><span data-stu-id="21603-138">If you set Requires New, your object is created in a separate synchronization boundary.</span></span> <span data-ttu-id="21603-139">In questo caso, le chiamate da altri oggetti possono essere elaborate contemporaneamente.</span><span class="sxs-lookup"><span data-stu-id="21603-139">In this case, calls from other objects can be processed at the same time.</span></span> <span data-ttu-id="21603-140">Tuttavia, le chiamate dall'oggetto Creator all'oggetto richiedono più risorse di sistema perché devono superare i limiti di sincronizzazione.</span><span class="sxs-lookup"><span data-stu-id="21603-140">However, calls from the creator object to your object require more system resources because they must cross synchronization boundaries.</span></span>
