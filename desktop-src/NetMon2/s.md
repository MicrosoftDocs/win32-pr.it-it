# <a name="s-network-monitor"></a>S (Network Monitor)

<dl> <dt>

<span id="_netmon_sap_gly"></span><span id="_NETMON_SAP_GLY"></span>**SAP**
</dt> <dd>

Vedere punto di accesso al servizio.

</dd> <dt>

<span id="_netmon_server_message_block_gly"></span><span id="_NETMON_SERVER_MESSAGE_BLOCK_GLY"></span>**SMB (Server Message Block)**
</dt> <dd>

Protocollo a livello di file che contiene primitive e viene utilizzato per operazioni quali **apertura**, **chiusura**, **lettura** e **scrittura**.

</dd> <dt>

<span id="_netmon_service_access_point_gly"></span><span id="_NETMON_SERVICE_ACCESS_POINT_GLY"></span>**punto di accesso al servizio (SAP)**
</dt> <dd>

Identificatore del frame.

</dd> <dt>

<span id="_netmon_session_gly"></span><span id="_NETMON_SESSION_GLY"></span>**sessione**
</dt> <dd>

Set di pacchetti di dati che una stazione invia a un'altra stazione durante un'acquisizione. Una sessione è uno scambio unidirezionale di pacchetti in cui ogni sessione ha un proprietario (la stazione che invia i pacchetti di dati) e un partner, ovvero la stazione che riceve i pacchetti di dati. Vedere anche \_ informazioni sulla sessione.

</dd> <dt>

<span id="_netmon_session_information_gly"></span><span id="_NETMON_SESSION_INFORMATION_GLY"></span>**informazioni sulla sessione**
</dt> <dd>

Informazioni che descrivono una sessione. Network Monitor salva le informazioni sulla sessione durante l'acquisizione dei dati. Le informazioni sulla sessione sono denominate statistiche di conversazione e possono essere recuperate da una chiamata al metodo **GetConversationStatistics** . Vedere anche [*\_ statistiche relative*](c.md)a sessioni, [*acquisizioni*](c.md)e conversazioni.

</dd> <dt>

<span id="_netmon_smb_gly"></span><span id="_NETMON_SMB_GLY"></span>**SMB**
</dt> <dd>

Vedere Server Message Block.

</dd> <dt>

<span id="_netmon_source_address_gly"></span><span id="_NETMON_SOURCE_ADDRESS_GLY"></span>**Indirizzo di origine**
</dt> <dd>

Numero esadecimale univoco che identifica il computer da cui vengono inviati i dati.

</dd> <dt>

<span id="_netmon_special_blobs_gly"></span><span id="_NETMON_SPECIAL_BLOBS_GLY"></span>**BLOB speciale**
</dt> <dd>

Un provider di pacchetti di rete (NPP) o un oggetto binario di grandi dimensioni (BLOB) di filtro che contiene informazioni aggiuntive non fornite da altri BLOB. Vedere anche [*provider di pacchetti di rete*](n.md) e [*BLOB*](b.md).

</dd> <dt>

<span id="_netmon_station_gly"></span><span id="_NETMON_STATION_GLY"></span>**Stazione**
</dt> <dd>

Una scheda di interfaccia di rete specifica installata in un computer che sta acquisendo dati. Vedere anche informazioni sulla stazione.

</dd> <dt>

<span id="_netmon_station_information_gly"></span><span id="_NETMON_STATION_INFORMATION_GLY"></span>**informazioni stazione**
</dt> <dd>

Informazioni che descrivono una stazione. Network Monitor salva le informazioni sulla stazione durante un'acquisizione dei dati. Le informazioni sulla stazione sono denominate anche statistiche di conversazione che è possibile recuperare tramite una chiamata al metodo **GetConversationStatistics** . Vedere anche [*statistiche relative*](c.md)a stazioni e conversazioni.

</dd> <dt>

<span id="_netmon_summary_property_gly"></span><span id="_NETMON_SUMMARY_PROPERTY_GLY"></span>**Proprietà Summary**
</dt> <dd>

Proprietà definita dal parser utilizzata per raggruppare le proprietà di un protocollo per la visualizzazione nell'interfaccia utente di Network Monitor. Una struttura **PROPERTYINFO** definisce una proprietà di riepilogo nello stesso modo in cui vengono definite altre proprietà definite dal parser. La proprietà Summary è in genere la prima definizione di proprietà associata a un frame quando Network Monitor chiama la funzione **AttachProperties** .

</dd> <dt>

<span id="_netmon_summary_pane_gly"></span><span id="_NETMON_SUMMARY_PANE_GLY"></span>**riquadro Riepilogo**
</dt> <dd>

Il riquadro in una finestra Network Monitor Frame Viewer che elenca i frame contenuti in un file di acquisizione. Per ogni fotogramma, il riquadro Riepilogo elenca il numero di frame, l'ora in cui viene acquisito il frame, l'indirizzo di origine e di destinazione del frame, i protocolli usati per inviare il frame e una breve descrizione del frame. Se a un indirizzo non elaborato è associato un indirizzo IP o IPX/XNS, questo indirizzo viene visualizzato in **src other addr** e **DST other addr**. Il tipo di indirizzo (IP o IPX/XNS) viene visualizzato nel **tipo altro addr**.

</dd> </dl>
