# <a name="iris-gl-and-opengl-differences"></a>Differenze tra IRIS GL e OpenGL

Questa appendice elenca le differenze tra OpenGL e IRIS GL. Viene specificato un termine per ogni differenza, seguito da una descrizione.



| Termine                                     | Descrizione                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|--------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| wrapping dell'accumulo                | L'operazione di buffer di accumulo OpenGL non è definita quando i valori dei componenti superano 1,0 o scendeno al di sotto di 1,0.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| righe con antialias                    | OpenGL consente di convertire le righe con antialias. IRIS GL no.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| arco                                  | OpenGL supporta gli archi nella libreria di utilità.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| elenchi di attributi                      | Gli attributi inseriti dagli attributi push IRIS GL differiscono da qualsiasi set di attributi di cui è stato push da OpenGL glPushAttrib. Poiché tutti gli stati OpenGL possono essere letti, tuttavia, è possibile implementare qualsiasi semantica push/pop desiderata usando OpenGL.                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ridimensionamento automatico delle trame            | L'interfaccia di trama OpenGL non supporta il ridimensionamento automatico delle immagini alla potenza di due dimensioni. GLU supporta tuttavia il ridimensionamento delle immagini.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| bbox                                 | OpenGL non supporta l'esecuzione condizionale degli elenchi di visualizzazione.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| callfunc                             | OpenGL non supporta il callback dagli elenchi di visualizzazione. Si noti che IRIS GL non supporta questa funzionalità neanche quando client e server si verificano su piattaforme diverse.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| circle                               | OpenGL supporta i cerchi con GLU. In OpenGL sia cerchi che archi (dischi e dischi parziali) possono avere fori. È anche possibile modificare la suddivisione delle primitive in OpenGL, le cui normali di superficie sono disponibili per l'illuminazione.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| opzioni di cancellazione                        | OpenGL cancella effettivamente i buffer. Non applica le operazioni sui pixel attualmente specificate, ad esempio blending e logicop, indipendentemente dalle modalità. Per cancellare usando queste funzionalità, è necessario eseguire il rendering di un poligono di dimensioni della finestra.                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| righe chiuse                         | OpenGL esegue il rendering di tutte le linee con alias a larghezza singola in modo che le linee di scommettimento non concivano pixel. Ciò significa che l'ultimo pixel di una linea indipendente non viene disegnato.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| flag color/normal                    | L'illuminazione OpenGL è abilitata o disabilitata in modo esplicito. Se abilitata, è efficace indipendentemente dall'ordine in cui vengono specificati i colori e le normali. Non è possibile abilitare o disabilitare l'illuminazione tra i comandi OpenGL glBegin e glEnd. Per disabilitare l'illuminazione tra glBegin e glEnd, specificare la riflettura del materiale di ambiente, diffusa e speculare, quindi impostare l'emissione del materiale sul colore desiderato.                                                                                                                                                                                                                                                            |
| poligoni concavi                     | L'API OpenGL principale non gestisce i poligoni concavi, ma GLU supporta la scomposizione di contorni concavi non auto-intersecati in triangoli. Questi triangoli possono essere disegnati immediatamente o restituiti.                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| colore calcolato corrente               | OpenGL non equivale a un colore calcolato corrente. Se si usa OpenGL come motore di illuminazione, è possibile usare il feedback per ottenere i colori generati dai calcoli di illuminazione.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| posizione grafica corrente            | OpenGL non mantiene una posizione grafica corrente. I comandi IRIS GL che dipendono dalla posizione grafica corrente, ad esempio linee relative e poligoni, non sono inclusi in OpenGL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| curve                               | OpenGL non supporta le curve IRIS GL. È consigliabile usare curve NURBS.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| defs/binds                           | OpenGL non ha il concetto di oggetti materiale, luce o trama. solo delle proprietà di materiale, luce e trama. È tuttavia possibile usare gli elenchi di visualizzazione per creare i propri oggetti.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| depthcue                             | OpenGL non offre alcun supporto diretto per il segnale di profondità, ma il supporto di questo strumento è una funzionalità più generale che è possibile usare facilmente per emulare la funzione depthcue IRIS GL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| visualizzazione della modifica dell'elenco                 | Gli elenchi di visualizzazione OpenGL non possono essere modificati solo creati ed distrutti. Poiché è possibile specificare i nomi degli elenchi visualizzati, tuttavia, è possibile ridefinire i singoli elenchi visualizzati in una gerarchia. Gli elenchi di visualizzazione OpenGL sono progettati per la memorizzazione dei dati nella cache, non per la gestione del database. È garantito che siano archiviati nel server in ambienti client/server, quindi non sono limitati dalla larghezza di banda di rete durante l'esecuzione. Gli elenchi di visualizzazione OpenGL possono essere chiamati tra i comandi glBegin e glEnd, in modo che la gerarchia dell'elenco di visualizzazione possa essere modificata in modo sufficientemente fine da poter essere modificata.                                                                                              |
| controllo degli errori                       | OpenGL verifica la presenza di errori con maggiore attenzione rispetto a IRIS GL. Ad esempio, tutte le funzioni OpenGL non accettate tra glBegin e glEnd vengono rilevate come errori e non hanno altri effetti.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| valori restituiti da error                  | Quando un comando OpenGL che restituisce un valore rileva un errore, restituisce sempre zero. I comandi OpenGL che restituiscono dati tramite puntatori passati non apportano modifiche al contenuto della matrice se viene rilevato un errore.                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| effetti collaterali degli errori                   | Quando un comando OpenGL comporta un errore, l'unico effetto collaterale è l'aggiornamento del flag di errore al valore appropriato. Non vengono apportate altre modifiche di stato. Un'eccezione è l'out \_ Errore \_ OF MEMORY, che è irreversibile.                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| feedback                             | Il feedback è standardizzato in OpenGL in modo che non cambi da computer a computer.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| tipi di carattere e stringhe                    | OpenGL richiede che i glifi di caratteri siano manipolati come singoli elenchi di visualizzazione. Fornisce una funzione di chiamata dell'elenco di visualizzazione che accetta un elenco di nomi di elenco visualizzati, ognuno dei quali rappresentato come 1, 2 o 4 byte. La funzione glCallLists aggiunge un offset specificato separatamente a ogni nome dell'elenco visualizzato prima della chiamata, consentendo agli elenchi di nomi di elenco visualizzati di essere considerati come stringhe. Questo meccanismo offre tutte le funzionalità dei tipi di carattere IRIS GL e molto altro ancora. Ad esempio, i caratteri costituiti da triangoli possono essere facilmente manipolati.                                                                                                                  |
| frontbuffer                          | IRIS GL ha regole complesse per il rendering nel front buffer in modalità buffer singolo. OpenGL gestisce il rendering nel front buffer in modo semplice.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| polygons polygons                      | È possibile usare la capacità degli stencil OpenGL per eseguire il rendering dei poligoni vuoti. OpenGL non supporta altri mezzi per la creazione di poligoni vuoti.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| chiusura dell'indice                       | Dove possibile, OpenGL considera gli indici dei colori e degli stencil come campi di bit anziché numeri. Di conseguenza, gli indici vengono mascherati, invece che con il controllo, all'intervallo supportato di framebuffer.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| colori integer                       | I componenti di colore di interi con segno (rosso, verde, blu o alfa) vengono mappati in modo lineare ai punti mobili, in modo che il numero intero più negativo sia mappato a 1,0 e il numero intero più positivo a 1,0. Questo mapping si verifica quando si specifica il colore prima che OpenGL sostituisca il colore corrente. I componenti di colore di interi senza segno vengono mappati in modo lineare a virgola mobile, in modo che 0 sia mappato a 0,0 e il numero intero più grande sia mappato a 1,0. Questo mapping si verifica quando si specifica il colore prima che OpenGL sostituisca il colore corrente.                                                                                                                                               |
| valori normali interi                      | I componenti normali interi vengono mappati esattamente come i componenti di colore con segno. Il numero intero più negativo viene mappato a 1,0 e il numero intero più positivo viene mappato a 1,0. frammenti di pixel. I pixel disegnati da glDrawPixels o glCopyPixels vengono sempre rasterizzati e convertiti in frammenti. I frammenti risultanti sono trame, innevati, con profondità memorizzata nel buffer, sfusi e così via, come se fossero stati generati da punti geometrici. I dati di frammento non forniti dai pixel di origine vengono aumentati dalla posizione raster corrente. Ad esempio, i pixel RGBA accettano la posizione raster Z e le coordinate della trama. I pixel di profondità accettano le coordinate di colore e trama della posizione raster. |
| Invarianza                           | OpenGL garantisce una certa coerenza che non è garantita da IRIS GL. OpenGL, ad esempio, garantisce che le sequenze di codice identiche inviate allo stesso sistema, con differenze solo nella funzione di fusione specificata, generino gli stessi frammenti di pixel. I frammenti sono tuttavia diversi se la fusione è abilitata e quindi disabilitata.                                                                                                                                                                                                                                                                                                                                                 |
| equazione di illuminazione                    | L'equazione di illuminazione OpenGL è leggermente diversa dall'equazione IRIS GL. OpenGL supporta l'attenuazione separata per ogni sorgente di luce, anziché una singola attenuazione per tutte le sorgenti di luce come IRIS GL. OpenGL regola l'equazione in modo che i contributi di illuminazione ambientale, diffusa e speculare siano tutti attenuati. OpenGL consente inoltre di specificare colori separati per l'intensità ambientale, diffusa e speculare delle sorgenti di luce, nonché per la riflettenza ambientale, diffusa e speculare dei materiali. Tutti i colori di luce e materiale OpenGL includono alfa. Se si imposta l'esponente speculare su zero, l'illuminazione speculare non viene persa in OpenGL.    |
| mapw                                 | Le utilità OpenGL supportano il mapping tra coordinate di oggetti e finestre.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| modalità matrice                          | Quando le funzioni ortodo, orto2, prospettiva e finestra IRIS GL operano su una determinata matrice, tutte le operazioni della matrice OpenGL funzionano sulla matrice corrente. Tutte le operazioni di matrice OpenGL, ad eccezione di glLoadIdentity e glLoadMatrix, moltiplicano la matrice corrente anziché sostituirla (come orto, orto2, prospettiva e finestra nella gl IRIS).                                                                                                                                                                                                                                                                                                                       |
| mipmaps, generazione automatica        | L'interfaccia di trama OpenGL non supporta la generazione automatica di immagini mipmap. GLU supporta tuttavia la generazione automatica di immagini mipmap per trame 1D e 2D.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| move/draw/pmove/pdraw/pclos          | OpenGL supporta solo grafica di tipo Begin/End, perché non mantiene una posizione grafica corrente. La specifica dei parametri scalari dei comandi di spostamento/disegno precedente è tuttavia accettata da OpenGL per tutti i comandi correlati ai vertici.                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| modalità mprojection                     | IRIS GL non trasforma la geometria in base alla matrice della visualizzazione modello in modalità matrice di proiezione. OpenGL trasforma sempre sia in base alla visualizzazione del modello che alla matrice di proiezione, indipendentemente dalla modalità matrice.                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| disegno con più buffer                 | OpenGL esegue il rendering in ogni buffer dei colori singolarmente, anziché calcolare un singolo nuovo valore di colore in base al contenuto di un buffer di colore e scriverlo in tutti i buffer di colore abilitati, come fa IRIS GL.                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Nurbs                                | OpenGL supporta NURBS con una combinazione di funzionalità di base (analizzatori) e supporto GLU. Sono supportate tutte le funzionalità NURBS IRIS GL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| modalità poligono precedente                     | I poligoni OpenGL con alias vengono sempre campionati a punti. La modalità di compatibilità poligono di IRIS GL, in cui i pixel all'esterno del perimetro del poligono sono inclusi nella rasterizzazione, non è supportata. Se il codice usa questa modalità poligono, è probabilmente per i rettangoli. I rettangoli in modalità poligono precedente sono più larghi di un pixel e superiori.                                                                                                                                                                                                                                                                                                                                                |
| formati di colore packed                 | OpenGL accetta i colori come componenti a 8 bit, ma questi componenti vengono considerati come una matrice di byte anziché come byte compressi in parole più grandi. Promuovendo l'indicizzazione delle matrici anziché lo spostamento, OpenGL promuove la programmazione invariante endian. Così come IRIS GL accetta colori imballati per il rendering geometrico e pixel, OpenGL accetta matrici di componenti di colore per il rendering geometrico e pixel.                                                                                                                                                                                                                                                              |
| patches                              | OpenGL non supporta le patch IRIS GL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| writemask colore per bit              | Le maschera di scrittura OpenGL per i componenti di colore abilitano o disabilitano le modifiche all'intero componente (rosso, verde, blu o alfa), non ai singoli bit di componenti. Si noti tuttavia che le maschera di scrittura per bit sono supportate sia per gli indici dei colori che per gli indici degli stencil.                                                                                                                                                                                                                                                                                                                                                                                                              |
| writemask profondità per bit              | Le maschera di scrittura OpenGL per i componenti di profondità abilitano o disabilitano le modifiche all'intero componente, non ai singoli bit del componente di profondità.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Scegliere                                 | La libreria OpenGL Utility include il supporto per la generazione di una matrice di selezione.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| coordinate pixel                    | Sia in OpenGL che in IRIS GL, l'origine del sistema di coordinate di una finestra si trova nell'angolo inferiore sinistro. OpenGL posiziona l'origine nell'angolo inferiore sinistro di questo pixel, mentre IRIS GL la posiziona al centro del pixel in basso a sinistra.                                                                                                                                                                                                                                                                                                                                                                                                                              |
| zoom pixel                           | I fattori di zoom negativi OpenGL riflettono la posizione grafica corrente. IRIS GL non definisce il funzionamento dei fattori di zoom negativi e fornisce invece pixmode di reflection RIGHT TO LEFT e \_ \_ TOP TO \_ \_ BOTTOM. Queste modalità di reflection si riflettono sul posto, anziché sulla posizione raster corrente. OpenGL non definisce le modalità di reflection.                                                                                                                                                                                                                                                                                                                    |
| pixmode                              | I trasferimenti di pixel OpenGL operano su singoli componenti di colore, anziché su gruppi di gruppi di quattro componenti a 8 bit come IRIS GL. Anche se OpenGL offre sostanzialmente più funzionalità pixel rispetto a IRIS GL, non supporta i costrutti di colore impacchetto e non consente la riassegnazione dei componenti colore (da rosso a verde, dal rosso al blu e così via) durante le operazioni di copia dei pixel.                                                                                                                                                                                                                                                                                    |
| polif/poly                            | OpenGL non offre supporto diretto per gli elenchi di vertici diversi dagli elenchi di visualizzazione. Tuttavia, funzioni come polf e poly possono essere implementate facilmente usando l'API OpenGL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| vertice di revoca poligono             | I poligoni IRIS GL con ombreggiatura piatta accettano il colore dell'ultimo vertice specificato, mentre i poligoni OpenGL accettano il colore del primo vertice specificato.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| stipple poligono                      | Con IRIS GL il modello a stipple del poligono è relativo allo schermo. Con OpenGL è relativo a una finestra.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| numero di vertici poligoni                 | Non esiste alcun limite al numero di vertici tra glBegin e glEnd con OpenGL, anche per glBegin(POLYGON). Con IRIS GL, i poligoni sono limitati a non più di 255 vertici.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| readdisplay                          | La lettura di pixel al di fuori dei limiti della finestra è una funzionalità del sistema di finestre, anziché una funzionalità di rendering. Usare le funzioni di Windows per sostituire il comando readdisplay IRIS GL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| relative move/draw/pmove/pdraw/pclos | OpenGL non mantiene una posizione grafica corrente e pertanto non supporta le operazioni sui vertici relativi.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| RGBA logicop()                       | OpenGL non supporta operazioni logiche sui buffer RGBA.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| sbox()                               | sbox è una primitiva rettangolo IRIS GL ben definita solo se trasformata senza rotazione. È progettato per essere sottoposto a rendering più velocemente rispetto ai rettangoli standard. Anche se OpenGL non supporta una primitiva di questo tipo, può essere ottimizzata per eseguire il rendering dei rettangoli molto rapidamente quando le matrici e le altre modalità sono in stati che semplificano i calcoli.                                                                                                                                                                                                                                                                                                                             |
| argomenti scalari                     | Tutti i comandi OpenGL accettati tra glBegin e glEnd hanno punti di ingresso che accettano argomenti scalari. Ad esempio, glColor4f (rosso, verde, blu, alfa).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Forbice                              | La funzione OpenGL glScissor non tiene traccia del viewport. Il comando viewport IRIS GL aggiorna automaticamente la scrmask.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| scrbox()                             | OpenGL non supporta il calcolo del rettangolo di selezione.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| scrsubdivide()                       | OpenGL non supporta la suddivisione dello schermo.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| modalità matrice singola                   | OpenGL gestisce sempre due matrici: ModelView e Projection. Anche se un'implementazione OpenGL può consolidarli in un'unica matrice per motivi di prestazioni, deve sempre presentare il modello a due matrici al programmatore.                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| modalità subpixel                        | Tutto il rendering OpenGL è la modalità subpixel positionedsubpixel è sempre attiva.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| swaptmesh()                          | OpenGL non supporta la funzionalità swaptmesh. Offre tuttavia due tipi di mesh triangolari: una che corrisponde al comportamento "strip" predefinito dell'IRIS GL e un'altra che corrisponde alla chiamata di swaptmesh prima del terzo e di tutti i vertici successivi quando si usa IRIS GL.                                                                                                                                                                                                                                                                                                                                                                           |
| argomenti di vettore                     | Tutti i comandi OpenGL accettati tra glBegin e glEnd hanno punti di ingresso che accettano argomenti vettoriali. Ad esempio, glColor4fv .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| gestione delle finestre                    | OpenGL non include comandi di sistema della finestra. È sempre supportato come estensione di una finestra o di un sistema operativo che include funzionalità per il controllo di dispositivi e finestre. Ogni estensione fornisce un meccanismo specifico del sistema per la creazione, l'eliminazione e la modifica di contesti di rendering OpenGL. Ad esempio, l'estensione OpenGL per il sistema windows X (GLX) include circa 10 comandi a questo scopo. I comandi IRIS GL, ad esempio gconfig e drawmode, non sono implementati da OpenGL.                                                                                                                                                                            |
| offset della finestra                        | IRIS GL restituisce le posizioni del riquadro di visualizzazione e dei caratteri nello schermo, anziché nelle coordinate della finestra. OpenGL usa sempre le coordinate della finestra.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Rendering z                          | OpenGL non supporta il rendering dei colori nel buffer di profondità. Consente buffer di colore aggiuntivi, che possono essere implementati usando la stessa memoria usata per i buffer di profondità in altre configurazioni di finestra. Tuttavia, questi buffer di colore aggiuntivi non possono condividere memoria con il buffer di profondità in un'unica configurazione.                                                                                                                                                                                                                                                                                                                                          |
