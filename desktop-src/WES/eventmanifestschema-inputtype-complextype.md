# <a name="inputtype-complex-type"></a>Tipo complesso InputType

Definisce un tipo di dati di input.

``` syntax
<xs:complexType name="InputType">
    <xs:sequence>
        <xs:element name="description"
            type="string"
         />
        <xs:element name="outType"
            type="OutputType"
            minOccurs="0"
            maxOccurs="unbounded"
         />
        <xs:any
            processContents="lax"
            namespace="##other"
         />
    </xs:sequence>
    <xs:attribute name="name"
        type="QName"
        use="required"
     />
    <xs:attribute name="value"
        type="string"
        use="required"
     />
    <xs:attribute name="symbol"
        type="CSymbolType"
        use="required"
     />
</xs:complexType>
```

## <a name="child-elements"></a>Elementi figlio



| Elemento                                                                  | Tipo                                                             | Descrizione                                                                                                                                                                                                                                                                                                                                                                                                   |
|--------------------------------------------------------------------------|------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**Descrizione**](eventmanifestschema-description-inputtype-element.md) | string                                                           | Descrizione del tipo di input.<br/>                                                                                                                                                                                                                                                                                                                                                                   |
| [**outtype**](eventmanifestschema-outtype-inputtype-element.md)         | [**OutputType**](eventmanifestschema-outputtype-complextype.md) | Elenco di tipi di dati di output che determinano la modalità di rendering dei dati. È necessario specificare almeno un tipo di output. Specificare un elenco se è possibile eseguire il rendering del tipo di input come tipi di output diversi. È ad esempio possibile eseguire il rendering del tipo di input Win: UInt16 come valore short senza segno, numero di porta o numero esadecimale. Uno dei tipi di output specificati deve avere l'attributo predefinito impostato su true.<br/> |

Il [**compilatore di messaggi (MC.exe)**](message-compiler--mc-exe-.md) usa il simbolo per creare una costante per il tipo di input nel file di intestazione generato dal compilatore.<br/> |

La colonna "formato XML standard" indica il tipo canonico per il tipo di input specifico. La colonna "tipo di output" elenca i formati possibili in cui è possibile eseguire il rendering dei dati.
