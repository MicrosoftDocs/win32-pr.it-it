# <a name="compression-api"></a>API di compressione

## <a name="purpose"></a>Scopo

L'API di compressione espone gli algoritmi di compressione Windows MSZIP, XPRESS, XPRESS \_ Huff e LZMS. Ciò consente agli sviluppatori di applicazioni Windows di gestire versioni, servizi ed estendere gli algoritmi di compressione esposti.

## <a name="developer-audience"></a>Sviluppatori

L'API di compressione è progettata per l'uso da parte degli sviluppatori C/C++ professionali delle applicazioni Windows. L'API di compressione espone gli algoritmi di compressione senza perdita di tempo usati da Windows tramite un'interfaccia pubblica e un'API in modalità utente. L'API di compressione è il metodo consigliato per gli sviluppatori Windows per controllare gli algoritmi di compressione. Questa funzionalità è 64 bit su Windows a 64 bit e 32 bit su Windows a 32 bit.

## <a name="run-time-requirements"></a>Requisiti di runtime

L'API di compressione è disponibile a partire da Windows 8 o Windows Server 2012.
