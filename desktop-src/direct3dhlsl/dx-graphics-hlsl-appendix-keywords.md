## <a name="remarks"></a>Commenti

Questi tipi numerici hanno espansioni di parole chiave scalari, vettoriali e matrici:

-   float, int, uint, bool
-   min10float, min16float
-   min12int, min16int
-   min16uint

Le espansioni di questi tipi numerici seguono questo modello, che usa float come esempio:

-   Scalare

    <dl> float  
    </dl>

-   Vettore

    <dl> float1, float2, float3, float4  
    </dl>

-   Matrice

    <dl> float1x1, float1x2, float1x3, float1x4  
    float2x1, float2x2, float2x3, float2x4  
    float3x1, float3x2, float3x3, float3x4  
    float4x1, float4x2, float4x3, float4x4  
    </dl>

HLSL supporta la [trama](dx-graphics-hlsl-texture.md) e il [campionatore](dx-graphics-hlsl-sampler.md) in minuscolo per motivi legacy. Per le nuove app è invece consigliabile usare i nuovi oggetti texture di HLSL ([Texture2D](sm5-object-texture2d.md), [Texture3D](sm5-object-texture3d.md)e così via) e gli oggetti Sampler ([SamplerState e SamplerComparisonState](dx-graphics-hlsl-sampler.md)).

### <a name="export"></a>esportare

Usare **Export** per contrassegnare le funzioni che si impaccano in una libreria.

Ecco un esempio:

``` syntax
export float identity(float x)
{
    return x;
}
```

Contrassegnando la funzione **Identity** con la parola chiave **Export** , la funzione **Identity** sarà disponibile da una libreria per il collegamento successivo. Senza il contrassegno di **esportazione** , la funzione **Identity** non è disponibile per il collegamento successivo.

Il compilatore ignora la parola chiave **Export** per la compilazione non della libreria.

> [!Note]  
> La parola chiave **Export** richiede la \_47.dll D3dcompiler o una versione successiva della dll.
