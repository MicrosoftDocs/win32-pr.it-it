# <a name="devices"></a>Dispositivi

## <a name="in-this-section"></a>Contenuto della sezione

| Argomento | Descrizione |
|---|---|
| [Accesso dei dispositivi](deviceaccess/device-access-broker-api-portal.md)<br/> | Le API di accesso ai dispositivi consentono agli sviluppatori di scrivere app Windows Store per dispositivi personalizzati. L'API applica un criterio per concedere a un'app di Windows Store il livello di accesso appropriato a un dispositivo e, se viene concesso l'accesso, fornisce metodi per l'invio di codici di controllo per il funzionamento del dispositivo.<br/> |
| [AllJoyn API](/previous-versions/windows/desktop/alljoyn/alljoyn-api-portal)<br/> | AllJoyn è un framework e un protocollo di tipo DCOM per effettuare chiamate al metodo remotabili e/o inviare segnali unidireli tra applicazioni in un bus distribuito. È progettato per essere usato principalmente per scenari Internet delle cose tipo. <br/> |
| [API del punto di servizio](/uwp/api/Windows.Devices.PointOfService)<br/> | L'API Point of Service consente agli sviluppatori di applicazioni di accedere ai dispositivi periferici POS (Point of Service). Lo spazio dei nomi fornisce un'interfaccia indipendente dal fornitore per l'accesso ai dispositivi POS da un'app UWP.<br/> |
| [Risorse per le comunicazioni](/windows/desktop/DevIO/communications-resources)<br/>           | Una risorsa di comunicazione è un dispositivo fisico o logico che fornisce un singolo flusso di dati bidirezionale asincrono. Porte seriali, porte parallele, fax e modem sono esempi di risorse di comunicazione. Per ogni risorsa di comunicazione è disponibile un provider di servizi, costituito da una libreria o un driver, che consente alle applicazioni di accedere alla risorsa. <br/> |
| [Gestione dei dispositivi](/windows/desktop/DevIO/device-management)<br/>                         | La gestione dei dispositivi consente di notificare in modo uniforme a tutte le applicazioni e ai componenti di sistema le modifiche che possono influire sul funzionamento o sull'accesso alle risorse. Le applicazioni e il sistema usano ed elaborano gli eventi del dispositivo per sfruttare le nuove risorse quando diventano disponibili e per evitare la perdita di dati quando le risorse esistenti diventano non disponibili. <br/>                 |
| [Archiviazione avanzata](/previous-versions/windows/desktop/enstor/enhanced-storage-portal)<br/>                  | L'API Archiviazione avanzata consente l'autenticazione end-to-end coerente per i dispositivi di archiviazione personali che differiscono in base al fattore di forma. <br/>                                                                                                                                                                                                                                                 |
| [Individuazione delle funzioni](/previous-versions/windows/desktop/fundisc/fd-portal)<br/>                                 | Individuazione funzioni offre un'interfaccia a livello di codice uniforme per enumerare le risorse di sistema, ad esempio i dispositivi hardware, indipendentemente dal fatto che siano locali o connesse tramite una rete.<br/>                                                                                                                                                                                                 |
| [PnP-X](/previous-versions/windows/desktop/fundisc/pnp-x)<br/>                                                  | Plug and Play Extensions (PnP-X) consente a un sistema di computer di individuare i dispositivi di rete e di installarli nel sistema locale usando Plug and Play (PnP). <br/>                                                                                                                                                                                                                     |
| [API di posizione](/windows/desktop/LocationAPI/windows-location-api-portal)<br/>             | L'API Location consente di semplificare la programmazione in grado di riconoscere la posizione fornendo un modo standard per recuperare i dati sulla posizione utente e standardizzare i formati per i report dei dati sulla posizione. L'API Location gestisce automaticamente le transizioni tra i provider di dati di posizione e sceglie sempre il provider più accurato per la situazione corrente.<br/>                                 |
| [API per sensori](/windows/desktop/SensorsAPI/portal)<br/>                                     | L'API Sensor consente alle applicazioni di ottenere e usare i dati dai sensori in modo standardizzato.<br/>                                                                                                                                                                                                                                                                                     |
| [API per dispositivi UPnP](/windows/desktop/UPnP/universal-plug-and-play-start-page)<br/>                | Il framework UPnP consente la rete dinamica di appliance intelligenti, dispositivi wireless e PC. <br/>                                                                                                                                                                                                                                                                            |
| [Servizi Web nei dispositivi](/windows/desktop/WsdApi/wsd-portal)<br/>                           | L'API Servizi Web Microsoft nei dispositivi (WSDAPI) supporta l'implementazione di dispositivi e servizi controllati dal client e host di dispositivi conformi al profilo dispositivi per i servizi Web (DPWS). <br/>                                                                                                                                                                          |
| [Windows Media Device Manager 11 SDK](./wmdm/windows-media-device-manager-11-sdk.md)<br/> | Le applicazioni o i componenti di Windows Media Device Manager dispongono di un'API coerente per la comunicazione con un'ampia gamma di dispositivi, tra cui Media Transfer Protocol (MTP), Mass Archiviazione Class (MSC), RAPI e altri dispositivi compilati sia sulle versioni più recenti che precedenti della tecnologia Windows Media.<br/>                                                                    |
| [Windows Mixed Reality](https://msdn.microsoft.com/library/Mt844799(v=WIN.10).aspx)<br/>          | Windows Mixed Reality consente la creazione di un'esperienza di realtà mista per Microsoft HoloLens visori immersivi. Per altre informazioni, vedere [l'Dev Center](https://developer.microsoft.com/windows/mixed-reality).<br/>                                                                                                                                    |
| [Windows Dispositivi portatili](./windows-portable-devices.md)<br/>            | Windows I dispositivi portatili (WPD) consentono ai computer di comunicare con i supporti collegati e i dispositivi di archiviazione. La piattaforma WPD offre un modo flessibile e affidabile per comunicare con lettori musicali, dispositivi di archiviazione, telefoni cellulari, fotocamere e molti altri tipi di dispositivi connessi.<br/>                                                                                               |
