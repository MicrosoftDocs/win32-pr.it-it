# <a name="devices"></a>Dispositivi

## <a name="in-this-section"></a>Contenuto della sezione

| Argomento | Descrizione |
|---|---|
| [Accesso dei dispositivi](deviceaccess/device-access-broker-api-portal.md)<br/> | Le API di accesso ai dispositivi consentono agli sviluppatori di scrivere app di Windows Store per dispositivi personalizzati. L'API applica un criterio per concedere a un'app di Windows Store il livello di accesso appropriato a un dispositivo e, se viene concesso l'accesso, fornisce metodi per l'invio di codici di controllo per il funzionamento del dispositivo.<br/> |
| [API AllJoyn](/previous-versions/windows/desktop/alljoyn/alljoyn-api-portal)<br/> | AllJoyn è un Framework e un protocollo di tipo DCOM per l'esecuzione di chiamate a metodi utilizzabili in remoto e/o l'invio di segnali unidirezionali tra applicazioni su un bus distribuito. È destinato a essere utilizzato principalmente per gli scenari di tipo Internet delle cose. <br/> |
| [API punto di servizio](/uwp/api/Windows.Devices.PointOfService)<br/> | Il punto di servizio API consente agli sviluppatori di applicazioni di accedere ai dispositivi periferici POS (Point of Service). Lo spazio dei nomi fornisce un'interfaccia indipendente dal fornitore per accedere ai dispositivi POS da un'app UWP.<br/> |
| [Risorse di comunicazione](/windows/desktop/DevIO/communications-resources)<br/>           | Una risorsa di comunicazione è un dispositivo fisico o logico che fornisce un singolo flusso di dati asincrono bidirezionale. Le porte seriali, le porte parallele, le macchine fax e i modem sono esempi di risorse di comunicazione. Per ogni risorsa di comunicazione è disponibile un provider di servizi, costituito da una libreria o da un driver, che consente alle applicazioni di accedere alla risorsa. <br/> |
| [Gestione dei dispositivi](/windows/desktop/DevIO/device-management)<br/>                         | La gestione dei dispositivi consente di notificare in modo uniforme a tutte le applicazioni e i componenti di sistema delle modifiche che potrebbero influire sul funzionamento o sull'accesso alle risorse. Le applicazioni e il sistema usano ed elaborano gli eventi del dispositivo per sfruttare le nuove risorse quando diventano disponibili e per impedire la perdita di dati quando le risorse esistenti diventano non disponibili. <br/>                 |
| [Archiviazione avanzata](/previous-versions/windows/desktop/enstor/enhanced-storage-portal)<br/>                  | L'API di archiviazione avanzata consente l'autenticazione end-to-end coerente per i dispositivi di archiviazione personali che differiscono da un fattore di forma. <br/>                                                                                                                                                                                                                                                 |
| [Individuazione funzioni](/previous-versions/windows/desktop/fundisc/fd-portal)<br/>                                 | L'individuazione delle funzioni fornisce un'interfaccia a livello di codice uniforme per l'enumerazione delle risorse di sistema, ad esempio i dispositivi hardware, sia locali che connessi tramite una rete.<br/>                                                                                                                                                                                                 |
| [PnP-X](/previous-versions/windows/desktop/fundisc/pnp-x)<br/>                                                  | Plug and Play Extensions (PnP-X) consente a un sistema di computer di individuare i dispositivi di rete e di installarli nel sistema locale utilizzando Plug and Play (PnP). <br/>                                                                                                                                                                                                                     |
| [API location](/windows/desktop/LocationAPI/windows-location-api-portal)<br/>             | L'API location consente di semplificare la programmazione in base alla posizione fornendo un metodo standard per recuperare i dati sulla posizione dell'utente e i formati di standardizzazione per i report dei dati sulla posizione. L'API location gestisce automaticamente le transizioni tra i provider di dati della posizione e sceglie sempre il provider più accurato per la situazione corrente.<br/>                                 |
| [API per sensori](/windows/desktop/SensorsAPI/portal)<br/>                                     | L'API Sensor consente alle applicazioni di ottenere e usare i dati dei sensori in modo standardizzato.<br/>                                                                                                                                                                                                                                                                                     |
| [API per dispositivi UPnP](/windows/desktop/UPnP/universal-plug-and-play-start-page)<br/>                | Il Framework UPnP consente la rete dinamica di Appliance intelligenti, dispositivi wireless e PC. <br/>                                                                                                                                                                                                                                                                            |
| [Servizi Web nei dispositivi](/windows/desktop/WsdApi/wsd-portal)<br/>                           | L'API Microsoft Web Services on Devices (WSDAPI) supporta l'implementazione di dispositivi e servizi controllati dal client e degli host di dispositivi conformi al profilo dei dispositivi per i servizi Web (DPWS). <br/>                                                                                                                                                                          |
| [Windows Media Gestione dispositivi 11 SDK](./wmdm/windows-media-device-manager-11-sdk.md)<br/> | Le applicazioni o i componenti basati su Windows Media Gestione dispositivi hanno un'API coerente per la comunicazione con un'ampia gamma di dispositivi, tra cui il protocollo MTP (Media Transfer Protocol), la classe di archiviazione di massa (MSC), RAPI e altri dispositivi basati sulle versioni più recenti e precedenti della tecnologia Windows Media.<br/>                                                                    |
| [Windows Mixed Reality](https://msdn.microsoft.com/library/Mt844799(v=WIN.10).aspx)<br/>          | La realtà mista di Windows consente di creare un'esperienza di realtà mista per Microsoft HoloLens e altri auricolari immersivi. Per altre informazioni, vedere il [centro per sviluppatori di realtà miste](https://developer.microsoft.com/windows/mixed-reality).<br/>                                                                                                                                    |
| [Dispositivi portatili Windows](./windows-portable-devices.md)<br/>            | Dispositivi portatili Windows (WPD) consente ai computer di comunicare con i dispositivi di archiviazione e i supporti collegati. WPD offre una soluzione flessibile e affidabile per la comunicazione dei computer con lettori musicali, dispositivi di archiviazione, telefoni cellulari, fotocamere e molti altri tipi di dispositivi connessi.<br/>                                                                                               |
