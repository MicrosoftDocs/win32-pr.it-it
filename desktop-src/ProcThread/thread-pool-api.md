<span data-ttu-id="987b6-104">Il Application Programming Interface del pool di thread (API) utilizza una struttura basata su oggetti.</span><span class="sxs-lookup"><span data-stu-id="987b6-104">The thread pool application programming interface (API) uses an object-based design.</span></span> <span data-ttu-id="987b6-105">Ognuno degli oggetti seguenti è rappresentato da una struttura di dati in modalità utente:</span><span class="sxs-lookup"><span data-stu-id="987b6-105">Each of the following objects is represented by a user-mode data structure:</span></span>

-   <span data-ttu-id="987b6-109">Un gruppo di pulitura è associato a un set di oggetti che generano callback.</span><span class="sxs-lookup"><span data-stu-id="987b6-109">A clean-up group is associated with a set of callback-generating objects.</span></span> <span data-ttu-id="987b6-110">Sono presenti funzioni che attendono e rilasciano tutti gli oggetti che sono membri di ogni gruppo di pulizia.</span><span class="sxs-lookup"><span data-stu-id="987b6-110">Functions exist to wait on and release all objects that are members of each clean-up group.</span></span> <span data-ttu-id="987b6-111">In questo modo l'applicazione non tiene traccia di tutti gli oggetti creati.</span><span class="sxs-lookup"><span data-stu-id="987b6-111">This frees the application from keeping track of all the objects it has created.</span></span>

-   <span data-ttu-id="987b6-116">Un oggetto timer controlla la pianificazione dei callback.</span><span class="sxs-lookup"><span data-stu-id="987b6-116">A timer object controls the scheduling of callbacks.</span></span> <span data-ttu-id="987b6-117">Ogni volta che un timer scade, il relativo callback viene inserito nel pool di lavoro.</span><span class="sxs-lookup"><span data-stu-id="987b6-117">Each time a timer expires, its callback is posted to its worker pool.</span></span> <span data-ttu-id="987b6-118">L'impostazione di un timer non può avere esito negativo a causa di risorse insufficienti.</span><span class="sxs-lookup"><span data-stu-id="987b6-118">Setting a timer cannot fail due to lack of resources.</span></span>
-   <span data-ttu-id="987b6-119">Un oggetto Wait fa in modo che un thread del cameriere attenda un handle di attesa.</span><span class="sxs-lookup"><span data-stu-id="987b6-119">A wait object causes a waiter thread to wait on a waitable handle.</span></span> <span data-ttu-id="987b6-120">Dopo che l'attesa è stata soddisfatta o il periodo di timeout scade, il thread del cameriere invia il callback degli oggetti di attesa al pool di lavoro dell'attesa.</span><span class="sxs-lookup"><span data-stu-id="987b6-120">After the wait is satisfied or the time-out period expires, the waiter thread posts the wait objects' callback to the wait's worker pool.</span></span> <span data-ttu-id="987b6-121">L'impostazione di un'attesa non può avere esito negativo a causa di risorse insufficienti.</span><span class="sxs-lookup"><span data-stu-id="987b6-121">Setting a wait cannot fail due to lack of resources.</span></span>
-   <span data-ttu-id="987b6-122">Un oggetto I/O associa un handle di file alla porta di completamento I/O per il pool di thread.</span><span class="sxs-lookup"><span data-stu-id="987b6-122">An I/O object associates a file handle with the I/O completion port for the thread pool.</span></span> <span data-ttu-id="987b6-123">Quando un'operazione di I/O asincrona viene completata, un thread di lavoro preleva lo stato dell'operazione e chiama il callback dell'oggetto I/O.</span><span class="sxs-lookup"><span data-stu-id="987b6-123">When an asynchronous I/O operation completes, a worker thread picks up the status of the operation and calls the I/O object's callback.</span></span>
