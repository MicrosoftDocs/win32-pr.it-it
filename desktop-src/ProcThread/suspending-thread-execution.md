# <a name="suspending-thread-execution"></a><span data-ttu-id="52339-104">Sospensione dell'esecuzione del thread</span><span class="sxs-lookup"><span data-stu-id="52339-104">Suspending Thread Execution</span></span>

<span data-ttu-id="52339-105">Un thread può sospendere e riprendere l'esecuzione di un altro thread.</span><span class="sxs-lookup"><span data-stu-id="52339-105">A thread can suspend and resume the execution of another thread.</span></span> <span data-ttu-id="52339-106">Mentre un thread è sospeso, non è pianificato per l'ora del processore.</span><span class="sxs-lookup"><span data-stu-id="52339-106">While a thread is suspended, it is not scheduled for time on the processor.</span></span>

<span data-ttu-id="52339-107">Se un thread viene creato in uno stato sospeso (con il flag di [**creazione \_ sospesa**](process-creation-flags.md) ), non inizia l'esecuzione fino a quando un altro thread non chiama la funzione [**ResumeThread**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-resumethread) con un handle per il thread sospeso.</span><span class="sxs-lookup"><span data-stu-id="52339-107">If a thread is created in a suspended state (with the [**CREATE\_SUSPENDED**](process-creation-flags.md) flag), it does not begin to execute until another thread calls the [**ResumeThread**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-resumethread) function with a handle to the suspended thread.</span></span> <span data-ttu-id="52339-108">Questa operazione può essere utile per inizializzare lo stato del thread prima che inizi l'esecuzione.</span><span class="sxs-lookup"><span data-stu-id="52339-108">This can be useful for initializing the thread's state before it begins to execute.</span></span> <span data-ttu-id="52339-109">La sospensione di un thread in fase di creazione può essere utile per la sincronizzazione monouso, perché in questo modo il thread sospeso eseguirà il punto iniziale del codice quando si chiama **ResumeThread**.</span><span class="sxs-lookup"><span data-stu-id="52339-109">Suspending a thread at creation can be useful for one-time synchronization, because this ensures that the suspended thread will execute the starting point of its code when you call **ResumeThread**.</span></span>

<span data-ttu-id="52339-110">La funzione [**SuspendThread**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-suspendthread) non deve essere usata per la sincronizzazione dei thread perché non controlla il punto nel codice in cui viene sospesa l'esecuzione del thread.</span><span class="sxs-lookup"><span data-stu-id="52339-110">The [**SuspendThread**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-suspendthread) function is not intended to be used for thread synchronization because it does not control the point in the code at which the thread's execution is suspended.</span></span> <span data-ttu-id="52339-111">Questa funzione è progettata principalmente per essere utilizzata dai debugger.</span><span class="sxs-lookup"><span data-stu-id="52339-111">This function is primarily designed for use by debuggers.</span></span>

<span data-ttu-id="52339-112">Un thread può restituire temporaneamente l'esecuzione per un intervallo specificato chiamando le funzioni [**Sleep**](/windows/win32/api/synchapi/nf-synchapi-sleep) o [**SleepEx**](/windows/win32/api/synchapi/nf-synchapi-sleepex) . questa operazione è utile in particolare nei casi in cui il thread risponde all'interazione dell'utente, perché può ritardare l'esecuzione per un tempo sufficiente a consentire agli utenti di osservare i risultati delle proprie azioni.</span><span class="sxs-lookup"><span data-stu-id="52339-112">A thread can temporarily yield its execution for a specified interval by calling the [**Sleep**](/windows/win32/api/synchapi/nf-synchapi-sleep) or [**SleepEx**](/windows/win32/api/synchapi/nf-synchapi-sleepex) functions This is useful particularly in cases where the thread responds to user interaction, because it can delay execution long enough to allow users to observe the results of their actions.</span></span> <span data-ttu-id="52339-113">Durante l'intervallo di sospensione, il thread non è pianificato per l'ora del processore.</span><span class="sxs-lookup"><span data-stu-id="52339-113">During the sleep interval, the thread is not scheduled for time on the processor.</span></span>

<span data-ttu-id="52339-114">La funzione [**SwitchToThread**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-switchtothread) è simile a [**Sleep**](/windows/win32/api/synchapi/nf-synchapi-sleep) e [**SleepEx**](/windows/win32/api/synchapi/nf-synchapi-sleepex), ad eccezione del fatto che non è possibile specificare l'intervallo.</span><span class="sxs-lookup"><span data-stu-id="52339-114">The [**SwitchToThread**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-switchtothread) function is similar to [**Sleep**](/windows/win32/api/synchapi/nf-synchapi-sleep) and [**SleepEx**](/windows/win32/api/synchapi/nf-synchapi-sleepex), except that you cannot specify the interval.</span></span> <span data-ttu-id="52339-115">**SwitchToThread** consente al thread di rinunciare all'intervallo di tempo.</span><span class="sxs-lookup"><span data-stu-id="52339-115">**SwitchToThread** allows the thread to give up its time slice.</span></span>
