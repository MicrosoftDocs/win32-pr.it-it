## <a name="root-parameters-and-arguments"></a><span data-ttu-id="4f3ff-111">Parametri e argomenti radice</span><span class="sxs-lookup"><span data-stu-id="4f3ff-111">Root parameters and arguments</span></span>

<span data-ttu-id="4f3ff-112">Una **firma radice** è simile a una firma della funzione API, che determina i tipi di dati che gli shader dovrebbero aspettarsi, ma non definisce la memoria o i dati effettivi.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-112">A **root signature** is similar to an API function signature, it determines the types of data the shaders should expect, but does not define the actual memory or data.</span></span> <span data-ttu-id="4f3ff-113">Un **parametro radice** è una voce nella firma radice.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-113">A **root parameter** is one entry in the root signature.</span></span> <span data-ttu-id="4f3ff-114">I valori effettivi dei parametri radice impostati e modificati in fase di esecuzione sono detti **argomenti radice**.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-114">The actual values of root parameters set and changed at runtime are called **root arguments**.</span></span> <span data-ttu-id="4f3ff-115">La modifica degli argomenti radice modifica i dati letti dagli shader.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-115">Changing the root arguments changes the data that the shaders read.</span></span>

## <a name="root-constants-descriptors-and-tables"></a><span data-ttu-id="4f3ff-116">Costanti radice, descrittori e tabelle</span><span class="sxs-lookup"><span data-stu-id="4f3ff-116">Root constants, descriptors and tables</span></span>

<span data-ttu-id="4f3ff-117">La firma radice può contenere tre tipi di parametri. costanti radice (costanti inline negli argomenti radice), descrittori radice (descrittori inline negli argomenti radice) e tabelle dei descrittori (puntatori a un intervallo di descrittori nell'heap del descrittore).</span><span class="sxs-lookup"><span data-stu-id="4f3ff-117">The root signature can contain three types of parameters; root constants (constants inlined in the root arguments), root descriptors (descriptors inlined in the root arguments), and descriptor tables (pointers to a range of descriptors in the descriptor heap).</span></span>

<span data-ttu-id="4f3ff-118">Le costanti radice sono valori inline a 32 bit che vengono visualizzati nello shader come buffer costante.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-118">The root constants are inline 32-bit values that show up in the shader as a constant buffer.</span></span>

<span data-ttu-id="4f3ff-119">I descrittori radice resi inline devono contenere descrittori a cui si accede più spesso, sebbene sia limitato a CBVs e a buffer UAV o SRV non elaborati o strutturati.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-119">The inlined root descriptors should contain descriptors that are accessed most often, though is limited to CBVs, and raw or structured UAV or SRV buffers.</span></span> <span data-ttu-id="4f3ff-120">Un tipo più complesso, ad esempio una trama 2D SRV, non può essere utilizzato come descrittore radice.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-120">A more complex type, such as a 2D texture SRV, cannot be used as a root descriptor.</span></span> <span data-ttu-id="4f3ff-121">I descrittori radice non includono un limite di dimensioni, pertanto non è possibile eseguire il controllo out-of-Bounds, a differenza dei descrittori negli heap dei descrittori, che includono una dimensione.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-121">Root descriptors do not include a size limit, so there can be no out-of-bounds checking, unlike descriptors in descriptor heaps, which do include a size.</span></span>

<span data-ttu-id="4f3ff-122">Le voci della tabella descrittore nelle firme radice contengono il descrittore, il nome di associazione dello shader HLSL e il flag di visibilità.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-122">Descriptor table entries within root signatures contain the descriptor, HLSL shader bind name and visibility flag.</span></span> <span data-ttu-id="4f3ff-123">Per informazioni dettagliate sui nomi degli shader, vedere il [modello di shader 5,1](/windows/desktop/direct3dhlsl/shader-model-5-1) .</span><span class="sxs-lookup"><span data-stu-id="4f3ff-123">Refer to [Shader Model 5.1](/windows/desktop/direct3dhlsl/shader-model-5-1) for details of shader names.</span></span> <span data-ttu-id="4f3ff-124">In alcuni componenti hardware è possibile che si verifichi un miglioramento delle prestazioni limitando la visibilità dei descrittori alle fasi dello shader che le richiedono (vedere la [**\_ \_ visibilità dello shader D3D12**](/windows/desktop/api/d3d12/ne-d3d12-d3d12_shader_visibility)).</span><span class="sxs-lookup"><span data-stu-id="4f3ff-124">On some hardware, there can be a performance gain from only making descriptors visible to the shader stages that require them (refer to [**D3D12\_SHADER\_VISIBILITY**](/windows/desktop/api/d3d12/ne-d3d12-d3d12_shader_visibility)).</span></span>

![voce della tabella descrittore radice](images/root-descriptor-table.png)

<span data-ttu-id="4f3ff-126">Il layout della firma radice è piuttosto flessibile, con alcuni vincoli imposti su hardware meno idoneo.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-126">The layout of the root signature is quite flexible, with some constraints imposed on less capable hardware.</span></span> <span data-ttu-id="4f3ff-127">Indipendentemente dal livello di hardware, le applicazioni devono sempre provare a rendere la firma radice di dimensioni ridotte, in base alle esigenze, per la massima efficienza.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-127">Regardless of the level of hardware, applications should always try to make the root signature as small as needed for maximum efficiency.</span></span> <span data-ttu-id="4f3ff-128">Le applicazioni possono comportare la disattivazione della presenza di più tabelle dei descrittori nella firma radice, ma meno spazio per le costanti radice o viceversa.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-128">Applications can trade off having more descriptor tables in the root signature but less room for root constants, or vice versa.</span></span>

<span data-ttu-id="4f3ff-129">Il contenuto della firma radice (le tabelle dei descrittori, le costanti radice e i descrittori radice) a cui l'applicazione è associata automaticamente viene sottoposta a controllo delle versioni dal driver D3D12 ogni volta che una parte del contenuto cambia tra le chiamate di/dispatch (Compute).</span><span class="sxs-lookup"><span data-stu-id="4f3ff-129">The contents of the root signature (the descriptor tables, root constants and root descriptors) that the application has bound automatically get versioned by the D3D12 driver whenever any part of the contents change between draw (graphics)/dispatch (compute) calls.</span></span> <span data-ttu-id="4f3ff-130">Ogni operazione di estrazione/invio ottiene quindi un set univoco completo dello stato della firma radice.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-130">So each draw/dispatch gets a unique full set of root signature state.</span></span>

<span data-ttu-id="4f3ff-131">Idealmente, esistono gruppi di oggetti di stato della pipeline (PSO) che condividono la stessa firma radice.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-131">Ideally, there are groups of Pipeline State Objects (PSOs) that share the same root signature.</span></span> <span data-ttu-id="4f3ff-132">Dopo aver impostato una firma radice sulla pipeline, tutte le associazioni che definisce (tabelle descrittori, descrittori, costanti) possono essere impostate o modificate singolarmente, inclusa l'ereditarietà in bundle.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-132">After a root signature is set on the pipeline, all the bindings that it defines (descriptor tables, descriptors, constants) can each be individually set or changed, including inheritance into bundles.</span></span>

<span data-ttu-id="4f3ff-133">Un'app può comportare un proprio compromesso tra il numero di tabelle di descrittori che desiderano i descrittori inline (che prevedono una maggiore quantità di spazio, ma la rimozione di un riferimento indiretto), che desiderano nella firma radice.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-133">An app can make its own tradeoff between how many descriptor tables it wants verses inline descriptors (which take more space but remove an indirection) verses inline constants (which have no indirection) they want in the root signature.</span></span> <span data-ttu-id="4f3ff-134">Le applicazioni devono utilizzare la firma radice nel modo più sporadico possibile, basandosi sulla memoria controllata dall'applicazione, ad esempio heap e heap di descrittori che puntano a tali elementi per rappresentare i dati in blocco.</span><span class="sxs-lookup"><span data-stu-id="4f3ff-134">Applications should use the root signature as sparingly as possible, relying on application controlled memory such as heaps and descriptor heaps pointing into them to represent bulk data.</span></span>
