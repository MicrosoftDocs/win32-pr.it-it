# <a name="t-telephony-api"></a>T (API Di telefonia)

<dl> <dt>

<span id="tapi2.t1_e1_tapgloss"></span><span id="TAPI2.T1_E1_TAPGLOSS"></span>**T1/E1**
</dt> <dd>

Collegamento di trasmissione digitale con una capacità di 1.544 Mbps. Negli Stati Uniti questo collegamento è denominato *T1.* Nell'Unione Europea si chiama *E1*. Vedere [*megabit al secondo (Mbps).*](m-tapgloss.md)

</dd> <dt>

<span id="tapi2.tapi_tapgloss"></span><span id="TAPI2.TAPI_TAPGLOSS"></span>**Tapi**
</dt> <dd>

Vedere *Telephony Application Programming Interface (TAPI)*.

</dd> <dt>

<span id="tapi2.tapisrv_tapgloss"></span><span id="TAPI2.TAPISRV_TAPGLOSS"></span>**tapisrv**
</dt> <dd>

Un'applicazione di servizio di supporto che fornisce un contesto di esecuzione per la libreria a collegamento dinamico TAPI per svolgere compiti come l'allocazione di memoria o l'accesso a strutture di dati che sono state scambiate su disco.

</dd> <dt>

<span id="tapi2.tdd_tapgloss"></span><span id="TAPI2.TDD_TAPGLOSS"></span>**Tdd**
</dt> <dd>

Vedere *Dispositivi di telecomunicazione per sordi (TDD).*

</dd> <dt>

<span id="tapi2.telecommucations_devices_for_the_deaf_tdd__tapgloss"></span><span id="TAPI2.TELECOMMUCATIONS_DEVICES_FOR_THE_DEAF_TDD__TAPGLOSS"></span>**Dispositivi di telecomunicazione per sordi (TDD)**
</dt> <dd>

Dispositivo dati per la trasmissione di segnali codificati. In genere, il dispositivo è un semplice terminale del computer costituito da tastiera, coppia acustica e schermo e viene eseguito a 300 baud.

</dd> <dt>

<span id="tapi2.telephony_tapgloss"></span><span id="TAPI2.TELEPHONY_TAPGLOSS"></span>**Telefonia**
</dt> <dd>

La scienza della trasmissione di segnali vocali, dati, video o immagini su una distanza usando la tecnologia che integra i computer con la rete telefonica.

</dd> <dt>

<span id="tapi2.telephony_application_programming_interface_tapi__tapgloss"></span><span id="TAPI2.TELEPHONY_APPLICATION_PROGRAMMING_INTERFACE_TAPI__TAPGLOSS"></span>**TapI (Telephony Application Programming Interface)**
</dt> <dd>

Set di funzioni che consente la programmazione di dispositivi basati su linea telefonica in modo indipendente dal dispositivo, offrendo la telefonia personale agli utenti. TAPI supporta sia il riconoscimento vocale che la trasmissione dei dati, consente un'ampia gamma di dispositivi terminali e supporta tipi di connessione complessi e tecniche di gestione delle chiamate, ad esempio conferenze telefoniche, attesa chiamate e casella vocale. TAPI consente di controllare tutti gli elementi dell'utilizzo del telefono, dalla semplice chiamata in dial-and-speak alla posta elettronica internazionale, all'interno di applicazioni sviluppate per Microsoft Windows. Vedere Panoramica [della telefonia Microsoft](./microsoft-telephony-overview.md)

</dd> <dt>

<span id="tapi2.telephony_service_provider_interface_tspi__tapgloss"></span><span id="TAPI2.TELEPHONY_SERVICE_PROVIDER_INTERFACE_TSPI__TAPGLOSS"></span>**TSPI (Telephony Service Provider Interface)**
</dt> <dd>

Strumento per la creazione di provider di servizi per i sistemi Windows Microsoft. TSPI definisce il modo in cui la rete condivide le informazioni Windows telefonia, che a sua volta dialoga con l'API, che dialoga con le Windows telefonia. Per altre informazioni, vedere [Telephony Service Provider Interface (TSPI)](./telephony-service-provider-interface-tspi-.md).

</dd> <dt>

<span id="tapi2.third_party_call_control_tapgloss"></span><span id="TAPI2.THIRD_PARTY_CALL_CONTROL_TAPGLOSS"></span>**controllo delle chiamate di terze parti**
</dt> <dd>

Modello di controllo delle chiamate di entità che consente a un'applicazione di stabilire o rispondere a una chiamata tra due parti.

</dd> <dt>

<span id="tapi2.time_to_live_ttl__tapgloss"></span><span id="TAPI2.TIME_TO_LIVE_TTL__TAPGLOSS"></span>**durata (TTL)**
</dt> <dd>

Valore compreso nell'intervallo da 0 a 255 che definisce l'ambito all'interno del quale i pacchetti multicast devono essere inviati tramite una rete usando il protocollo IP (Internet Protocol). L'ambito è definito in termini di modalità di destinazione locale o remota di un pacchetto. Ogni router decrementa il TTL di uno. Quando il valore raggiunge un limite inferiore predefinito, il router genera il pacchetto. I requisiti di backbone multicast correnti, disponibili nel sito ftp://ftp.isi.edu/mbone/faq.txt, definiscono gli ambiti standard seguenti: rete locale, 1; sito locale, 15; region, 63; world, 127. Altre impostazioni possono avere un significato locale. Ad esempio, 31 potrebbe indicare tutti i siti all'interno di una determinata organizzazione.

</dd> <dt>

<span id="tapi2.tspi_tapgloss"></span><span id="TAPI2.TSPI_TAPGLOSS"></span>**TSPI**
</dt> <dd>

Vedere [Telephony Service Provider Interface (TSPI)](./telephony-service-provider-interface-tspi-.md).

</dd> <dt>

<span id="tapi2.ttl_tapgloss"></span><span id="TAPI2.TTL_TAPGLOSS"></span>**Ttl**
</dt> <dd>

Vedere *time to live (TTL)*.

</dd> <dt>

<span id="tapi2.twisted_pair_tapgloss"></span><span id="TAPI2.TWISTED_PAIR_TAPGLOSS"></span>**coppia contorta**
</dt> <dd>

Cablaggio telefonico regolare. Ogni cavo telefonico è in realtà una coppia di cavi.

</dd> </dl>
