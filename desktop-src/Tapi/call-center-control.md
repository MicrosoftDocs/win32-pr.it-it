# <a name="call-center-control"></a>Controllo Call Center

TAPI 2,2 di base può essere usato per gestire i Call Center e altri elementi dell'infrastruttura di rete di telefonia (ad esempio, i server IVR e Voice mail) tramite il controllo delle chiamate di terze parti. Per semplificare la creazione di proxy ACD che vengono eseguiti nei server, le funzioni sono state aggiunte a TAPI 2,2 per offrire assistenza aggiuntiva agli sviluppatori di applicazioni.

Gli argomenti seguenti descrivono le funzionalità di TAPI 2,2 che è possibile usare per implementare i controlli del Call Center:

-   [Modellazione di un Call Center](modeling-of-a-call-center.md)
-   [Stazioni](stations.md)
-   [Composizione predittiva](predictive-dialing.md)
-   [Code di chiamata e punti di route](call-queues-and-route-points.md)
-   [Monitoraggio e controllo degli agenti ACD](acd-agent-monitoring-and-control.md)
-   [Controllo stato stazione](station-status-control.md)
-   [Controllo stato stazione](station-status-control.md)
-   [Timer stato chiamata](call-state-timer.md)
-   [Timer eventi multimediali](media-event-timers.md)

TAPI 3. x è l'API preferita per la scrittura di applicazioni agente ACD. Per informazioni sull'uso di queste interfacce e metodi, vedere la sezione [controlli del Call Center](./about-call-center-controls.md) . Tuttavia, TAPI 2,2 rimane utilizzabile per i gruppi di applicazioni ACD completi, se lo si desidera.
