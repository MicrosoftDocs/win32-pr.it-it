<span data-ttu-id="4ccef-104">Le funzionalità multimediali sono diverse rispetto a TAPI 2,2 (TAPI/C) rispetto a TAPI 3 (COM), in gran parte perché l'API COM ha accesso ai provider di servizi multimediali (MSPs).</span><span class="sxs-lookup"><span data-stu-id="4ccef-104">Media features are different with TAPI 2.2 (TAPI/C) as opposed to TAPI 3 (COM), largely because the COM API has access to media service providers (MSPs).</span></span> <span data-ttu-id="4ccef-105">Per ulteriori informazioni su MSPs, vedere [informazioni sul provider di servizi multimediali (msp)](./about-the-media-service-provider-msp-.md).</span><span class="sxs-lookup"><span data-stu-id="4ccef-105">For more information about MSPs, see [About The Media Service Provider (MSP)](./about-the-media-service-provider-msp-.md).</span></span> <span data-ttu-id="4ccef-106">Per altre informazioni sulle operazioni dei flussi multimediali, vedere [controllo multimediale](./device-control.md).</span><span class="sxs-lookup"><span data-stu-id="4ccef-106">For more information about media stream operations, see [Media Control](./device-control.md).</span></span>

<span data-ttu-id="4ccef-107">I due concetti più importanti per un'applicazione sono il tipo di supporto (o la modalità) e il flusso.</span><span class="sxs-lookup"><span data-stu-id="4ccef-107">The two most important concepts for an application are the media type (or mode) and the stream.</span></span> <span data-ttu-id="4ccef-108">Il tipo è il formato in cui vengono trasmessi i dati.</span><span class="sxs-lookup"><span data-stu-id="4ccef-108">The type is the form in which data is transmitted.</span></span> <span data-ttu-id="4ccef-109">Per altre informazioni e per un elenco di tipi definiti da TAPI, [vedere \_ costanti LINEMEDIAMODE](linemediamode--constants.md).</span><span class="sxs-lookup"><span data-stu-id="4ccef-109">For more information and a list of TAPI-defined types, see [LINEMEDIAMODE\_ Constants](linemediamode--constants.md).</span></span> <span data-ttu-id="4ccef-110">Il flusso multimediale è il flusso di dati effettivo.</span><span class="sxs-lookup"><span data-stu-id="4ccef-110">The media stream is the actual stream of data.</span></span> <span data-ttu-id="4ccef-111">Un MSP può fornire accesso diretto al flusso.</span><span class="sxs-lookup"><span data-stu-id="4ccef-111">An MSP can provide direct access to the stream.</span></span> <span data-ttu-id="4ccef-112">Le applicazioni TAPI 2,2 hanno un certo accesso, ma principalmente fanno riferimento ad altre API per implementare tali controlli.</span><span class="sxs-lookup"><span data-stu-id="4ccef-112">TAPI 2.2 applications have some access, but primarily reference other APIs to implement such controls.</span></span>

<span data-ttu-id="4ccef-113">Queste API includono l'API di forma d'onda, l'API di comunicazione e l'interfaccia di controllo multimediale (MCI).</span><span class="sxs-lookup"><span data-stu-id="4ccef-113">These APIs include the Waveform API, the Comm API, and the Media Control Interface (MCI).</span></span> <span data-ttu-id="4ccef-114">L'API per la forma d'onda viene usata per la programmazione multimediale, l'API di comunicazione è il set di funzioni di comunicazione fornite da Platform Software Development Kit (SDK) e il MCI fornisce un'interfaccia generalizzata di alto livello per il controllo dei dispositivi multimediali.</span><span class="sxs-lookup"><span data-stu-id="4ccef-114">The Waveform API is used for multimedia programming, the Comm API is the set of communications functions provided by the Platform Software Development Kit (SDK), and the MCI provides a high-level generalized interface for controlling media devices.</span></span>

<span data-ttu-id="4ccef-115">Per i dispositivi linea, ad esempio, un'applicazione può usare TAPI 2,2 per stabilire una connessione a un'altra stazione.</span><span class="sxs-lookup"><span data-stu-id="4ccef-115">For example, for line devices, an application can use TAPI 2.2 to establish a connection to another station.</span></span> <span data-ttu-id="4ccef-116">Una volta stabilita la connessione, l'applicazione può quindi usare l'API di forma d'onda (o l'API MCI WaveAudio) sul dispositivo associato per riprodurre (inviare) e registrare (ricevere) dati audio sulla connessione.</span><span class="sxs-lookup"><span data-stu-id="4ccef-116">Once the connection is established, the application can then use the Waveform API (or the MCI Waveaudio API) on the associated device to play back (send) and record (receive) audio data over the connection.</span></span> <span data-ttu-id="4ccef-117">Analogamente, se il flusso del supporto di connessione è da un modem, un'applicazione utilizzerà le estensioni di configurazione del modem dell'API di comunicazione per controllare il flusso multimediale.</span><span class="sxs-lookup"><span data-stu-id="4ccef-117">Similarly, if the connection media stream is from a modem, an application would use the modem configuration extensions of the Communications API to control the media stream.</span></span>

<span data-ttu-id="4ccef-118">Per fornire a TAPI 2,2 l'accesso tramite flusso multimediale a un telefono o a una chiamata su un dispositivo a linee, il provider di servizi deve implementare sia la telefonia SPI che il flusso multimediale appropriato SPI o l'interfaccia DDI (Device-Driver Interface).</span><span class="sxs-lookup"><span data-stu-id="4ccef-118">To provide TAPI 2.2 with media-stream access to either a phone or a call on a line device, the service provider must implement both the Telephony SPI and the appropriate media stream SPI or device-driver interface (DDI).</span></span> <span data-ttu-id="4ccef-119">Il provider di servizi può supportare linee e telefoni simultaneamente.</span><span class="sxs-lookup"><span data-stu-id="4ccef-119">The service provider can support lines and phones simultaneously.</span></span>

<span data-ttu-id="4ccef-120">Poiché queste classi di dispositivi e operazioni di flusso multimediale funzionano indipendentemente l'una dall'altra, il coordinamento del loro utilizzo deve verificarsi a livello di applicazione.</span><span class="sxs-lookup"><span data-stu-id="4ccef-120">Because these device classes and media stream operations function independently of one another, coordination of their usage must occur at the application level.</span></span> <span data-ttu-id="4ccef-121">Per più applicazioni che condividono chiamate e flussi multimediali è probabile che sia necessario il coordinamento delle attività a livello di applicazione per impedire l'uso in conflitto di TAPI e dell'API del flusso multimediale.</span><span class="sxs-lookup"><span data-stu-id="4ccef-121">Multiple applications that share calls and media streams will likely require coordination of their activities at the application level to prevent conflicting usage of TAPI and the media stream API.</span></span>

<span data-ttu-id="4ccef-122">TAPI segnala le modifiche apportate al tipo di flusso multimediale (voce, fax, modem dati e così via) alle applicazioni partecipanti.</span><span class="sxs-lookup"><span data-stu-id="4ccef-122">TAPI reports changes in the type of media stream (voice, fax, data modem, and so on) to participating applications.</span></span> <span data-ttu-id="4ccef-123">Questo processo viene talvolta definito classificazione delle [*chiamate*](c-tapgloss.md).</span><span class="sxs-lookup"><span data-stu-id="4ccef-123">This process is sometimes referred to as [*call classification*](c-tapgloss.md).</span></span> <span data-ttu-id="4ccef-124">Il meccanismo utilizzato per determinare il tipo di flusso multimediale è specifico del provider di servizi.</span><span class="sxs-lookup"><span data-stu-id="4ccef-124">The mechanism used to determine the type of media stream is specific to the service provider.</span></span> <span data-ttu-id="4ccef-125">Un provider di servizi, ad esempio, può filtrare il flusso multimediale per l'energia o i toni che caratterizzano il tipo di supporto oppure può utilizzare la suoneria distinta, i dati scambiati nei messaggi in rete o la conoscenza del chiamante o l'ID chiamato per effettuare questa determinazione.</span><span class="sxs-lookup"><span data-stu-id="4ccef-125">For example, a service provider may filter the media stream for energy or tones that characterize the media type, or it may use distinctive ringing, data exchanged in messages over the network, or knowledge about the caller or called ID to make this determination.</span></span>
