# <a name="file-terminals"></a>Terminali file

In TAPI 3.1 l'oggetto terminale viene espanso per includere un terminale file. I terminali di file offrono un modo semplice per eseguire operazioni comuni, ad esempio la registrazione di un flusso audio o di flussi in un file o la successiva riproduzione dell'audio registrato in un flusso.

Per modificare i terminali di file vengono usati cinque oggetti di base:

-   Terminale di riproduzione file
-   Terminale di registrazione file
-   Traccia di riproduzione file
-   Traccia di registrazione file
-   Evento file

Gli oggetti Terminale file possono essere pensati come un'astrazione per l'archiviazione, che in genere è un file. Le tracce corrispondono ai singoli flussi di dati audio in tale archiviazione, ad esempio quando viene registrata la voce.

Per altre informazioni sui terminali di file, vedere:

-   [Terminale di riproduzione file e terminale di registrazione file](file-playback-terminal-and-file-recording-terminal.md)
-   [Tenere traccia dei terminali](track-terminals.md)

Per altre informazioni ed esempi di codice ed elenchi di attività correlati ai terminali di file, vedere [Uso dei terminali file](using-file-terminals.md).
