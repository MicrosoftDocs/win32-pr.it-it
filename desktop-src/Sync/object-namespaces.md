# <a name="object-namespaces"></a><span data-ttu-id="4cbbc-103">Spazi dei nomi degli oggetti</span><span class="sxs-lookup"><span data-stu-id="4cbbc-103">Object Namespaces</span></span>

<span data-ttu-id="4cbbc-106">Un processo può creare uno spazio dei nomi privato utilizzando la funzione [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) .</span><span class="sxs-lookup"><span data-stu-id="4cbbc-106">A process can create a private namespace using the [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) function.</span></span> <span data-ttu-id="4cbbc-107">Questa funzione richiede l'impostazione di un *limite* che definisce la modalità di isolamento degli oggetti nello spazio dei nomi.</span><span class="sxs-lookup"><span data-stu-id="4cbbc-107">This function requires that you specify a *boundary* that defines how the objects in the namespace are to be isolated.</span></span> <span data-ttu-id="4cbbc-108">Il chiamante deve essere all'interno del limite specificato affinché l'operazione di creazione abbia esito positivo.</span><span class="sxs-lookup"><span data-stu-id="4cbbc-108">The caller must be within the specified boundary for the create operation to succeed.</span></span> <span data-ttu-id="4cbbc-109">Per specificare un limite, usare le funzioni [**CreateBoundaryDescriptor**](/windows/desktop/api/WinBase/nf-winbase-createboundarydescriptora) e [**AddSIDToBoundaryDescriptor**](/windows/win32/api/namespaceapi/nf-namespaceapi-addsidtoboundarydescriptor) .</span><span class="sxs-lookup"><span data-stu-id="4cbbc-109">To specify a boundary, use the [**CreateBoundaryDescriptor**](/windows/desktop/api/WinBase/nf-winbase-createboundarydescriptora) and [**AddSIDToBoundaryDescriptor**](/windows/win32/api/namespaceapi/nf-namespaceapi-addsidtoboundarydescriptor) functions.</span></span>

<span data-ttu-id="4cbbc-110">Il parametro *lpAliasPrefix* di [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) funge da nome dello spazio dei nomi.</span><span class="sxs-lookup"><span data-stu-id="4cbbc-110">The *lpAliasPrefix* parameter of [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) serves as the name of the namespace.</span></span> <span data-ttu-id="4cbbc-111">Ogni spazio dei nomi viene identificato in modo univoco in base al nome e ai limiti.</span><span class="sxs-lookup"><span data-stu-id="4cbbc-111">Each namespace is uniquely identified by its name and boundaries.</span></span> <span data-ttu-id="4cbbc-112">Il sistema supporta più spazi dei nomi privati con lo stesso nome, purché specifichino limiti diversi.</span><span class="sxs-lookup"><span data-stu-id="4cbbc-112">The system supports multiple private namespaces with the same name, as long as they specify different boundaries.</span></span>

<span data-ttu-id="4cbbc-113">Si supponga che un processo richieda la creazione di uno spazio dei nomi, NS1, che definisce un limite contenente due elementi: il SID amministratore e il numero della sessione corrente.</span><span class="sxs-lookup"><span data-stu-id="4cbbc-113">Suppose that a process requests the creation of a namespace, NS1, that defines a boundary containing two elements: the <span data-ttu-id="4cbbc-115">Un altro processo può accedere a questo spazio dei nomi utilizzando la funzione [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) .</span><span class="sxs-lookup"><span data-stu-id="4cbbc-115">Another process can access this namespace using the [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) function.</span></span> <span data-ttu-id="4cbbc-116">Il nome e il limite specificati devono corrispondere se process is to open the namespace created by the first process.</span></span> <span data-ttu-id="4cbbc-117">Si noti che un processo può aprire uno spazio dei nomi esistente anche se non si trova data-stu-id="4cbbc-117">Note that a process can open an existing namespace even if it is not within the boundary unless the creator restricted access to the namespace using the *lpPrivateNamespaceAttributes* parameter.</span></span>

<span data-ttu-id="4cbbc-118">Gli oggetti creati in questo spazio dei nomi hanno nomi nel formato *prefisso* \\ *nomeoggetto*.</span><span class="sxs-lookup"><span data-stu-id="4cbbc-118">The objects that are created in this namespace have names that are of the form *prefix*\\*objectname*.</span></span> <span data-ttu-id="4cbbc-119">Il prefisso è il nome dello spazio dei nomi specificato dal parametro *lpAliasPrefix* di [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea).</span><span class="sxs-lookup"><span data-stu-id="4cbbc-119">The prefix is the namespace name specified by the *lpAliasPrefix* parameter of [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea).</span></span> <span data-ttu-id="4cbbc-120">Per creare, ad esempio, un oggetto evento denominato MyNamespace nello spazio dei nomi NS1, chiamare la funzione [**CreateEvent**](/windows/win32/api/synchapi/nf-synchapi-createeventa) con il parametro *LPNAME* impostato su NS1 \\ .</span><span class="sxs-lookup"><span data-stu-id="4cbbc-120">For example, to create an event object named MyEvent in the NS1 namespace, call the [**CreateEvent**](/windows/win32/api/synchapi/nf-synchapi-createeventa) function with the *lpName* parameter set to NS1\\MyEvent.</span></span>

<span data-ttu-id="4cbbc-121">Il processo che ha creato lo spazio dei nomi può utilizzare la funzione [**ClosePrivateNamespace**](/windows/win32/api/namespaceapi/nf-namespaceapi-closeprivatenamespace) per chiudere l'handle allo spazio dei nomi.</span><span class="sxs-lookup"><span data-stu-id="4cbbc-121">The process that created the namespace can use the [**ClosePrivateNamespace**](/windows/win32/api/namespaceapi/nf-namespaceapi-closeprivatenamespace) function to close the handle to the namespace.</span></span> <span data-ttu-id="4cbbc-122">L'handle viene chiuso anche quando termina il processo che ha creato lo spazio dei nomi.</span><span class="sxs-lookup"><span data-stu-id="4cbbc-122">The handle is also closed when the process that created the namespace terminates.</span></span> <span data-ttu-id="4cbbc-123">Dopo la chiusura dell'handle dello spazio dei nomi, le chiamate successive a [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) hanno esito negativo, ma tutte le operazioni sugli oggetti nello spazio dei nomi vengono eseguite correttamente</span><span class="sxs-lookup"><span data-stu-id="4cbbc-123">After the namespace handle is closed, subsequent calls to [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) fail, but all operations on objects in the namespace succeed.</span></span>
