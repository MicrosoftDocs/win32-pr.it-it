# <a name="u-security-glossary"></a>U (Glossario sulla sicurezza)

[A](a-gly.md) [B](b-gly.md) [C](c-gly.md) [d](d-gly.md) [E](e-gly.md) F [G](g-gly.md) [H](h-gly.md) [i](i-gly.md) J [K](k-gly.md) [L](l-gly.md) [M](m-gly.md) [N](n-gly.md) [O](o-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) [T](t-gly.md) U [V](v-gly.md) [W](w-gly.md) [X](x-gly.md) Y Z

<dl> <dt>

<span id="_security_ucs_gly"></span><span id="_SECURITY_UCS_GLY"></span>**UCS**
</dt> <dd>

Vedere *set di caratteri universali*.

</dd> <dt>

<span id="_security_unicode_gly"></span><span id="_SECURITY_UNICODE_GLY"></span>**Unicode**
</dt> <dd>

Standard di codifica dei caratteri in tutto il mondo che consente di contenere più informazioni in ogni stringa definendo stringhe di caratteri a 16 bit anziché le stringhe di caratteri a 8 bit standard. Unicode consente lo scambio di dati universali e migliora l'elaborazione di testo multilingue. Anche le stringhe Unicode sono denominate stringhe a caratteri *Wide*.

</dd> <dt>

<span id="_security_universal_character_set_gly"></span><span id="_SECURITY_UNIVERSAL_CHARACTER_SET_GLY"></span>**Set di caratteri universali**
</dt> <dd>

UCS Un set di caratteri A 16 bit.

</dd> <dt>

<span id="_security_universal_well_known_security_identifier_gly"></span><span id="_SECURITY_UNIVERSAL_WELL_KNOWN_SECURITY_IDENTIFIER_GLY"></span>**identificatore di sicurezza noto universale**
</dt> <dd>

Un [*ID di sicurezza*](s-gly.md) (SID) noto in tutti i sistemi che rispettano il modello di sicurezza basato su utenti autenticati e controllo di accesso discrezionale.

</dd> <dt>

<span id="_security_upn_gly"></span><span id="_SECURITY_UPN_GLY"></span>**UPN**
</dt> <dd>

Vedere *nome dell'entità utente*.

</dd> <dt>

<span id="_security_user_gly"></span><span id="_SECURITY_USER_GLY"></span>**utente**
</dt> <dd>

Un utente di un sistema è identificato da una procedura di accesso, che stabilisce un processo mediante il quale le applicazioni possono eseguire e accedere ad altri oggetti correlati alla sicurezza. Un esempio di procedura di accesso consiste nell'utilizzo di una smart card, possibilmente insieme a una password o un codice PIN (Personal Identification Number).

</dd> <dt>

<span id="_security_user_interface_smart_card_gly"></span><span id="_SECURITY_USER_INTERFACE_SMART_CARD_GLY"></span>**interfaccia utente (smart card)**
</dt> <dd>

Una finestra di dialogo comune che consente all'utente di connettersi a una smart card e utilizzarla in un'applicazione. Utilizzando la finestra di dialogo, l'utente può specificare una scheda specifica o cercare la smart card da aprire.

</dd> <dt>

<span id="_security_user_principal_name_gly"></span><span id="_SECURITY_USER_PRINCIPAL_NAME_GLY"></span>**nome entità utente**
</dt> <dd>

UPN Un nome di account utente (talvolta definito nome di accesso dell'utente) e un nome di dominio che identifica il dominio in cui si trova l'account utente. Questo nome rappresenta il formato standard dei nomi di accesso a un dominio Windows. Il formato è: someone@example.com (come per un indirizzo di posta elettronica).

</dd> </dl>
