# <a name="a-security-glossary"></a>A (Glossario della sicurezza)

<dl> <dt>

<span id="_security_absolute_security_descriptor_gly"></span><span id="_SECURITY_ABSOLUTE_SECURITY_DESCRIPTOR_GLY"></span>**descrittore di sicurezza assoluto**
</dt> <dd>

Struttura del descrittore di sicurezza che contiene puntatori alle informazioni di sicurezza associate a un oggetto .

Vedere anche [*descrittore di sicurezza*](s-gly.md) [*e descrittore di sicurezza self-relative.*](s-gly.md)

</dd> <dt>

<span id="_security_abstract_syntax_notation_one_gly"></span><span id="_SECURITY_ABSTRACT_SYNTAX_NOTATION_ONE_GLY"></span>**Notazione della sintassi astratta 1**
</dt> <dd>

(ASN.1) Metodo utilizzato per specificare oggetti astratti destinati alla trasmissione seriale.

</dd> <dt>

<span id="_security_access_block_gly"></span><span id="_SECURITY_ACCESS_BLOCK_GLY"></span>**blocco di accesso**
</dt> <dd>

BLOB di chiavi che contiene la chiave della crittografia simmetrica usata per crittografare un file o un messaggio. Il blocco di accesso può essere aperto solo con una chiave privata.

</dd> <dt>

<span id="_security_access_control_entry_gly"></span><span id="_SECURITY_ACCESS_CONTROL_ENTRY_GLY"></span>**voce di controllo di accesso**
</dt> <dd>

(ACE) Voce in un elenco di controllo di accesso (ACL). Una voce ACE contiene un set di diritti di accesso e un ID di sicurezza (SID) che identifica un fiduciare per cui i diritti sono consentiti, negati o controllati.

Vedere anche *elenco di controllo di accesso,* identificatore di [*sicurezza*](s-gly.md)e [*trustee*](t-gly.md).

</dd> <dt>

<span id="_security_access_control_list_gly"></span><span id="_SECURITY_ACCESS_CONTROL_LIST_GLY"></span>**elenco di controllo di accesso**
</dt> <dd>

(ACL) Elenco di protezioni di sicurezza applicabili a un oggetto . Un oggetto può essere un file, un processo, un evento o qualsiasi altro elemento con un descrittore di sicurezza. Una voce in un elenco di controllo di accesso (ACL) è una voce di controllo di accesso (ACE). Esistono due tipi di elenco di controllo di accesso, discrezionale e di sistema.

Vedere anche voce *di controllo di accesso*, elenco di controllo [*di*](d-gly.md)accesso discrezionale , [*descrittore*](s-gly.md)di sicurezza e [*elenco di controllo di accesso di sistema*](s-gly.md).

</dd> <dt>

<span id="_security_access_mask_gly"></span><span id="_SECURITY_ACCESS_MASK_GLY"></span>**maschera di accesso**
</dt> <dd>

Valore a 32 bit che specifica i diritti consentiti o negati in una voce di controllo di accesso (ACE). Una maschera di accesso viene usata anche per richiedere i diritti di accesso all'apertura di un oggetto.

Vedere anche voce *di controllo di accesso*.

</dd> <dt>

<span id="_security_access_token_gly"></span><span id="_SECURITY_ACCESS_TOKEN_GLY"></span>**token di accesso**
</dt> <dd>

Un token di accesso contiene le informazioni sulla protezione relative a una sessione di accesso. Il sistema crea un token di accesso quando un utente esegue l'accesso e quindi assegna una copia del token a ogni processo eseguito per conto di tale utente. Il token identifica l'utente nonché i relativi gruppi e privilegi. Il sistema utilizza il token per controllare l'accesso agli oggetti a protezione diretta e per autorizzare o impedire nell'ambito del computer locale l'esecuzione di varie operazioni di sistema. Esistono due tipi di token di accesso, primario e di rappresentazione.

Vedere anche [*token di rappresentazione,*](i-gly.md) [*token primario,*](p-gly.md) [*privilegio,*](p-gly.md) [*processo*](p-gly.md)e identificatore [*di sicurezza*](s-gly.md).

</dd> <dt>

<span id="_security_ace_gly"></span><span id="_SECURITY_ACE_GLY"></span>**asso**
</dt> <dd>

Vedere voce *di controllo di accesso*.

</dd> <dt>

<span id="_security_acl_gly"></span><span id="_SECURITY_ACL_GLY"></span>**Acl**
</dt> <dd>

Vedere elenco *di controllo di accesso*.

</dd> <dt>

<span id="_security_aes_gly"></span><span id="_SECURITY_AES_GLY"></span>**Advanced Encryption Standard**
</dt> <dd>

(AES) Algoritmo di crittografia specificato dal National Institute of Standards and Technology (NIST) per proteggere le informazioni riservate.

</dd> <dt>

<span id="_security_alg_class_data_encrypt_gly"></span><span id="_SECURITY_ALG_CLASS_DATA_ENCRYPT_GLY"></span>**CRITTOGRAFIA DEI DATI DELLA CLASSE ALG \_ \_ \_**
</dt> <dd>

Classe di algoritmo CryptoAPI per gli algoritmi di crittografia dei dati. Gli algoritmi di crittografia dei dati tipici includono RC2 e RC4.

</dd> <dt>

<span id="_security_alg_class_hash_gly"></span><span id="_SECURITY_ALG_CLASS_HASH_GLY"></span>**HASH DELLA CLASSE ALG \_ \_**
</dt> <dd>

Classe di algoritmo CryptoAPI per algoritmi hash. Gli algoritmi hash tipici includono MD2, MD5, SHA-1 e MAC.

</dd> <dt>

<span id="_security_alg_class_key_exchange_gly"></span><span id="_SECURITY_ALG_CLASS_KEY_EXCHANGE_GLY"></span>**SCAMBIO DI \_ CHIAVI DI \_ CLASSE ALG \_**
</dt> <dd>

Classe di algoritmo CryptoAPI per gli algoritmi di scambio delle chiavi. Un tipico algoritmo di scambio di chiavi è RSA \_ KEYX.

</dd> <dt>

<span id="_security_alg_class_signature_gly"></span><span id="_SECURITY_ALG_CLASS_SIGNATURE_GLY"></span>**FIRMA DELLA CLASSE ALG \_ \_**
</dt> <dd>

Classe di algoritmo CryptoAPI per gli algoritmi di firma. Un tipico algoritmo di firma digitale è RSA \_ SIGN.

</dd> <dt>

<span id="_security_apdu_gly"></span><span id="_SECURITY_APDU_GLY"></span>**APDU**
</dt> <dd>

Vedere *unità dati del protocollo dell'applicazione*.

</dd> <dt>

<span id="_security_application_protocol_data_unit_gly"></span><span id="_SECURITY_APPLICATION_PROTOCOL_DATA_UNIT_GLY"></span>**unità dati del protocollo dell'applicazione**
</dt> <dd>

(APDU) Sequenza di comando (un'unità dati del protocollo dell'applicazione) che può essere inviata dal smart card o restituita dall'applicazione.

Vedere anche [*rispondere a APDU.*](r-gly.md)

</dd> <dt>

<span id="_security_application_protocol_gly"></span><span id="_SECURITY_APPLICATION_PROTOCOL_GLY"></span>**protocollo dell'applicazione**

Protocollo che in genere si trova sopra il livello di trasporto. Ad esempio, HTTP, TELNET, FTP e SMTP sono tutti protocolli dell'applicazione.

</dd> <dt>

<span id="_security_asn.1_gly"></span><span id="_SECURITY_ASN.1_GLY"></span>**ASN.1**
</dt> <dd>

Vedere *Notazione della sintassi astratta 1.*

</dd> <dt>

<span id="_security_ascii_gly"></span><span id="_SECURITY_ASCII_GLY"></span>**Ascii**
</dt> <dd>

American Standard Code for Information Interchange. Schema di codifica che assegna valori numerici a lettere, numeri, segni di punteggiatura e altri caratteri.

</dd> <dt>

<span id="_security_asymmetric_algorithm_gly"></span><span id="_SECURITY_ASYMMETRIC_ALGORITHM_GLY"></span>**algoritmo asimmetrico**
</dt> <dd>

Vedere [*algoritmo a chiave pubblica*](p-gly.md).

</dd> <dt>

<span id="_security_asymmetric_key_gly"></span><span id="_SECURITY_ASYMMETRIC_KEY_GLY"></span>**chiave asimmetrica**
</dt> <dd>

Una delle due chiavi usate con un algoritmo di crittografia asimmetrico. Tale algoritmo usa due chiavi crittografiche: una "chiave pubblica" per la crittografia e una "chiave privata" per la decrittografia. Nella firma e nella verifica, i ruoli vengono invertati: la chiave pubblica viene usata per la verifica e la chiave privata viene usata per la generazione della firma. La funzionalità più importante di tali algoritmi è che la sicurezza non dipende dal mantenimento del segreto della chiave pubblica (anche se può richiedere una certa garanzia di autenticità delle chiavi pubbliche, ad esempio, che siano ottenute da un'origine attendibile). È necessaria la segretezza della chiave privata. Esempi di algoritmi a chiave pubblica sono [*Digital Signature Algorithm (DSA),*](d-gly.md)Elliptic Curve Digital Signature Algorithm (ECDSA) e la famiglia di algoritmi Rivest-Shamir-Adleman (RSA).

</dd> <dt>

<span id="_security_atr_string_gly"></span><span id="_SECURITY_ATR_STRING_GLY"></span>**Stringa ATR**
</dt> <dd>

Sequenza di byte restituita da un smart card quando è attivata. Questi byte vengono usati per identificare la scheda nel sistema.

</dd> <dt>

<span id="_security_attribute_gly"></span><span id="_SECURITY_ATTRIBUTE_GLY"></span>**Attributo**
</dt> <dd>

Elemento di un nome distinto relativo (RDN). Alcuni attributi tipici includono nome comune, cognome, indirizzo di posta elettronica, indirizzo postale e nome paese/area geografica.

</dd> <dt>

<span id="_security_attribute_blob_gly"></span><span id="_SECURITY_ATTRIBUTE_BLOB_GLY"></span>**BLOB di attributi**
</dt> <dd>

Rappresentazione codificata delle informazioni sugli attributi archiviate in una richiesta di certificato.

</dd> <dt>

<span id="_security_audit_security_object_gly"></span><span id="_SECURITY_AUDIT_SECURITY_OBJECT_GLY"></span>**Controllare l'oggetto di sicurezza**
</dt> <dd>

Descrittore di sicurezza che controlla l'accesso al sottosistema dei criteri di controllo.

</dd> <dt>

<span id="_security_authentication_gly"></span><span id="_SECURITY_AUTHENTICATION_GLY"></span>**Autenticazione**
</dt> <dd>

Processo di verifica dell'attestazione di identità di un utente, un computer, un servizio o un processo.

</dd> <dt>

<span id="_security_authentication_package_gly"></span><span id="_SECURITY_AUTHENTICATION_PACKAGE_GLY"></span>**pacchetto di autenticazione**
</dt> <dd>

DLL che incapsula la logica di autenticazione usata per determinare se consentire a un utente di accedere. LSA autentica un accesso utente inviando la richiesta a un pacchetto di autenticazione. Il pacchetto di autenticazione esamina quindi le informazioni di accesso e autentica o rifiuta il tentativo di accesso utente.

</dd> <dt>

<span id="_security_authenticode_gly"></span><span id="_SECURITY_AUTHENTICODE_GLY"></span>**Authenticode**
</dt> <dd>

Funzionalità di sicurezza di Internet Explorer. Authenticode consente ai fornitori di codice eseguibile scaricabile (plug-in o controlli ActiveX, ad esempio) di collegare certificati digitali ai propri prodotti per assicurare agli utenti finali che il codice è dello sviluppatore originale e non è stato modificato. Authenticode consente agli utenti finali di decidere se accettare o rifiutare i componenti software pubblicati su Internet prima di iniziare il download.

</dd> </dl>
