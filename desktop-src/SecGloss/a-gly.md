<dl> <dt>

<span id="_security_absolute_security_descriptor_gly"></span><span id="_SECURITY_ABSOLUTE_SECURITY_DESCRIPTOR_GLY"></span>**descrittore di sicurezza assoluto**
</dt> <dd>

Struttura del descrittore di sicurezza che contiene i puntatori alle informazioni di sicurezza associate a un oggetto.

Vedere anche [*descrittore di sicurezza*](s-gly.md) e [*descrittore di sicurezza autonomo*](s-gly.md).

</dd> <dt>

<span id="_security_abstract_syntax_notation_one_gly"></span><span id="_SECURITY_ABSTRACT_SYNTAX_NOTATION_ONE_GLY"></span>**Notazione di sintassi astratta uno**
</dt> <dd>

(ASN. 1) metodo utilizzato per specificare oggetti astratti destinati alla trasmissione seriale.

</dd> <dt>

<span id="_security_access_block_gly"></span><span id="_SECURITY_ACCESS_BLOCK_GLY"></span>**blocco di accesso**
</dt> <dd>

BLOB della chiave che contiene la chiave della crittografia simmetrica utilizzata per crittografare un file o un messaggio. Il blocco di accesso può essere aperto solo con una chiave privata.

</dd> <dt>

<span id="_security_access_control_entry_gly"></span><span id="_SECURITY_ACCESS_CONTROL_ENTRY_GLY"></span>**voce di controllo di accesso**
</dt> <dd>

Ace Una voce in un elenco di controllo di accesso (ACL). Una voce ACE contiene un set di diritti di accesso e un ID di sicurezza (SID) che identifica un trustee per il quale i diritti sono consentiti, negati o controllati.

Vedere anche *elenco di controllo di accesso*, [*ID di sicurezza*](s-gly.md)e [*trustee*](t-gly.md).

</dd> <dt>

<span id="_security_access_control_list_gly"></span><span id="_SECURITY_ACCESS_CONTROL_LIST_GLY"></span>**elenco di controllo di accesso**
</dt> <dd>

ACL Elenco di protezioni di sicurezza che si applicano a un oggetto. Un oggetto può essere un file, un processo, un evento o qualsiasi altro elemento con un descrittore di sicurezza. Una voce in un elenco di controllo di accesso (ACL) è una voce di controllo di accesso (ACE). Esistono due tipi di elenco di controllo di accesso, discrezionali e di sistema.

Vedere anche *voce di controllo di accesso*, [*elenco di controllo di accesso discrezionale*](d-gly.md), [*descrittore di sicurezza*](s-gly.md)e elenco di controllo di accesso di [*sistema*](s-gly.md).

</dd> <dt>

<span id="_security_access_mask_gly"></span><span id="_SECURITY_ACCESS_MASK_GLY"></span>**maschera di accesso**
</dt> <dd>

Valore a 32 bit che specifica i diritti consentiti o negati in una voce di controllo di accesso (ACE). Viene inoltre utilizzata una maschera di accesso per richiedere i diritti di accesso quando un oggetto viene aperto.

Vedere anche la *voce di controllo di accesso*.

</dd> <dt>

<span id="_security_access_token_gly"></span><span id="_SECURITY_ACCESS_TOKEN_GLY"></span>**token di accesso**
</dt> <dd>

Un token di accesso contiene le informazioni sulla protezione relative a una sessione di accesso. Il sistema crea un token di accesso quando un utente esegue l'accesso e quindi assegna una copia del token a ogni processo eseguito per conto di tale utente. Il token identifica l'utente nonché i relativi gruppi e privilegi. Il sistema utilizza il token per controllare l'accesso agli oggetti a protezione diretta e per autorizzare o impedire nell'ambito del computer locale l'esecuzione di varie operazioni di sistema. Esistono due tipi di token di accesso, primario e di rappresentazione.

Vedere anche [*token di rappresentazione*](i-gly.md), [*token primario*](p-gly.md), [*privilegio*](p-gly.md), [*processo*](p-gly.md)e ID di [*sicurezza*](s-gly.md).

</dd> <dt>

<span id="_security_ace_gly"></span><span id="_SECURITY_ACE_GLY"></span>**ACE**
</dt> <dd>

Vedere *voce di controllo di accesso*.

</dd> <dt>

<span id="_security_acl_gly"></span><span id="_SECURITY_ACL_GLY"></span>**ACL**
</dt> <dd>

Vedere *l'elenco di controllo di accesso*.

</dd> <dt>

<span id="_security_aes_gly"></span><span id="_SECURITY_AES_GLY"></span>**Advanced Encryption Standard**
</dt> <dd>

AES Algoritmo di crittografia specificato dal National Institute of Standards and Technology (NIST) per proteggere le informazioni riservate.

</dd> <dt>

<span id="_security_alg_class_data_encrypt_gly"></span><span id="_SECURITY_ALG_CLASS_DATA_ENCRYPT_GLY"></span>**\_ \_ crittografia dei dati della classe ALG \_**
</dt> <dd>

Classe dell'algoritmo CryptoAPI per gli algoritmi di crittografia dei dati. Gli algoritmi di crittografia dei dati tipici includono RC2 e RC4.

</dd> <dt>

<span id="_security_alg_class_hash_gly"></span><span id="_SECURITY_ALG_CLASS_HASH_GLY"></span>**\_hash della classe ALG \_**

Classe dell'algoritmo CryptoAPI per gli algoritmi di hashing. Gli algoritmi di hash tipici includono MD2, MD5, SHA-1 e MAC.

</dd> <dt>

<span id="_security_alg_class_key_exchange_gly"></span><span id="_SECURITY_ALG_CLASS_KEY_EXCHANGE_GLY"></span>**\_scambio di \_ chiavi di classe ALG \_**
</dt> <dd>

Classe dell'algoritmo CryptoAPI per gli algoritmi di scambio di chiave. Un tipico algoritmo di scambio delle chiavi è RSA \_ KEYX.

</dd> <dt>

<span id="_security_alg_class_signature_gly"></span><span id="_SECURITY_ALG_CLASS_SIGNATURE_GLY"></span>**\_firma della classe ALG \_**
</dt> <dd>

Classe dell'algoritmo CryptoAPI per gli algoritmi di firma. Un tipico algoritmo di firma digitale è il \_ segno RSA.

</dd> <dt>

<span id="_security_apdu_gly"></span><span id="_SECURITY_APDU_GLY"></span>**APDU**
</dt> <dd>

Vedere *unità dati del protocollo di applicazione*.

</dd> <dt>

<span id="_security_application_protocol_data_unit_gly"></span><span id="_SECURITY_APPLICATION_PROTOCOL_DATA_UNIT_GLY"></span>**unità dati del protocollo applicativo**
</dt> <dd>

APDU Sequenza di comandi (un'unità dati del protocollo applicativa) che può essere inviata dalla smart card o restituita dall'applicazione.

Vedere anche [*Reply APDU*](r-gly.md).

</dd> <dt>

<span id="_security_application_protocol_gly"></span><span id="_SECURITY_APPLICATION_PROTOCOL_GLY"></span>**protocollo applicativo**

Protocollo che in genere si trova sopra il livello di trasporto. Ad esempio, HTTP, TELNET, FTP e SMTP sono tutti protocolli applicativi.

</dd> <dt>

<span id="_security_asn.1_gly"></span><span id="_SECURITY_ASN.1_GLY"></span>**ASN. 1**

Vedere *notazione della sintassi astratta uno*.

</dd> <dt>

<span id="_security_ascii_gly"></span><span id="_SECURITY_ASCII_GLY"></span>**ASCII**
</dt> <dd>

American Standard Code for Information Interchange. Schema di codifica che assegna valori numerici a lettere, numeri, segni di punteggiatura e altri caratteri.

</dd> <dt>

<span id="_security_asymmetric_algorithm_gly"></span><span id="_SECURITY_ASYMMETRIC_ALGORITHM_GLY"></span>**algoritmo asimmetrico**
</dt> <dd>

Vedere [*algoritmo a chiave pubblica*](p-gly.md).

</dd> <dt>

<span id="_security_asymmetric_key_gly"></span><span id="_SECURITY_ASYMMETRIC_KEY_GLY"></span>**chiave asimmetrica**
</dt> <dd>

Una delle due chiavi utilizzate con un algoritmo di crittografia asimmetrico. Tale algoritmo utilizza due chiavi crittografiche: una "chiave pubblica" per la crittografia e una "chiave privata" per la decrittografia. Per la firma e la verifica, i ruoli vengono invertiti: la chiave pubblica viene utilizzata per la verifica e la chiave privata viene utilizzata per la generazione della firma. La funzionalità più importante di questi algoritmi è che la loro sicurezza non dipende dal mantenimento del segreto della chiave pubblica (sebbene possa richiedere una garanzia di autenticità delle chiavi pubbliche, ad esempio, che vengono ottenute da una fonte attendibile). La segretezza della chiave privata è obbligatoria. Esempi di algoritmi a chiave pubblica sono l' [*algoritmo di firma digitale (DSA)*](d-gly.md), l'algoritmo di firma digitale a curva ellittica (ECDSA) e la famiglia di algoritmi RSA (Rivest-Shamir-Adleman).

</dd> <dt>

<span id="_security_atr_string_gly"></span><span id="_SECURITY_ATR_STRING_GLY"></span>**Stringa ATR**
</dt> <dd>

Sequenza di byte restituita da una smart card quando è accesa. Questi byte vengono usati per identificare la scheda al sistema.

</dd> <dt>

<span id="_security_attribute_gly"></span><span id="_SECURITY_ATTRIBUTE_GLY"></span>**attributo**
</dt> <dd>

Elemento di un nome distinto relativo (RDN). Alcuni attributi tipici includono il nome comune, il cognome, l'indirizzo di posta elettronica, l'indirizzo postale e il nome del paese/area geografica.

</dd> <dt>

<span id="_security_attribute_blob_gly"></span><span id="_SECURITY_ATTRIBUTE_BLOB_GLY"></span>**BLOB attributo**
</dt> <dd>

Rappresentazione codificata delle informazioni sugli attributi archiviate in una richiesta di certificato.

</dd> <dt>

<span id="_security_audit_security_object_gly"></span><span id="_SECURITY_AUDIT_SECURITY_OBJECT_GLY"></span>**Controllo di sicurezza-oggetto**
</dt> <dd>

Descrittore di sicurezza che controlla l'accesso al sottosistema dei criteri di controllo.

</dd> <dt>

<span id="_security_authentication_gly"></span><span id="_SECURITY_AUTHENTICATION_GLY"></span>**autenticazione**
</dt> <dd>

Processo di verifica dell'attestazione di identità di un utente, un computer, un servizio o un processo.

</dd> <dt>

<span id="_security_authentication_package_gly"></span><span id="_SECURITY_AUTHENTICATION_PACKAGE_GLY"></span>**pacchetto di autenticazione**
</dt> <dd>

DLL che incapsula la logica di autenticazione utilizzata per determinare se consentire a un utente di effettuare l'accesso. LSA autentica un accesso utente inviando la richiesta a un pacchetto di autenticazione. Il pacchetto di autenticazione esamina quindi le informazioni di accesso e autentica o rifiuta il tentativo di accesso dell'utente.

</dd> <dt>

<span id="_security_authenticode_gly"></span><span id="_SECURITY_AUTHENTICODE_GLY"></span>**Authenticode**
</dt> <dd>

Funzionalità di sicurezza di Internet Explorer. Authenticode consente ai fornitori di codice eseguibile scaricabile, ad esempio plug-in o controlli ActiveX, di collegare certificati digitali ai propri prodotti per garantire agli utenti finali che il codice provenga dallo sviluppatore originale e non sia stato modificato. Authenticode consente agli utenti finali di decidere autonomamente se accettare o rifiutare i componenti software pubblicati su Internet prima di iniziare il download.

</dd> </dl>
