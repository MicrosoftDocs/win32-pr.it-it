# <a name="types-of-errors-logged-by-the-http-server-api"></a>Tipi di errori registrati dall'API del server HTTP

L'API server HTTP consente di registrare le risposte di errore a client, timeout di connessione, richieste orfane e connessioni interrotte gestite in modo non corretto.

La tabella seguente elenca gli errori registrati da HTTP.



| Errore                                                                                                                                                  | Descrizione                                                                                                                                                                                                                         |
|--------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="Responses_to_clients"></span><span id="responses_to_clients"></span><span id="RESPONSES_TO_CLIENTS"></span>"Risposte ai client"<br/> | L'API del server HTTP invia una risposta di errore a un client, ad esempio, un errore 400 causato da un errore di analisi nell'ultima richiesta ricevuta. Dopo l'invio della risposta di errore, l'API del server HTTP termina la connessione.<br/> |
| <span id="Connection_time-outs"></span><span id="connection_time-outs"></span><span id="CONNECTION_TIME-OUTS"></span>"Timeout della connessione"<br/> | L'API del server HTTP scade una connessione. Se una richiesta è in sospeso al timeout della connessione, viene utilizzata per fornire ulteriori informazioni sulla connessione nel log degli errori di.<br/>                                         |
| <span id="Orphaned_requests"></span><span id="orphaned_requests"></span><span id="ORPHANED_REQUESTS"></span>"Richieste orfane"<br/>             | Un processo in modalità utente viene terminato in modo imprevisto mentre sono ancora presenti richieste in coda indirizzate a tale processo. L'API server HTTP registra le richieste orfane nel log degli errori di.<br/>                                      |
