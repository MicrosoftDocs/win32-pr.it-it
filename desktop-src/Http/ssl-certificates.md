# <a name="ssl-certificates"></a><span data-ttu-id="28381-104">Certificati SSL</span><span class="sxs-lookup"><span data-stu-id="28381-104">SSL Certificates</span></span>

<span data-ttu-id="28381-105">Secure Sockets Layer (SSL), noto anche come Transport Layer Security (TLS), è diventato uno standard per la protezione delle connessioni Internet e viene usato per impedire l'intercettazione sulla rete.</span><span class="sxs-lookup"><span data-stu-id="28381-105">Secure Sockets Layer (SSL), also known as Transport Layer Security (TLS), has become a standard for securing Internet connections and is used to prevent eavesdropping on the network.</span></span> <span data-ttu-id="28381-106">Il protocollo SSL/TLS consente a un client e a un server di autenticarsi a vicenda e di negoziare gli algoritmi di crittografia.</span><span class="sxs-lookup"><span data-stu-id="28381-106">The SSL/TLS protocol allows a client and server to authenticate each other and negotiate encryption algorithms.</span></span>

<span data-ttu-id="28381-107">SSL usa una chiave di crittografia e un algoritmo di crittografia per proteggere la connessione HTTP.</span><span class="sxs-lookup"><span data-stu-id="28381-107">SSL uses an encryption key and an encryption algorithm to secure the HTTP connection.</span></span> <span data-ttu-id="28381-108">Le chiavi di crittografia sono contenute nei certificati SSL usati dal client e dal server.</span><span class="sxs-lookup"><span data-stu-id="28381-108">The encryption keys are contained in SSL certificates used by both the client and the server.</span></span> <span data-ttu-id="28381-109">Il certificato è in genere un documento X. 509 ([RFC 2459](https://www.ietf.org/rfc/rfc2459.txt)).</span><span class="sxs-lookup"><span data-stu-id="28381-109">The certificate is typically an X.509 ([RFC 2459](https://www.ietf.org/rfc/rfc2459.txt)) document.</span></span> <span data-ttu-id="28381-110">Il server fornisce il certificato SSL per la sessione e invia il certificato al client nella fase di handshake.</span><span class="sxs-lookup"><span data-stu-id="28381-110">The server provides the SSL certificate for the session and sends the certificate to the client in the handshake phase.</span></span> <span data-ttu-id="28381-111">Il client invia il certificato al server solo se il server invia una richiesta al client per un certificato.</span><span class="sxs-lookup"><span data-stu-id="28381-111">The client sends its certificate to the server only if the server sends a request to the client for a certificate.</span></span> <span data-ttu-id="28381-112">Il client autentica quindi sempre il server, ma il server ha l'opzione per autenticare il client.</span><span class="sxs-lookup"><span data-stu-id="28381-112">Thus the client always authenticates the server but the server has the option whether or not to authenticate the client.</span></span>

<span data-ttu-id="28381-113">I certificati del server devono essere archiviati nell'archivio permanente locale dell'API del server HTTP, per l'utilizzo ogni volta che viene creata una connessione protetta.</span><span class="sxs-lookup"><span data-stu-id="28381-113">Server certificates must be stored in the HTTP Server API's local persistent storage, for use each time a secure connection is created.</span></span> <span data-ttu-id="28381-114">Ogni voce dell'archivio certificati contiene anche l'indirizzo IP e la porta del server, l'hash del certificato (usato per firmare i messaggi) e l'ID applicazione.</span><span class="sxs-lookup"><span data-stu-id="28381-114">Each certificate store entry also contains the IP Address and port of the server, the certificate hash (used for signing the messages) and the application ID.</span></span> <span data-ttu-id="28381-115">L'ID applicazione viene usato per identificare l'applicazione proprietaria del certificato.</span><span class="sxs-lookup"><span data-stu-id="28381-115">The application ID is used to identify the application that owns the certificate.</span></span>

<span data-ttu-id="28381-116">Gli amministratori di sistema possono archiviare le informazioni sul certificato del server SSL con le API di configurazione.</span><span class="sxs-lookup"><span data-stu-id="28381-116">System administrators can store SSL server certificate information with the configuration APIs.</span></span> <span data-ttu-id="28381-117">Uno strumento di amministrazione chiama la funzione [**HttpSetServiceConfiguration**](/windows/desktop/api/Http/nf-http-httpsetserviceconfiguration) e specifica il valore HttpServiceConfigSSLCertInfo per il parametro di configurazione del servizio per impostare le informazioni per un certificato SSL.</span><span class="sxs-lookup"><span data-stu-id="28381-117">An administrative tool calls the [**HttpSetServiceConfiguration**](/windows/desktop/api/Http/nf-http-httpsetserviceconfiguration) function and specifies HttpServiceConfigSSLCertInfo value for the service configuration parameter to set information for an SSL certificate.</span></span> <span data-ttu-id="28381-118">È possibile configurare un solo certificato server per ogni coppia di porta e indirizzo IP nel computer.</span><span class="sxs-lookup"><span data-stu-id="28381-118">Only one server certificate can be configured for each IP Address and port pair on the machine.</span></span> <span data-ttu-id="28381-119">L'API server HTTP fornisce anche funzioni di [**query**](/windows/desktop/api/Http/nf-http-httpqueryserviceconfiguration) e di [**eliminazione**](/windows/desktop/api/Http/nf-http-httpdeleteserviceconfiguration) per accedere ai certificati esistenti o eliminarli.</span><span class="sxs-lookup"><span data-stu-id="28381-119">The HTTP Server API also provides [**query**](/windows/desktop/api/Http/nf-http-httpqueryserviceconfiguration) and [**delete**](/windows/desktop/api/Http/nf-http-httpdeleteserviceconfiguration) functions to access or delete existing certificates.</span></span>
