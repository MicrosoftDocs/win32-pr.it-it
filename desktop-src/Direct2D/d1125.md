# <a name="d1125-dxgi-format-mismatch"></a>D1125: mancata corrispondenza del formato DXGI

Il formato pixel passato a questa API non è compatibile con il formato pixel della \[ superficie \] .

## <a name="placeholders"></a>Segnaposto

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*area*
</dt> <dd>

La destinazione di rendering o la bitmap specificata.

</dd> </dl>

## <a name="possible-causes"></a>Possibili cause

Operazioni come ID2D1RenderTarget:: CreateBitmapFromWicBitmap o ID2D1RenderTarget:: CreateDxgiSurfaceRendertarget richiedono sia una superficie che un formato di superficie specificato. Direct2D verifica che il tipo di superficie specificato sia compatibile con la superficie. Se non è compatibile, Direct2D restituirà questo messaggio.

## <a name="possible-fixes"></a>Correzioni possibili

Verificare che il formato DXGI specificato \_ corrisponda o sia compatibile con il tipo della superficie sottostante. Per le risorse di superficie Direct3D create dall'applicazione, usare lo stesso \_ formato DXGI usato per creare la risorsa. Per altre istruzioni, vedere [formati di pixel supportati e modalità Alpha](supported-pixel-formats-and-alpha-modes.md) .
