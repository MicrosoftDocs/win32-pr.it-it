# <a name="d1125-dxgi-format-mismatch"></a>D1125: Formato DXGI non corrispondente

Il formato pixel passato a questa API non è compatibile con il formato pixel della \[ superficie \] .

## <a name="placeholders"></a>Segnaposto

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*Superficie*
</dt> <dd>

Destinazione di rendering o bitmap specificata.

</dd> </dl>

## <a name="possible-causes"></a>Possibili cause

Operazioni come ID2D1RenderTarget::CreateBitmapFromWicBitmap o ID2D1RenderTarget::CreateDxgiSurfaceRendertarget richiedono sia una superficie che un formato di superficie specificato. Direct2D verifica che il tipo di superficie specificato sia compatibile con la superficie. Se non è compatibile, Direct2D restituisce questo messaggio.

## <a name="possible-fixes"></a>Possibili correzioni

Verificare che il formato DXGI specificato corrisponda o sia compatibile \_ con il tipo della superficie sottostante. Per le risorse surface Direct3D create dall'applicazione, usare lo stesso formato DXGI \_ usato per creare la risorsa. Per altre [informazioni, vedere Formati pixel supportati e modalità alfa.](supported-pixel-formats-and-alpha-modes.md)
