# <a name="d1180-begindraw-incorrect-state"></a>D1180: BeginDraw stato non corretto

BeginDraw è stato chiamato più volte, senza una chiamata corrispondente a EndDraw.






 

## <a name="possible-causes"></a>Possibili cause

L'applicazione ha eseguito più chiamate a BeginDraw senza chiamare EndDraw nella stessa destinazione. Quando si usa Direct2D, le applicazioni devono corrispondere a ogni chiamata a BeginDraw con un EndDraw successivo. In alternativa, questo messaggio può verificarsi se la destinazione è in uno stato errato e la chiamata BeginDraw non ha consentito il rendering della destinazione.

## <a name="possible-fixes"></a>Correzioni possibili

Verificare che ogni chiamata a BeginDraw corrisponda a un EndDraw successivo. Quando si inviano comandi a una destinazione usando Flush o EndDraw, verificare che il risultato restituito sia S \_ OK prima di provare a usare nuovamente la destinazione.
