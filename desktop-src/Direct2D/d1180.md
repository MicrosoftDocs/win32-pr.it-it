# <a name="d1180-begindraw-incorrect-state"></a>D1180: BeginDraw Stato non corretto

BeginDraw è stato chiamato più volte, senza una chiamata corrispondente a EndDraw.






 

## <a name="possible-causes"></a>Possibili cause

L'applicazione ha effettuato più chiamate a BeginDraw senza chiamare EndDraw sulla stessa destinazione. Durante l'uso di Direct2D, le applicazioni devono corrispondere a ogni chiamata a BeginDraw con un enddraw successivo. In alternativa, questo messaggio può verificarsi se la destinazione si trova in uno stato erre e la chiamata BeginDraw non consente il rendering della destinazione.

## <a name="possible-fixes"></a>Possibili correzioni

Assicurarsi che ogni chiamata a BeginDraw corrisponda a un enddraw successivo. Quando si inviano comandi a una destinazione usando Flush o EndDraw, verificare che il risultato restituito sia S OK prima di provare a \_ usare di nuovo la destinazione.
