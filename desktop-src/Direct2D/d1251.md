# <a name="d1251-d3d-state-issue"></a>D1251: problema di stato D3D

Lo stato del dispositivo Direct3D è stato modificato quando Direct2D lo stava ancora usando. Questo può essere indicativo di una violazione del threading perché solo un thread può usare il dispositivo alla volta.






 

## <a name="possible-causes"></a>Possibili cause

È possibile che l'applicazione usi una factory D2D a thread singolo su due thread diversi contemporaneamente.

## <a name="possible-fixes"></a>Correzioni possibili

Assicurarsi che l'applicazione rispetti correttamente i criteri di threading di le convenzioni.
