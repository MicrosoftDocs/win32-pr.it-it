# <a name="d1251-d3d-state-issue"></a>D1251: Problema di stato D3D

Lo stato del dispositivo Direct3D è cambiato quando Direct2D lo stava ancora usando. Ciò può essere indicativo di una violazione del threading perché solo un thread può usare il dispositivo alla volta.






 

## <a name="possible-causes"></a>Possibili cause

L'applicazione potrebbe usare una factory D2D a thread singolo in due thread diversi contemporaneamente.

## <a name="possible-fixes"></a>Possibili correzioni

Assicurarsi che l'applicazione rispetti correttamente i criteri di threading di Direct2D.
