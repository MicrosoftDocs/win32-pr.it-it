# <a name="d1194-flush-outstanding-dc"></a>D1194: Scarica il controller di dominio in attesa

La chiamata di Flush o EndDraw tra una coppia GetDC/ReleaseDC non è valida.






 

## <a name="possible-causes"></a>Possibili cause

Quando si usa un RenderTarget interoperato GDI, un'applicazione può controllare la proprietà del controller di dominio di destinazione usando IGdiInteropRenderTarget:: GetDC o IGdiInteropRenderTarget:: ReleaseDC. Tuttavia, non è possibile chiamare ID2D1RenderTarget:: Flush o ID2D1RenderTarget:: EndDraw mentre l'applicazione ha il controllo del controller di dominio. Flush e EndDraw possono causare l'esecuzione del rendering sulla risorsa di destinazione e Direct2D non deve tentare di modificare la risorsa nello stesso momento in cui si trovano.

## <a name="possible-fixes"></a>Correzioni possibili

Per risolvere questo problema, assicurarsi che l'applicazione non chiami ID2D1RenderTarget:: Flush o ID2D1RenderTarget:: EndDraw mentre è proprietario del controller di dominio di IGdiInteropRenderTarget. Se possibile, organizzare l'applicazione per separare il comportamento di rendering Direct2D dal comportamento GDI per distinguere la proprietà del controller di dominio in modo più chiaro.
