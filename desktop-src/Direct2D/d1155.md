# <a name="d1155-fillopacitymask-requires-aliased-rendering"></a>D1155: FillOpacityMask richiede il rendering con alias

FillOpacityMask richiede che la modalità antialias sia impostata su D2D1 \_ ANTIALIAS \_ MODE \_ ALIASED.





 

## <a name="possible-causes"></a>Possibili cause

Quando un'applicazione chiama ID2D1RenderTarget::FillOpacityMask, la modalità antialias della destinazione di rendering coinvolta deve essere D2D1 \_ ANTIALIAS \_ MODE \_ ALIASED. Il comportamento predefinito di una destinazione di rendering è l'uso dell'antialiasing, quindi è necessario chiamare ID2D1RenderTarget::SetAntialiasMode prima di compilare una maschera di opacità.

## <a name="possible-fixes"></a>Possibili correzioni

Usare ID2D1RenderTarget::SetAntialiasMode per garantire che una destinazione di rendering usi il rendering con alias durante il riempimento di una maschera di opacità. Se le operazioni successive richiedono l'antialiasing, impostare di nuovo la modalità antialias della destinazione su D2D1 \_ ANTIALIAS \_ MODE \_ PER \_ PRIMITIVE. Assicurarsi che le chiamate a ID2D1RenderTarget::SetAntialiasMode e ID2D1RenderTarget::FillOpacityMask siano emesse nella stessa destinazione.
