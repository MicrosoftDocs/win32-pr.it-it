# <a name="d1201-pushaxisalignedclip-with-antialiased-multisampled-target"></a>D1201: PushAxisAlignedClip con destinazione multicampionata antialias

PushAxisAlignedClip è stato usato con una destinazione multicampionata. La chiamata verrà comunque emessa, ma potrebbero verificarsi artefatti se i limiti del rettangolo di ritaglio non sono allineati al pixel. Inoltre, il risultato sottoposto a rendering potrebbe non rappresentare il modello multicampionamento previsto.






 

## <a name="possible-causes"></a>Possibili cause

L'applicazione ha interoperato una trama Direct3D multicampionata come destinazione Direct2D e ha chiamato PushAxisAlignedClip nella destinazione.

## <a name="possible-fixes"></a>Possibili correzioni

Evitare l'interoperabilità di trame multicampionamento se la funzionalità di ritaglio è importante. L'uso di livelli o clip con questi tipi di risorse può causare una penalità delle prestazioni o risultati di rendering imprevisti. Ciò vale in particolare se i limiti della clip non sono allineati al pixel. Se possibile, usare un metodo alternativo per mascherare il contenuto anziché una clip, ad esempio una forbice Direct3D. In alternativa, usare una risorsa intermedia anziché una risorsa multicampionata. Se la risorsa multicampionata è destinata all'anti-aliasing del contenuto 2D, valutare se l'anti-aliasing predefinito per primitivo di Direct2D sia sufficiente.
