# <a name="d1201-pushaxisalignedclip-with-antialiased-multisampled-target"></a>D1201: PushAxisAlignedClip con destinazione multicampionata con anti-aliasing

PushAxisAlignedClip è stato usato con una destinazione multicampionata. La chiamata verrà comunque eseguita, ma gli artefatti possono verificarsi se i limiti del rettangolo di ritaglio sono allineati in pixel. Inoltre, il risultato di cui è stato eseguito il rendering potrebbe non rappresentare il modello multisample previsto.






 

## <a name="possible-causes"></a>Possibili cause

L'applicazione ha interoperato una trama Direct3D multicampionata come destinazione Direct2D e ha chiamato PushAxisAlignedClip sulla destinazione.

## <a name="possible-fixes"></a>Correzioni possibili

Evitare l'interoperabilità di trame multicampionate se è importante la funzionalità di ritaglio. L'uso di livelli o clip con questi tipi di risorse può comportare una riduzione delle prestazioni o risultati imprevisti del rendering. Questa operazione è particolarmente vera se i limiti della clip non sono allineati ai pixel. Se possibile, usare un metodo alternativo per mascherare il contenuto anziché una clip, ad esempio una forbice Direct3D. In alternativa, usare un intermedio anziché una risorsa multicampionata. Se la risorsa multicampionata è progettata per l'anti-aliasing di contenuto 2D, valutare se è sufficiente l'anti-aliasing per primitiva di le convenzioni.
