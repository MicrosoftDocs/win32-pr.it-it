# <a name="d1154-invalid-bitmap-input-format"></a>D1154: formato di input bitmap non valido

Questa operazione non è compatibile con il formato pixel della bitmap.






 

## <a name="possible-causes"></a>Possibili cause

Direct2D supporta formati di superficie solo Alpha come DXGI \_ Format \_ a8 \_ UNORM, ma possono essere utilizzati solo come pennelli di opacità o per la maschera. Se un'applicazione crea un pennello bitmap la cui bitmap ha un formato solo Alpha, non deve tentare di utilizzare il pennello con operazioni che richiedono un pennello colori. ID2D1RenderTarget::D rawLine e ID2D1RenderTarget:: FillRectangle, ad esempio, usare un pennello colori.

## <a name="possible-fixes"></a>Correzioni possibili

Per risolvere questo problema, assicurarsi che le bitmap con formati solo alfa vengano utilizzate solo per i pennelli o la maschera di opacità. Se una bitmap deve contenere informazioni sul colore, crearla utilizzando un formato di superficie che includa i canali dei colori.
