# <a name="d1154-invalid-bitmap-input-format"></a>D1154: Formato di input bitmap non valido

Questa operazione non è compatibile con il formato pixel della bitmap.






 

## <a name="possible-causes"></a>Possibili cause

Direct2D supporta formati di superficie solo alfa, ad esempio DXGI \_ FORMAT \_ A8 UNORM, ma possono essere usati solo come pennelli di opacità o \_ per la maschera. Se un'applicazione crea un pennello bitmap la cui bitmap ha un formato solo alfa, non deve tentare di usare il pennello con operazioni che richiedono un pennello colore. ID2D1RenderTarget::D rawLine e ID2D1RenderTarget::FillRectangle, ad esempio, usano un pennello colore.

## <a name="possible-fixes"></a>Possibili correzioni

Per risolvere questo messaggio, assicurarsi che le bitmap con formati solo alfa siano usate solo per i pennelli di opacità o la maschera. Se una bitmap deve contenere informazioni sul colore, crearla usando un formato di superficie che include canali di colore.
