# <a name="d1127-alpha-mode-mismatch"></a>D1127: mancata corrispondenza della modalità Alpha

La modalità Alpha passata a questa API non è compatibile con il formato pixel della \[ *superficie* \] .

## <a name="placeholders"></a>Segnaposto

<span id="surface"></span><span id="SURFACE"></span>*area*
</dt> <dd>

Superficie Direct3D che non supporta la modalità Alpha.

</dd> </dl> 

|             |             |
|-------------|-------------|
| Livello di errore | Informazioni |



 

## <a name="possible-causes"></a>Possibili cause

Operazioni come ID2D1RenderTarget:: CreateBitmapFromWicBitmap o ID2D1RenderTarget:: CreateDxgiSurfaceRendertarget richiedono sia una superficie che una modalità Alpha di superficie specificata. Direct2D verifica che la modalità Alpha specificata sia compatibile con la superficie. Se non è compatibile, Direct2D restituirà questo messaggio.

## <a name="possible-fixes"></a>Correzioni possibili

Modificare l'applicazione per utilizzare una \_ modalità Alpha corrispondente o compatibile con la modalità Alpha della superficie sottostante. Per ulteriori informazioni sulle modalità Alpha esposte da Direct2D, vedere [formati pixel supportati e modalità Alpha](supported-pixel-formats-and-alpha-modes.md) .
