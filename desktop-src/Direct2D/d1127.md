# <a name="d1127-alpha-mode-mismatch"></a>D1127: Mancata corrispondenza della modalità alfa

La modalità alfa passata in questa API non è compatibile con il formato pixel della \[ *superficie* \] .

## <a name="placeholders"></a>Segnaposto

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*Superficie*
</dt> <dd>

Superficie Direct3D che non supporta la modalità alfa.

</dd> </dl> 



 

## <a name="possible-causes"></a>Possibili cause

Operazioni come ID2D1RenderTarget::CreateBitmapFromWicBitmap o ID2D1RenderTarget::CreateDxgiSurfaceRendertarget richiedono sia una superficie che una modalità alfa della superficie specificata. Direct2D verifica che la modalità alfa specificata sia compatibile con la superficie. Se non è compatibile, Direct2D restituisce questo messaggio.

## <a name="possible-fixes"></a>Possibili correzioni

Modificare l'applicazione in modo che usi una modalità ALFA che corrisponda o sia compatibile con la \_ modalità alfa della superficie sottostante. Per altre indicazioni sulle modalità alfa [esposte](supported-pixel-formats-and-alpha-modes.md) da Direct2D, vedere Formati di pixel e modalità alfa supportati.
