# <a name="d1123-dxgi-format-unknown-not-allowed"></a>D1123: Formato DXGI sconosciuto non consentito

DXGI \_ FORMAT UNKNOWN non è consentito come parametro per questa \_ API.






 

## <a name="possible-causes"></a>Possibili cause

Per la funzionalità Direct2D che accetta un formato di superficie come parametro, il formato della superficie deve essere specificato completamente. In alcuni casi, può essere lasciato sconosciuto. Ad esempio, quando si chiama ID2D1RenderTarget:: CreateHwndRenderTarget o ID2D1RenderTarget::CreateDxgiSurfaceRenderTarget, Direct2D può dedurre un formato di superficie dal formato della catena di scambio o dal formato della superficie Direct3D di supporto, come nel caso di .

## <a name="possible-fixes"></a>Possibili correzioni

Nei casi in cui non è stato possibile dedurre il formato di superficie, Direct2D non creerà la risorsa e restituisce questo messaggio. Per risolvere questo problema, modificare l'applicazione in modo da specificare un formato di superficie diverso da UNKNOWN per l'operazione che ha causato il messaggio. Un formato di superficie predefinito ragionevole da usare, in caso di dubbi, è DXGI \_ FORMAT \_ B8G8R8A8 \_ UNORM. Se l'applicazione deve creare una risorsa Direct2D per interagire con una risorsa Direct3D o WIC, scegliere un formato corrispondente alla risorsa interoperabile. La semantica per le risorse WIC è leggermente diversa dalle risorse DXGI. Per altre [indicazioni, vedere](supported-pixel-formats-and-alpha-modes.md) Formati di pixel e modalità alfa supportati.
