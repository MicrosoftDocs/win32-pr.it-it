<span data-ttu-id="ed326-113">In questo argomento viene presentata la funzionalità Direct2D, un'API che consente agli sviluppatori Win32 di eseguire attività di rendering di grafica 2D con prestazioni e qualità visive superiori.</span><span class="sxs-lookup"><span data-stu-id="ed326-113">This topic introduces Direct2D, an API that provides Win32 developers with the ability to perform 2-D graphics rendering tasks with superior performance and visual quality.</span></span>

## <a name="what-is-direct2d"></a><span data-ttu-id="ed326-114">Che cos'è Direct2D?</span><span class="sxs-lookup"><span data-stu-id="ed326-114">What is Direct2D?</span></span>

<span data-ttu-id="ed326-115">Direct2D è un'API grafica 2D in modalità immediata e con accelerazione hardware che fornisce prestazioni elevate e rendering di alta qualità per la geometria 2D, le bitmap e il testo.</span><span class="sxs-lookup"><span data-stu-id="ed326-115">Direct2D is a hardware-accelerated, immediate-mode 2-D graphics API that provides high performance and high-quality rendering for 2-D geometry, bitmaps, and text.</span></span> <span data-ttu-id="ed326-116">L'API Direct2D è progettata per interagire con il codice esistente che usa GDI, GDI+ o Direct3D.</span><span class="sxs-lookup"><span data-stu-id="ed326-116">The Direct2D API is designed to interoperate with existing code that uses GDI, GDI+, or Direct3D.</span></span>

<span data-ttu-id="ed326-117">Direct2D è progettato principalmente per l'uso da parte delle classi di sviluppatori seguenti:</span><span class="sxs-lookup"><span data-stu-id="ed326-117">Direct2D is designed primarily for use by the following classes of developers:</span></span>

-   <span data-ttu-id="ed326-118">Sviluppatori di applicazioni native di grandi dimensioni e di livello aziendale.</span><span class="sxs-lookup"><span data-stu-id="ed326-118">Developers of large, enterprise-scale, native applications.</span></span>

-   <span data-ttu-id="ed326-121">Gli sviluppatori che utilizzano la grafica Direct3D e necessitano di rendering semplice e a prestazioni elevate 2D e di testo per menu, elementi dell'interfaccia utente e visualizzazioni Heads-up (HUDs).</span><span class="sxs-lookup"><span data-stu-id="ed326-121">Developers who use Direct3D graphics and need simple, high-performance 2-D and text rendering for menus, user interface (UI) elements, and Heads-up Displays (HUDs).</span></span>

## <a name="why-direct2d"></a><span data-ttu-id="ed326-122">Perché Direct2D?</span><span class="sxs-lookup"><span data-stu-id="ed326-122">Why Direct2D?</span></span>

<span data-ttu-id="ed326-123">Le motivazioni principali per la creazione di una nuova API grafica 2D in Microsoft Windows sono le seguenti:</span><span class="sxs-lookup"><span data-stu-id="ed326-123">The primary motivations for creating a new 2-D graphics API in Microsoft Windows include the following:</span></span>

-   <span data-ttu-id="ed326-124">Per restare al passo con l'aumento del livello di ricchezza visiva a cui gli utenti di Windows sono abituati.</span><span class="sxs-lookup"><span data-stu-id="ed326-124">To keep pace with the increasing level of visual richness that Windows users are accustomed to.</span></span>
-   <span data-ttu-id="ed326-125">Per consentire agli sviluppatori di scrivere codice di rendering 2D scalabile direttamente con l'hardware di elaborazione grafica del PC in cui è in esecuzione.</span><span class="sxs-lookup"><span data-stu-id="ed326-125">To enable developers to write 2-D rendering code that scales directly with the graphics processing hardware of the PC it is running on.</span></span>
-   <span data-ttu-id="ed326-126">Per consentire agli sviluppatori di scrivere codice per il rendering di immagini 2D che possono essere eseguite nel contesto di un servizio.</span><span class="sxs-lookup"><span data-stu-id="ed326-126">To enable developers to write code for rendering 2-D graphics that can run in the context of a service.</span></span>

<span data-ttu-id="ed326-127">Negli ultimi anni, gli utenti finali hanno iniziato a aspettarsi una maggiore fedeltà visiva nelle esperienze digitali.</span><span class="sxs-lookup"><span data-stu-id="ed326-127">In recent years, end users have begun to expect greater visual fidelity in digital experiences.</span></span> <span data-ttu-id="ed326-128">Questa tendenza si riflette nell'elettronica consumer.</span><span class="sxs-lookup"><span data-stu-id="ed326-128">This trend is reflected in consumer electronics.</span></span> <span data-ttu-id="ed326-129">I dispositivi GPS, i dispositivi di riproduzione multimediale, i telefoni cellulari e le fotocamere digitali offrono esperienze costantemente più ricche di anno dopo anno.</span><span class="sxs-lookup"><span data-stu-id="ed326-129">GPS devices, media playback devices, mobile phones, and digital cameras deliver consistently richer experiences year after year.</span></span> <span data-ttu-id="ed326-130">Questa tendenza può essere considerata anche nella diversità di contenuti grafici in film, televisori, giochi video e sul Web.</span><span class="sxs-lookup"><span data-stu-id="ed326-130">This trend can also be seen in the diversity of graphical content in film, television, video games, and on the Web.</span></span> <span data-ttu-id="ed326-131">Per rimanere al passo con queste modifiche, gli sviluppatori vengono costantemente invitati a portare le proprie applicazioni Windows esistenti al livello successivo di ricchezza visiva.</span><span class="sxs-lookup"><span data-stu-id="ed326-131">To keep pace with these changes, developers are consistently asked to take their existing Windows applications to the next level of visual richness.</span></span>

<span data-ttu-id="ed326-132">I processori grafici nei PC Windows moderni si sono evoluti costantemente, grazie ai miglioramenti apportati alla grafica dei giochi video e alle parti dell'esperienza Windows come Windows Media Center e Aero.</span><span class="sxs-lookup"><span data-stu-id="ed326-132">Graphics processors in modern Windows PCs have also been evolving steadily, driven by advances in video game graphics and parts of the Windows experience like Windows Media Center and Aero.</span></span> <span data-ttu-id="ed326-133">Alcune applicazioni Windows possono sfruttare le GPU moderne usando Microsoft Direct3D e Windows Presentation Foundation (WPF).</span><span class="sxs-lookup"><span data-stu-id="ed326-133">Some Windows applications can take advantage of modern GPUs by using Microsoft Direct3D and Windows Presentation Foundation (WPF).</span></span> <span data-ttu-id="ed326-134">Sebbene Direct3D soddisfi le applicazioni grafiche 3D di fascia alta e WPF soddisfi le esigenze degli sviluppatori .NET, esistono Gap per gli sviluppatori che dispongono di codebase esistenti di grandi dimensioni per il rendering del codice basato su GDI e GDI+ o che desiderano incorporare grafica 2D di alta qualità nelle applicazioni basate su Direct3D.</span><span class="sxs-lookup"><span data-stu-id="ed326-134">While Direct3D serves high-end 3-D graphics applications and WPF addresses the needs of .NET developers, there are gaps for developers who have large existing codebases of rendering code based on GDI and GDI+ or who want to incorporate high-quality 2-D graphics in their Direct3D-based applications.</span></span>

<span data-ttu-id="ed326-135">Infine, la necessità di un'API grafica che può essere usata in un servizio è diventata un requisito emergente per gli sviluppatori che lavorano in scenari aziendali e di sviluppo Web.</span><span class="sxs-lookup"><span data-stu-id="ed326-135">Finally, the need for a graphics API that can be used in a service has become an emerging requirement for developers working in enterprise and Web-development scenarios.</span></span> <span data-ttu-id="ed326-136">Le API di rendering esistenti si concentrano sul rendering lato client in una singola sessione utente.</span><span class="sxs-lookup"><span data-stu-id="ed326-136">Existing rendering APIs focus on client-side rendering in a single user session.</span></span> <span data-ttu-id="ed326-137">Di conseguenza, possono essere suddivisi in aree di solidità e scalabilità quando vengono usati in un contesto del servizio.</span><span class="sxs-lookup"><span data-stu-id="ed326-137">As such, they can fall short in areas of robustness and scalability when used in a service context.</span></span> <span data-ttu-id="ed326-138">Per risolvere il problema, è necessaria una nuova API.</span><span class="sxs-lookup"><span data-stu-id="ed326-138">A new API is required to address this.</span></span>

<span data-ttu-id="ed326-164">L'integrazione del rendering basato su Direct2D è semplificata per gli sviluppatori grazie all'interoperabilità a livello di superficie con GDI e Direct3D.</span><span class="sxs-lookup"><span data-stu-id="ed326-164">Integrating Direct2D-based rendering is made easier for developers through surface-level interoperability with GDI and Direct3D.</span></span> <span data-ttu-id="ed326-165">Le applicazioni che eseguono il rendering del contenuto principalmente con GDI, GDI+ o Direct3D possono iniziare utilizzando Direct2D per eseguire il rendering di aree specifiche dell'applicazione e nel tempo passare a un modello in cui il rendering viene eseguito principalmente tramite Direct2D, usando GDI principalmente per i plug-in o l'estendibilità legacy.</span><span class="sxs-lookup"><span data-stu-id="ed326-165">Applications that render content primarily with GDI, GDI+, or Direct3D, can begin by using Direct2D to render specific areas of their application, and over time move to a model where rendering is performed primarily via Direct2D, using GDI primarily for plug-ins or legacy extensibility.</span></span>
