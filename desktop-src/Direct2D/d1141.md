# <a name="d1141-print-requires-srgb-color-space"></a>D1141: la stampa richiede spazio colore sRGB

Stampa Direct2D: lo spazio colori personalizzato non è supportato. Uso di sRGB.






 

## <a name="possible-causes"></a>Possibili cause

Quando si crea un controllo di stampa Direct2D, un'applicazione usa un oggetto D2D1 \_ PRINT CONTROL PROPERTIES con un attributo dello spazio \_ \_ colore. Direct2D supporta [**D2D1 \_ COLOR SPACE \_ \_ CUSTOM**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_color_space) per alcune operazioni, ad esempio la creazione di un contesto di colore usando dati ICC. Durante la creazione di un controllo di stampa, Tuttavia, Direct2D supporta solo spazi colori predefiniti, ad esempio sRGB o scRGB.

## <a name="possible-fixes"></a>Possibili correzioni

Per risolvere il messaggio, specificare uno spazio colore non personalizzato, ad esempio sRGB o scRGB durante la creazione di un controllo di stampa. Il messaggio non indica un errore, ma indica che Direct2D ignorerà l'identificatore CUSTOM e creerà il controllo di stampa usando uno spazio colori sRGB.
