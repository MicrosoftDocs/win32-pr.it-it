# <a name="d1141-print-requires-srgb-color-space"></a>D1141: la stampa richiede lo spazio colore sRGB

Stampa Direct2D: lo spazio colore personalizzato non è supportato. In alternativa, usare sRGB.






 

## <a name="possible-causes"></a>Possibili cause

Quando si crea un controllo di stampa Direct2D, un'applicazione utilizza \_ \_ le proprietà di un controllo di stampa d2d1 \_ con un attributo dello spazio dei colori. Direct2D supporta [**\_ lo spazio dei colori d2d1 \_ \_ personalizzato**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_color_space) per alcune operazioni, ad esempio la creazione di un contesto di colore usando i dati ICC. Durante la creazione di un controllo di stampa, tuttavia, Direct2D supporta solo spazi colori predefiniti, ad esempio sRGB o scRGB.

## <a name="possible-fixes"></a>Correzioni possibili

Per risolvere il messaggio, specificare uno spazio di colore non personalizzato, ad esempio sRGB o scRGB, durante la creazione di un controllo di stampa. Il messaggio non indica un errore, ma indica che Direct2D ignorerà l'identificatore personalizzato e creerà il controllo di stampa utilizzando uno spazio dei colori di sRGB.
