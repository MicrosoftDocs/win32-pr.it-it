# <a name="d1135-bitmap-option-must-be-subset"></a>D1135: l'opzione Bitmap deve essere subset

Il valore delle \[ *opzioni bitmap* \] deve essere un subset dei flag associati alla superficie DXGI.






 

## <a name="placeholders"></a>Segnaposto

<dl> <dt>

<span id="value"></span><span id="VALUE"></span>*Valore*
</dt> <dd>

Valore UINT dell'enumerazione delle opzioni bitmap.

</dd> </dl>

## <a name="possible-causes"></a>Possibili cause

Durante l'interoperabilità delle trame Direct3D con Direct2D come mappe ID2D1Bitmap, le opzioni specificate nel campo OPZIONI BITMAP D2D1 devono corrispondere agli attributi \_ \_ Direct3D della trama. In particolare:

-   [**L'opzione D2D1 \_ BITMAP OPTIONS CPU \_ \_ \_ READ**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) richiede l'opzione DIRECT3D CPU \_ ACCESS \_ READ.
-   [**L'opzione D2D1 \_ BITMAP OPTIONS \_ \_ TARGET**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) richiede l'opzione BIND \_ RENDER \_ TARGET.
-   [**L'opzione D2D1 \_ BITMAP OPTIONS \_ \_ NONE**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) non è compatibile con l'opzione DIRECT3D USAGE \_ STAGING solo con ACCESSO ALLA CPU \_ \_ WRITE.

## <a name="possible-fixes"></a>Possibili correzioni

Per risolvere questo problema, assicurarsi che la creazione di [**ID2D1Bitmap**](/windows/win32/api/d2d1/nn-d2d1-id2d1bitmap) usi identificatori [**D2D1 \_ BITMAP \_ OPTIONS**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) che concordano con le proprietà della trama Direct3D sottostante. Per usare **D2D1 \_ BITMAP \_ OPTIONS,** questo può comportare l'uso di [**ID2D1DeviceContext::CreateBitmap**](id2d1devicecontext-createbitmap-overload.md) anziché [**ID2D1RenderTarget::CreateBitmap**](id2d1rendertarget-createbitmap.md), perché è un membro di [**D2D1 \_ BITMAP \_ PROPERTIES1.**](/windows/desktop/api/D2D1_1/ns-d2d1_1-d2d1_bitmap_properties1)
