# <a name="d1181-enddraw-incorrect-state"></a>D1181: EndDraw stato non corretto

EndDraw è stato chiamato, ma lo stato di renderTarget non è valido. Questo può derivare dalla chiamata a EndDraw senza un BeginDraw corrispondente.






 

## <a name="possible-causes"></a>Possibili cause

L'applicazione ha eseguito più chiamate a EndDraw senza chiamare BeginDraw nella stessa destinazione. Quando si usa Direct2D, le applicazioni devono corrispondere a ogni chiamata a EndDraw con una chiamata precedente a BeginDraw. In alternativa, questo messaggio può verificarsi se la destinazione è in uno stato errato e non è stato eseguito il rendering.

## <a name="possible-fixes"></a>Correzioni possibili

Verificare che ogni chiamata a BeginDraw corrisponda a un EndDraw successivo. Quando si inviano comandi a una destinazione usando Flush o EndDraw, verificare che il risultato restituito sia S \_ OK prima di provare a usare nuovamente la destinazione.
