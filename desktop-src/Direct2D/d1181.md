# <a name="d1181-enddraw-incorrect-state"></a>D1181: Stato non corretto di EndDraw

È stato chiamato EndDraw, ma la destinazione di rendering non era in uno stato valido. Ciò può derivare dalla chiamata di EndDraw senza un BeginDraw corrispondente.






 

## <a name="possible-causes"></a>Possibili cause

L'applicazione ha effettuato più chiamate a EndDraw senza chiamare BeginDraw nella stessa destinazione. Quando si usa Direct2D, le applicazioni devono corrispondere a ogni chiamata a EndDraw con una chiamata precedente a BeginDraw. In alternativa, questo messaggio può verificarsi se la destinazione è in uno stato eroso e non è possibile eseguirne il rendering.

## <a name="possible-fixes"></a>Possibili correzioni

Assicurarsi che ogni chiamata a BeginDraw corrisponda a un enddraw successivo. Quando si inviano comandi a una destinazione usando Flush o EndDraw, verificare che il risultato restituito sia OK prima di provare a \_ usare nuovamente la destinazione.
