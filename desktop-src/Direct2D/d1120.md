# <a name="d1120-invalid-attempt-to-draw-bitmap"></a>D1120: Tentativo non valido di disegnare una bitmap

Un'operazione non è riuscita perché la bitmap ha l'opzione D2D1 \_ BITMAP \_ OPTIONS CANNOT \_ \_ DRAW.





 

## <a name="possible-causes"></a>Possibili cause

Questo messaggio viene visualizzato quando un'applicazione tenta di disegnare una bitmap contrassegnata con D2D1 \_ BITMAP \_ OPTIONS CANNOT \_ \_ DRAW. Sono incluse le chiamate dirette a DrawBitmap, nonché il disegno di primitive usando un pennello bitmap.

## <a name="possible-fixes"></a>Possibili correzioni

Il flag CANNOT DRAW viene usato per annotare correttamente le risorse che possono essere usate solo come destinazioni, ad esempio backbuffer della catena \_ di scambio. Tuttavia, è anche consentito omettere il flag CANNOT DRAW per alcune bitmap di destinazione se possono essere usate \_ anche come origine. In questo caso, è sufficiente omettere il flag CANNOT \_ DRAW. Se la bitmap deve essere annotata con CANNOT DRAW a causa dell'uso esclusivo come destinazione o per consentire il blocco della CPU, è consigliabile usare una trama di staging con le stesse dimensioni dell'origine desiderata, ma senza la \_ designazione CANNOT \_ DRAW. Copiare nella trama di staging usando ID3D11DeviceContext::CopyResource e usare la trama di staging come origine per l'operazione di disegno. L'uso di una trama di staging causa implicazioni in termini di prestazioni; possono essere accettabili se la bitmap di origine non cambia spesso.
