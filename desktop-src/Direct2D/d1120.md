# <a name="d1120-invalid-attempt-to-draw-bitmap"></a>D1120: tentativo non valido di creare una bitmap

Un'operazione non è riuscita perché nella bitmap sono \_ disponibili le opzioni bitmap d2d1 \_ \_ \_ . Impossibile creare l'opzione.



|             |             |
|-------------|-------------|
| Livello di errore | Informazioni |



 

## <a name="possible-causes"></a>Possibili cause

Questo messaggio si verifica quando un'applicazione tenta di creare una bitmap contrassegnata con \_ le \_ opzioni bitmap d2d1 \_ non è in grado di \_ creare. Sono incluse le chiamate dirette a DrawBitmap, nonché il disegno di primitive usando un pennello bitmap.

## <a name="possible-fixes"></a>Correzioni possibili

Il \_ flag di traccia non può essere usato per annotare correttamente le risorse che sono utilizzabili come destinazioni, ad esempio i buffer della catena di scambio. Tuttavia, è anche consentito omettere il flag non è possibile \_ creare per alcune bitmap di destinazione se possono anche essere utilizzate come origine. In tal caso, è sufficiente omettere il flag non è possibile \_ creare. Se la bitmap deve essere annotata con la proprietà non può essere creata a \_ causa dell'uso esclusivo come destinazione o per consentire la blocco della CPU, provare a usare una trama di staging con le stesse dimensioni dell'origine desiderata, ma senza la designazione non può essere \_ disegnato. Copiare nella trama di gestione temporanea usando sul ID3D11DeviceContext:: CopyResource e usare la trama di staging come origine per l'operazione di disegno. L'uso di una trama di gestione temporanea offre implicazioni sulle prestazioni; potrebbero essere accettabili se la bitmap di origine non cambia spesso.
