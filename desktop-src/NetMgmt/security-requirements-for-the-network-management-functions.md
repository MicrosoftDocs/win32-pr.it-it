# <a name="network-management-function-security-requirements"></a>Requisiti di sicurezza delle funzioni di gestione di rete

La chiamata di alcune delle funzioni di gestione di rete non richiede l'appartenenza a gruppi speciali. Altre funzioni richiedono che gli utenti abbia un livello di privilegio specifico per l'esecuzione corretta. Se applicabile, la sezione Osservazioni nella pagina di riferimento di una funzione indica il livello di privilegio che un utente deve avere per eseguire la funzione specifica.

I requisiti di sicurezza che si applicano ai controller di dominio di Active Directory possono differire da quelli che si applicano a server e workstation.

Per altre informazioni e un elenco delle funzioni interessate, vedere gli argomenti seguenti:

-   [Requisiti per le funzioni di gestione della rete nei controller di dominio di Active Directory](requirements-for-network-management-functions-on-active-directory-domain-controllers.md)
-   [Requisiti per le funzioni di gestione della rete in server e workstation](requirements-for-network-management-functions-on-servers-and-workstations.md)

Per altre informazioni sul controllo dell'accesso agli oggetti a protezione diretta, vedere [Controllo](/windows/desktop/SecAuthZ/access-control)di accesso , [privilegi](/windows/desktop/SecAuthZ/privileges)e [oggetti a protezione diretta](/windows/desktop/SecAuthZ/securable-objects).

Per altre informazioni sui descrittori di sicurezza specifici usati durante i controlli di accesso, vedere la pagina di riferimento delle singole funzioni. Per altre informazioni sulla chiamata di funzioni che richiedono privilegi di amministratore, vedere [Esecuzione con privilegi speciali.](/windows/desktop/SecBP/running-with-special-privileges)
