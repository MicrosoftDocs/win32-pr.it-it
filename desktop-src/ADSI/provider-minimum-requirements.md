# <a name="provider-minimum-requirements"></a>Requisiti minimi del provider

Per scrivere un provider ADSI, implementare le interfacce del servizio Active Directory come descritto nelle sezioni riportate di seguito.

-   L' [implementazione di base](core-implementation.md) Elenca brevemente gli elementi software che un provider ADSI deve supportare.
-   L' [implementazione facoltativa](optional-implementation.md) Elenca brevemente gli elementi software che un provider ADSI è fortemente consigliato per supportare.
-   Le [proprietà personalizzate](custom-properties.md) sono le proprietà e le interfacce esposte da ogni provider per fornire la programmabilità delle funzionalità native del servizio directory.
-   Le [interfacce duali](dual-interfaces.md) descrivono la convenzione ADSI per fornire interfacce accessibili sia per i controller di automazione che per quelli non conformi all'automazione.
