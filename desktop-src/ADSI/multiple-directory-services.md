# <a name="multiple-directory-services"></a>Servizi directory multipli

Una sfida dell'uso di un ambiente di elaborazione distribuita è l'identificazione e l'individuazione di risorse quali utenti, gruppi, code di stampa e documenti. Un servizio directory fa parte di un ambiente di elaborazione distribuita che consente di individuare e identificare gli utenti e le risorse disponibili nel sistema. Un servizio directory è simile a una directory telefonica. Dato un nome per una persona o una risorsa, fornisce i dati necessari per accedere a tale persona o risorsa. Non è necessario iniziare con l'associazione di dati per accedere a una risorsa in rete.

La maggior parte delle aziende dispone già di molte directory diverse. Ad esempio, i sistemi operativi di rete, i sistemi di posta elettronica e i prodotti "groupware" hanno tutte le proprie directory. Quando una singola azienda distribuisce più directory, si verificano molti problemi. Questi problemi includono usabilità, coerenza dei dati, costi di sviluppo e costi di supporto, tra gli altri.

Questi problemi vengono risolti in ADSI, in quanto viene fornito un unico set coerente e aperto di interfacce che possono essere utilizzate per la gestione e l'uso di qualsiasi servizio directory che dispone di un provider ADSI.
