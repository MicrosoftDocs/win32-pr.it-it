# <a name="registry-key-security-and-access-rights"></a>Diritti di accesso e sicurezza della chiave del registro di sistema

Il modello di sicurezza di Windows consente di controllare l'accesso alle chiavi del registro di sistema. Per altre informazioni sulla sicurezza, vedere [modello di controllo di accesso](/windows/desktop/SecAuthZ/access-control-model).

È possibile specificare un [descrittore di sicurezza](/windows/desktop/SecAuthZ/security-descriptors) per una chiave del registro di sistema quando si chiama la funzione [**RegCreateKeyEx**](/windows/desktop/api/Winreg/nf-winreg-regcreatekeyexa) o [**RegSetKeySecurity**](/windows/desktop/api/winreg/nf-winreg-regsetkeysecurity) . Se si specifica **null**, la chiave ottiene un descrittore di sicurezza predefinito. Gli ACL in un descrittore di sicurezza predefinito per una chiave vengono ereditati dalla relativa chiave padre diretta.

Per ottenere il descrittore di sicurezza di una chiave del registro di sistema, chiamare la funzione [**RegGetKeySecurity**](/windows/desktop/api/winreg/nf-winreg-reggetkeysecurity), [**GetNamedSecurityInfo**](/windows/desktop/api/aclapi/nf-aclapi-getnamedsecurityinfoa)o [**GetSecurityInfo**](/windows/desktop/api/aclapi/nf-aclapi-getsecurityinfo) .

I diritti di accesso validi per le chiavi del registro di sistema includono i diritti di accesso di tipo eliminazione, lettura \_ \_ e scrittura DAC e scrittura \_ proprietario. [](/windows/desktop/SecAuthZ/standard-access-rights) Le chiavi del registro di sistema non supportano il diritto di sincronizzazione dell'accesso standard.

Quando si chiama la funzione [**RegOpenKeyEx**](/windows/desktop/api/Winreg/nf-winreg-regopenkeyexa) , il sistema controlla i diritti di accesso richiesti rispetto al descrittore di sicurezza della chiave. Se l'utente non dispone dell'accesso corretto alla chiave del registro di sistema, l'operazione di apertura non riesce. Se un amministratore deve accedere alla chiave, la soluzione consiste nell'abilitare il privilegio SE \_ accetta il \_ nome di proprietà \_ e aprire la chiave del registro di sistema con l'accesso in scrittura al \_ proprietario. Per ulteriori informazioni, vedere [Abilitazione e disabilitazione dei privilegi](/windows/desktop/SecAuthZ/enabling-and-disabling-privileges-in-c--).

È possibile richiedere il \_ \_ diritto di accesso di sicurezza del sistema di accesso a una chiave del registro di sistema se si desidera leggere o scrivere l'elenco di controllo di accesso di sistema (SACL) della chiave. Per altre informazioni, vedere [elenchi di controllo di accesso (ACL)](/windows/desktop/SecAuthZ/access-control-lists) e [accesso SACL diritto](/windows/desktop/SecAuthZ/sacl-access-right).

Per visualizzare i diritti di accesso correnti per una chiave, incluse le chiavi predefinite, utilizzare l'editor del registro di sistema (Regedt32.exe). Dopo aver eseguito la navigazione alla chiave desiderata, passare al menu **modifica** e selezionare **autorizzazioni**.
