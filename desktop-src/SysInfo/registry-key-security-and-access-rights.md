# <a name="registry-key-security-and-access-rights"></a>Sicurezza e diritti di accesso delle chiavi del Registro di sistema

Il Windows di sicurezza consente di controllare l'accesso alle chiavi del Registro di sistema. Per altre informazioni sulla sicurezza, vedere [Modello di controllo di accesso.](/windows/desktop/SecAuthZ/access-control-model)

È possibile specificare un [descrittore di sicurezza](/windows/desktop/SecAuthZ/security-descriptors) per una chiave del Registro di sistema quando si chiama la [**funzione RegCreateKeyEx**](/windows/desktop/api/Winreg/nf-winreg-regcreatekeyexa) o [**RegSetKeySecurity.**](/windows/desktop/api/winreg/nf-winreg-regsetkeysecurity) Se si specifica **NULL,** la chiave ottiene un descrittore di sicurezza predefinito. Gli ACL in un descrittore di sicurezza predefinito per una chiave vengono ereditati dalla relativa chiave padre diretta.

Per ottenere il descrittore di sicurezza di una chiave del Registro di sistema, chiamare la funzione [**RegGetKeySecurity,**](/windows/desktop/api/winreg/nf-winreg-reggetkeysecurity) [**GetNamedSecurityInfo**](/windows/desktop/api/aclapi/nf-aclapi-getnamedsecurityinfoa)o [**GetSecurityInfo.**](/windows/desktop/api/aclapi/nf-aclapi-getsecurityinfo)

I diritti di accesso validi per le chiavi del Registro di sistema includono i diritti di accesso standard DELETE, READ \_ CONTROL, WRITE DAC e \_ WRITE \_ [](/windows/desktop/SecAuthZ/standard-access-rights)OWNER. Le chiavi del Registro di sistema non supportano il diritto di accesso standard SYNCHRONIZE.

Quando si chiama la [**funzione RegOpenKeyEx,**](/windows/desktop/api/Winreg/nf-winreg-regopenkeyexa) il sistema controlla i diritti di accesso richiesti rispetto al descrittore di sicurezza della chiave. Se l'utente non ha l'accesso corretto alla chiave del Registro di sistema, l'operazione di apertura non riesce. Se un amministratore deve accedere alla chiave, la soluzione consiste nell'abilitare il privilegio TAKE OWNERSHIP NAME di edizione Standard e aprire la chiave del Registro di sistema con accesso \_ \_ WRITE \_ \_ OWNER. Per altre informazioni, vedere [Abilitazione e disabilitazione dei privilegi.](/windows/desktop/SecAuthZ/enabling-and-disabling-privileges-in-c--)

È possibile richiedere il diritto di accesso ACCESS SYSTEM SECURITY a una chiave del Registro di sistema se si vuole leggere o scrivere l'elenco di controllo di accesso di sistema \_ \_ (SACL) della chiave. Per altre informazioni, vedere [Elenchi di controllo di accesso (ACL)](/windows/desktop/SecAuthZ/access-control-lists) e Diritto di accesso [SACL.](/windows/desktop/SecAuthZ/sacl-access-right)

Per visualizzare i diritti di accesso correnti per una chiave, incluse le chiavi predefinite, usare l'editor del Registro di sistema (Regedt32.exe). Dopo aver selezionato la chiave desiderata, passare al menu **Modifica** e selezionare **Autorizzazioni.**
