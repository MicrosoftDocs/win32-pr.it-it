# <a name="how-to-design-a-domain-shader"></a><span data-ttu-id="75bc2-103">Procedura: progettare un Domain shader</span><span class="sxs-lookup"><span data-stu-id="75bc2-103">How To: Design a Domain Shader</span></span>

<span data-ttu-id="75bc2-104">Un Domain shader è il terzo di tre fasi che interagiscono per implementare lo [schema a mosaico](direct3d-11-advanced-stages-tessellation.md).</span><span class="sxs-lookup"><span data-stu-id="75bc2-104">A domain shader is the third of three stages that work together to implement [tessellation](direct3d-11-advanced-stages-tessellation.md).</span></span> <span data-ttu-id="75bc2-105">Il Domain shader genera la geometria della superficie dai punti di controllo trasformati da uno scafo dello shader e dalle coordinate UV.</span><span class="sxs-lookup"><span data-stu-id="75bc2-105">The domain shader generates the surface geometry from the transformed control points from a hull shader and the UV coordinates.</span></span> <span data-ttu-id="75bc2-106">In questo argomento viene illustrato come progettare un Domain shader.</span><span class="sxs-lookup"><span data-stu-id="75bc2-106">This topics shows how to design a domain shader.</span></span>

<span data-ttu-id="75bc2-107">Un Domain shader viene richiamato una volta per ogni punto generato dalla funzione fissa mosaico.</span><span class="sxs-lookup"><span data-stu-id="75bc2-107">A domain shader is invoked once for each point generated by the fixed function tessellator.</span></span> <span data-ttu-id="75bc2-108">Gli input sono le \[ coordinate UV W \] del punto della patch, nonché tutti i dati di output dello scafo shader, inclusi i punti di controllo e le costanti di patch.</span><span class="sxs-lookup"><span data-stu-id="75bc2-108">The inputs are the UV\[W\] coordinates of the point on the patch, as well as all of the output data from the hull shader including control points and patch constants.</span></span> <span data-ttu-id="75bc2-109">L'output è un vertice definito nel modo desiderato.</span><span class="sxs-lookup"><span data-stu-id="75bc2-109">The output is a vertex defined in whatever way is desired.</span></span> <span data-ttu-id="75bc2-110">Se l'output viene inviato al pixel shader, l'output deve includere una posizione (indicata con una \_ semantica di posizione SV).</span><span class="sxs-lookup"><span data-stu-id="75bc2-110">If the output is being sent to the pixel shader, the output must include a position (denoted with a SV\_Position semantic).</span></span>

<span data-ttu-id="75bc2-111">**Per progettare un Domain shader**</span><span class="sxs-lookup"><span data-stu-id="75bc2-111">**To design a domain shader**</span></span>

1.  <span data-ttu-id="75bc2-112">Definire l'attributo di dominio.</span><span class="sxs-lookup"><span data-stu-id="75bc2-112">Define the domain attribute.</span></span>

    ```
    [domain("quad")]
    ```

    

    <span data-ttu-id="75bc2-113">Il dominio è definito per una patch quad.</span><span class="sxs-lookup"><span data-stu-id="75bc2-113">The domain is defined for a quad patch.</span></span>

2.  <span data-ttu-id="75bc2-114">Dichiarare il percorso sullo scafo con il valore del sistema del [percorso del dominio](/windows/desktop/direct3dhlsl/sv-domainlocation) .</span><span class="sxs-lookup"><span data-stu-id="75bc2-114">Declare the location on the hull with the [domain location](/windows/desktop/direct3dhlsl/sv-domainlocation) system value.</span></span>

    -   <span data-ttu-id="75bc2-115">Per una patch quad, usare un float2.</span><span class="sxs-lookup"><span data-stu-id="75bc2-115">For a quad patch, use a float2.</span></span>
    -   <span data-ttu-id="75bc2-116">Per una patch Tri, usare float3 (per le coordinate baricentrica)</span><span class="sxs-lookup"><span data-stu-id="75bc2-116">For a tri patch, use a float3 (for barycentric coordinates)</span></span>
    -   <span data-ttu-id="75bc2-117">Per un, usare un float2.</span><span class="sxs-lookup"><span data-stu-id="75bc2-117">For an isoline, use a float2.</span></span>

    <span data-ttu-id="75bc2-118">Il percorso del dominio per una patch Quad è pertanto simile al seguente:</span><span class="sxs-lookup"><span data-stu-id="75bc2-118">Therefore, the domain location for a quad patch looks like this:</span></span>

    ```
    float2 UV : SV_DomainLocation
    ```

    

3.  <span data-ttu-id="75bc2-119">Definire gli altri input.</span><span class="sxs-lookup"><span data-stu-id="75bc2-119">Define the other inputs.</span></span>

    <span data-ttu-id="75bc2-120">Gli altri input provengono dallo scafo dello shader e sono definiti dall'utente.</span><span class="sxs-lookup"><span data-stu-id="75bc2-120">The other inputs come from the hull shader and are user defined.</span></span> <span data-ttu-id="75bc2-121">Sono inclusi i punti di controllo di input per la patch, tra i quali possono essere presenti da 1 a 32 punti e i dati costanti della patch di input.</span><span class="sxs-lookup"><span data-stu-id="75bc2-121">This includes the input control points for patch, of which there can be between 1 and 32 points, and input patch constant data.</span></span>

    <span data-ttu-id="75bc2-122">I punti di controllo sono definiti dall'utente, in genere con una struttura come questa (definita in [procedura: progettare uno scafo shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span><span class="sxs-lookup"><span data-stu-id="75bc2-122">The control points are user defined, usually with a structure such as this one (defined in [How To: Design a Hull Shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span></span>

    ```
    const OutputPatch<BEZIER_CONTROL_POINT, 16> bezpatch
    ```

    

    <span data-ttu-id="75bc2-123">Anche i dati delle costanti patch sono definiti dall'utente e possono avere un aspetto simile al seguente (definito in [procedura: progettare uno scafo shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span><span class="sxs-lookup"><span data-stu-id="75bc2-123">The patch constant data is also user defined, and might look like this one (defined in [How To: Design a Hull Shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span></span>

    ```
    HS_CONSTANT_DATA_OUTPUT input
    ```

    

4.  <span data-ttu-id="75bc2-124">Aggiungere codice definito dall'utente per calcolare gli output; Ciò costituisce il corpo dello shader del dominio.</span><span class="sxs-lookup"><span data-stu-id="75bc2-124">Add user-defined code to compute the outputs; this makes up the body of the domain shader.</span></span>

    <span data-ttu-id="75bc2-125">Questa struttura contiene output di Domain shader definiti dall'utente.</span><span class="sxs-lookup"><span data-stu-id="75bc2-125">This structure contains user-defined domain shader outputs.</span></span>

    ```
    struct DS_OUTPUT
    {
        float3 vNormal    : NORMAL;
        float2 vUV        : TEXCOORD;
        float3 vTangent   : TANGENT;
        float3 vBiTangent : BITANGENT;
        
        float4 vPosition  : SV_POSITION;
    };
    ```

    

    <span data-ttu-id="75bc2-126">La funzione accetta ogni UV di input (mosaico) e valuta la patch di Bezier in questa posizione.</span><span class="sxs-lookup"><span data-stu-id="75bc2-126">The function takes each input UV (from the tessellator) and evaluates the Bezier patch at this position.</span></span>

    ```
    [domain("quad")]
    DS_OUTPUT BezierEvalDS( HS_CONSTANT_DATA_OUTPUT input, 
                            float2 UV : SV_DomainLocation,
                            const OutputPatch<BEZIER_CONTROL_POINT, 16> bezpatch )
    {
        DS_OUTPUT Output;

        // Insert code to compute the output here.

        return Output;    
    }
    ```

    

    <span data-ttu-id="75bc2-127">La funzione viene richiamata una volta per ogni punto generato dalla funzione fissa mosaico.</span><span class="sxs-lookup"><span data-stu-id="75bc2-127">The function is invoked once for each point generated by the fixed function tessellator.</span></span> <span data-ttu-id="75bc2-128">Poiché in questo esempio viene usata una patch quad, il percorso del dominio di input ([SV \_ DomainLocation](/windows/desktop/direct3dhlsl/sv-domainlocation)) è un float2 (UV). una patch Tri avrà un percorso di input float3 (UVW baricentrica coordinate) e un percorso di dominio float2 input.</span><span class="sxs-lookup"><span data-stu-id="75bc2-128">Since this example uses a quad patch, the input domain location ([SV\_DomainLocation](/windows/desktop/direct3dhlsl/sv-domainlocation)) is a float2 (UV); a tri patch would have a float3 input location (UVW barycentric coordinates), and an isoline would have a float2 input domain location.</span></span>

    <span data-ttu-id="75bc2-129">Gli altri input per la funzione provengono direttamente dallo scafo dello shader.</span><span class="sxs-lookup"><span data-stu-id="75bc2-129">The other inputs for the function come from the hull shader directly.</span></span> <span data-ttu-id="75bc2-130">In questo esempio si tratta di 16 punti di controllo ognuno dei quali è un **\_ \_ punto di controllo Bezier**, nonché i dati delle costanti patch (**\_ \_ \_ output di dati costanti HS**).</span><span class="sxs-lookup"><span data-stu-id="75bc2-130">In this example it is 16 control points each one being a **BEZIER\_CONTROL\_POINT**, as well as patch constant data (**HS\_CONSTANT\_DATA\_OUTPUT**).</span></span> <span data-ttu-id="75bc2-131">L'output è un vertice che contiene i dati desiderati, in questo esempio l' **\_ output di DS** .</span><span class="sxs-lookup"><span data-stu-id="75bc2-131">The output is a vertex containing any data desired - **DS\_OUTPUT** in this example.</span></span>
