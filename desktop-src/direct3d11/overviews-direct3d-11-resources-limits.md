 Un'app può provare ad allocare più memoria per una risorsa rispetto alle dimensioni massime specificate. Ciò significa che il runtime Direct3D 11 consente questi tentativi di allocazione di memoria nel caso in cui l'hardware possa supportarli. Tuttavia, il runtime Direct3D 11 garantisce solo che le allocazioni entro le dimensioni massime delle risorse siano supportate da tutto l'hardware [di livello](overviews-direct3d-11-devices-downlevel-intro.md) 11 della funzionalità. Se un'app tenta di allocare memoria per una risorsa entro le dimensioni massime della risorsa, il runtime ha esito negativo solo se il sistema operativo esaurisce le risorse. Se un'app tenta di allocare memoria per una risorsa oltre le dimensioni massime della risorsa, il runtime può non riuscire perché il sistema operativo è estenso o l'hardware non supporta allocazioni superiori alle dimensioni massime della risorsa. Il livello di debug controlla solo D3D11 \_ REQ \_ RESOURCE SIZE IN \_ \_ \_ MEGABYTES EXPRESSION A TERM \_ \_ \_ (128) MB.

 Gli pixel shader di output vengono condivisi tra registri di output pixel (quattro componenti a 32 bit) e UAV.

 Il numero totale di componenti per tutti i punti di controllo da hull shader a domain shader è limitato a 3968, ovvero 128 meno dei punti di controllo massimi per il numero massimo di registri del punto di controllo per quattro componenti.

4Per i profili compute shader CS \_ 4 \_ 0 e CS \_ 4 \_ 1 è disponibile solo 1 UAV. Per altre informazioni sui profili shader, vedere [Modello shader 5.](/windows/desktop/direct3dhlsl/d3d11-graphics-reference-sm5)
