# <a name="purpose-of-the-printticket"></a>Scopo di PrintTicket

Questo argomento non è corrente. Per le informazioni più aggiornate, vedere Print [Schema Specification](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Lo scopo principale di PrintTicket è esprimere le configurazioni dei dispositivi. A differenza del documento PrintCapabilities, che definisce tutte le possibili configurazioni che il dispositivo può assumere (lo spazio di configurazione del dispositivo), lo scopo di PrintTicket è definire una singola configurazione specifica che viene applicata all'oggetto processo (in questo caso la pagina) che viene elaborato. Per promuovere la portabilità e la coerenza delle configurazioni, le parole chiave e la gerarchia specifiche presentate nella sezione Parole chiave dello schema di stampa vengono usate dalle tecnologie PrintCapabilities e PrintTicket. Il framework dello [schema](print-schema-framework.md) di stampa definisce inoltre i costrutti di base di entrambe le tecnologie, per promuovere una comunicazione non ambigua, aperta ed estendibile della formattazione dei processi e di PrintCapabilities.
