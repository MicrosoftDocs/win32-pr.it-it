# <a name="terms-used-in-the-print-schema"></a>Termini usati nello schema di stampa

Questo argomento non è corrente. Per le informazioni più aggiornate, vedere Print [Schema Specification](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

L'elenco seguente contiene termini specifici dello schema di stampa e delle tecnologie correlate.



| Termine                                                                                                                                                                 | Descrizione                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>nome dell'attributo<br/>                                                                          | Un nome di attributo è il token a sinistra del segno di uguale in un attributo XML standard. La rappresentazione standard di un attributo XML è *il nome dell'attributo*="*valore dell'attributo*". Anche se simile, il termine "nome attributo" non deve essere confuso con l'attributo name definito in Print Schema Framework. L'attributo name definito in Print Schema Framework è un'istanza specifica di un nome di attributo. <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>Candidato<br/>                                                                                         | Opzione. Qualsiasi opzione che appartiene all'elenco valutato durante il processo di assegnazione dei punteggi. Vedere i termini, l'opzione di assegnazione dei punteggi o il mapping e l'opzione di riferimento.<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>Configurazione<br/>                                             | Descrive uno degli stati in cui un dispositivo può presupporre o essere inserito. Il dispositivo esegue l'attività in modo diverso in ogni stato. Esiste una differenza distinta e identificabile tra il comportamento del dispositivo in ognuno di questi stati. Ad esempio, in uno stato, il supporto viene estratto da un vassoio e finisce in un altro vassoio. Una modifica in uno o entrambi questi vassoi determina uno stato diverso.<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>Vincolo<br/>                                                         | Informazioni necessarie per determinare la validità di una configurazione. Un vincolo è spesso nel formato: FeatureA impostato su Option1 non è compatibile con FeatureB impostato su Option2.<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>elementi corrispondenti<br/>                                                  | Due elementi che hanno la stessa posizione rispetto a una coppia specificata di elementi di origine. Gli elementi di origine sono spesso due elementi Option che vengono confrontati.<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>Predefinito<br/>                                                                                               | Opzione. Opzione designata dallo schema di stampa o dal provider PrintCapabilities come opzione assegnata a una funzionalità quando PrintTicket non contiene un'impostazione esplicita per tale funzionalità. Ogni funzionalità ha un'opzione predefinita designata.<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>attributo di configurazione del dispositivo<br/>                          | Attributo del dispositivo che contribuisce alla definizione della configurazione del dispositivo. Un attributo di configurazione del dispositivo è rappresentato in Print Schema Framework da costrutti di funzionalità/opzione o parametri. Si noti che l'attributo utilizzato in questo contesto non ha alcuna relazione con gli attributi XML o con gli elementi Attribute definiti dal framework dello schema di stampa.<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>tipo di elemento<br/>                                                                                | Il tipo di ogni elemento XML definito in Print Schema Framework è identificato dal relativo tag di elemento XML.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>elementi in comune<br/>                                                              | Vedere gli elementi corrispondenti.<br/> Funzionalità. Costrutto definito dal framework dello schema di stampa che rappresenta un attributo di configurazione del dispositivo specifico. Ogni istanza di funzionalità rappresenta un attributo del dispositivo. Un'istanza feature viene definita definendo in modo esplicito ogni stato.<br/> Opzione Identity. Opzione che non ha alcun effetto osservabile quando è selezionata; in effetti, un "no-op". Il risultato è sempre lo stesso indipendentemente dal numero di volte in cui questa opzione è selezionata.<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>nome dell'istanza<br/>                                                                             | Attributo XML che identifica l'istanza specifica di un tipo di elemento. Vedere l'attributo name.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>client di interfaccia<br/>                                                                    | Entità che utilizza documenti PrintCapabilities e che costruisce e utilizza PrintTicket. Esempi comuni di client di interfaccia includono moduli di interfaccia utente e moduli di rendering.<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>provider di interfaccia<br/>                                                              | Entità che implementa ed espone funzionalità tramite un'interfaccia. Nello schema di stampa si tratta della creazione di documenti PrintCapabilities e della convalida di PrintTicket. <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>posizione di un elemento<br/>                                                  | Posizione di una particolare istanza di un elemento all'interno della gerarchia degli elementi. La posizione è definita sia dall'elemento specifico che da tutti i predecessori di tale elemento. In altre parole, location è l'equivalente di un percorso completo in un file system. <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>Attributo name<br/>                                                                          | Designazione data all'attributo XML con il nome di attributo "name". Questo attributo viene visualizzato nella maggior parte dei tipi di elemento definiti in Print Schema Framework. Identifica e definisce le istanze di ogni tipo di elemento. I due termini seguenti fanno riferimento agli attributi del nome: regola di univocità di pari livello e convenzione dello spazio dei nomi.<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>Convenzione dello spazio dei nomi<br/>                 | Indica che gli attributi dei nomi definiti da schema, nonché qualsiasi oggetto definito da schema, esistono in uno spazio dei nomi dedicato, nello spazio dei nomi pubblico e che gli attributi del nome definiti privatamente devono esistere in uno spazio dei nomi diverso, che può essere identificato come appartenente all'entità che definisce.<br/> Opzione. Costrutto Print Schema Framework usato per rappresentare gli stati che ogni funzionalità può presupporre o in cui può essere inserito. Ad esempio, se la funzionalità rappresenta le dimensioni dei supporti, i valori tipici di Option potrebbero essere Letter, Legal, A4, tra gli altri.<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>Parametro<br/>                                                             | Lo schema di stampa supporta la possibilità di descrivere i dati con parametri ai fini dell'input dell'utente. Esistono tre tipi di elemento print schema framework che svolgono ruoli nell'input parametrico: ParameterDef, ParameterInit e ParameterRef. Un elemento ParameterDef caratterizza un parametro ,ovvero l'intervallo di valori, il tipo di dati e altre caratteristiche. Un elemento ParameterInit viene usato per inizializzare un parametro. Un elemento ParameterRef viene usato per fare riferimento a una particolare istanza di un elemento ParameterInit.<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>Parametri<br/>                                                                             | Opzione. Istanza Option che contiene almeno un elemento ScoredProperty che a sua volta contiene un elemento ParameterRef anziché un elemento Value.<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>perfettamente corrispondenti<br/>                                                              | Opzioni. Due istanze option vengono considerate perfettamente corrispondenti se per ogni elemento ScoredProperty in un'istanza Option è presente un'istanza ScoredProperty corrispondente nell'altra istanza option ed entrambi gli elementi Value delle istanze ScoredProperty corrispondenti sono identici.<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>PrintCapabilities Schema<br/> | Schema che definisce la sintassi e la semantica usate da un documento PrintCapabilities. Lo schema PrintCapabilities e lo schema PrintTicket sono subset dello schema print. Vedere anche Print Schema Framework.<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>PrintTicket Schema<br/>                         | Schema che definisce la sintassi e la semantica usate da PrintTicket. Lo schema PrintTicket e lo schema PrintCapabilities sono subset dello schema print. Vedere anche Print Schema Framework.<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Framework dello schema di stampa<br/>         | Schema che definisce la sintassi e la semantica utilizzate dalle tecnologie dello schema di stampa. Ad esempio, Print Schema Framework definisce una raccolta di tipi di elemento e i relativi utilizzi, ma non enumera tutte le istanze degli elementi.<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>Parole chiave dello schema di stampa<br/>             | Schema che definisce le singole istanze degli elementi Property, Feature e Option comunemente usati per descrivere i dispositivi e la formattazione dei processi. Questo schema definisce un linguaggio comune che rende portabili documenti PrintCapabilities e PrintTicket. Noto anche come schema pubblico.<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>definita privatamente<br/>                                                                 | Qualsiasi istanza di elementi presenti in un documento PrintCapabilities o PrintTicket che non sono definiti in modo esplicito nelle parole chiave dello schema di stampa. Queste istanze di elemento devono essere qualificate in modo esplicito con uno spazio dei nomi privato per fornitore.<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>Riferimento<br/>                                                                                         | Opzione. Opzione con cui viene confrontata ogni opzione candidata. Un'opzione di riferimento ha in genere origine da PrintTicket. Vedere i termini, l'opzione di assegnazione dei punteggi o il mapping e l'opzione candidata.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>livello radice<br/>                                                                                      | Il livello radice è la base della struttura gerarchica di Print Schema Framework. Un elemento di livello radice è definito come figlio di un elemento PrintCapabilities o di un elemento PrintTicket.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>Prefisso di ambito<br/>                                                                          | Etichetta testuale preaccodata a una parola chiave dello schema per fornire l'ambito contestuale. I prefissi di ambito consentiti per lo schema di stampa sono Job, Doc e Page. Ad esempio, il prefisso di ambito per JobInputBin è "Job".<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>ScoredProperty<br/>                                         | Costrutto di Print Schema Framework usato per rappresentare le qualità che distinguono o differenziano un'istanza option da un'altra. Un esempio è la differenza tra letter e legal sized paper.<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>assegnazione di punteggi o mapping<br/>                                                              | Opzione. Processo di selezione da un elenco dell'istanza opzione (candidata) che meglio corrisponde a una determinata istanza di opzione (riferimento). I termini strettamente correlati sono opzione candidata e opzione di riferimento. <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>Regola di univocità di pari livello<br/>     | Indica che tutti gli elementi di pari livello dello stesso tipo di elemento devono avere attributi di nome univoci. Alcuni tipi di elementi sono esentati da questa regola. Queste eccezioni sono definite in Print Schema Framework.<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>a valore singolo e multivalore<br/>                    | Proprietà. Una proprietà il cui valore è invariante (non dipende dalla configurazione) viene considerata a valore singolo. Una proprietà di questo tipo ha un solo valore in qualsiasi configurazione possibile. Una proprietà che dipende dalla configurazione viene considerata multivalore.<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>suffissi/suffissi<br/>                                                                       | Parola chiave dello schema che rappresenta il nome della funzionalità o del parametro. Ad esempio, il suffisso per JobInputBin è 'InputBin'.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>Convalida<br/>                                                         | Processo di verifica della conformità di printTicket allo schema e rappresenta una configurazione che può essere eseguita da un dispositivo specifico.<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_Indefinito\_<br/>                                                   | I dati specifici rappresentati qui sono un segnaposto per il driver, il dispositivo o l'applicazione specifici da produrre. Non indica che i dati devono essere una stringa.<br/>                                                                                                                                                                                                                                                                                                                                                            |
