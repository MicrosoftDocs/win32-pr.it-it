# <a name="using-capicom"></a>Uso di CAPICOM

\[CAPICOM è un componente solo a 32 bit disponibile per l'uso nei sistemi operativi seguenti: Windows Server 2008, Windows Vista e Windows XP. Usare invece il .NET Framework per implementare le funzionalità di sicurezza. Per altre informazioni, vedere [Alternative all'uso di CAPICOM.](alternatives-to-using-capicom.md)\]

Questa sezione include scenari che usano procedure CAPICOM.

> [!Note]  
> La [](../secgloss/d-gly.md) creazione di firme digitali e messaggi non inviluppi con CAPICOM viene eseguita usando la crittografia PKI (Public Key Infrastructure) e può essere eseguita solo se il firmatario o l'utente che decrittografa un messaggio in busta ha accesso a un certificato con una chiave privata associata disponibile. Per decrittografare un messaggio in busta, è necessario che nell'archivio MY sia presente un certificato con accesso alla chiave privata.
