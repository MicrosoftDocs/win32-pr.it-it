# <a name="using-capicom"></a>Uso di CAPICOM

\[CAPICOM è un componente solo a 32 bit disponibile per l'uso nei sistemi operativi seguenti: Windows Server 2008, Windows Vista e Windows XP. Usare invece il .NET Framework per implementare le funzionalità di sicurezza. Per altre informazioni, vedere [alternative all'uso di CAPICOM](alternatives-to-using-capicom.md).\]

Questa sezione include scenari che usano le procedure di CAPICOM.

> [!Note]  
> La creazione di [*firme digitali*](../secgloss/d-gly.md) e la disattivazione di messaggi con CAPICOM viene eseguita usando la crittografia a chiave pubblica (PKI) e può essere eseguita solo se il firmatario o l'utente che decrittografa un messaggio in busta ha accesso a un certificato con una chiave privata disponibile associata. Per decrittografare un messaggio in busta digitale, un certificato con accesso alla chiave privata deve trovarsi nell'archivio personale.
