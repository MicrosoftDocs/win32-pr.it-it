# <a name="prov_dss"></a>PROVA \_ DSS

Il \_ tipo di provider prov DSS, ad esempio prov \_ RSA \_ sig, supporta solo [*hash*](../secgloss/h-gly.md) e [*firme digitali*](../secgloss/d-gly.md). L'algoritmo di firma specificato dal \_ tipo di provider prov DSS è l' [*algoritmo di firma digitale*](../secgloss/d-gly.md) (DSA).

## <a name="algorithms-supported"></a>Algoritmi supportati

Per le descrizioni di ognuno di questi algoritmi, vedere il glossario.



| Scopo      | Algoritmi supportati                                                                                                              |
|--------------|-----------------------------------------------------------------------------------------------------------------------------------|
| Scambio di chiave | nessuno                                                                                                                              |
| Firma    | [*DSS*](../secgloss/d-gly.md)                                                                       |
| Crittografia   | nessuno                                                                                                                              |
| Hashing      | [](../secgloss/m-gly.md)[*Sha* MD5](../secgloss/s-gly.md)<br/> |



 

## <a name="related-documentation"></a>Related Documentation (Documentazione correlata)

Il DSA è stato proposto dal [*National Institute of Standards and Technology*](../secgloss/n-gly.md) (NIST). Una descrizione dell'algoritmo è reperibile nel riferimento governativo seguente: "proposed Federal Information Processing Standard for Digital Signature Standard (DSS)," Federal Register, v. 57, no. 21, 31 gen 1992, pp. 3747-3749.
