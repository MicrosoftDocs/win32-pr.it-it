<span data-ttu-id="eacfa-111">Quando si implementa il server di stato, è necessario prendere in considerazione quanto segue:</span><span class="sxs-lookup"><span data-stu-id="eacfa-111">The following should be taken into consideration in implementing the state server:</span></span>

-   <span data-ttu-id="eacfa-112">Il server di stato deve prendere la decisione online entro pochi secondi.</span><span class="sxs-lookup"><span data-stu-id="eacfa-112">The state server must make the decision online in a few seconds.</span></span> <span data-ttu-id="eacfa-113">Per questo motivo, il server di stato richiede un'infrastruttura scalabile in grado di supportare molti aggiornamenti e query al secondo.</span><span class="sxs-lookup"><span data-stu-id="eacfa-113">For this reason the state server requires a scaleable infrastructure that can support many updates and queries per second.</span></span> <span data-ttu-id="eacfa-114">I database relazionali non sono appropriati per le query di grandi dimensioni con aggiornamenti simultanei.</span><span class="sxs-lookup"><span data-stu-id="eacfa-114">Relational databases are not appropriate for such large queries with simultaneous updates.</span></span> <span data-ttu-id="eacfa-115">I database relazionali vengono creati principalmente per garantire la coerenza dei dati e fornire una visualizzazione coerente dei dati a tutti i consumer.</span><span class="sxs-lookup"><span data-stu-id="eacfa-115">Relational databases are primarily built to keep data consistent and to provide a consistent view of the data to all consumers.</span></span> <span data-ttu-id="eacfa-116">Non sono compilati per aggiornamenti rapidi.</span><span class="sxs-lookup"><span data-stu-id="eacfa-116">They are not built for quick updates.</span></span>
-   <span data-ttu-id="eacfa-117">La coerenza transazionale sugli aggiornamenti tra più oggetti non è importante.</span><span class="sxs-lookup"><span data-stu-id="eacfa-117">Transactional consistency on updates between multiple objects is not important.</span></span> <span data-ttu-id="eacfa-118">Questo è dovuto al fatto che il server di stato può tollerare una piccola finestra di opportunità.</span><span class="sxs-lookup"><span data-stu-id="eacfa-118">This is because the state server can tolerate a small window of opportunity.</span></span> <span data-ttu-id="eacfa-119">Tuttavia, la coerenza transazionale di un singolo aggiornamento è importante per ridurre le possibilità di lasciare il server di stato in uno stato incoerente se uno dei server RADIUS viene arrestato durante l'aggiornamento.</span><span class="sxs-lookup"><span data-stu-id="eacfa-119">However, transactional consistency of a single update is important to reduce the chances of leaving the state server in an inconsistent state if one of the RADIUS servers is shut down in the middle of the update.</span></span>
-   <span data-ttu-id="eacfa-120">La persistenza (il salvataggio dello stato della rete in un archivio permanente) non è importante perché le informazioni persistenti non risulteranno rapidamente sincronizzate con lo stato effettivo della rete.</span><span class="sxs-lookup"><span data-stu-id="eacfa-120">Persistence (saving the state of the network to persistent storage) is not important because the persistent information will quickly fall out of sync with the actual state of the network.</span></span>

<span data-ttu-id="eacfa-122">Una possibile progettazione consiste nell'implementare sia una DLL dell'estensione di autenticazione che una DLL di estensione di autorizzazione.</span><span class="sxs-lookup"><span data-stu-id="eacfa-122">One possible design is to implement both an Authentication Extension DLL and an Authorization Extension DLL.</span></span> <span data-ttu-id="eacfa-123">Ognuna di queste dll può comunicare in rete con un database.</span><span class="sxs-lookup"><span data-stu-id="eacfa-123">Each of these DLLs can communicate over the network with a database.</span></span> <span data-ttu-id="eacfa-124">La DLL dell'estensione di autorizzazione può aggiornare il database con informazioni sugli utenti attualmente connessi alla rete.</span><span class="sxs-lookup"><span data-stu-id="eacfa-124">The Authorization Extension DLL can update the database with information about who is currently logged onto the network.</span></span> <span data-ttu-id="eacfa-125">La DLL dell'estensione di autenticazione può eseguire una query sul database per ottenere queste informazioni per decidere se accettare o rifiutare la richiesta di autenticazione di un utente specifico. Se l'utente è già connesso, la richiesta viene rifiutata.</span><span class="sxs-lookup"><span data-stu-id="eacfa-125">The Authentication Extension DLL can query the database for this information to decide whether to accept or reject a particular user's authentication request; if the user is already logged on, the request is rejected.</span></span>

<span data-ttu-id="eacfa-126">Il vantaggio della DLL dell'estensione di autorizzazione che aggiorna il database del server di stato è che la DLL dell'estensione di autorizzazione ha accesso ad altre informazioni sull'utente autenticato.</span><span class="sxs-lookup"><span data-stu-id="eacfa-126">The advantage of having the Authorization Extension DLL update the state-server database is that the Authorization Extension DLL has access to more information about the authenticated user.</span></span> <span data-ttu-id="eacfa-127">La DLL dell'estensione di autorizzazione ha accesso a tutti gli attributi di autorizzazione dal meccanismo di autorizzazione NPS.</span><span class="sxs-lookup"><span data-stu-id="eacfa-127">The Authorization Extension DLL has access to all of the authorization attributes from the NPS Authorization mechanism.</span></span> <span data-ttu-id="eacfa-128">Ad esempio, alcuni utenti possono disporre di autorizzazioni che consentono loro di avere più sessioni.</span><span class="sxs-lookup"><span data-stu-id="eacfa-128">For example, some users may have authorizations that allow them to have multiple sessions.</span></span> <span data-ttu-id="eacfa-129">Il server di stato deve considerare tali utenti come un caso speciale.</span><span class="sxs-lookup"><span data-stu-id="eacfa-129">The state server should treat such users as a special case.</span></span>
