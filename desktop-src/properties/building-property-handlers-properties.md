## <a name="metadata"></a><span data-ttu-id="60bd3-119">Metadati</span><span class="sxs-lookup"><span data-stu-id="60bd3-119">Metadata</span></span>

<span data-ttu-id="60bd3-120">In Windows Vista e versioni successive, un nuovo sistema di proprietà basato sui metadati è fondamentale per organizzare elementi quali file, messaggi di posta elettronica e contatti.</span><span class="sxs-lookup"><span data-stu-id="60bd3-120">In Windows Vista and later, a new property system based on metadata is central to organizing items such as files, e-mail, and contacts.</span></span> <span data-ttu-id="60bd3-121">In questo nuovo sistema di proprietà è possibile cercare gli elementi in base ai relativi metadati e gli utenti possono leggere o scrivere tali metadati.</span><span class="sxs-lookup"><span data-stu-id="60bd3-121">In this new property system, items can be searched based on their metadata, and users can read or write that metadata.</span></span> <span data-ttu-id="60bd3-122">I metadati in questo sistema sono rappresentati da un set estendibile di *proprietà* implementate come coppie nome/valore.</span><span class="sxs-lookup"><span data-stu-id="60bd3-122">Metadata in this system is represented by an extensible set of *properties* implemented as name/value pairs.</span></span>

## <a name="legacy-technology"></a><span data-ttu-id="60bd3-148">Tecnologia legacy</span><span class="sxs-lookup"><span data-stu-id="60bd3-148">Legacy Technology</span></span>

<span data-ttu-id="60bd3-149">È stata sviluppata la tecnologia di flusso secondario Microsoft Windows NT File System (NTFS) per supportare la persistenza di informazioni aggiuntive con il file tramite un flusso alternativo impostato a file system livello.</span><span class="sxs-lookup"><span data-stu-id="60bd3-149">Microsoft Windows NT File System (NTFS) secondary stream technology was developed to support the persistence of extra information with the file through an alternate stream set at the file system layer.</span></span> <span data-ttu-id="60bd3-150">Ci si potrebbe chiedere perché questi flussi secondari non vengono usati come metodo principale per archiviare le proprietà, in particolare con il supporto dei metadati aperti.</span><span class="sxs-lookup"><span data-stu-id="60bd3-150">One might wonder why those secondary streams are not used as the main method to store properties, especially with open metadata support.</span></span> <span data-ttu-id="60bd3-151">Il motivo principale è la trasportabilità di queste informazioni aggiuntive.</span><span class="sxs-lookup"><span data-stu-id="60bd3-151">The primary reason is the transportability of this extra information.</span></span> <span data-ttu-id="60bd3-152">Sfortunatamente, questi flussi alternativi vengono rimossi in molti scenari, tra cui il supporto della memorizzazione nella cache sul lato client (CSC), l'invio di file come allegati e la copia di file in un archivio non NTFS.</span><span class="sxs-lookup"><span data-stu-id="60bd3-152">Unfortunately, these alternate streams are removed in many scenarios including client-side caching support (CSC), sending files as attachments, and copying files to a non-NTFS store.</span></span>

<span data-ttu-id="60bd3-153">I flussi secondari non offrono una soluzione affidabile in cui le proprietà possono essere trasmesse con il file e, di conseguenza, il sistema di proprietà di Windows Vista non fornisce un meccanismo predefinito che sfrutta i flussi NTFS secondari per l'archiviazione delle proprietà.</span><span class="sxs-lookup"><span data-stu-id="60bd3-153">Secondary streams do not provide a robust solution in which properties are guaranteed to travel with the file, and hence, the Windows Vista property system does not provide a built-in mechanism that exploits the secondary NTFS streams for property storage.</span></span> <span data-ttu-id="60bd3-154">È anche sconsigliato creare gestori di proprietà che usano flussi secondari per l'archiviazione delle proprietà.</span><span class="sxs-lookup"><span data-stu-id="60bd3-154">ISVs are also highly discouraged from building property handlers that use secondary streams for the storage of properties.</span></span> <span data-ttu-id="60bd3-155">Esistono naturalmente scenari in cui i flussi secondari NTFS sono appropriati, in particolare quando le applicazioni possono garantire che il file che gestiscono sia sempre archiviato in un volume NTFS e non si spostano in seguito all'interazione dell'utente finale.</span><span class="sxs-lookup"><span data-stu-id="60bd3-155">Of course there are scenarios where NTFS secondary streams are appropriate, particularly when applications can guarantee that the file they are dealing with is always stored in an NTFS volume and will not travel as a result of end user interaction.</span></span>
