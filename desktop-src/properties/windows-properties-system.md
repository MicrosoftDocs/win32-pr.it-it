# <a name="windows-property-system"></a>Sistema di proprietà di Windows

## <a name="purpose"></a>Scopo

Il sistema di proprietà di Windows è un sistema di lettura/scrittura estendibile di definizioni dei dati che fornisce un modo uniforme per esprimere i metadati sugli elementi della shell. Il sistema di proprietà di Windows in Windows Vista e versioni successive consente di archiviare e recuperare metadati per gli elementi della shell. Un elemento della shell è costituito da qualsiasi singolo contenuto, ad esempio un file, una cartella, un messaggio di posta elettronica o un contatto. Una proprietà è una singola parte di metadati associata a un elemento della shell.

## <a name="developer-audience"></a>Sviluppatori

Prima di iniziare a leggere la documentazione relativa all'SDK del sistema di proprietà di Windows, è necessario avere una conoscenza di base dei seguenti elementi:

-   Component Object Model (COM)
-   Programmazione dello spazio dei nomi della shell

Per un'introduzione a COM, vedere [nozioni fondamentali su com](../com/com-fundamentals.md). Per un'introduzione alla programmazione dello spazio dei nomi della shell, vedere [Introduzione con lo spazio dei nomi della shell](../shell/namespace-intro.md).

Per gli usi del sistema di proprietà di Windows, vedere [Panoramica del sistema di proprietà: scenari di sviluppo](property-system-overview.md).

## <a name="run-time-requirements"></a>Requisiti di runtime

L'ambiente di runtime supportato per l'utilizzo del sistema di proprietà di Windows è Windows Vista o versione successiva e Windows Software Development Kit (SDK). Windows XP e Microsoft Windows Desktop Search (WDS) 3,0 o versioni successive includono anche un subset del sistema di proprietà di Windows. Per il download di Windows 7 o Windows Vista SDK aggiornato, vedere la [Windows SDK](https://msdn.microsoft.com/windowsvista/bb980924.aspx).
