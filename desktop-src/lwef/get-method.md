## <a name="remarks"></a>Commenti

Se si carica un carattere usando il protocollo HTTP (un. File ACF), è necessario usare il metodo **Get** per recuperare i dati di animazione prima di poter riprodurre l'animazione. Questo metodo non viene usato se il carattere è stato caricato usando il protocollo UNC (un. File ACS). Non è inoltre possibile recuperare i dati HTTP per un carattere usando **Get** se il carattere è stato caricato usando il protocollo UNC (. File di caratteri ACS).

Se si dichiara un riferimento a un oggetto e lo si imposta su questo metodo, viene restituito un oggetto [**Request**](/windows/desktop/lwef/the-request-object) . Se non è possibile caricare l'animazione associata, il server imposta la proprietà [**status**](status-property.md) dell'oggetto **richiesta** su "failed" con un numero di errore appropriato. È possibile usare l'evento [**RequestComplete**](requestcomplete-event.md) per controllare lo stato e determinare l'azione da eseguire.

I dati di animazione o audio recuperati con il metodo **Get** vengono archiviati nella cache del browser. Le chiamate successive verificheranno la cache e, se i dati di animazione sono già presenti, il controllo caricherà i dati direttamente dalla cache. Una volta caricati, i dati di animazione o audio possono essere riprodotti con i metodi [**Play**](play-method.md) o [**Speak**](speak-method.md) .
