# <a name="the-listening-tip"></a>Il suggerimento di ascolto

\[Microsoft Agent è stato deprecato a partire da Windows 7 e potrebbe non essere disponibile nelle versioni successive di Windows.\]

Il suggerimento di ascolto è un altro servizio di input vocale fornito da Microsoft Agent. Quando viene installato l'input vocale, Agent include una speciale finestra descrizione comando che viene visualizzata quando l'utente preme il tasto di scelta rapida in ascolto o chiama il metodo Listen. Il suggerimento di ascolto viene visualizzato solo quando i servizi di riconoscimento vocale sono disponibili. Se nessun client ha creato un comando Voice o ha caricato correttamente un motore di riconoscimento vocale, il suggerimento di ascolto non viene visualizzato. Inoltre, è necessario abilitare sia l'input vocale che l'opzione Visualizza suggerimenti in ascolto nelle opzioni dei caratteri avanzati affinché il suggerimento appaia.

Il suggerimento di ascolto scade automaticamente dopo la presentazione. Se il timeout del testo "udito" viene completato mentre l'utente continua a tenere premuto il tasto di scelta rapida, il suggerimento torna al testo "in ascolto", a meno che il server non riceva un altro espressione corrispondente. In questo caso, il suggerimento Visualizza il nuovo testo "udito" e inizia il timeout per il testo del suggerimento. Se l'utente rilascia il tasto di scelta rapida e il server Visualizza il testo "udito", il timeout continua e la finestra del suggerimento di ascolto viene nascosta quando scade l'intervallo di timeout.

Se il server non ha ancora tentato di caricare un motore di riconoscimento vocale, il suggerimento di ascolto non viene visualizzato. Analogamente, se l'utente ha disabilitato la visualizzazione del suggerimento di ascolto o l'input vocale disabilitato nelle opzioni dei caratteri avanzati, il suggerimento di ascolto non verrà visualizzato.

Il suggerimento di ascolto non viene visualizzato quando il puntatore è posizionato sull'icona della barra delle applicazioni del carattere. Viene invece visualizzata la finestra del suggerimento di notifica standard e viene visualizzato il nome del carattere.

Le applicazioni client non possono scrivere direttamente nel suggerimento di ascolto, ma è possibile specificare il testo alternativo visualizzato dal server al riconoscimento di un comando Voice corrispondente. A tale scopo, impostare la proprietà [**confidenza**](confidence-property.md) e la nuova proprietà [**ConfidenceText**](confidencetext-property.md)per il comando. Se l'input vocale corrisponde al comando, ma la corrispondenza migliore non supera l'impostazione di attendibilità, il server utilizza il set di testo nella proprietà **ConfidenceText** nella finestra del suggerimento. Se il client non fornisce questo valore, il server Visualizza il testo (grammatica) a cui corrisponde.

Il testo del suggerimento di ascolto viene visualizzato nella lingua in base all'impostazione dell'ID lingua del client di input-attivo, indipendentemente dal fatto che sia disponibile un motore di riconoscimento vocale compatibile con la lingua.
