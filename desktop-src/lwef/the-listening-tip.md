# <a name="the-listening-tip"></a>Suggerimento per l'ascolto

\[Microsoft Agent è deprecato a Windows 7 e potrebbe non essere disponibile nelle versioni successive di Windows.\]

Il suggerimento per l'ascolto è un altro servizio di input vocale fornito da Microsoft Agent. Quando è installato l'input vocale, Agent include una speciale finestra della descrizione comando che viene visualizzata quando l'utente preme il tasto di scelta rapida Listening o chiama il metodo Listen. Il suggerimento per l'ascolto viene visualizzato solo quando i servizi voce sono disponibili. Se nessun client ha creato un comando vocale o carica correttamente un motore di riconoscimento vocale, il suggerimento per l'ascolto non viene visualizzato. Inoltre, sia l'input vocale che l'opzione Display Listening Suggerimenti in Opzioni avanzate per i caratteri devono essere abilitate per la visualizzazione del suggerimento.

Il suggerimento per l'ascolto si verifica automaticamente dopo la presentazione. Se il timeout del testo "Heard" viene completato mentre l'utente tiene premuto il tasto di scelta rapida, il suggerimento ripristina il testo di "ascolto" a meno che il server non riceva un'altra espressione corrispondente. In questo caso, il suggerimento visualizza il nuovo testo "Heard" e inizia il timeout per tale testo della mancia. Se l'utente rilascia il tasto di scelta rapida e il server visualizza il testo "Heard", il timeout continua e la finestra Suggerimento in ascolto viene nascosta al termine dell'intervallo di timeout.

Se il server non ha ancora tentato di caricare un motore di riconoscimento vocale, il suggerimento per l'ascolto non verrà visualizzato. Analogamente, se l'utente ha disabilitato la visualizzazione della descrizione comandi vocali o disabilitato l'input vocale in Opzioni avanzate per i caratteri, il suggerimento per l'ascolto non verrà visualizzato.

Il suggerimento per l'ascolto non viene visualizzato quando il puntatore si trova sull'icona della barra delle applicazioni del carattere. Viene invece visualizzata la finestra del suggerimento di notifica standard con il nome del carattere.

Le applicazioni client non possono scrivere direttamente nel suggerimento per l'ascolto, ma è possibile specificare testo alternativo che il server visualizza al riconoscimento di un comando vocale corrispondente. A tale scopo, impostare la [**proprietà Confidence**](confidence-property.md) e la nuova [**proprietà ConfidenceText**](confidencetext-property.md)per il comando. Se l'input vocale corrisponde al comando, ma la corrispondenza migliore non supera l'impostazione di attendibilità, il server usa il testo impostato nella **proprietà ConfidenceText** nella finestra del suggerimento. Se il client non fornisce questo valore, il server visualizza il testo (grammatica) corrispondente.

Il testo del suggerimento di ascolto viene visualizzato nella lingua in base all'impostazione dell'ID lingua dei caratteri del client attivo per l'input, indipendentemente dal fatto che sia disponibile un motore di riconoscimento vocale compatibile con la lingua.
