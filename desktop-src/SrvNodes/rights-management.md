# <a name="microsoft-rights-management-sdks"></a>SDK di Microsoft Rights Management

Sono ora disponibili tre generazioni di Rights Management SDK: [Rights Management sdk 4,2](/previous-versions/windows/desktop/msipcthin2/active-directory-rights-management-services-multi-platform-thin-client-sdk-portal), [Microsoft Rights Management sdk 2,1](/previous-versions/windows/desktop/msipc/microsoft-information-protection-and-control-client-portal) e [Active Directory Rights Management Services SDK](/previous-versions/windows/desktop/adrms_sdk/active-directory-rights-management-services-sdk-portal), nonché un'API di scripting, [Active Directory Rights Management Services API di scripting](/previous-versions/windows/desktop/adrms_script/adrms-script-portal), che consente l'amministrazione personalizzata di un server RMS.

## <a name="in-this-section"></a>Contenuto della sezione

<dl> <dt>

[Esempi di codice e strumenti](rms-code-samples-and-tools.md)
</dt> <dd>

Raccolta di esempi di codice RMS forniti da Microsoft e strumenti di supporto per gli sviluppatori in tutti i sistemi operativi supportati. Android, iOS/OS X, Windows Phone e desktop di Windows.

</dd> <dd>

RMS SDK 4,2 è un set di strumenti di nuova generazione semplificato che offre un'esperienza di sviluppo semplice per abilitare le app per dispositivi Android, iOS e Mac OS X con la protezione delle informazioni tramite Microsoft Rights Management Services.

</dd> <dd>

RMS SDK 2,1 è una potente offerta SDK per gli sviluppatori di applicazioni desktop di Windows e i provider di soluzioni basate su server per abilitare i prodotti con Rights Management.

</dd> <dt>

[Active Directory Rights Management Services SDK](/previous-versions/windows/desktop/adrms_sdk/active-directory-rights-management-services-sdk-portal)
</dt> <dd>

> [!Note]  
> AD RMS SDK che sfruttano le funzionalità esposte dal client in Msdrm.dll è disponibile per l'uso in Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows 7, Windows Server 2008 e Windows Vista. È possibile che in versioni successive sia stata modificata o non sia più disponibile. Usare invece [Microsoft Rights Management Services SDK 2,1](/previous-versions/windows/desktop/msipc/microsoft-information-protection-and-control-client-portal), che sfrutta le funzionalità esposte dal client in Msipc.dll.

 

</dd> <dt>

[API di script di Active Directory Rights Management Services](/previous-versions/windows/desktop/adrms_script/adrms-script-portal)
</dt> <dd>

L'API di scripting di Windows Active Directory Rights Management Services (AD RMS) può essere usata per creare script per amministrare un'installazione di AD RMS.

</dd> </dl>
