# <a name="system-administration"></a>Amministrazione sistema

> [!Note]  
> Questi argomenti sono destinati agli sviluppatori di software. Per i problemi relativi al supporto tecnico, vedere [supporto-Guida di Windows](https://windows.microsoft.com/windows/support#1TC=windows-10).

 

## <a name="in-this-section"></a>Contenuto della sezione

<dl> <dt>

[Provider WMI dati configurazione di avvio](/previous-versions/windows/desktop/bcd/boot-configuration-data-portal)
</dt> <dd>

Il provider di dati configurazione di avvio (BCD) Strumentazione gestione Windows (WMI) consente l'accesso programmatico agli archivi BCD, che descrivono le applicazioni di avvio e le impostazioni dell'applicazione di avvio.

</dd> <dt>

[Criteri di gruppo](group-policy.md)
</dt> <dd>

Criteri di gruppo consente l'amministrazione basata su criteri utilizzando Microsoft Active Directory Directory Services. Criteri di gruppo USA i servizi di directory e l'appartenenza ai gruppi di sicurezza per fornire flessibilità e supportare informazioni di configurazione estese.

</dd> <dt>

[Microsoft Management Console (MMC)](microsoft-management-console--mmc-.md)
</dt> <dd>

Microsoft Management Console (MMC) fornisce un'interfaccia utente e un modello di amministrazione semplici, coerenti e integrati.

</dd> <dt>

[Provider di applicazioni per la gestione dei dispositivi mobili](/previous-versions/windows/desktop/mdmappprov/mobile-device-management-application-provider-portal)
</dt> <dd>

Il provider di applicazioni MDM (Mobile Device Management) gestisce le applicazioni nei dispositivi sottoscritti a un servizio MDM.

</dd> <dt>

[Registrazione della gestione dei dispositivi mobili](/windows/desktop/MDMReg/mobile-device-management-registration-portal)
</dt> <dd>

La registrazione MDM registra i dispositivi in un servizio MDM.

</dd> <dt>

[Provider di impostazioni di gestione dei dispositivi mobili](/previous-versions/windows/desktop/mdmsettingsprov/mobile-device-management-settings-provider-portal)
</dt> <dd>

Il provider di impostazioni MDM consente la gestione delle impostazioni nei dispositivi sottoscritti a un servizio MDM.

</dd> <dt>

[NetShell](/previous-versions/windows/desktop/netshell/netshell-start-page)
</dt> <dd>

NetShell è uno strumento basato su riga di comando che consente agli amministratori di amministrare e configurare in remoto i servizi di rete critici.

</dd> <dt>

[Infrastruttura di gestione delle impostazioni](/previous-versions/windows/desktop/smi/settings-management-infrastructure--smi-)
</dt> <dd>

L'infrastruttura SMI (Settings Management Infrastructure) fornisce un'infrastruttura standardizzata per accedere e modificare le impostazioni modificabili da utenti e applicazioni.

</dd> <dt>

[Registrazione inventario software](/previous-versions/windows/desktop/sil/software-inventory-logging-portal)
</dt> <dd>

Registrazione inventario software raccoglie i dati sulle licenze per il software installato in un server Windows e fornisce l'accesso remoto ai dati in modo che possano essere aggregati facilmente da un Data Center.

</dd> <dt>

[API di gestione licenze software](/previous-versions/windows/desktop/secslapi/software-licensing-api-portal)
</dt> <dd>

L'API di gestione licenze software (SLAPi) può essere usata per determinare un'installazione di Microsoft Windows originale, installare e registrare una licenza di gestione delle risorse e recuperare informazioni sui criteri di licenza di un componente software.

</dd> <dt>

[Ripristino configurazione di sistema](/windows/desktop/sr/system-restore-portal)
</dt> <dd>

Ripristino configurazione di sistema monitora e registra automaticamente le modifiche di sistema principali nel computer di un utente. È progettato per ridurre i costi di supporto e aumentare la soddisfazione dei clienti consentendo a un utente di annullare una modifica che potrebbe avere causato un problema con il sistema o ripristinare un giorno in cui il sistema è stato in grado di garantire prestazioni ottimali.

</dd> <dt>

[Arresto del sistema](/windows/desktop/Shutdown/system-shutdown)
</dt> <dd>

Le funzioni e i messaggi di arresto del sistema consentono alle applicazioni di disconnettere l'utente corrente, arrestare il sistema o bloccare la workstation.

</dd> <dt>

[Utilità di pianificazione](/windows/desktop/TaskSchd/task-scheduler-start-page)
</dt> <dd>

Il Utilità di pianificazione consente di eseguire automaticamente le attività di routine in un computer scelto. Il Utilità di pianificazione esegue questa operazione monitorando tutti i criteri scelti per avviare le attività (denominate trigger) e quindi eseguire le attività quando i criteri vengono soddisfatti.

</dd> <dt>

[Registrazione accesso utenti](/previous-versions/windows/desktop/ual/user-access-logging)
</dt> <dd>

Registrazione accesso utenti (registrazione accesso utenti) è un Framework comune per i ruoli di Windows Server per segnalare le rispettive metriche di consumo. Questo framework registrazione accesso utenti è un componente fondamentale e essenziale della soluzione di gestione delle licenze più ampia.

</dd> <dt>

[Servizi di distribuzione Windows](/windows/desktop/Wds/windows-deployment-services-portal)
</dt> <dd>

Servizi di distribuzione Windows (WDS) è la versione aggiornata di servizi di installazione remota (RIS). WDS consente la distribuzione dei sistemi operativi Windows. È possibile utilizzare WDS per configurare nuovi client con un'installazione basata su rete senza richiedere agli amministratori di visitare ogni computer o di installare direttamente da supporti CD o DVD.

</dd> <dt>

[API Windows Genuine Advantage](/previous-versions/windows/desktop/wingen/windows-genuine-advantage-api-portal)
</dt> <dd>

L'API Windows Genuine Advantage viene utilizzata per determinare se il sistema operativo Windows in esecuzione nel sistema corrente è una copia originale.

</dd> <dt>

[Strumentazione gestione Windows (WMI)](/windows/desktop/WmiSdk/wmi-start-page)
</dt> <dd>

Strumentazione gestione Windows (WMI) è l'infrastruttura per i dati e le operazioni di gestione nei sistemi operativi basati su Windows. È possibile scrivere script WMI o applicazioni per automatizzare le attività amministrative sui computer remoti.

</dd> <dt>

[API WMIv2 (Windows Defender)](/previous-versions/windows/desktop/defender/windows-defender-wmiv2-apis-portal)
</dt> <dd>

È possibile utilizzare le API WMI di Windows Defender per gestire Malware Protection tramite script o applicazioni.

</dd> <dt>

[Windows PowerShell](https://msdn.microsoft.com/library/Dd835506(v=VS.85).aspx)
</dt> <dd>

Windows PowerShell è una shell da riga di comando basata su attività e un linguaggio di scripting progettato appositamente per l'amministrazione del sistema. Basato su .NET Framework, Windows PowerShell consente ai professionisti IT e agli utenti esperti di controllare e automatizzare l'amministrazione del sistema operativo Windows e delle applicazioni eseguite al suo interno.

</dd> <dt>

[Gestione remota Windows](/windows/desktop/WinRM/portal)
</dt> <dd>

Il Gestione remota Windows (WinRM) è l'implementazione Microsoft del protocollo WS-Management, un protocollo standard basato su Simple Object Access Protocol (SOAP), basato sul firewall, che consente l'interoperabilità di hardware e sistemi operativi di fornitori diversi. La specifica del protocollo WS-Management consente ai sistemi di accedere alle informazioni di gestione e scambiarle attraverso un'infrastruttura IT in modo comune.

</dd> <dt>

[Protezione risorse di Windows](/windows/desktop/Wfp/windows-resource-protection-portal)
</dt> <dd>

Protezione risorse di Windows (WRP) impedisce la sostituzione dei file di sistema essenziali, delle cartelle e delle chiavi del registro di sistema installate come parte del sistema operativo.

</dd> <dt>

[Windows Server Update Services](windows-server-update-services.md)
</dt> <dd>

Gli amministratori di sistema possono utilizzare l'API Windows Server Update Services (WSUS) per determinare quali aggiornamenti si applicano a un computer o a un gruppo di computer, scaricare tali aggiornamenti e installarli senza interventi da parte dell'utente.

Fornitori di software indipendenti e sviluppatori di utenti finali possono integrare le funzionalità WSUS nel software di gestione dei computer o degli aggiornamenti per offrire un ambiente operativo semplice.

</dd> <dt>

[Strumento valutazione sistema Windows](/windows/desktop/WinSAT/windows-system-assessment-tool-portal)
</dt> <dd>

Come utilizzare WinSAT per valutare le caratteristiche e le funzionalità delle prestazioni di un computer.

</dd> <dt>

[Cartelle di lavoro](/previous-versions/windows/desktop/syncshareservermgmt/sync-share-server-management-portal)
</dt> <dd>

Cartelle di lavoro consente agli utenti di archiviare e accedere ai file di lavoro su computer e dispositivi personali.

</dd> </dl>
