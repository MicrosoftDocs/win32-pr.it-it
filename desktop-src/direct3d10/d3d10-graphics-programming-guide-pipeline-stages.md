-   <span data-ttu-id="dd0df-113">[Fase vertex-shader](https://www.bing.com/search?q=Vertex-Shader+Stage). -   <span data-ttu-id="dd0df-115">[Fase geometry-shader](https://www.bing.com/search?q=Geometry-Shader+Stage) : la fase geometry-shader elabora intere primitive.</span><span class="sxs-lookup"><span data-stu-id="dd0df-115">[Geometry-Shader Stage](https://www.bing.com/search?q=Geometry-Shader+Stage) - The geometry-shader stage processes entire primitives.</span></span> <span data-ttu-id="dd0df-116">L'input è una primitiva completa (tre vertici per un triangolo, due vertici per una linea o un singolo vertice per un punto).</span><span class="sxs-lookup"><span data-stu-id="dd0df-116">Its input is a full primitive (which is three vertices for a triangle, two vertices for a line, or a single vertex for a point).</span></span> <span data-ttu-id="dd0df-117">Inoltre, ogni primitiva può includere anche i dati dei vertici per tutte le primitive adiacenti Edge.</span><span class="sxs-lookup"><span data-stu-id="dd0df-117">In addition, each primitive can also include the vertex data for any edge-adjacent primitives.</span></span> <span data-ttu-id="dd0df-118">Questo può includere al massimo tre vertici aggiuntivi per un triangolo o due vertici aggiuntivi per una riga.</span><span class="sxs-lookup"><span data-stu-id="dd0df-118">This could include at most an additional three vertices for a triangle or an additional two vertices for a line.</span></span> <span data-ttu-id="dd0df-119">Geometry shader supporta anche l'amplificazione e la deamplificazione di geometria limitate.</span><span class="sxs-lookup"><span data-stu-id="dd0df-119">The Geometry Shader also supports limited geometry amplification and de-amplification.</span></span> <span data-ttu-id="dd0df-120">Data una primitiva di input, geometry shader può rimuovere la primitiva oppure creare una o più nuove primitive.</span><span class="sxs-lookup"><span data-stu-id="dd0df-120">Given an input primitive, the Geometry Shader can discard the primitive, or emit one or more new primitives.</span></span>

-   <span data-ttu-id="dd0df-124">[Fase di rasterizzazione](../direct3d11/d3d10-graphics-programming-guide-rasterizer-stage.md) : il rasterizzatore è responsabile per le primitive di ritaglio, per preparare le primitive per l'pixel shader e per determinare come richiamare i pixel shader.</span><span class="sxs-lookup"><span data-stu-id="dd0df-124">[Rasterizer Stage](../direct3d11/d3d10-graphics-programming-guide-rasterizer-stage.md) - The rasterizer is responsible for clipping primitives, preparing primitives for the pixel shader and determining how to invoke pixel shaders.</span></span>
-   <span data-ttu-id="dd0df-125">[Fase pixel-shader](https://www.bing.com/search?q=Pixel-Shader+Stage). -   <span data-ttu-id="dd0df-126">[Fase di Unione dell'output](../direct3d11/d3d10-graphics-programming-guide-output-merger-stage.md) : la fase di Unione dell'output è responsabile della combinazione di diversi tipi di dati di output (pixel shader valori, informazioni di profondità e stencil) con il contenuto della destinazione di rendering e i buffer di profondità/stencil per generare il risultato finale della pipeline.</span><span class="sxs-lookup"><span data-stu-id="dd0df-126">[Output-Merger Stage](../direct3d11/d3d10-graphics-programming-guide-output-merger-stage.md) - The output-merger stage is responsible for combining various types of output data (pixel shader values, depth and stencil information) with the contents of the render target and depth/stencil buffers to generate the final pipeline result.</span></span>
