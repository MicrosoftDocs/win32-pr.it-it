# <a name="resources-direct3d-10"></a><span data-ttu-id="2fe1d-103">Risorse (Direct3D 10)</span><span class="sxs-lookup"><span data-stu-id="2fe1d-103">Resources (Direct3D 10)</span></span>

<span data-ttu-id="2fe1d-104">Una risorsa è un'area della memoria accessibile tramite la pipeline Direct3D.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-104">A resource is an area in memory that can be accessed by the Direct3D pipeline.</span></span> <span data-ttu-id="2fe1d-105">Per consentire alla pipeline di accedere in modo efficiente alla memoria, i dati forniti alla pipeline, ad esempio la geometria di input, le risorse dello shader, le trame e così via, devono essere archiviati in una risorsa.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-105">In order for the pipeline to access memory efficiently, data that is provided to the pipeline (such as input geometry, shader resources, textures etc) must be stored in a resource.</span></span> <span data-ttu-id="2fe1d-106">Esistono due tipi di risorse da cui derivano tutte le risorse Direct3D: buffer e trama.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-106">There are two types of resources from which all Direct3D resources derive: a buffer or a texture.</span></span> <span data-ttu-id="2fe1d-107">Per ogni fase della pipeline possono essere attive fino a 128 risorse.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-107">Up to 128 resources can be active for each pipeline stage.</span></span>

<span data-ttu-id="2fe1d-108">In ogni applicazione vengono in genere create molte risorse.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-108">Each application will typically create many resources.</span></span> <span data-ttu-id="2fe1d-109">Esempi di risorse includono: buffer di vertici, buffer di indice, buffer costante, trame e risorse dello shader.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-109">Examples of resource include: vertex buffers, index buffer, constant buffer, textures, and shader resources.</span></span> <span data-ttu-id="2fe1d-110">Sono disponibili diverse opzioni che determinano il modo in cui è possibile usare le risorse.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-110">There are several options that determine how resources can be used.</span></span> <span data-ttu-id="2fe1d-111">È possibile creare risorse fortemente tipizzate o di tipo less; è possibile controllare se le risorse hanno accesso sia in lettura che in scrittura; è possibile rendere le risorse accessibili solo alla CPU, alla GPU o a entrambe.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-111">You can create resources that are strongly typed or type less; you can control whether resources have both read and write access; you can make resources accessible to only the CPU, GPU, or both.</span></span> <span data-ttu-id="2fe1d-112">Naturalmente, ci sarà un compromesso tra velocità e funzionalità. maggiore è il numero di funzionalità consentite per una risorsa, minore sarà la prestazione prevista.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-112">Naturally, there will be speed vs. functionality tradeoff - the more functionality you allow a resource to have, the less performance you should expect.</span></span>

<span data-ttu-id="2fe1d-113">Poiché un'applicazione spesso usa molte trame, Direct3D introduce anche il concetto di una matrice di trame per semplificare la gestione delle trame.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-113">Since an application often uses many textures, Direct3D also introduces the concept of a texture array to simplify texture management.</span></span> <span data-ttu-id="2fe1d-114">Una matrice di trame contiene una o più trame (tutte dello stesso tipo e dimensioni) che possono essere indicizzate dall'interno di un'applicazione o da shader.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-114">A texture array contains one or more textures (all of the same type and dimensions) that can be indexed from within an application or by shaders.</span></span> <span data-ttu-id="2fe1d-115">Le matrici di trame consentono di usare una singola interfaccia con più indici per accedere a molte trame.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-115">Texture arrays allow you to use a single interface with multiple indexes to access many textures.</span></span> <span data-ttu-id="2fe1d-116">È possibile creare un numero così elevato di matrici di trame per gestire i diversi tipi di trama necessari.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-116">You can create as many texture arrays to manage different texture types as you need.</span></span>

<span data-ttu-id="2fe1d-117">Dopo aver creato le risorse che l'applicazione userà, è possibile connettere o associare ogni risorsa alle fasi della pipeline che li useranno.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-117">Once you have created the resources that your application will use, you connect or bind each resource to the pipeline stages that will use them.</span></span> <span data-ttu-id="2fe1d-118">Questa operazione viene eseguita chiamando un'API bind, che accetta un puntatore alla risorsa.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-118">This is accomplished by calling a bind API, which takes a pointer to the resource.</span></span> <span data-ttu-id="2fe1d-119">Poiché è possibile che più fasi della pipeline debbano accedere alla stessa risorsa, Direct3D 10 introduce il concetto di visualizzazione delle risorse.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-119">Since more than one pipeline stage may need access to the same resource, Direct3D 10 introduces the concept of a resource view.</span></span> <span data-ttu-id="2fe1d-120">Una visualizzazione identifica la parte di una risorsa a cui è possibile accedere.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-120">A view identifies the portion of a resource that can be accessed.</span></span> <span data-ttu-id="2fe1d-121">È possibile creare visualizzazioni m o una risorsa e associarle a n fasi della pipeline, presupponendo che si seguano le regole di binding per la risorsa condivisa (il runtime genererà errori in fase di compilazione se non lo è).</span><span class="sxs-lookup"><span data-stu-id="2fe1d-121">You can create m views or a resource and bind them to n pipeline stages, assuming you follow binding rules for shared resource (the runtime will generate errors at compile time if you don't).</span></span>

<span data-ttu-id="2fe1d-122">Una visualizzazione risorse fornisce un modello generale per l'accesso a una risorsa (trame, buffer e così via).</span><span class="sxs-lookup"><span data-stu-id="2fe1d-122">A resource view provides a general model for access to a resource (textures, buffers, etc.).</span></span> <span data-ttu-id="2fe1d-123">Poiché è possibile usare una visualizzazione per indicare al runtime i dati a cui accedere e come accedervi, le visualizzazioni delle risorse consentono di creare meno risorse del tipo.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-123">Because you can use a view to tell the runtime what data to access and how to access it, resource views allow you create type less resources.</span></span> <span data-ttu-id="2fe1d-124">Ovvero, è possibile creare risorse per una determinata dimensione in fase di compilazione e quindi dichiarare il tipo di dati all'interno della risorsa quando la risorsa viene associata alla pipeline.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-124">That is, you can create resources for a given size at compile time, and then declare the data type within the resource when the resource gets bound to the pipeline.</span></span> <span data-ttu-id="2fe1d-125">Le visualizzazioni espongono numerose nuove funzionalità per l'utilizzo delle risorse, ad esempio la possibilità di leggere le superfici dello shader nello shader, di generare CubeMaps dinamici in un singolo passaggio e di eseguire il rendering simultaneo in più sezioni di un volume.</span><span class="sxs-lookup"><span data-stu-id="2fe1d-125">Views expose many new capabilities for using resources, such as the ability to read back depth/stencil surfaces in the shader, generating dynamic cubemaps in a single pass, and rendering simultaneously to multiple slices of a volume.</span></span>
