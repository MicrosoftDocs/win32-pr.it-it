# <a name="implementing-in-place-activation"></a><span data-ttu-id="187fd-103">Implementazione di In-Place attivazione</span><span class="sxs-lookup"><span data-stu-id="187fd-103">Implementing In-Place Activation</span></span>

<span data-ttu-id="187fd-104">L'attivazione sul posto consente a un utente di interagire con un oggetto incorporato senza uscire dal documento contenitore.</span><span class="sxs-lookup"><span data-stu-id="187fd-104">In-place activation enables a user to interact with an embedded object without leaving the container document.</span></span> <span data-ttu-id="187fd-105">Quando un utente attiva l'oggetto, una *barra dei menu composita* che include elementi dalle barre dei menu delle applicazioni del contenitore e dell'applicazione Server sostituisce la barra dei menu principale del contenitore.</span><span class="sxs-lookup"><span data-stu-id="187fd-105">When a user activates the object, a *composite menu bar* comprising elements from both the container application's and server application's menu bars replaces the container's main menu bar.</span></span> <span data-ttu-id="187fd-106">I comandi e le funzionalità di entrambe le applicazioni sono quindi disponibili per l'utente, inclusa la Guida sensibile al contesto per l'oggetto attivo.</span><span class="sxs-lookup"><span data-stu-id="187fd-106">Commands and features from both applications are thus available to the user, including context sensitive help for the active object.</span></span> <span data-ttu-id="187fd-107">Quando un utente inizia a lavorare con una parte non oggetto del documento, l'oggetto viene disattivato, causando il menu originale del documento contenitore per sostituire il menu composito.</span><span class="sxs-lookup"><span data-stu-id="187fd-107">When a user begins working with some non-object portion of the document, the object is deactivated, causing the container document's original menu to replace the composite menu.</span></span>

<span data-ttu-id="187fd-108">Questa funzionalità è stata originariamente denominata *modifica sul posto*.</span><span class="sxs-lookup"><span data-stu-id="187fd-108">This capability originally went by the name of *in-place editing*.</span></span> <span data-ttu-id="187fd-109">Il nome è stato modificato perché un utente può interagire con un oggetto in esecuzione solo un modo.</span><span class="sxs-lookup"><span data-stu-id="187fd-109">The name was changed because editing is only one way for a user to interact with a running object.</span></span> <span data-ttu-id="187fd-110">I clip audio, ad esempio, possono essere ascoltati anziché modificarli.</span><span class="sxs-lookup"><span data-stu-id="187fd-110">Sound clips, for example, can be listened to instead of editing.</span></span> <span data-ttu-id="187fd-111">È possibile visualizzare i clip video anziché modificarli.</span><span class="sxs-lookup"><span data-stu-id="187fd-111">Video clips can be viewed instead of editing.</span></span> <span data-ttu-id="187fd-112">L'attivazione sul posto è particolarmente adatta nel caso di clip video perché consente l'esecuzione sul posto, senza chiamare una finestra separata.</span><span class="sxs-lookup"><span data-stu-id="187fd-112">In-place activation is particularly apt in the case of video clips because it allows them to run in place, without calling up a separate window.</span></span> <span data-ttu-id="187fd-113">Questo potrebbe essere fondamentale se il video venisse visualizzato, ad esempio, in combinazione con i dati di testo adiacenti nel documento contenitore.</span><span class="sxs-lookup"><span data-stu-id="187fd-113">This could be critical if the video were to be viewed, say, in conjunction with adjacent text data in the container document.</span></span>

<span data-ttu-id="187fd-114">L'implementazione dell'attivazione sul posto è strettamente facoltativa per le applicazioni contenitore e server.</span><span class="sxs-lookup"><span data-stu-id="187fd-114">Implementing in-place activation is strictly optional for both container and server applications.</span></span> <span data-ttu-id="187fd-115">OLE supporta ancora il modello in cui l'attivazione di un oggetto comporta l'apertura di una finestra separata da parte dell'applicazione server.</span><span class="sxs-lookup"><span data-stu-id="187fd-115">OLE still supports the model in which activating an object causes the server application to open a separate window.</span></span> <span data-ttu-id="187fd-116">Gli oggetti collegati vengono sempre aperti in una finestra separata per evidenziare che si trovano in un documento separato.</span><span class="sxs-lookup"><span data-stu-id="187fd-116">Linked objects always open in a separate window to emphasize that they reside in a separate document.</span></span>

<span data-ttu-id="187fd-117">L'attivazione sul posto inizia con l'oggetto in risposta a una chiamata di [**Overb IOleObject::D**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-doverb) dal relativo contenitore.</span><span class="sxs-lookup"><span data-stu-id="187fd-117">In-place activation begins with the object in response to an [**IOleObject::DoVerb**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-doverb) call from its container.</span></span> <span data-ttu-id="187fd-118">Questa chiamata viene in genere eseguita in risposta a un utente che fa doppio clic sull'oggetto o seleziona un comando (verbo) dal menu modifica dell'applicazione contenitore.</span><span class="sxs-lookup"><span data-stu-id="187fd-118">This call usually happens in response to a user double-clicking the object or selecting a command (verb) from the container application's Edit menu.</span></span>

<span data-ttu-id="187fd-119">La finestra sul posto riceve l'input della tastiera e del mouse mentre l'oggetto incorporato è attivo.</span><span class="sxs-lookup"><span data-stu-id="187fd-119">The in-place window receives keyboard and mouse input while the embedded object is active.</span></span> <span data-ttu-id="187fd-120">Quando un utente seleziona i comandi dalla barra dei menu composita, il comando e i messaggi di menu associati vengono inviati al contenitore o all'applicazione oggetto, a seconda del proprietario del menu a discesa particolare selezionato.</span><span class="sxs-lookup"><span data-stu-id="187fd-120">When a user selects commands from the composite menu bar, the command and associated menu messages are sent to the container or object application, depending on which owns the particular drop-down menu selected.</span></span> <span data-ttu-id="187fd-121">L'input per mezzo di righelli, barre degli strumenti o aree di controllo di un oggetto passa direttamente all'oggetto incorporato che possiede le finestre.</span><span class="sxs-lookup"><span data-stu-id="187fd-121">Input by means of an object's rulers, toolbars, or frame adornments go directly to the embedded object, which owns these windows.</span></span>

<span data-ttu-id="187fd-122">Un oggetto incorporato attivato sul posto rimane attivo fino a quando il contenitore non lo disattiva in risposta all'input dell'utente oppure l'oggetto volontariamente lo stato attivo, come può fare, ad esempio, un clip video.</span><span class="sxs-lookup"><span data-stu-id="187fd-122">An in-place-activated embedded object remains active until either the container deactivates it in response to user input or the object voluntarily gives up the active state, as a video clip might do, for example.</span></span> <span data-ttu-id="187fd-123">Un utente può disattivare un oggetto facendo clic all'interno del documento del contenitore ma al di fuori della finestra di attivazione sul posto dell'oggetto o semplicemente facendo clic su un altro oggetto.</span><span class="sxs-lookup"><span data-stu-id="187fd-123">A user can deactivate an object by clicking inside the container document but outside the object's in-place-activation window, or simply by clicking another object.</span></span> <span data-ttu-id="187fd-124">Un oggetto attivato sul posto rimane attivo, tuttavia, se l'utente fa clic sulla barra del titolo del contenitore, sulla barra di scorrimento o, in particolare, sulla barra dei menu.</span><span class="sxs-lookup"><span data-stu-id="187fd-124">An in-place-activated object remains active, however, if the user clicks the container's title bar, scroll bar or, in particular, menu bar.</span></span>

<span data-ttu-id="187fd-125">È possibile implementare un server oggetto di attivazione sul posto come un server in-process (DLL) o un server locale (EXE).</span><span class="sxs-lookup"><span data-stu-id="187fd-125">You can implement an in-place-activation-object server either as an in-process server (DLL) or a local server (EXE).</span></span> <span data-ttu-id="187fd-126">In entrambi i casi, la barra dei menu composita contiene elementi, in genere menu a discesa, sia dal processo del server che da quello del contenitore.</span><span class="sxs-lookup"><span data-stu-id="187fd-126">In both cases, the composite menu bar contains items (typically drop-down menus) from both the server and container processes.</span></span> <span data-ttu-id="187fd-127">Nel caso di un server in-process, la finestra di attivazione sul posto è semplicemente un'altra finestra figlio nella gerarchia della finestra del contenitore, ricevendo l'input tramite il message pump dell'applicazione contenitore.</span><span class="sxs-lookup"><span data-stu-id="187fd-127">In the case of a in-process server, the in-place activation window is simply another child window in the container's window hierarchy, receiving its input through the container application's message pump.</span></span>

<span data-ttu-id="187fd-128">Nel caso di un server locale, la finestra di attivazione sul posto appartiene al processo dell'applicazione server dell'oggetto incorporato, ma la finestra padre appartiene al contenitore.</span><span class="sxs-lookup"><span data-stu-id="187fd-128">In the case of a local server, the in-place activation window belongs to the embedded object's server application process, but its parent window belongs to the container.</span></span> <span data-ttu-id="187fd-129">L'input per la finestra attivazione sul posto viene visualizzato nella coda di messaggi del server e viene inviato dal ciclo di messaggi del server.</span><span class="sxs-lookup"><span data-stu-id="187fd-129">Input for the in-place-activation window appears in the server's message queue and is dispatched by the server's message loop.</span></span> <span data-ttu-id="187fd-130">Le librerie OLE sono responsabili del modo in cui i comandi di menu e i messaggi vengono inviati correttamente.</span><span class="sxs-lookup"><span data-stu-id="187fd-130">The OLE libraries are responsible for seeing to it that menu commands and messages are dispatched correctly.</span></span>
