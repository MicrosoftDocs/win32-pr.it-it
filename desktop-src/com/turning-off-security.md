# <a name="turning-off-security"></a>Disattivazione della sicurezza

Esistono diversi motivi per cui è possibile decidere di disattivare la sicurezza fornita da COM. Quando si sviluppano applicazioni, potrebbe essere utile disattivare inizialmente la protezione per consentire all'utente di concentrarsi sulle altre funzionalità all'interno dell'applicazione. Quindi, quando si è pronti, è possibile aggiungere funzionalità di sicurezza alla progettazione. Per semplificare il debug, potrebbe essere necessario disattivare la sicurezza e quindi aggiungere le funzionalità di sicurezza una alla volta. Infine, alcune applicazioni non necessitano di sicurezza e potrebbero essere eseguite più velocemente se non vengono eseguiti controlli di sicurezza.

Negli argomenti seguenti vengono fornite procedure dettagliate:

-   [Disattivazione della sicurezza di attivazione](turning-off-activation-security.md)
-   [Disattivazione della sicurezza delle chiamate](turning-off-call-security.md)
