# <a name="scripting-com-objects-in-custom-applications"></a>Scripting di oggetti COM in applicazioni personalizzate

Microsoft offre diversi ambienti per lo scripting di oggetti COM: Active Server, Windows Host script e così via. I fornitori di software indipendenti (ISV) possono anche licenzare i motori di scripting Microsoft per l'uso nelle applicazioni. Ad esempio, un ISV che crea un elaboratore di testo potrebbe voler aggiungere un linguaggio macro all'applicazione in modo che gli utenti potrebbero automatizzare attività semplici. Con la licenza di un motore di scripting, l'ISV può usare un linguaggio come VBScript o JScript come linguaggio macro dell'applicazione.

Oltre a licenze per VBScript e JScript di scripting, gli ISV possono scrivere i propri ActiveX di scripting. Questi motori di scripting possono quindi essere collegati a qualsiasi ambiente host che supporti lo standard ActiveX del motore di scripting. Ad esempio, un ISV potrebbe scrivere un motore di scripting PerlScript e installarlo in un server ASP, consentendo così al server ASP di elaborare gli script PerlScript.
