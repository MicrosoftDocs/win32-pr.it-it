# <a name="the-component-object-model"></a><span data-ttu-id="bb17e-103">Component Object Model (COM)</span><span class="sxs-lookup"><span data-stu-id="bb17e-103">The Component Object Model</span></span>

<span data-ttu-id="bb17e-104">Microsoft Component Object Model (COM) è un sistema indipendente dalla piattaforma, distribuito e orientato a oggetti per la creazione di componenti software binari in grado di interagire.</span><span class="sxs-lookup"><span data-stu-id="bb17e-104">The Microsoft Component Object Model (COM) is a platform-independent, distributed, object-oriented system for creating binary software components that can interact.</span></span> <span data-ttu-id="bb17e-105">COM è la tecnologia di base per Microsoft OLE (documenti composti), ActiveX (componenti abilitati per Internet) e altri.</span><span class="sxs-lookup"><span data-stu-id="bb17e-105">COM is the foundation technology for Microsoft's OLE (compound documents), ActiveX (Internet-enabled components), as well as others.</span></span>

<span data-ttu-id="bb17e-106">Per comprendere COM (e pertanto tutte le tecnologie basate su COM), è fondamentale comprendere che non si tratta di un linguaggio orientato a oggetti, ma di uno standard.</span><span class="sxs-lookup"><span data-stu-id="bb17e-106">To understand COM (and therefore all COM-based technologies), it is crucial to understand that it is not an object-oriented language but a standard.</span></span> <span data-ttu-id="bb17e-107">Né COM specificano la struttura di un'applicazione; la lingua, la struttura e i dettagli di implementazione vengono lasciati allo sviluppatore di applicazioni.</span><span class="sxs-lookup"><span data-stu-id="bb17e-107">Nor does COM specify how an application should be structured; language, structure, and implementation details are left to the application developer.</span></span> <span data-ttu-id="bb17e-108">COM specifica invece un modello a oggetti e requisiti di programmazione che consentono agli oggetti COM (detti anche componenti COM o semplicemente a *oggetti*) di interagire con altri oggetti.</span><span class="sxs-lookup"><span data-stu-id="bb17e-108">Rather, COM specifies an object model and programming requirements that enable COM objects (also called COM components, or sometimes simply *objects*) to interact with other objects.</span></span> <span data-ttu-id="bb17e-109">Questi oggetti possono trovarsi all'interno di un singolo processo, in altri processi, e possono anche trovarsi su computer remoti.</span><span class="sxs-lookup"><span data-stu-id="bb17e-109">These objects can be within a single process, in other processes, and can even be on remote computers.</span></span> <span data-ttu-id="bb17e-110">Possono essere scritti in linguaggi diversi e possono essere strutturalmente molto diversi, motivo per cui COM viene definito *standard binario*. standard applicato dopo che un programma è stato convertito in codice macchina binario.</span><span class="sxs-lookup"><span data-stu-id="bb17e-110">They can be written in different languages, and they may be structurally quite dissimilar, which is why COM is referred to as a *binary standard*; a standard that applies after a program has been translated to binary machine code.</span></span>

<span data-ttu-id="bb17e-111">L'unico requisito per il linguaggio COM è che il codice viene generato in un linguaggio in grado di creare strutture di puntatori e, in modo esplicito o implicito, chiamare funzioni tramite puntatori.</span><span class="sxs-lookup"><span data-stu-id="bb17e-111">The only language requirement for COM is that code is generated in a language that can create structures of pointers and, either explicitly or implicitly, call functions through pointers.</span></span> <span data-ttu-id="bb17e-112">I linguaggi orientati a oggetti, ad esempio C++ e Smalltalk, forniscono meccanismi di programmazione che semplificano l'implementazione di oggetti COM, ma è possibile utilizzare linguaggi quali C, Java e VBScript per creare e utilizzare oggetti COM.</span><span class="sxs-lookup"><span data-stu-id="bb17e-112">Object-oriented languages such as C++ and Smalltalk provide programming mechanisms that simplify the implementation of COM objects, but languages such as C, Java, and VBScript can be used to create and use COM objects.</span></span>

<span data-ttu-id="bb17e-113">COM definisce la natura essenziale di un oggetto COM.</span><span class="sxs-lookup"><span data-stu-id="bb17e-113">COM defines the essential nature of a COM object.</span></span> <span data-ttu-id="bb17e-114">In generale, un oggetto software è costituito da un set di dati e dalle funzioni che modificano i dati.</span><span class="sxs-lookup"><span data-stu-id="bb17e-114">In general, a software object is made up of a set of data and the functions that manipulate the data.</span></span> <span data-ttu-id="bb17e-115">Un oggetto COM è un oggetto in cui l'accesso ai dati di un oggetto viene effettuato esclusivamente tramite uno o più set di funzioni correlate.</span><span class="sxs-lookup"><span data-stu-id="bb17e-115">A COM object is one in which access to an object's data is achieved exclusively through one or more sets of related functions.</span></span> <span data-ttu-id="bb17e-116">Questi set di funzioni sono denominati *interfacce* e le funzioni di un'interfaccia sono denominate *Metodi*.</span><span class="sxs-lookup"><span data-stu-id="bb17e-116">These function sets are called *interfaces*, and the functions of an interface are called *methods*.</span></span> <span data-ttu-id="bb17e-117">COM richiede inoltre che l'unico modo per accedere ai metodi di un'interfaccia sia tramite un puntatore all'interfaccia.</span><span class="sxs-lookup"><span data-stu-id="bb17e-117">Further, COM requires that the only way to gain access to the methods of an interface is through a pointer to the interface.</span></span>

<span data-ttu-id="bb17e-118">Oltre a specificare lo standard dell'oggetto binario di base, COM definisce alcune interfacce di base che forniscono funzioni comuni a tutte le tecnologie basate su COM e fornisce un numero ridotto di funzioni richieste da tutti i componenti.</span><span class="sxs-lookup"><span data-stu-id="bb17e-118">Besides specifying the basic binary object standard, COM defines certain basic interfaces that provide functions common to all COM-based technologies, and it provides a small number of functions that all components require.</span></span> <span data-ttu-id="bb17e-119">COM definisce anche il modo in cui gli oggetti interagiscono in un ambiente distribuito e ha aggiunto funzionalità di sicurezza che consentono di garantire l'integrità del sistema e dei componenti.</span><span class="sxs-lookup"><span data-stu-id="bb17e-119">COM also defines how objects work together over a distributed environment and has added security features to help provide system and component integrity.</span></span>
