# <a name="events-in-com-and-connectable-objects"></a><span data-ttu-id="62c0f-103">Eventi in COM e oggetti collegabili</span><span class="sxs-lookup"><span data-stu-id="62c0f-103">Events in COM and Connectable Objects</span></span>

<span data-ttu-id="62c0f-104">Quando un programma rileva un evento che si è verificato, può inviare una notifica ai client.</span><span class="sxs-lookup"><span data-stu-id="62c0f-104">When a program detects something that has happened, it can notify its clients.</span></span> <span data-ttu-id="62c0f-105">Se, ad esempio, un programma di titolo azionario rileva una modifica nel prezzo di una borsa, può notificare a tutti i client la modifica.</span><span class="sxs-lookup"><span data-stu-id="62c0f-105">For example, if a stock ticker program detects a change in the price of a stock, it can notify all clients of the change.</span></span> <span data-ttu-id="62c0f-106">Questo processo di notifica viene definito *generazione di un evento*.</span><span class="sxs-lookup"><span data-stu-id="62c0f-106">This notification process is referred to as *firing an event*.</span></span>

<span data-ttu-id="62c0f-107">Con COM, gli oggetti server possono usare gli eventi COM per generare eventi senza informazioni sugli oggetti a cui verrà inviata una notifica.</span><span class="sxs-lookup"><span data-stu-id="62c0f-107">With COM, server objects can use COM events to fire events without any information about what objects will be notified.</span></span> <span data-ttu-id="62c0f-108">Gli oggetti possono inoltre utilizzare *oggetti collegabili* per mantenere informazioni dettagliate sui client che hanno richiesto le notifiche.</span><span class="sxs-lookup"><span data-stu-id="62c0f-108">Objects can also use *connectable objects* to maintain detailed information about clients who have requested notifications.</span></span>

<span data-ttu-id="62c0f-109">Gli oggetti COM collegabili forniscono interfacce in uscita ai client oltre alle interfacce in ingresso.</span><span class="sxs-lookup"><span data-stu-id="62c0f-109">COM connectable objects provide outgoing interfaces to their clients in addition to their incoming interfaces.</span></span> <span data-ttu-id="62c0f-110">Di conseguenza, gli oggetti e i relativi client possono interagire con la comunicazione bidirezionale.</span><span class="sxs-lookup"><span data-stu-id="62c0f-110">As a result, objects and their clients can engage in bidirectional communication.</span></span> <span data-ttu-id="62c0f-111">Le interfacce in ingresso vengono implementate in un oggetto e ricevono chiamate da client esterni di un oggetto, mentre le interfacce in uscita vengono implementate nel sink del client e ricevono chiamate dall'oggetto.</span><span class="sxs-lookup"><span data-stu-id="62c0f-111">Incoming interfaces are implemented on an object and receive calls from external clients of an object, while outgoing interfaces are implemented on the client's sink and receive calls from the object.</span></span> <span data-ttu-id="62c0f-112">L'oggetto definisce un'interfaccia che si desidera utilizzare e il client la implementa.</span><span class="sxs-lookup"><span data-stu-id="62c0f-112">The object defines an interface it would like to use, and the client implements it.</span></span>

<span data-ttu-id="62c0f-113">Un oggetto definisce le interfacce in ingresso e fornisce le implementazioni di tali interfacce.</span><span class="sxs-lookup"><span data-stu-id="62c0f-113">An object defines its incoming interfaces and provides implementations of these interfaces.</span></span> <span data-ttu-id="62c0f-114">Le interfacce in entrata sono disponibili per i client tramite il metodo [**IUnknown:: QueryInterface**](/windows/desktop/api/Unknwn/nf-unknwn-iunknown-queryinterface(q)) dell'oggetto.</span><span class="sxs-lookup"><span data-stu-id="62c0f-114">Incoming interfaces are available to clients through the object's [**IUnknown::QueryInterface**](/windows/desktop/api/Unknwn/nf-unknwn-iunknown-queryinterface(q)) method.</span></span> <span data-ttu-id="62c0f-115">I client chiamano i metodi di un'interfaccia in ingresso nell'oggetto e l'oggetto esegue le azioni desiderate per conto del client.</span><span class="sxs-lookup"><span data-stu-id="62c0f-115">Clients call the methods of an incoming interface on the object, and the object performs desired actions on behalf of the client.</span></span>

<span data-ttu-id="62c0f-116">Le interfacce in uscita sono definite anche da un oggetto, ma il client fornisce le implementazioni delle interfacce in uscita su un oggetto sink creato dal client.</span><span class="sxs-lookup"><span data-stu-id="62c0f-116">Outgoing interfaces are also defined by an object, but the client provides the implementations of the outgoing interfaces on a sink object that the client creates.</span></span> <span data-ttu-id="62c0f-117">L'oggetto chiama quindi i metodi dell'interfaccia in uscita sull'oggetto sink per notificare al client le modifiche apportate all'oggetto, per attivare eventi nel client, per richiedere qualcosa dal client o, di fatto, per qualsiasi scopo con cui il creatore dell'oggetto viene visualizzato.</span><span class="sxs-lookup"><span data-stu-id="62c0f-117">The object then calls methods of the outgoing interface on the sink object to notify the client of changes in the object, to trigger events in the client, to request something from the client, or, in fact, for any purpose the object creator comes up with.</span></span>

<span data-ttu-id="62c0f-118">Un esempio di interfaccia in uscita è un'interfaccia IButtonSink definita da un controllo pulsante di push per notificare ai client i relativi eventi.</span><span class="sxs-lookup"><span data-stu-id="62c0f-118">An example of an outgoing interface is an IButtonSink interface defined by a push button control to notify its clients of its events.</span></span> <span data-ttu-id="62c0f-119">Ad esempio, l'oggetto Button chiama IButtonSink:: OnClick sull'oggetto sink del client quando l'utente fa clic sul pulsante sullo schermo.</span><span class="sxs-lookup"><span data-stu-id="62c0f-119">For example, the button object calls IButtonSink::OnClick on the client's sink object when the user clicks the button on the screen.</span></span> <span data-ttu-id="62c0f-120">Il controllo Button definisce l'interfaccia in uscita.</span><span class="sxs-lookup"><span data-stu-id="62c0f-120">The button control defines the outgoing interface.</span></span> <span data-ttu-id="62c0f-121">Affinché un client del pulsante gestisca l'evento, il client deve implementare tale interfaccia in uscita su un oggetto sink, quindi connettere il sink al controllo Button.</span><span class="sxs-lookup"><span data-stu-id="62c0f-121">For a client of the button to handle the event, the client must implement that outgoing interface on a sink object and then connect that sink to the button control.</span></span> <span data-ttu-id="62c0f-122">Quindi, quando si verificano eventi nel pulsante, il pulsante chiamerà il sink, a quel punto il client può eseguire qualsiasi azione da assegnare a tale pulsante.</span><span class="sxs-lookup"><span data-stu-id="62c0f-122">Then, when events occur in the button, the button will call the sink, at which time the client can execute whatever action it wishes to assign to that button click.</span></span>

<span data-ttu-id="62c0f-123">Gli oggetti collegabili forniscono un meccanismo generale per la comunicazione da oggetto a client.</span><span class="sxs-lookup"><span data-stu-id="62c0f-123">Connectable objects provide a general mechanism for object-to-client communication.</span></span> <span data-ttu-id="62c0f-124">Tutti gli oggetti che desiderano esporre eventi o notifiche di qualsiasi tipo possono utilizzare questa tecnologia.</span><span class="sxs-lookup"><span data-stu-id="62c0f-124">Any object that wishes to expose events or notifications of any kind can use this technology.</span></span> <span data-ttu-id="62c0f-125">Oltre alla tecnologia di oggetti collegabile generale, COM offre numerose interfacce di sink e siti per scopi specifici usati dagli oggetti per notificare ai client di eventi specifici di interesse per il client.</span><span class="sxs-lookup"><span data-stu-id="62c0f-125">In addition to the general connectable object technology, COM provides many special purpose sink and site interfaces used by objects to notify clients of specific events of interest to the client.</span></span> <span data-ttu-id="62c0f-126">Ad esempio, [**IAdviseSink**](/windows/desktop/api/ObjIdl/nn-objidl-iadvisesink) può essere utilizzato dagli oggetti per notificare ai client i dati e visualizzare le modifiche nell'oggetto.</span><span class="sxs-lookup"><span data-stu-id="62c0f-126">For example, [**IAdviseSink**](/windows/desktop/api/ObjIdl/nn-objidl-iadvisesink) may be used by objects to notify clients of data and view changes in the object.</span></span>
