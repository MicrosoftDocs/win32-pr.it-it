# <a name="application-identity"></a>Identità applicazione

L'identità di un'applicazione è l'account usato per eseguire l'applicazione. L'identità può essere quella dell'utente attualmente connesso (utente interattivo), dell'utente che ha avviato il server, di un utente specificato o di un servizio. Per informazioni su come impostare l'identità dell'applicazione Dcomcnfg.exe, vedere Impostazione della sicurezza a livello di processo [tramite DCOMCNFG.](setting-processwide-security-using-dcomcnfg.md)

Gli argomenti seguenti descrivono ognuno dei quattro tipi di identità dell'applicazione e illustrano le limitazioni di ogni tipo:

-   [Utente interattivo](interactive-user.md)
-   [Avvio dell'utente](launching-user.md)
-   [Utente specificato](specified-user.md)
-   [Identità del servizio](service-identity.md)
