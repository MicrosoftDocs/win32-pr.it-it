# <a name="user-interface-issues-for-ipv6-winsock-applications"></a><span data-ttu-id="72f17-104">Problemi dell'interfaccia utente per le applicazioni Winsock IPv6</span><span class="sxs-lookup"><span data-stu-id="72f17-104">User Interface Issues for IPv6 Winsock Applications</span></span>

<span data-ttu-id="72f17-105">Una delle modifiche più ovvie da IPv4 a IPv6 è la dimensione dell'indirizzo IP.</span><span class="sxs-lookup"><span data-stu-id="72f17-105">One of the most obvious changes from IPv4 to IPv6 is the size of the IP address.</span></span> <span data-ttu-id="72f17-106">Molte interfacce utente forniscono finestre di dialogo che consentono a un utente di immettere un indirizzo IP, come esemplificato nella figura seguente.</span><span class="sxs-lookup"><span data-stu-id="72f17-106">Many user interfaces provide dialog boxes that enable a user to enter an IP address, as exemplified in the following figure.</span></span>

![casella indirizzo IPv4 comune in un'interfaccia utente](images/portingguide001.jpg)

<span data-ttu-id="72f17-108">L'indirizzamento in IPv6, a causa di molti fattori quali la lunghezza, la complessità e il significato delle sezioni nello spazio degli indirizzi IPv6, non è favorevole alla modifica o alla specifica da parte degli utenti.</span><span class="sxs-lookup"><span data-stu-id="72f17-108">Addressing in IPv6, due to many factors such as length, complexity, and the significance of sections within the IPv6 address space, is not conducive to modification or specification by users.</span></span> <span data-ttu-id="72f17-109">Pertanto, la necessità di fornire agli utenti la possibilità di specificare il proprio indirizzo è ridotta.</span><span class="sxs-lookup"><span data-stu-id="72f17-109">Therefore, the need to provide users with the capability of specifying their own address is reduced.</span></span> <span data-ttu-id="72f17-110">Inoltre, a causa della complessità associata all'indirizzamento IPv6, la possibilità per gli amministratori di specificare le informazioni sugli indirizzi IPv6 non può essere eseguita in base ai singoli nodi.</span><span class="sxs-lookup"><span data-stu-id="72f17-110">Additionally, due to the complexity associated with IPv6 addressing, providing administrators with the capability of specifying IPv6 address information is not likely to occur on a per-node basis.</span></span>

<span data-ttu-id="72f17-111">La visualizzazione di un indirizzo IPv6 nell'interfaccia utente non è inutilizzabile e pertanto gli sviluppatori devono considerare la variabilità delle dimensioni di un indirizzo IPv6 quando si modifica un'applicazione per supportare IPv6.</span><span class="sxs-lookup"><span data-stu-id="72f17-111">Displaying an IPv6 address in the UI is not inconceivable, and therefore developers should consider the variability in the size of an IPv6 address when modifying an application to support IPv6.</span></span>

<span data-ttu-id="72f17-112">Nella parte restante di questa sezione viene illustrata la differenza tra la prevedibilità degli indirizzi IPv4 e la lunghezza degli indirizzi IPv6.</span><span class="sxs-lookup"><span data-stu-id="72f17-112">The rest of this section discusses the difference between IPv4 address length predictability and IPv6 address length considerations.</span></span> <span data-ttu-id="72f17-113">Questa sezione presuppone che gli indirizzi IPv6 siano visualizzati nella relativa rappresentazione esadecimale.</span><span class="sxs-lookup"><span data-stu-id="72f17-113">This section presumes IPv6 addresses are being displayed in their hexadecimal representation.</span></span>

<span data-ttu-id="72f17-114">Gli indirizzi IPv4 sono di dimensioni stimabili, perché seguono rigidamente la notazione decimale punteggiata, come illustrato nell'esempio di indirizzo seguente:</span><span class="sxs-lookup"><span data-stu-id="72f17-114">IPv4 addresses are predictable in size, because they rigidly follow dotted decimal notation, as the following address example illustrates:</span></span>

``` syntax
10.10.256.1
```

<span data-ttu-id="72f17-115">Gli indirizzi IPv6 non sono così stimabili, a causa della convenzione degli indirizzi IPv6 che consente l'uso di un doppio segno di due punti (::) per rappresentare una serie di zeri.</span><span class="sxs-lookup"><span data-stu-id="72f17-115">IPv6 addresses are not so predictable, due to the IPv6 address convention that enables the use of a double-colon (::) to represent a series of zeros.</span></span> <span data-ttu-id="72f17-116">Di conseguenza, le rappresentazioni di indirizzo IPv6 seguenti equivalgono allo stesso indirizzo IPv6:</span><span class="sxs-lookup"><span data-stu-id="72f17-116">As such, the following IPv6 address representations equate to the same IPv6 address:</span></span>

``` syntax
1040:0:0:0:0:0:0:1
1040::1
```

<span data-ttu-id="72f17-117">La capacità di rappresentare una serie di zeri con un doppio segno di due punti restituisce una lunghezza imprevedibile per qualsiasi IPv6 specificato, che richiede ai programmatori di prendere in considerazione questa funzionalità quando si creano visualizzazioni dell'interfaccia utente degli indirizzi IPv6.</span><span class="sxs-lookup"><span data-stu-id="72f17-117">The capability to represent a series of zeros with a double-colon results in an unpredictable length for any given IPv6, which requires programmers to take this capability into consideration when creating user interface displays of IPv6 addresses.</span></span> <span data-ttu-id="72f17-118">Certamente, gli sviluppatori devono assicurarsi che l'interfaccia utente sia in grado di visualizzare gli indirizzi IP che non utilizzano un doppio segno di due punti per rappresentare una serie di zeri (il primo indirizzo riportato di seguito), oltre a essere in grado di visualizzare l'indirizzo IPv6 più lungo possibile (secondo indirizzo, con l'indirizzo IPv4 incorporato) quando si crea l'interfaccia utente che supporta IPv6.</span><span class="sxs-lookup"><span data-stu-id="72f17-118">Certainly, developers should ensure that the user interface is capable of displaying IP addresses that do not use a double-colon to represent a series of zeros (first address below), as well as being capable of displaying the longest possible IPv6 address (second address below, with the embedded IPv4 address) when creating their IPv6-capable user interface.</span></span> <span data-ttu-id="72f17-119">Si noti, inoltre, che l'aggiunta dell'identificatore di ambito (ID) all'indirizzo seguente comporta un aumento della lunghezza di un numero di altri undici caratteri:</span><span class="sxs-lookup"><span data-stu-id="72f17-119">Note, too, that adding the Scope identifier (ID) to the following address would increase its length by as much as another eleven characters:</span></span>

``` syntax
21DA:00D3:0010:2F3B:02AA:00FF:FE28:9C5A
0000:0000:0000:0000:0000:ffff:123.123.123.123
```

<span data-ttu-id="72f17-120">Un altro aspetto importante è che gli indirizzi basati sul nome siano più appropriati degli indirizzi IPv6 basati su numeri.</span><span class="sxs-lookup"><span data-stu-id="72f17-120">Another important consideration is whether name-based addresses are more appropriate than number-based IPv6 addresses.</span></span> <span data-ttu-id="72f17-121">Se gli indirizzi basati sul nome sono più appropriati, è necessario considerare le convenzioni di denominazione nell'interfaccia utente, inclusi eventuali controlli degli errori di input appropriati per l'attività.</span><span class="sxs-lookup"><span data-stu-id="72f17-121">If name-based addresses are more appropriate, consideration for naming conventions should be built into the user interface, including any input error-checking appropriate for the task.</span></span>

<span data-ttu-id="72f17-122">Esistono altre complessità associate alla visualizzazione degli indirizzi IPv6 che devono essere presi in considerazione dagli sviluppatori durante la modifica dell'applicazione e durante la progettazione delle rappresentazioni dell'interfaccia utente degli indirizzi IPv6.</span><span class="sxs-lookup"><span data-stu-id="72f17-122">There are other complexities associated with displaying IPv6 addresses that developers must take into consideration when modifying their application, and when designing user interface representations of IPv6 addresses.</span></span> <span data-ttu-id="72f17-123">Di seguito sono riportate alcune considerazioni:</span><span class="sxs-lookup"><span data-stu-id="72f17-123">Some of these considerations are the following:</span></span>

-   <span data-ttu-id="72f17-124">L'indirizzo deve contenere tutte le sequenze di zeri oppure utilizzare la notazione con due punti.</span><span class="sxs-lookup"><span data-stu-id="72f17-124">Should the address contain all sequences of zeros, or use the double-colon notation?</span></span>
-   <span data-ttu-id="72f17-125">È più appropriato usare una rappresentazione di indirizzo basata su numeri o una rappresentazione basata su nome?</span><span class="sxs-lookup"><span data-stu-id="72f17-125">Is it more appropriate to use a number-based address representation or a name-based representation?</span></span>
-   <span data-ttu-id="72f17-126">L'utente è interessato a discernere un certo aspetto dello schema di indirizzamento, ad esempio il prefisso della subnet, l'identificatore dell'ambito o altri campi sottocampi?</span><span class="sxs-lookup"><span data-stu-id="72f17-126">Is the user interested in discerning a certain aspect of the addressing scheme, such as the subnet prefix, scope identifier, or other subfields?</span></span>
-   <span data-ttu-id="72f17-127">L'utente è interessato a determinare altri aspetti dell'indirizzo, ad esempio l'identificatore TLA, l'identificatore NLA o l'identificatore del contratto di contratto?</span><span class="sxs-lookup"><span data-stu-id="72f17-127">Is the user interested in determining other aspects of the address, such as the TLA identifier, the NLA identifier, or the SLA identifier?</span></span>
-   <span data-ttu-id="72f17-128">L'interfaccia utente sarà in grado di discernere gli indirizzi IPv6 incorporati e, in caso affermativo, come verranno gestiti e visualizzati?</span><span class="sxs-lookup"><span data-stu-id="72f17-128">Will your user interface be capable of discerning embedded IPv6 addresses, and if so, how will those be handled and displayed?</span></span> <span data-ttu-id="72f17-129">Quando si visualizzano le informazioni sull'indirizzo per l'utente, è possibile discernere tra gli indirizzi compatibili con IPv4 e gli indirizzi IPv6 con mapping IPv4?</span><span class="sxs-lookup"><span data-stu-id="72f17-129">Will you discern between IPv4-compatible addresses and IPv4-mapped IPv6 addresses when displaying address information to the user?</span></span>

<span data-ttu-id="72f17-130">Esistono anche altre considerazioni e gli sviluppatori devono considerare attentamente i destinatari dei clienti quando sviluppano interfacce utente di indirizzi IP.</span><span class="sxs-lookup"><span data-stu-id="72f17-130">There are other considerations as well, and developers should carefully consider their customer audience when developing IP address user interfaces.</span></span>

<span data-ttu-id="72f17-131">Procedure consigliate</span><span class="sxs-lookup"><span data-stu-id="72f17-131">Best Practices</span></span>

-   <span data-ttu-id="72f17-132">Gli sviluppatori devono considerare l'approccio appropriato per ogni interfaccia utente quando modificano l'applicazione in modo da supportare IPv6.</span><span class="sxs-lookup"><span data-stu-id="72f17-132">Developers must consider the appropriate approach to each user interface when modifying their application to support IPv6.</span></span> <span data-ttu-id="72f17-133">Verificare che l'interfaccia utente contenga una lunghezza sufficiente per visualizzare gli indirizzi IPv6 è imperativa, in quanto determina se l'indirizzo è basato sul numero o sul nome.</span><span class="sxs-lookup"><span data-stu-id="72f17-133">Ensuring that the user interface contains sufficient length to display IPv6 addresses is imperative, as is determining whether that address is number or name based.</span></span>

<span data-ttu-id="72f17-136">Codice da evitare</span><span class="sxs-lookup"><span data-stu-id="72f17-136">Code To Avoid</span></span>

-   <span data-ttu-id="72f17-137">Gli elementi dell'interfaccia utente che dipendono da un indirizzo di dimensione IPv4 devono essere sottoposti a controllo e parte del controllo deve includere se le informazioni fornite (in IPv4) sono appropriate per IPv6.</span><span class="sxs-lookup"><span data-stu-id="72f17-137">User interface elements that depend on an IPv4-sized address must undergo scrutiny, and part of that scrutiny should include whether the information you were providing (under IPv4) is appropriate for IPv6.</span></span>

1.  <span data-ttu-id="72f17-142">Eseguire una revisione visiva dell'interfaccia utente, cercando qualsiasi elemento che dipenda da una lunghezza specifica per la stringa dell'indirizzo IP.</span><span class="sxs-lookup"><span data-stu-id="72f17-142">Perform a visual review of the user interface, looking for any element that is dependent on a specific length for the IP address string.</span></span> <span data-ttu-id="72f17-143">I controlli con la notazione decimale tratteggiata a quattro sezioni facilmente identificabili sono facili da individuare, mentre altri non lo sono.</span><span class="sxs-lookup"><span data-stu-id="72f17-143">Controls with the easily identified four-section dotted decimal notation are easy to spot, but others are not.</span></span> <span data-ttu-id="72f17-144">È possibile che vengano visualizzati indirizzi IP, ad esempio nelle finestre di dialogo, in cui un indirizzo IPv6 potrebbe esaurire lo spazio di visualizzazione.</span><span class="sxs-lookup"><span data-stu-id="72f17-144">There may be places where IP addresses could be displayed, such as in dialog boxes, where an IPv6 address might run out of display room.</span></span>
