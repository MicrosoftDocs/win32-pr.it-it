# <a name="upcalls-exposed-by-windows-sockets-2-dll"></a>Upcalls Exposed by Windows Sockets 2 DLL

Questa sezione elenca le upcall che i provider di servizi possono effettuare nel client Windows Sockets. I provider di servizi ricevono una tabella upcall dispatch come parametro per la funzione [**WSPStartup**](/windows/desktop/api/Ws2spi/nf-ws2spi-wspstartup) e usano le voci in questa tabella per effettuare le upcall. Pertanto, un client non deve esportare le funzioni WPU.

Non è obbligatorio che i provider usino tutte queste upcall. La tabella seguente indica quali upcall devono essere usate e quali sono facoltative.

| Funzione                                                               | Descrizione                                                                                                              | Stato                         | Significato                                                                                                                                                                          |
|------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------|--------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**WPUCloseEvent**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpucloseevent)                               | Chiude un handle dell'oggetto evento aperto.                                                                                      | facoltativo.                      | Il provider può usare una chiamata Windows appropriata.                                                                                                                     |
| [**WPUCloseSocketHandle**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpuclosesockethandle)                 | Chiude un handle socket allocato dalla DLL Windows Sockets.                                                             | Obbligatorio.                      | Il32.dll Ws2 deve eseguire query e/o modificare le informazioni sullo stato \_ interno associate all'handle del socket.                                                                       |
| [**WPUCloseThread**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpuclosethread)                             | Chiude un ID thread per un thread del servizio interno.                                                                       |                                |                                                                                                                                                                                  |
| [**WPUCompleteOverlappedRequest**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpucompleteoverlappedrequest) | Recapita una notifica di completamento di I/O sovrapposta in cui il meccanismo di completamento è diverso da APC in modalità utente.    |                                |                                                                                                                                                                                  |
| [**WPUCreateEvent**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpucreateevent)                             | Crea un nuovo oggetto evento.                                                                                              | facoltativo.                      | Il provider può usare una chiamata Windows appropriata.                                                                                                                        |
| [**WPUCreateSocketHandle**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpucreatesockethandle)               | Crea un nuovo handle socket per i provider nonIFS.                                                                        | Obbligatorio per i provider nonIFS. | Il32.dll Ws2 deve eseguire query e/o modificare le informazioni sullo stato \_ interno associate all'handle del socket.                                                                       |
| [**WPUFDIsSet**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpufdisset)                                     | Controlla l'appartenenza dell'handle di socket specificato.                                                                    | facoltativo.                      | Si tratta solo di una funzione di praticità che sa come scorrere le strutture [**\_ dei set fd.**](/windows/desktop/api/winsock/nf-winsock-fd_set) Un provider potrebbe dover comunque eseguire in modo esplicito queste strutture. |
| [**WPUGetProviderPath**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpugetproviderpath)                     | Recupera il percorso della DLL per il provider specificato.                                                                       | Obbligatorio.                      | Solo il32.dll Ws2 saprebbe dove è stato installato un livello di protocollo adiacente (potenzialmente di \_ un altro fornitore).                                                           |
| [**WPUModifyIFSHandle**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpumodifyifshandle)                     | Riceve un handle IFS (possibilmente) modificato dalla DLL Windows Sockets.                                                  | Obbligatorio per i provider IFS.    | Il32.dll Ws2 deve eseguire query e/o modificare le informazioni sullo stato \_ interno associate all'handle del socket.                                                                       |
| [**WPUPostMessage**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpupostmessage)                             | Esegue la funzione [**PostMessage**](/windows/win32/api/winuser/nf-winuser-postmessagea) standard in modo da mantenere la compatibilità con le versioni precedenti. | Obbligatorio.                      | Windows 2000 e Windows NT solo. Windows 95 consente di pubblicare un messaggio dalla modalità kernel.                                                                                               |
| [**WPUQueryBlockingCallback**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpuqueryblockingcallback)         | Restituisce un puntatore alla funzione hook di blocco di un thread.                                                                  | Obbligatorio.                      | Non esiste alcuna funzionalità di Windows corrispondente. Solo il32.dll Ws2 \_ ha le informazioni per eseguire questa operazione.                                                                    |
| [**WPUQuerySocketHandleContext**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpuquerysockethandlecontext)   | Ottiene il valore di contesto di un socket (solo provider nonIFS).                                                                   | Obbligatorio per i provider nonIFS. | Il32.dll Ws2 deve eseguire query e/o modificare le informazioni sullo stato \_ interno associate all'handle del socket.                                                                       |
| [**WPUQueueApc**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpuqueueapc)                                   | Accoda un APC in modalità utente al thread specificato.                                                                          | facoltativo.                      | È [**anche possibile usare QueueUserApc.**](/windows/win32/api/processthreadsapi/nf-processthreadsapi-queueuserapc)                                                                                                                      |
| [**WPUResetEvent**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpuresetevent)                               | Reimposta un oggetto evento.                                                                                                  | facoltativo.                      | Il provider può usare una chiamata Windows appropriata.                                                                                                                        |
| [**WPUSetEvent**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpusetevent)                                   | Imposta un oggetto evento.                                                                                                    | facoltativo.                      | Il provider può usare una chiamata Windows appropriata.                                                                                                                        |
