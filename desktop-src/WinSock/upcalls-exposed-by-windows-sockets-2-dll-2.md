# <a name="upcalls-exposed-by-windows-sockets-2-dll"></a>Chiamate esposte dalla DLL di Windows Sockets 2

In questa sezione vengono elencate le chiamate che i provider di servizi possono apportare al client Windows Sockets. I provider di servizi ricevono una tabella dispatch di chiamata come parametro per la funzione [**WSPStartup**](/windows/desktop/api/Ws2spi/nf-ws2spi-wspstartup) e usano le voci di questa tabella per effettuare le chiamate. Pertanto, non è necessario che un client esporti le funzioni WPU.

Non è obbligatorio che i provider usino tutti questi richiami. La tabella seguente indica quali chiamate devono essere utilizzate e quali sono facoltative.

|------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------|--------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**WPUCloseEvent**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpucloseevent)                               | Chiude un handle di oggetto evento aperto.                                                                                      | facoltativo.                      | Il provider può invece usare una chiamata di Windows appropriata.                                                                                                                     |
| [**WPUCloseSocketHandle**](/windows/desktop/api/Ws2spi/nf-ws2spi-wpuclosesockethandle)                 | Chiude un handle di socket allocato dalla DLL di Windows Sockets.                                                             | Obbligatorio.                      | Il \_32.dll WS2 deve eseguire una query e/o modificare le informazioni sullo stato interno associate all'handle del socket.                                                                       |
