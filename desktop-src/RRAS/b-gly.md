<dl> <dt>

<span id="_rras_best_route_gly"></span><span id="_RRAS_BEST_ROUTE_GLY"></span>**route migliore**
</dt> <dd>

Il percorso migliore è una route ottimale a una destinazione di rete, in base ai criteri specificati. Questo concetto si basa sul fatto che ci sono alcuni "costi" necessari per l'esecuzione di una route in una rete.

Il percorso migliore da eseguire è quello con il costo più basso, in base a un criterio specificato. Questo criterio può includere il numero di reti attraversate, il tipo di rete incrociato (ad esempio, pubblico o privato) o un limite di valuta o di larghezza di banda.

</dd> <dt>

<span id="_rras_border_gateway_protocol_bgp__gly"></span><span id="_RRAS_BORDER_GATEWAY_PROTOCOL_BGP__GLY"></span>**protocollo BGP (Border Gateway Protocol)**
</dt> <dd>

</dd> <dt>
