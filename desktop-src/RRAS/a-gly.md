<dl> <dt>

<span id="_rras_aged_out_gly"></span><span id="_RRAS_AGED_OUT_GLY"></span>**obsoleto**
</dt> <dd>

Inoltre, *obsoleto dalla cache*. In RRAS una route è obsoleta quando viene rimossa dalla tabella di routing dopo un periodo di tempo specificato.

</dd> <dt>

<span id="_rras_authentication_gly"></span><span id="_RRAS_AUTHENTICATION_GLY"></span>**autenticazione**
</dt> <dd>

L'autenticazione è il processo di verifica che un utente sia colui che dichiara di essere. L'autenticazione viene in genere eseguita utilizzando un nome utente e una password.

</dd> <dt>

<span id="_rras_autonomous_system_gly"></span><span id="_RRAS_AUTONOMOUS_SYSTEM_GLY"></span>**sistema autonomo (AS)**
</dt> <dd>

Un sistema autonomo è un gruppo di router che condividono un singolo criterio amministrativo. Questi router usano tutti lo stesso protocollo di routing per comunicare, denominato [*protocollo gateway interno*](i-gly.md). Vedere anche [*protocollo gateway esterno*](e-gly.md).

</dd> <dt>

<span id="_rras_autostatic_route_gly"></span><span id="_RRAS_AUTOSTATIC_ROUTE_GLY"></span>**Route autostatic**
</dt> <dd>

Una route autostatic è una route ottenuta in modo dinamico da un protocollo di routing. Una volta ottenuta una route in questo modo, si comporta come un [*STATICROUTE*](s-gly.md).

</dd> </dl>
