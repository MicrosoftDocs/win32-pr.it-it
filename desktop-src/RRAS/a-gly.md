<dl> <dt>

<span id="_rras_aged_out_gly"></span><span id="_RRAS_AGED_OUT_GLY"></span>**obsoleto**
</dt> <dd>

Inoltre, *obsoleto dalla cache.* In RRAS una route viene obsoleta quando viene rimossa dalla tabella di routing dopo un periodo di tempo specificato.

</dd> <dt>

<span id="_rras_authentication_gly"></span><span id="_RRAS_AUTHENTICATION_GLY"></span>**Autenticazione**
</dt> <dd>

L'autenticazione è il processo di verifica che un utente sia l'utente che dichiara di essere. L'autenticazione viene in genere eseguita usando un nome utente e una password.

</dd> <dt>

<span id="_rras_autonomous_system_gly"></span><span id="_RRAS_AUTONOMOUS_SYSTEM_GLY"></span>**sistema autonomo (AS)**
</dt> <dd>

Un sistema autonomo è un gruppo di router che condividono un singolo criterio amministrativo. Tutti questi router usano lo stesso protocollo di routing per comunicare (denominato [*protocollo gateway interno).*](i-gly.md) Vedere anche [*il protocollo gateway esterno.*](e-gly.md)

</dd> <dt>

<span id="_rras_autostatic_route_gly"></span><span id="_RRAS_AUTOSTATIC_ROUTE_GLY"></span>**route autostatic**
</dt> <dd>

Una route autostatic è una route ottenuta dinamicamente da un protocollo di routing. Dopo essere stata ottenuta in questo modo, una route si comporta come [*staticroute.*](s-gly.md)

</dd> </dl>
