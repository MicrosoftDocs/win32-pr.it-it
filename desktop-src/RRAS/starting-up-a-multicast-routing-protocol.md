# <a name="starting-up-a-multicast-routing-protocol"></a>Avvio di un protocollo di routing multicast

La tabella seguente riepiloga una serie di passaggi di un'interazione di avvio tra un protocollo di routing e il gestore di gruppi multicast. La prima colonna descrive le azioni eseguite dal protocollo di routing e le risposte del protocollo di routing al gestore di gruppi multicast. La seconda colonna descrive le risposte del gestore del gruppo multicast al protocollo di routing e le azioni eseguite dal gestore di gruppi multicast, ad esempio i callback. La terza colonna presenta eventuali informazioni aggiuntive.

Ogni riga della tabella rappresenta un passaggio.



| Azione del protocollo di routing                                                                                                                                                                                                                                                                                                                 | Azione di gestione del gruppo multicast                                                                                                                                                                                                                                                                                                                                                                                                                                            | Note                                                                                                                                                                                                                                   |
|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Eseguire la registrazione con il gestore di gruppi multicast usando la [**funzione MgmRegisterMProtocol.**](/windows/desktop/api/Mgm/nf-mgm-mgmregistermprotocol)                                                                                                                                                                                                                      | Restituire al protocollo di routing un handle che il protocollo deve usare per identificarsi nelle chiamate API MGM successive.                                                                                                                                                                                                                                                                                                                                                        |                                                                                                                                                                                                                                         |
| Determinare se un'interfaccia è già di proprietà e il protocollo di routing proprietario usando la [**funzione MgmGetProtocolOnInterface.**](/windows/desktop/api/Mgm/nf-mgm-mgmgetprotocoloninterface)                                                                                                                                                                    |                                                                                                                                                                                                                                                                                                                                                                                                                                                                           | IGMP può usare questa funzione per determinare il proprietario di un'interfaccia ed eseguire l'elaborazione specifica del protocollo con le informazioni restituite da questa funzione.                                                                             |
| Assumere la proprietà di tutte le interfacce in cui il protocollo è abilitato usando la [**funzione MgmTakeInterfaceOwnership.**](/windows/desktop/api/Mgm/nf-mgm-mgmtakeinterfaceownership)                                                                                                                                                                                | Se IGMP ha già assunto la proprietà di un'interfaccia e la chiamata di funzione [**MgmTakeInterfaceOwnership**](/windows/desktop/api/Mgm/nf-mgm-mgmtakeinterfaceownership) viene ricevuta per la stessa interfaccia, contattare IGMP usando il callback [**\_ CALLBACK \_ IGMP \_ DISABLE DI PMGM.**](/windows/win32/api/mgm/nc-mgm-pmgm_disable_igmp_callback) Dopo aver apportato tutte le modifiche ai dati multicast relativi alla proprietà dell'interfaccia, contattare di nuovo IGMP usando [**\_ PMGM ENABLE \_ IGMP \_ CALLBACK**](/windows/desktop/api/Mgm/nc-mgm-pmgm_enable_igmp_callback).<br/> | Un solo protocollo può essere proprietario di un'interfaccia in un determinato momento, oltre a IGMP.                                                                                                                                                            |
| Determinare lo stato corrente dell'appartenenza al gruppo nel router. Questa azione viene eseguita usando le funzioni di enumerazione dell'appartenenza al [**gruppo: MgmGroupEnumerationStart**](/windows/desktop/api/Mgm/nf-mgm-mgmgroupenumerationstart), [**MgmGroupEnumerationGetNext**](/windows/desktop/api/Mgm/nf-mgm-mgmgroupenumerationgetnext)e [**MgmGroupEnumerationEnd**](/windows/desktop/api/Mgm/nf-mgm-mgmgroupenumerationend). | Restituisce l'elenco di gruppi.                                                                                                                                                                                                                                                                                                                                                                                                                                                | I protocolli di routing possono usare i risultati per determinare le azioni da eseguire in base ai gruppi che sono già stati aggiunti. Per [un esempio completo dell'uso](enumerating-groups.md) di queste funzioni, vedere Enumerazione dei gruppi.<br/> |
