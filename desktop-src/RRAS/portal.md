# <a name="routing-and-remote-access-service"></a>Routing and Remote Access Service

## <a name="purpose"></a>Scopo

Il servizio di accesso remoto (RAS) può essere usato per creare applicazioni client. Queste applicazioni visualizzano finestre di dialogo comuni RAS, gestiscono le connessioni e i dispositivi di accesso remoto e modificano le voci del libro telefonico.

Le API di routing rendono possibile la creazione di applicazioni per amministrare le funzionalità di routing del sistema operativo.

Gli sviluppatori possono utilizzare le API del protocollo di routing per implementare i protocolli di routing.

## <a name="developer-audience"></a>Sviluppatori

Le API del servizio Routing e accesso remoto sono progettate per l'uso da parte dei programmatori C/C++. I programmatori devono inoltre acquisire familiarità con i concetti relativi alla rete.
