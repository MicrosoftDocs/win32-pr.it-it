# <a name="routing-protocols"></a>Protocolli di routing

## <a name="purpose"></a>Scopo

Gli sviluppatori possono utilizzare le API del protocollo di routing per implementare i protocolli di routing.

## <a name="developer-audience"></a>Sviluppatori

Le API di routing sono progettate per l'uso da parte dei programmatori C/C++. I programmatori devono inoltre acquisire familiarità con i concetti relativi alla rete.

## <a name="run-time-requirements"></a>Requisiti di runtime

Il protocollo di routing è una tecnologia basata su server. Tutte le funzionalità dei protocolli di routing sono incorporate in Windows Server 2008, Windows Server 2003 e Windows 2000 Server. Le applicazioni di routing non possono essere eseguite su sistemi operativi client, ad esempio Windows XP o Windows Vista. Per informazioni più specifiche sui sistemi operativi che supportano una particolare funzione, vedere le sezioni requisiti nella documentazione.
