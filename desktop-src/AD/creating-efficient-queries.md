# <a name="creating-efficient-queries"></a>Creazione di query efficienti

La tabella seguente identifica i concetti importanti da considerare quando si crea una query efficiente.



| Area                  | Descrizione                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|-----------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Indicizzazione              | Verificare che il filtro query contenga almeno un attributo indicizzato.<br/> Per ulteriori informazioni, vedere [attributi indicizzati](indexed-attributes.md).<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Classe rispetto alla categoria    | L'istruzione "objectClass = XYZ" si riferisce agli oggetti directory in cui "xyz" rappresenta una classe della gerarchia di classi di oggetti, mentre "objectCategory = XYZ" si riferisce agli oggetti directory in cui "xyz" identifica una classe specifica nella gerarchia di classi di oggetti. La proprietà [**objectClass**](/windows/desktop/ADSchema/a-objectclass) può assumere più valori, mentre [**objectCategory**](/windows/desktop/ADSchema/a-objectcategory) accetta un solo valore e, di conseguenza, è più adatta per la corrispondenza dei tipi di oggetti in una ricerca di directory.<br/>                                                                                                                                                                                          |
| Ricerca di testo        | Evitare la ricerca di testo al centro e alla fine di una stringa.<br/> Ad esempio, "CN = \* Hille \* " o "CN = \* Larouse".<br/> L'utilizzo di criteri di corrispondenza più specifici tende a migliorare le prestazioni di ricerca. Questo perché Active Directory Domain Services valuterà tutti i predicati, identificherà gli indici e quindi sceglierà un indice con la probabilità di ottenere il set più piccolo di valori restituiti. Questa tecnica non funziona bene con le ricerche con stringhe Mid-and-end. Se non si dispone di altre opzioni diverse dall'utilizzo di queste ricerche, è possibile definire un indice di tupla per l'attributo.<br/> Per ulteriori informazioni sugli indici di tupla, vedere funzionamento dell' [indicizzazione delle tuple](how-tuple-indexing-works.md).<br/> |
| Ricerca di sottoalbero     | Usare il catalogo globale se si stanno valutando ricerche di sottoalbero. Per la ricerca di riferimenti sono necessarie risorse estese.<br/> Per ulteriori informazioni, vedere [specifica di altre opzioni di ricerca](specifying-other-search-options.md).<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Uso del paging         | Si supponga che una ricerca del sottoalbero restituisca un set di risultati di grandi dimensioni. Usare il paging quando si eseguono ricerche di sottoalbero. Il server produrrà un set di risultati di grandi dimensioni in blocchi riducendo le risorse di memoria sul lato server. In questo modo è possibile ridurre l'utilizzo della rete e ridurre la necessità di inviare blocchi di dati di grandi dimensioni in una rete.<br/> Per ulteriori informazioni, vedere [specifica di altre opzioni di ricerca](specifying-other-search-options.md).<br/>                                                                                                                                                                                                                                                                                        |
| Combinare le ricerche      | Usare più attributi per una ricerca. Una ricerca di un oggetto che legge due attributi è più efficiente di due ricerche dello stesso oggetto, ciascuna delle quali restituisce un attributo.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Uso efficiente dell'associazione | Eseguire un'associazione a un oggetto una sola volta e mantenere il punto di controllo dell'associazione per il resto della sessione. Non associare e annullare il binding per ogni chiamata. Se si utilizza ADO o OLE DB, non creare molti oggetti connessione.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Caching RoodDSE       | Leggere il rootDSE una sola volta e ricordarne il contenuto per il resto della sessione.<br/> Per ulteriori informazioni, vedere [binding senza server e RootDSE](serverless-binding-and-rootdse.md).<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Salvataggio permanente riferimenti | Consente di salvare in modo permanente i riferimenti agli oggetti come GUID, non nomi distinti, per poter essere rinominati ed eliminati in modo sicuro.<br/> Per ulteriori informazioni, vedere [utilizzo di objectGUID per l'associazione a un oggetto](using-objectguid-to-bind-to-an-object.md).<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
