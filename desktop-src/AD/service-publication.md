# <a name="service-publication"></a>Pubblicazione di un servizio

I servizi si annunciano usando oggetti archiviati in Active Directory Domain Services. Questa operazione è nota come *pubblicazione del servizio*. I client eseguono una query sulla directory per individuare i servizi. Questa operazione è denominata *Rendezvous del servizio client*. In questa sezione vengono illustrati i tipi di oggetti directory utilizzati per la pubblicazione del servizio e viene illustrata la relativa modalità di utilizzo per gli Rendezvous del servizio client.

In questa sezione vengono illustrati gli argomenti seguenti:

-   [Problemi di sicurezza per la pubblicazione del servizio](security-issues-for-service-publication.md)
-   [Oggetti punto di connessione](connection-points.md)
-   [Pubblicazione con i punti di connessione del servizio (convergenza)](publishing-with-service-connection-points.md)
-   [Informazioni da archiviare in un punto di connessione del servizio](service-connection-point-properties.md)
-   [Dove creare un punto di connessione del servizio](where-to-create-a-service-connection-point.md)
-   [Come pubblicare servizi replicabili, basati su host e database usando i punti di connessione del servizio](service-connection-points-for-replicated-host-based-and-database-services.md)
-   [Creazione e gestione di un punto di connessione del servizio](creating-and-maintaining-a-service-connection-point.md)
-   [Come un client esegue una query per un SCP e lo usa per eseguire il binding a un'istanza del servizio](how-clients-find-and-use-a-service-connection-point.md)
-   [Uso delle API RpcNs (RPC Name Service) per pubblicare un servizio RPC](publishing-with-the-rpc-name-service-rpcns.md)

-   [Pubblicazione di servizi basati su COM nell'archivio di classi COM+](publishing-com-services.md)

Per ulteriori informazioni sul modo in cui i servizi e i client si autenticano [reciprocamente, vedere l'autenticazione reciproca tramite Kerberos](mutual-authentication-using-kerberos.md). Per ulteriori informazioni sui contesti di sicurezza del servizio e gli account di accesso, vedere [account di accesso al servizio](service-logon-accounts.md).
