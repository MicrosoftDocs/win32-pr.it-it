<span data-ttu-id="938bd-106">Quando si crea un nuovo oggetto in Active Directory Domain Services, è possibile creare in modo esplicito un descrittore di sicurezza e quindi impostare il descrittore di sicurezza come proprietà **ntSecurityDescriptor** dell'oggetto.</span><span class="sxs-lookup"><span data-stu-id="938bd-106">When you create a new object in Active Directory Domain Services, you can explicitly create a security descriptor and then set that security descriptor as the object's **nTSecurityDescriptor** property.</span></span> <span data-ttu-id="938bd-107">Per ulteriori informazioni, vedere [creazione di un descrittore di sicurezza per un nuovo oggetto directory](creating-a-security-descriptor-for-a-new-directory-object.md).</span><span class="sxs-lookup"><span data-stu-id="938bd-107">For more information, see [Creating a Security Descriptor for a New Directory Object](creating-a-security-descriptor-for-a-new-directory-object.md).</span></span>

<span data-ttu-id="938bd-108">Active Directory Domain Services utilizzare le seguenti regole per impostare l'elenco DACL nel descrittore di sicurezza del nuovo oggetto:</span><span class="sxs-lookup"><span data-stu-id="938bd-108">Active Directory Domain Services use the following rules to set the DACL in the new object's security descriptor:</span></span>

-   <span data-ttu-id="938bd-109">Se si specifica in modo esplicito un descrittore di sicurezza quando si crea l'oggetto, il sistema unisce eventuali ACE ereditabili dall'oggetto padre nell'elenco DACL specificato, a meno che il bit non **\_ \_ protetto DACL** non sia impostato nei bit di controllo del descrittore di sicurezza.</span><span class="sxs-lookup"><span data-stu-id="938bd-109">If you explicitly specify a security descriptor when you create the object, the system merges any inheritable ACEs from the parent object into the specified DACL unless the **SE\_DACL\_PROTECTED** bit is set in the security descriptor's control bits.</span></span>
-   <span data-ttu-id="938bd-110">Se non si specifica un descrittore di sicurezza, il sistema compila l'elenco DACL dell'oggetto unendo eventuali ACE ereditabili dall'oggetto padre nell'elenco DACL predefinito dall'oggetto **classSchema** per la classe dell'oggetto.</span><span class="sxs-lookup"><span data-stu-id="938bd-110">If you do not specify a security descriptor, the system builds the object's DACL by merging any inheritable ACEs from the parent object into the default DACL from the **classSchema** object for the object's class.</span></span>
-   <span data-ttu-id="938bd-111">Se lo schema non dispone di un DACL predefinito, il DACL dell'oggetto è l'elenco DACL predefinito del token primario o di rappresentazione del creatore.</span><span class="sxs-lookup"><span data-stu-id="938bd-111">If the schema does not have a default DACL, the object's DACL is the default DACL from the primary or impersonation token of the creator.</span></span>
-   <span data-ttu-id="938bd-112">Se non è presente alcun DACL specificato, ereditato o predefinito, il sistema crea l'oggetto senza DACL, che consente a tutti gli utenti di accedere completamente all'oggetto.</span><span class="sxs-lookup"><span data-stu-id="938bd-112">If there is no specified, inherited, or default DACL, the system creates the object with no DACL, which allows everyone full access to the object.</span></span>

<span data-ttu-id="938bd-113">Il sistema usa un algoritmo simile per creare un SACL per un oggetto servizio directory.</span><span class="sxs-lookup"><span data-stu-id="938bd-113">The system uses a similar algorithm to build a SACL for a directory service object.</span></span>
