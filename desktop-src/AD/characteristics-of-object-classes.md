# <a name="characteristics-of-object-classes"></a>Caratteristiche delle classi di oggetti

Ogni classe di oggetti Active Directory Domain Services è definita da un **oggetto classSchema** nel contenitore dello schema. Gli attributi di un **oggetto classSchema** specificano le caratteristiche della classe, ad esempio:

-   Identificatori di classe: le classi hanno diversi identificatori, tra cui **ldapDisplayName**, che vengono usati dai client LDAP per identificare la classe nei filtri di ricerca, **e schemaIDGUID**, che vengono usati nei descrittori di sicurezza per controllare l'accesso alla classe.
-   Attributi possibili: una definizione di classe di oggetti include elenchi degli attributi obbligatori e facoltativi che possono essere impostati in un'istanza della classe.
-   Possibili elementi padre: ogni istanza dell'oggetto, ad eccezione della radice della gerarchia di directory, ha esattamente un elemento padre. Una definizione di classe di oggetti include elenchi di possibili elementi padre, ad esempio delle classi di oggetti che possono contenere un'istanza della classe.
-   Superclassi e classi ausiliarie: ogni classe di oggetti (ad eccezione di [*top*](/previous-versions/windows/desktop/legacy/ms681941(v=vs.85))) è derivata da un'altra classe. Una classe eredita i possibili attributi e possibili elementi padre dalle classi sopra di essa nella gerarchia di classi. Una classe può anche avere un numero qualsiasi di classi ausiliarie da cui eredita elenchi di possibili attributi. Per altre informazioni, vedere [Ereditarietà delle classi nello schema di Active Directory.](class-inheritance-in-the-active-directory-schema.md)

La tabella seguente elenca **lDAPDisplayName e** la descrizione degli attributi chiave di un **oggetto classSchema.** Per altre informazioni e un elenco completo degli attributi obbligatori e facoltativi di un **oggetto classSchema,** vedere [**classSchema.**](/windows/desktop/ADSchema/c-classschema)

L'attributo di <strong>denominazione per</strong> gli oggetti <strong>classSchema è cn</strong> (Common-Name). Il valore assegnato a <strong>cn</strong> è il valore che la classe dell'oggetto avrà come RDN. Ad esempio, il <strong>cn</strong> della classe di oggetto <strong>organizationalUnit</strong> è Organizational-Unit, che verrà visualizzato in un nome distinto come CN=Organizational-Unit. Il <strong>cn</strong> deve essere univoco nel contenitore dello schema. |

| <strong>schemaIDGUID</strong> | GUID archiviato come stringa ottetto. Questo GUID identifica in modo univoco la classe. Se si crea una nuova classe, generare il proprio GUID per ogni classe in modo che tutte le installazioni dell'estensione usino lo stesso <strong>schemaIDGUID</strong> per fare riferimento alla classe.<br /> |

Se <strong>adminDisplayName</strong> non viene specificato quando viene creata una classe, il sistema usa il valore Common-Name come nome visualizzato. Questo nome visualizzato viene usato solo se non esiste un mapping nella proprietà <strong>classDisplayName</strong> dell'identificatore di visualizzazione per la classe. Per altre informazioni, vedere <a href="display-specifiers.md">Visualizzare identificatori e</a> Nomi visualizzati di classi e <a href="class-and-attribute-display-names.md">attributi.</a><br /> | 
| <strong>governsID</strong> | OID della classe. Questo valore deve essere univoco tra <strong>gli elementi governsIDs</strong> di tutti gli oggetti <strong>classSchema</strong> e <strong>gli attributeID</strong> di tutti <strong>gli oggetti attributeSchema.</strong> Per altre informazioni, vedere <a href="object-identifiers.md">Identificatori di oggetto.</a> | 
| <strong>rDnAttId</strong> | Identifica l'attributo di denominazione, ovvero l'attributo che fornisce il nome RDN per questa classe se diverso dal valore predefinito (<strong>cn</strong>). L'uso di un attributo di denominazione diverso <strong>da cn</strong> è sconsigliato. Gli attributi di denominazione devono essere ricavati dal set noto (OU, CN, O, L e DC) riconosciuto da tutti i client LDAP versione 3. Per altre informazioni, vedere <a href="object-names-and-identities.md">Object Names and Identities</a> and <a href="syntaxes-for-attributes-in-active-directory-domain-services.md">Syntaxes for Attributes in Active Directory Domain Services</a>. | <strong>mustContain</strong>, <strong>systemMustContain</strong> | Coppia di proprietà multivalore che specificano gli attributi che devono essere presenti nelle istanze di questa classe. Si tratta di attributi obbligatori che devono essere presenti durante la creazione e non possono essere cancellati dopo la creazione. Dopo la creazione della classe, queste proprietà non possono essere modificate. Il set completo di attributi obbligatori per una classe è l'unione dei valori <strong>systemMustContain</strong> e <strong>mustContain</strong> in questa classe e in tutte le classi ereditate.<br /> | 
| <strong>mayContain</strong>, <strong>systemMayContain</strong> | Coppia di proprietà multivalore che specificano gli attributi che possono essere presenti nelle istanze di questa classe. Si tratta di attributi facoltativi che non sono obbligatori e, pertanto, possono essere presenti o meno in un'istanza di questa classe. È possibile aggiungere o rimuovere <strong>valori mayContain</strong> da un oggetto <strong>classSchema</strong> di categoria 1 o di categoria 2 esistente. Prima di rimuovere un <strong>valore mayContain</strong> da un oggetto <strong>classSchema,</strong> è necessario cercare le istanze della classe di oggetti e cancellare tutti i valori per l'attributo che si sta rimuovendo. Dopo la creazione della classe, la proprietà <strong>systemMayContain</strong> non può essere modificata. Il set completo di attributi facoltativi per una classe è l'unione dei valori <strong>systemMayContain</strong> e <strong>mayContain</strong> in questa classe e in tutte le classi ereditate.<br /> | 
| <strong>possSuperiors</strong>, <strong>systemPossSuperiors</strong> | Coppia di proprietà multivalore che specificano le classi strutturali che possono essere elementi padre legali di istanze di questa classe. Il set completo di possibili superiori è l'unione dei valori <strong>systemPossSuperiors</strong> e <strong>possSuperiors</strong> in questa classe e di tutte le classi strutturali o astratte ereditate. <strong>I valori systemPossSuperiors</strong> e <strong>possSuperiors</strong> non vengono ereditati dalle classi ausiliarie. È possibile aggiungere o rimuovere <strong>valori possSuperiors</strong> da un oggetto <strong>classSchema</strong> di categoria 1 o categoria 2 esistente. Dopo la creazione della classe, la <strong>proprietà systemPossSuperiors</strong> non può essere modificata.<br /> | 
| <strong>objectClassCategory</strong> | Valore intero che specifica la categoria della classe , che può essere una delle seguenti:<ul><li>Strutturale, ovvero è possibile crearne un'istanza nella directory .</li><li>Astratto, ovvero la classe fornisce una definizione di base di una classe che può essere usata per formare classi strutturali.</li><li>Ausiliario, ovvero una classe che può essere usata per estendere la definizione di una classe che eredita da essa, ma non può essere usata per formare una classe da sola.</li></ul>Per altre informazioni, vedere <a href="structural-abstract-and-auxiliary-classes.md">Classi strutturali, astratte e ausiliarie.</a><br /> | 
| <strong>subClassOf</strong> | OID per la superclasse immediata di questa classe, ovvero la classe da cui deriva questa classe. Per le classi strutturali, <strong>subClassOf</strong> può essere una classe strutturale o astratta.<br /> Per le classi <strong>astratte, subClassOf</strong> può essere solo una classe astratta.<br /> Per le classi ausiliarie, <strong>subClassOf</strong> può essere una classe astratta o ausiliaria.<br /> Se si definisce una nuova classe, assicurarsi che la <strong>classe subClassOf</strong> esista o esista quando la nuova classe viene scritta nella directory. Se la classe non esiste, <strong>l'oggetto classSchema</strong> non viene aggiunto alla directory.<br /> | 
| <strong>auxiliaryClass</strong>, <strong>systemAuxiliaryClass</strong> | Coppia di proprietà multivalore che specificano le classi ausiliarie da cui eredita questa classe. Il set completo di classi ausiliarie è l'unione dei valori <strong>systemAuxiliaryClass</strong> <strong>e auxiliaryClass</strong> in questa classe e in tutte le classi ereditate. Per un <strong>oggetto classSchema</strong> esistente, i valori possono essere aggiunti alla <strong>proprietà auxiliaryClass</strong> ma non rimossi. Dopo la creazione della classe, la <strong>proprietà systemAuxiliaryClass</strong> non può essere modificata.<br /> | 
| <strong>defaultObjectCategory</strong> | Nome distinto di questa classe di oggetti o di una delle relative superclassi. Quando viene creata un'istanza di questa classe di oggetti, il sistema imposta la proprietà <strong>objectCategory</strong> della nuova istanza sul valore specificato nella proprietà <strong>defaultObjectCategory</strong> della relativa classe di oggetti. La <strong>proprietà objectCategory</strong> è una proprietà indicizzata usata per aumentare l'efficienza delle ricerche nelle classi di oggetti. Se <strong>defaultObjectCategory</strong> non viene specificato quando viene creata una classe, il sistema la imposta sul nome distinto (DN) dell'oggetto <strong>classSchema</strong> per questa classe. Se l'oggetto viene spesso sottoposto a query in base al valore di una superclasse anziché alla classe dell'oggetto, è possibile impostare <strong>defaultObjectCategory</strong> sul DN della superclasse. Ad esempio, se si sta sottoclassando una classe predefinita (categoria 1), la procedura consigliata consiste nell'impostare <strong>defaultObjectCategory</strong> sullo stesso valore della superclasse. In questo modo l'interfaccia utente standard "trova" la sottoclasse.<br /> Per altre informazioni, vedere <a href="object-class-and-object-category.md">Classe object e categoria di oggetti.</a><br /> |

Per evitare che questi oggetti ingombrino l'interfaccia utente, ogni oggetto ha un attributo booleano denominato <strong>showInAdvancedViewOnly.</strong> Se <strong>defaultHidingValue</strong> è impostato su <strong>TRUE,</strong>le nuove istanze dell'oggetto vengono nascoste negli snap-in amministrativi e nella shell Windows. Una voce di menu per la <strong></strong> classe di oggetti non verrà visualizzata nel menu di scelta rapida Nuovo degli snap-in Amministrativi anche se le proprietà appropriate della creazione guidata sono impostate sull'oggetto <strong>displaySpecifier</strong> della classe di oggetti.<br /> Se <strong>defaultHidingValue</strong> è impostato su <strong>FALSE,</strong>le nuove istanze dell'oggetto vengono visualizzate negli snap-in amministrativi e nella shell Windows. Impostare questa proprietà su <strong>FALSE</strong> per visualizzare le istanze della classe negli snap-in amministrativi e nella shell e abilitare una creazione guidata e la relativa voce di <strong>menu</strong> nel menu Nuovo degli snap-in amministrativi.<br /> Se il <strong>valore defaultHidingValue</strong> non è impostato, il valore predefinito è <strong>TRUE.</strong><br /> |

Il 0x10 bit identifica una classe di categoria 1 (una classe che fa parte dello schema di base incluso nel sistema). Non è possibile impostare questo bit, ovvero il bit non è impostato nelle classi di categoria 2 ,che sono estensioni dello schema. | 
| <strong>systemOnly</strong> | Valore booleano che specifica se solo il server Active Directory può modificare la classe. Le classi solo di sistema possono essere create o eliminate solo da Directory System Agent (DSA). Le classi solo di sistema sono quelle da cui dipende il sistema per le normali operazioni. | 
| <strong>defaultSecurityDescriptor</strong> | Specifica il descrittore di sicurezza predefinito per i nuovi oggetti di questa classe. Per altre informazioni, vedere <a href="default-security-descriptor.md">Descrittore di sicurezza predefinito</a> e Impostazione dei descrittori di sicurezza <a href="how-security-descriptors-are-set-on-new-directory-objects.md">nei nuovi oggetti directory.</a> | 
| <strong>isDefunct</strong> | Valore booleano che indica se la classe è in defezione. Per altre informazioni, vedere <a href="disabling-existing-classes-and-attributes.md">Disabilitazione di classi e attributi esistenti.</a> | 
| <strong>description</strong> | Descrizione testuale della classe che può essere utilizzata dalle applicazioni amministrative. | 
| <strong>Objectclass</strong> | Identifica la classe di oggetti di cui questo oggetto è un'istanza, ovvero la classe dell'oggetto <strong>classSchema</strong> per tutte le definizioni di classe e la classe di oggetti <strong>attributeSchema</strong> per tutte le definizioni di attributo. |
