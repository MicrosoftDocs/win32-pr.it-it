# <a name="dynamic-objects"></a><span data-ttu-id="7889b-104">Oggetti dinamici</span><span class="sxs-lookup"><span data-stu-id="7889b-104">Dynamic Objects</span></span>

<span data-ttu-id="7889b-105">In Windows Server 2003 e versioni successive di Windows Active Directory Domain Services fornire supporto per l'archiviazione di voci dinamiche nella directory.</span><span class="sxs-lookup"><span data-stu-id="7889b-105">In Windows Server 2003 and later versions of Windows, Active Directory Domain Services provide support for storing dynamic entries in the directory.</span></span> <span data-ttu-id="7889b-106">Una voce dinamica è un oggetto nella directory a cui è associato un valore TTL (time-to-Live).</span><span class="sxs-lookup"><span data-stu-id="7889b-106">A dynamic entry is an object in the directory which has an associated time-to-live (TTL) value.</span></span> <span data-ttu-id="7889b-107">Il valore TTL per una voce viene impostato al momento della creazione della voce.</span><span class="sxs-lookup"><span data-stu-id="7889b-107">The TTL for an entry is set when the entry is created.</span></span> <span data-ttu-id="7889b-108">Il time-to-Live viene decrementato automaticamente e quando scade la voce dinamica scompare.</span><span class="sxs-lookup"><span data-stu-id="7889b-108">The time-to-live is automatically decremented, and when it expires the dynamic entry disappears.</span></span> <span data-ttu-id="7889b-109">Il client può causare la permanenza di una voce dinamica più lunga rispetto alla durata rimanente aggiornando (modificando) il valore TTL.</span><span class="sxs-lookup"><span data-stu-id="7889b-109">The client can cause a dynamic entry to remain longer than its current remaining life by refreshing (modifying) its TTL value.</span></span> <span data-ttu-id="7889b-110">I client che archiviano i dati dinamici nella directory devono aggiornare periodicamente tali dati per impedirne la visualizzazione.</span><span class="sxs-lookup"><span data-stu-id="7889b-110">Clients that store dynamic data in the directory must periodically refresh that data to prevent it from disappearing.</span></span>

<span data-ttu-id="7889b-111">Molti servizi e applicazioni che usano LDAP per archiviare e accedere a dati relativamente statici e interessanti a livello globale da un server di Active Directory preferiscono anche continuare a usare l'accesso LDAP per le proprie esigenze di Dynamic Data.</span><span class="sxs-lookup"><span data-stu-id="7889b-111">Many applications and services that use LDAP to store and access relatively static and globally interesting data from an Active Directory server would also prefer to continue using LDAP access for their dynamic data needs.</span></span> <span data-ttu-id="7889b-112">Per alcune applicazioni, inoltre, potrebbe essere opportuno non caricare l'attività di Garbage Collection di oggetti in DS che hanno una vita utile limitata al servizio directory.</span><span class="sxs-lookup"><span data-stu-id="7889b-112">Moreover, for some applications, it may be desirable to off-load the task of garbage-collecting objects in the DS that have a limited useful life to the directory service.</span></span> <span data-ttu-id="7889b-113">Le partizioni di directory applicative (con la possibilità di posizionare le repliche controllate) e TTLs per oggetto offrono la possibilità di ospitare dati dinamici in Active Directory Domain Services, consentendo in tal modo l'accesso LDAP.</span><span class="sxs-lookup"><span data-stu-id="7889b-113">Application directory partitions (with the ability for controlled placement of replicas) and per-object TTLs together will provide the capability of hosting dynamic data in Active Directory Domain Services, thus allowing LDAP access to it.</span></span>

<span data-ttu-id="7889b-114">Una nuova classe di oggetti ausiliaria denominata **DynamicObject** con OID = 1.3.6.1.4.1.1466.101.119.2 verrà definita nello schema ad di base per l'uso da parte delle voci dinamiche.</span><span class="sxs-lookup"><span data-stu-id="7889b-114">A new auxiliary object class called **dynamicObject** with OID = 1.3.6.1.4.1.1466.101.119.2 will be defined in the base AD schema to be used by dynamic entries.</span></span> <span data-ttu-id="7889b-115">Questa classe ausiliaria contiene l'attributo denominato **entryTTL** con OID = 1.3.6.1.4.1.1466.101.119.3 come attributo di sistema-può contenere.</span><span class="sxs-lookup"><span data-stu-id="7889b-115">This auxiliary class contains the attribute called **entryTTL** with OID = 1.3.6.1.4.1.1466.101.119.3 as a system-may-contain attribute.</span></span> <span data-ttu-id="7889b-116">Il valore di questo attributo è il "tempo in secondi" durante il quale la voce di directory corrispondente continuerà a esistere prima di scomparire dalla directory.</span><span class="sxs-lookup"><span data-stu-id="7889b-116">The value of this attribute is the "time in seconds" that the corresponding directory entry will continue to exist before disappearing from the directory.</span></span> <span data-ttu-id="7889b-117">Se il client non fornisce un valore per questo attributo in modo esplicito al momento della creazione dell'oggetto, DS fornisce un valore predefinito come specificato in un secondo momento.</span><span class="sxs-lookup"><span data-stu-id="7889b-117">If the client does not supply a value for this attribute explicitly at object creation, then the DS provides a default value as specified later.</span></span>

<span data-ttu-id="7889b-118">Una nuova operazione LDAP estesa con OID = 1.3.6.1.4.1.1466.101.119.1 per l'aggiornamento client di una voce dinamica nella directory verrà definita e pubblicata nell'attributo **supportedExtension** dell'oggetto DSE radice.</span><span class="sxs-lookup"><span data-stu-id="7889b-118">A new extended LDAP operation with OID = 1.3.6.1.4.1.1466.101.119.1 for client refresh of a dynamic entry in the directory will be defined and published in the **supportedExtension** attribute of the root DSE object.</span></span>

<span data-ttu-id="7889b-119">Nell'implementazione effettiva, **entryTTL** è un attributo costruito.</span><span class="sxs-lookup"><span data-stu-id="7889b-119">In the actual implementation, **entryTTL** is a constructed attribute.</span></span> <span data-ttu-id="7889b-120">L'ora di scadenza dell'oggetto reale viene archiviata come tempo assoluto quando l'oggetto può essere eliminato in un attributo solo di sistema denominato **ms-DS-entry-time-to-Live**.</span><span class="sxs-lookup"><span data-stu-id="7889b-120">The real object expiration time is stored as an absolute time when the object can be destroyed in a system-only attribute called **ms-DS-Entry-Time-To-Live**.</span></span>

<span data-ttu-id="7889b-121">Tutti gli oggetti dinamici presentano le limitazioni seguenti:</span><span class="sxs-lookup"><span data-stu-id="7889b-121">All dynamic objects have the following limitations:</span></span>

-   <span data-ttu-id="7889b-126">Le voci dinamiche vengono gestite in modo simile alle voci non dinamiche durante l'elaborazione delle operazioni di ricerca, confronto, aggiunta, eliminazione, modifica e MODIFYDN del.</span><span class="sxs-lookup"><span data-stu-id="7889b-126">Dynamic entries are handled similar to non-dynamic entries when processing search, compare, add, delete, modify, and modifyDN operations.</span></span>

-   <span data-ttu-id="7889b-128">Non è possibile aggiungere una voce non dinamica a una voce dinamica.</span><span class="sxs-lookup"><span data-stu-id="7889b-128">A non-dynamic entry cannot be added subordinate to a dynamic entry.</span></span>
