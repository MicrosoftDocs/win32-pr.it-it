# <a name="session-monikers"></a>Moniker di sessione

Distributed COM (DCOM) consente l'attivazione degli oggetti per sessione tramite un moniker di sessione fornito dal sistema. Per altre informazioni sulla creazione e sull'uso dei moniker di sessione, incluso un esempio di codice che illustra come attivare un processo del server locale, vedere gli argomenti seguenti.

[Attivazione da sessione a sessione con un moniker di sessione](session-to-session-activation-with-a-session-moniker.md)
</dt> <dd>

L'attivazione da sessione a sessione (detta anche attivazione tra sessioni) consente a un processo client di avviare (attivare) un processo server locale in una sessione specificata.

</dd> <dt>
