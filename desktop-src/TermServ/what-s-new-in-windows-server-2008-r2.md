# <a name="whats-new-in-windows-server-2008-r2"></a>Novità di Windows Server 2008 R2

Windows Server 2008 R2 introduce i nuovi elementi di programmazione seguenti per Servizi Desktop remoto.



<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<tbody>
<tr class="odd">
<td>Servizi terminal è ora Servizi Desktop remoto<br/></td>
<td>Servizi terminal è stato rinominato Servizi Desktop remoto. Un server in cui è installato il servizio ruolo host sessione Desktop remoto (host sessione Desktop remoto) viene ora definito server Host sessione Desktop remoto anziché server terminal.<br/>
<ul>
<li><a href="terminal-services-is-now-remote-desktop-services.md">Servizi terminal è ora Servizi Desktop remoto</a></li>
</ul></td>
</tr>
<tr class="even">
<td>API di virtualizzazione Desktop remoto<br/></td>
<td>L'API di virtualizzazione Desktop remoto fornisce enumerazioni, interfacce e strutture che è possibile utilizzare per creare plug-in personalizzati che eseguono l'override della logica di reindirizzamento standard di Connessione Desktop remoto broker (gestore connessione Desktop remoto). Gestore connessione Desktop remoto (in precedenza noto come broker di sessione di servizi Terminal) è stato espanso per supportare le connessioni alle macchine virtuali.<br/>
<ul>
<li><a href="using-the-remote-desktop-virtualization-api.md">Uso dell'API di virtualizzazione Desktop remoto</a></li>
<li><a href="terminal-services-virtualization-api-reference.md">Informazioni di riferimento sulle API di virtualizzazione Desktop remoto</a></li>
</ul></td>
</tr>
<tr class="odd">
<td>Provider di Remote Desktop Protocol<br/></td>
<td>L'API del provider Remote Desktop Protocol può essere usata per creare un protocollo che Personalizza l'interazione tra il servizio Servizi Desktop remoto e i client desktop.<br/>
<ul>
<li><a href="creating-a-custom-remote-protocol.md">Creazione di un provider di Remote Desktop Protocol</a></li>
<li><a href="custom-remote-protocol-reference.md">Riferimento al provider Remote Desktop Protocol</a></li>
</ul></td>
</tr>
<tr class="even">
<td>API Servizi Desktop remoto AudioEndpoint<br/></td>
<td>L'API Servizi Desktop remoto AudioEndpoint supporta l'enumerazione, le interfacce e le strutture per la registrazione dell'endpoint audio e il trasporto di dati.<br/>
<ul>
<li><a href="terminal-services-audioendpoint-api-reference.md">Informazioni di riferimento sull'API Servizi Desktop remoto AudioEndpoint</a></li>
</ul></td>
</tr>
<tr class="odd">
<td>API del servizio Servizio di gestione Connessione RemoteApp e desktop<br/></td>
<td>L'API del servizio Servizio di gestione Connessione RemoteApp e desktop supporta le interfacce e le strutture che è possibile usare per eseguire il filtro personalizzato delle risorse e fornire supporto per i tipi di file che non sono supportati in modo nativo in Windows Server 2008 R2.<br/>
<ul>
<li><a href="centralized-publishing-api-reference.md">Riferimento all'API del servizio Servizio di gestione Connessione RemoteApp e desktop</a></li>
</ul></td>
</tr>
</tbody>
</table>
